1. Machine Learning Fundamentals

Q1: What is Machine Learning?

Machine Learning (ML) is a subset of artificial intelligence that enables systems to learn and improve from experience without being explicitly programmed. It focuses on developing algorithms that can access data and learn from it.

Key Components:

  • Data: Training data for learning patterns
  • Features: Measurable properties of data
  • Algorithms: Mathematical models for learning
  • Model: Trained algorithm that makes predictions
  • Evaluation: Metrics to assess model performance
Q2: What is the difference between Machine Learning and Deep Learning?
Aspect Machine Learning Deep Learning
Data Structured data Unstructured data
Features Manual feature extraction Automatic feature learning
Architecture Simple algorithms Neural networks with many layers
Compute Less compute intensive Highly compute intensive
Use Case Tabular data, simple problems Images, text, complex problems
Q3: What is the difference between AI, ML, and DL?
  • Artificial Intelligence (AI): Broad field of creating intelligent machines. Includes ML, DL, rule-based systems.
  • Machine Learning (ML): Subset of AI. Systems learn from data. Includes supervised, unsupervised, reinforcement learning.
  • Deep Learning (DL): Subset of ML. Uses neural networks with many layers. Learns complex patterns automatically.

2. Types of Machine Learning

Q4: What are the main types of Machine Learning?
  • Supervised Learning: Learning from labeled data. Predict outcomes. Examples: Classification, Regression.
  • Unsupervised Learning: Learning from unlabeled data. Discover patterns. Examples: Clustering, Dimensionality Reduction.
  • Reinforcement Learning: Learning through trial and error. Optimize decisions. Examples: Game playing, robotics.
  • Semi-Supervised Learning: Combination of labeled and unlabeled data. Improves learning with limited labels.
Q5: What is supervised learning?

Supervised Learning is a type of ML where the model learns from labeled training data. The algorithm learns to map input features to output labels.

Types:

  • Classification: Predict discrete labels (spam/not spam)
  • Regression: Predict continuous values (house price)
Q6: What is unsupervised learning?

Unsupervised Learning is a type of ML where the model learns from unlabeled data. The algorithm discovers hidden patterns and structures in the data.

Types:

  • Clustering: Group similar data points (K-means)
  • Dimensionality Reduction: Reduce features (PCA)
  • Association: Discover relationships (Apriori)

3. Data Preprocessing

Q7: What is data preprocessing?

Data Preprocessing is the process of cleaning and transforming raw data into a format suitable for ML algorithms.

Key Steps:

  • Data Cleaning: Handle missing values, outliers
  • Data Transformation: Normalize, standardize
  • Feature Encoding: Convert categorical to numerical
  • Feature Scaling: Scale features to same range
  • Feature Selection: Select relevant features
Q8: How do you handle missing values?
  • Deletion: Remove rows or columns with missing values
  • Imputation: Fill missing values with mean, median, mode
  • Forward/Backward Fill: Use previous or next values
  • Predictive Imputation: Use ML to predict missing values
  • Indicator Variables: Add flag for missing values
Q9: What is the difference between normalization and standardization?
Aspect Normalization Standardization
Formula (X - min) / (max - min) (X - mean) / std
Range 0 to 1 No fixed range
Mean Not zero Zero
Use Case Neural networks, image data Linear models, Gaussian data

4. Feature Engineering

Q10: What is feature engineering?

Feature Engineering is the process of using domain knowledge to create new features from raw data to improve ML model performance.

Techniques:

  • Feature Creation: Create new features from existing ones
  • Feature Transformation: Transform features (log, sqrt)
  • Feature Selection: Select most important features
  • Feature Extraction: Extract features from raw data
Q11: What is one-hot encoding?

One-Hot Encoding is a technique to convert categorical variables into binary vectors. Each category becomes a new binary feature.

# Example: Color column with values Red, Green, Blue
# After one-hot encoding:
Color_Red  Color_Green  Color_Blue
1          0            0
0          1            0
0          0            1

5. Model Evaluation

Q13: What are common evaluation metrics for classification?
  • Accuracy: Correct predictions / Total predictions
  • Precision: TP / (TP + FP)
  • Recall: TP / (TP + FN)
  • F1 Score: Harmonic mean of precision and recall
  • ROC-AUC: Area under ROC curve
  • Confusion Matrix: Table of actual vs predicted
Q14: What are common evaluation metrics for regression?
  • MSE (Mean Squared Error): Average of squared errors
  • RMSE (Root MSE): Square root of MSE
  • MAE (Mean Absolute Error): Average of absolute errors
  • R² Score: Coefficient of determination
  • Adjusted R²: R² adjusted for number of features
Q15: What is the difference between precision and recall?
  • Precision: Of all predicted positive, how many are actually positive. Measures quality of positive predictions. Important when false positives are costly.
  • Recall: Of all actual positive, how many are predicted positive. Measures quantity of positive predictions. Important when false negatives are costly.

6. Supervised Learning Algorithms

Q16: What is Linear Regression?

Linear Regression is a supervised learning algorithm used for regression tasks. It models the relationship between input features and continuous output using a linear equation.

# Linear Regression equation
y = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ + ε

# Where:
# y = predicted value
# β₀ = intercept
# β₁...βₙ = coefficients
# x₁...xₙ = features
# ε = error term
Q17: What is Logistic Regression?

Logistic Regression is a classification algorithm that uses the logistic function to model the probability of a binary outcome.

# Logistic Regression equation
P(y=1) = 1 / (1 + e^(-z))

# Where:
# z = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ
# P(y=1) = probability of class 1
Q18: What is Decision Tree?

Decision Tree is a supervised learning algorithm that uses a tree-like model of decisions. It splits data based on features to make predictions.

Key Concepts:

  • Root Node: Top node representing entire dataset
  • Internal Nodes: Decision nodes with splits
  • Leaf Nodes: Terminal nodes with predictions
  • Splitting Criteria: Gini impurity, Information gain
Q19: What is Random Forest?

Random Forest is an ensemble learning method that constructs multiple decision trees and combines their predictions through voting or averaging.

Benefits:

  • Reduces Overfitting: Averages multiple trees
  • Handles Non-linearity: Complex relationships
  • Feature Importance: Provides feature importance
  • Robust: Handles noise and outliers
Q20: What is Support Vector Machine (SVM)?

Support Vector Machine (SVM) is a supervised learning algorithm used for classification and regression. It finds the optimal hyperplane that separates classes.

Key Concepts:

  • Hyperplane: Decision boundary
  • Support Vectors: Data points closest to hyperplane
  • Kernel Trick: Transform data to higher dimensions
  • Margin: Distance between hyperplane and support vectors

7. Unsupervised Learning Algorithms

Q21: What is K-Means Clustering?

K-Means Clustering is an unsupervised learning algorithm that partitions data into K clusters based on feature similarity.

Algorithm Steps:

  1. Initialize K cluster centroids randomly
  2. Assign each data point to nearest centroid
  3. Recalculate centroids as mean of assigned points
  4. Repeat until convergence
Q22: What is Hierarchical Clustering?

Hierarchical Clustering is an unsupervised learning algorithm that builds a hierarchy of clusters. It creates a tree-like structure called a dendrogram.

Types:

  • Agglomerative: Bottom-up approach. Start with each point as cluster, merge closest.
  • Divisive: Top-down approach. Start with one cluster, split recursively.
Q23: What is Principal Component Analysis (PCA)?

Principal Component Analysis (PCA) is a dimensionality reduction technique that transforms data into a lower-dimensional space while preserving maximum variance.

Key Concepts:

  • Principal Components: Orthogonal directions of maximum variance
  • Eigenvalues: Amount of variance explained
  • Eigenvectors: Direction of principal components
  • Variance Explained: Proportion of total variance

8. Neural Networks and Deep Learning

Q24: What is a Neural Network?

Neural Network is a computational model inspired by biological neural networks. It consists of interconnected nodes (neurons) organized in layers.

Components:

  • Input Layer: Receives input features
  • Hidden Layers: Process and transform data
  • Output Layer: Produces predictions
  • Weights: Learnable parameters
  • Bias: Offset terms
  • Activation Functions: Non-linear transformations
Q25: What are common activation functions?
  • Sigmoid: Maps to (0, 1). Used in output layer for binary classification.
  • Tanh: Maps to (-1, 1). Zero-centered. Used in hidden layers.
  • ReLU: max(0, x). Most popular. Solves vanishing gradient.
  • Leaky ReLU: max(αx, x). Allows small negative values.
  • Softmax: Converts to probabilities. Used in multi-class classification.
Q26: What is backpropagation?

Backpropagation is the algorithm used to train neural networks. It calculates gradients of the loss function with respect to weights using the chain rule.

Process:

  1. Forward pass: Compute predictions
  2. Calculate loss: Compare predictions to actual
  3. Backward pass: Compute gradients
  4. Update weights: Use gradient descent
Q27: What is the vanishing gradient problem?

Vanishing Gradient Problem occurs when gradients become extremely small as they propagate backward through deep networks, making training difficult.

Solutions:

  • ReLU Activation: Avoids saturation
  • Better Initialization: Xavier, He initialization
  • Batch Normalization: Normalizes layer inputs
  • Residual Connections: Skip connections

9. Model Optimization

Q28: What is overfitting?

Overfitting occurs when a model learns the training data too well, including noise and outliers, resulting in poor generalization to new data.

Solutions:

  • More Training Data: Increase dataset size
  • Regularization: L1, L2 regularization
  • Dropout: Randomly drop neurons during training
  • Early Stopping: Stop training when validation loss increases
  • Simpler Model: Reduce model complexity
Q29: What is underfitting?

Underfitting occurs when a model is too simple to capture the underlying pattern in the data, resulting in poor performance on both training and test data.

Solutions:

  • More Complex Model: Increase model capacity
  • More Features: Add relevant features
  • Reduce Regularization: Decrease regularization strength
  • Train Longer: Increase training epochs
Q30: What is regularization?

Regularization is a technique to prevent overfitting by adding a penalty term to the loss function to discourage complex models.

Types:

  • L1 Regularization (Lasso): Adds absolute value of weights. Encourages sparsity.
  • L2 Regularization (Ridge): Adds squared value of weights. Encourages small weights.
  • Elastic Net: Combination of L1 and L2.

10. ML Pipelines and MLOps

Q31: What is an ML pipeline?

ML Pipeline is a sequence of data processing and modeling steps that automate the machine learning workflow from data ingestion to model deployment.

Components:

  • Data Ingestion: Collect and load data
  • Data Preprocessing: Clean and transform data
  • Feature Engineering: Create and select features
  • Model Training: Train ML models
  • Model Evaluation: Assess model performance
  • Model Deployment: Deploy to production
Q32: What is MLOps?

MLOps (Machine Learning Operations) is a set of practices that aims to deploy and maintain machine learning models in production reliably and efficiently.

Key Practices:

  • CI/CD for ML: Automated training and deployment
  • Model Monitoring: Track model performance
  • Data Drift Detection: Monitor data distribution changes
  • Model Versioning: Track model versions
  • Automated Retraining: Retrain models periodically
Q33: What is A/B testing for ML models?

A/B Testing is a method to compare two versions of a model by randomly assigning users to each version and measuring performance metrics.

Process:

  1. Define hypothesis and metrics
  2. Split users into control and test groups
  3. Deploy different models to each group
  4. Collect and analyze results
  5. Choose winning model

11. Popular ML Frameworks

Q34: What is TensorFlow?

TensorFlow is an open-source ML framework developed by Google. It provides tools for building and deploying ML models across various platforms.

Key Features:

  • Flexible Architecture: Build and deploy anywhere
  • Keras API: High-level API for easy model building
  • TensorBoard: Visualization and debugging tool
  • TensorFlow Serving: Model deployment
  • TensorFlow Lite: Mobile and edge deployment
Q35: What is PyTorch?

PyTorch is an open-source ML framework developed by Facebook. It provides a dynamic computational graph and is popular in research and production.

Key Features:

  • Dynamic Computation Graph: Build models on the fly
  • Pythonic: Easy to use and debug
  • TorchScript: Production deployment
  • Rich Ecosystem: Extensive libraries
  • Strong Community: Active research community
Q36: What is Scikit-learn?

Scikit-learn is a Python ML library built on NumPy, SciPy, and matplotlib. It provides simple and efficient tools for data mining and data analysis.

Key Features:

  • Classification: SVM, Random Forest, KNN
  • Regression: Linear, Ridge, Lasso
  • Clustering: K-Means, DBSCAN
  • Dimensionality Reduction: PCA, t-SNE
  • Model Selection: Cross-validation, grid search

12. Advanced Topics

Q37: What is Transfer Learning?

Transfer Learning is a technique where a model trained on one task is reused as the starting point for a model on a second task.

Benefits:

  • Faster Training: Less training time
  • Better Performance: Better with limited data
  • Lower Cost: Less compute required
  • Pre-trained Models: Leverage existing models
Q38: What is Ensemble Learning?

Ensemble Learning is a technique that combines multiple models to improve predictive performance compared to individual models.

Methods:

  • Bagging: Bootstrap Aggregating (Random Forest)
  • Boosting: Sequential model improvement (XGBoost, AdaBoost)
  • Stacking: Combine predictions from multiple models
  • Voting: Majority voting or averaging
Q39: What is Gradient Boosting?

Gradient Boosting is an ensemble learning technique that builds models sequentially, with each new model correcting errors of the previous ones.

Popular Implementations:

  • XGBoost: Extreme Gradient Boosting
  • LightGBM: Light Gradient Boosting Machine
  • CatBoost: Categorical Boosting

13. Common Interview Questions

Q40: What is the difference between bias and variance?
  • Bias: Error from erroneous assumptions. High bias = underfitting. Model too simple.
  • Variance: Error from sensitivity to small fluctuations. High variance = overfitting. Model too complex.
Q41: What is the bias-variance tradeoff?

Bias-Variance Tradeoff is the balance between underfitting (high bias) and overfitting (high variance). The goal is to find the sweet spot that minimizes total error.

Q42: What is cross-validation?

Cross-Validation is a technique to assess model performance by splitting data into multiple subsets and training/evaluating on different combinations.

Types:

  • K-Fold CV: Split into K folds, train on K-1, test on 1
  • Stratified K-Fold: Preserves class distribution
  • Leave-One-Out: K = number of samples
  • Time Series CV: Respects temporal order

Q48: What are the best practices for ML model deployment?
  • Model Versioning: Track model versions and metadata
  • Monitoring: Monitor model performance and data drift
  • Scalability: Design for scale and load
  • Security: Secure model and data
  • Testing: Test thoroughly before deployment
  • Rollback: Plan for rollback if needed
  • Documentation: Document model and API
  • API Design: Design clear and consistent API
Q49: What are the best practices for ML model monitoring?
  • Performance Metrics: Track accuracy, precision, recall
  • Data Drift: Monitor input data distribution
  • Prediction Drift: Monitor prediction distribution
  • Latency: Monitor prediction latency
  • Throughput: Monitor request volume
  • Errors: Monitor error rates
  • Alerts: Set up alerts for anomalies
  • Retraining: Schedule periodic retraining
Q50: What are the common challenges in Machine Learning?
  • Data Quality: Poor quality data leads to poor models
  • Data Quantity: Insufficient data limits performance
  • Feature Engineering: Requires domain knowledge
  • Model Selection: Choosing the right model is difficult
  • Hyperparameter Tuning: Time-consuming process
  • Overfitting: Models overfit training data
  • Interpretability: Complex models are hard to interpret
  • Deployment: Deploying models is challenging
  • Maintenance: Models require ongoing maintenance
  • Ethics: Bias and fairness concerns
