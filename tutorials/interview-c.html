C Interview Guide

Comprehensive guide covering C language features with detailed explanations and code examples

1. C Fundamentals

Q1: What is C and what are its key features?

C is a general-purpose, procedural programming language created by Dennis Ritchie at Bell Labs in 1972. Key features include:

  • Simple: Minimal set of keywords and straightforward syntax
  • Portable: Code can run on different platforms with minimal changes
  • Efficient: Produces fast and optimized code
  • Low-level Access: Direct memory manipulation via pointers
  • Structured: Supports structured programming constructs
  • Extensible: Can be extended with user-defined functions and types
  • Foundation: Basis for many modern languages (C++, Java, Python)

2. Data Types and Variables

Q4: What is the difference between printf and scanf?
Aspect printf scanf
Purpose Output data to console Input data from console
Direction Program to user User to program
Parameters Values or expressions Addresses of variables
Return Number of characters printed Number of items read
code.c
int age; printf("Enter age: "); scanf("%d", &age); // Needs address printf("Age: %d", age);
Q5: What is the difference between %d and %i in printf?

In printf, both %d and %i are identical - they both print integers.

In scanf, %d expects a decimal integer, while %i can accept decimal, octal (with 0 prefix), or hexadecimal (with 0x prefix).

code.c
int num1, num2; scanf("%d", &num1); // Decimal only scanf("%i", &num2); // Decimal, octal, or hex printf("num1: %d, num2: %d", num1, num2);
Q6: What is the difference between ++i and i++?

++i (Pre-increment): Increments the value first, then uses it.

i++ (Post-increment): Uses the value first, then increments it.

code.c
int i = 5; int j = ++i; // i=6, j=6 (pre-increment) i = 5; j = i++; // j=5, i=6 (post-increment)

3. Control Structures

Q7: What are the different control structures in C?

1. Sequential: Statements executed in order

2. Selection (Decision Making): if, if-else, switch

3. Iteration (Looping): for, while, do-while

4. Jump: break, continue, goto, return

code.c
// Selection if (condition) { // code } else if (condition) { // code } else { // code } switch (expression) { case 1: // code break; default: // code } // Iteration for (int i = 0; i < 10; i++) { // code } while (condition) { // code } do { // code } while (condition); // Jump break; // Exit loop/switch continue; // Skip to next iteration goto label; // Jump to label return; // Return from function

Aspect while do-while

Syntax while (condition) { } do { } while (condition);
code.c
int i = 5; while (i < 5) { } // Won't execute do { } while (i < 5); // Executes once

4. Functions

Q9: What is the difference between call by value and call by reference?
Aspect Call by Value Call by Reference
Parameter Value is copied Address is passed
Modification Original not affected Original is affected
Memory Separate memory for copy Same memory accessed
Performance Slower for large data Faster (no copying)
code.c
// Call by value - copy void swapByValue(int a, int b) { int temp = a; a = b; b = temp; } // Call by reference - pointer void swapByReference(int *a, int *b) { int temp = *a; *a = *b; *b = temp; } int x = 10, y = 20; swapByValue(x, y); // x=10, y=20 (unchanged) swapByReference(&x, &y); // x=20, y=10 (changed)
Q10: What is recursion in C?

Recursion is when a function calls itself to solve a problem. Each recursive call works on a smaller instance of the problem.

code.c
int factorial(int n) { if (n <= 1) return 1; return n * factorial(n - 1); } int fibonacci(int n) { if (n <= 1) return n; return fibonacci(n - 1) + fibonacci(n - 2); }
Tip: Always have a base case to prevent infinite recursion. Recursion can lead to stack overflow if not properly handled.

5. Arrays and Strings

Q11: What is the difference between array and pointer?
Aspect Array Pointer
Definition Collection of same-type elements Variable that stores address
Memory Static allocation (usually) Dynamic allocation possible
Size Fixed at declaration Can point to different sizes
Operation Cannot be reassigned Can be reassigned
code.c
int arr[5] = {1, 2, 3, 4, 5}; int *ptr = arr; // Array decays to pointer arr[2] = 10; // OK *(ptr + 2) = 10; // OK // arr = ptr; // Error - array can't be reassigned ptr = &arr[0]; // OK - pointer can be reassigned
Q12: What is a null-terminated string in C?

In C, strings are arrays of characters terminated by a null character ('\0'). The null character marks the end of the string.

code.c
char str1[] = "Hello"; // 'H','e','l','l','o','\0' char str2[] = {'H', 'i', '\0'}; char str3[] = {'H', 'e', 'l', 'l', 'o'}; // Not null-terminated strlen(str1); // 5 (doesn't count '\0')
Important: Always ensure strings are null-terminated. Functions like strlen, strcpy, etc. rely on the null terminator.

6. Pointers

Q13: What is a pointer and how is it used?

A pointer is a variable that stores the memory address of another variable. Pointers are used for:

  • Direct memory access
  • Dynamic memory allocation
  • Pass by reference
  • Data structures (linked lists, trees)
  • Efficient array handling
code.c
int num = 10; int *ptr = # *ptr = 20; // Modify value through pointer printf("%d", num); // 20
Q14: What is the difference between *ptr and &ptr?

*ptr (Dereference operator): Gets the value stored at the address pointed to by ptr.

&ptr (Address-of operator): Gets the memory address of the variable ptr.

code.c
int num = 10; int *ptr = # *ptr; // Value at address (10) # // Address of num ptr; // Address stored in ptr &ptr; // Address of ptr variable
Q15: What is pointer arithmetic?

Pointer arithmetic allows you to perform arithmetic operations on pointers. The operations are scaled by the size of the data type.

code.c
int arr[] = {10, 20, 30, 40, 50}; int *ptr = arr; *ptr; // 10 *(ptr + 1); // 20 (scaled by sizeof(int)) ptr++; // Moves to next element ptr += 2; // Moves ahead by 2 elements

7. Memory Management

Q16: What is dynamic memory allocation in C?

Dynamic memory allocation allows you to allocate memory at runtime using functions from stdlib.h:

  • malloc: Allocates uninitialized memory
  • calloc: Allocates and initializes memory to zero
  • realloc: Reallocates memory to new size
  • free: Deallocates previously allocated memory
code.c
int *ptr1 = malloc(5 * sizeof(int)); // Uninitialized int *ptr2 = calloc(5, sizeof(int)); // Initialized to zero ptr1 = realloc(ptr1, 10 * sizeof(int)); // Resize free(ptr1); free(ptr2); // Deallocate
Important: Always check if malloc/calloc returns NULL (allocation failed). Always free allocated memory to prevent memory leaks.
Q17: What is the difference between malloc and calloc?
Aspect malloc calloc
Initialization Uninitialized (garbage values) Initialized to zero
Parameters Size in bytes Number of elements, size of each
Speed Faster Slower (due to initialization)
Return void* void*
code.c
#include #include int main() { int *ptr1 = (int*)malloc(5 * sizeof(int)); int *ptr2 = (int*)calloc(5, sizeof(int)); // malloc - values are garbage for (int i = 0; i < 5; i++) { printf("malloc[%d]: %d\n", i, ptr1[i]); } // calloc - values are zero for (int i = 0; i < 5; i++) { printf("calloc[%d]: %d\n", i, ptr2[i]); } free(ptr1); free(ptr2); return 0; }

10. Preprocessor Directives

Q21: What is the difference between #define and const?
Aspect #define const
Type Preprocessor directive Language keyword
Type Checking No type checking Type checking
Scope Global (unless undefined) Respects scope
Memory No memory allocation (text substitution) Memory allocated
Debugging Harder (no symbol in debugger) Easier (symbol exists)
code.c
#define PI 3.14159 const float pi = 3.14159f; #undef PI // Can undefine #define // pi = 3.0; // Error - can't modify const
Q22: What is the difference between #include and #include "file"?

#include : Searches for the file in standard system directories (compiler's include paths).

#include "file": Searches for the file in the current directory first, then in standard system directories.

code.c
// Standard library - use <> #include #include #include // User-defined headers - use "" #include "myheader.h" #include "utils.h"
Tip: Use <> for standard library headers and "" for your own header files.
Q23: What is a segmentation fault (SIGSEGV) in C?

Segmentation fault occurs when program attempts to access memory it doesn't have permission to access. Common causes: dereferencing NULL pointer, accessing array out of bounds, writing to read-only memory, using freed memory (dangling pointer), stack overflow.

code.c
#include int main() { int *ptr = NULL; *ptr = 10; // Segmentation fault return 0; }
Q24: Does sizeof() evaluate the expression passed to it?

No, sizeof() does not evaluate the expression. It only determines the type at compile-time. Example: sizeof(a++) doesn't increment a - only the type is considered for size calculation.

code.c
#include int main() { int a = 5; printf("%zu", sizeof(a++)); // a remains 5 printf("%d", a); // Still 5 return 0; }
Q25: What is the difference between NULL, 0, and '\0'?

NULL: Null pointer ((void*)0). 0: Integer constant (can represent null pointer in pointer context). '\0': Null character used to terminate strings (ASCII 0).

code.c
int *ptr = NULL; // Null pointer int num = 0; // Integer char ch = '\0'; // Null character (ASCII 0)
Q26: What are UART, SPI, and I2C communication protocols?

UART: Asynchronous, 2 wires (TX, RX), simple, moderate speed. SPI: Synchronous, 4 wires (MOSI, MISO, SCLK, SS), high speed, full-duplex. I2C: Synchronous, 2 wires (SDA, SCL), slower, multi-device addressing.

code.c
// UART - Simple, 2 wires // SPI - Fast, 4 wires, full-duplex // I2C - Slower, 2 wires, multi-device // UART initialization (pseudocode) UART->BAUD = 9600; UART->CTRL = UART_ENABLE | UART_TX_ENABLE;
Q27: What is a watchdog timer (WDT) in embedded systems?

Hardware timer that resets microcontroller if software becomes unresponsive. Program must periodically "kick" watchdog before timeout expires. Used for safety-critical systems to recover from hangs or deadlocks.

code.c
// Watchdog timer (pseudocode) WDT->CTRL = WDT_ENABLE | WDT_TIMEOUT_2S; while (1) { do_work(); WDT->REFRESH = 0xAA55; // Kick watchdog }
Q28: What is memory-mapped I/O in Embedded C?

Hardware registers assigned fixed memory addresses, accessed like variables. Use volatile keyword to prevent compiler optimization. Example: #define PORTB (*(volatile unsigned char*)0x25).

code.c
#define PORTB (*(volatile unsigned char*)0x25) // Access hardware register PORTB = 0xFF; // Write to register unsigned char value = PORTB; // Read from register
Q29: What is the difference between deep copy and shallow copy in C?

Shallow copy: Copies pointer addresses, not data. Both pointers point to same memory. Deep copy: Copies actual data, allocates new memory. Changes don't affect original.

code.c
#include #include // Shallow copy (problematic) char *ptr1 = malloc(10); strcpy(ptr1, "Hello"); char *ptr2 = ptr1; // Same memory // Deep copy (correct) char *ptr3 = malloc(10); strcpy(ptr3, ptr1); // New memory
Q30: What is a buffer overflow?

Writing data beyond allocated memory boundaries. Can crash program, corrupt memory, or enable security exploits. Prevented by bounds checking, using safe functions (fgets, strncpy).

code.c
#include // Unsafe - buffer overflow char buffer[5]; strcpy(buffer, "HelloWorld"); // Overflow! // Safe - use fgets fgets(buffer, sizeof(buffer), stdin);
Q31: What is undefined behavior in C?

Code with no specified behavior in C standard. Results unpredictable: may crash, work incorrectly, or appear to work. Examples: dereferencing NULL, signed integer overflow, using uninitialized variables.

code.c
int *ptr = NULL; // Undefined: dereferencing NULL int arr[5]; arr[10] = 5; // Undefined: out of bounds int x; // Undefined: uninitialized
Q32: What is the difference between char *s and char s[]?

char *s: Pointer to string (modifiable address, may point to read-only memory). char s[]: Array of characters (modifiable content, allocated on stack).

code.c
char *p = "Hello"; // May be read-only char a[] = "Hello"; // Modifiable p[0] = 'h'; // Undefined behavior a[0] = 'h'; // OK
Q33: What is a restrict keyword in C?

Tells compiler that pointer is only way to access object. Enables compiler optimizations. Used in function parameters to promise no aliasing.

code.c
void copy(int *restrict dest, int *restrict src, int n) { // Compiler knows dest and src don't overlap for (int i = 0; i < n; i++) { dest[i] = src[i]; } }
Q34: What is the difference between struct and class in C++?

struct: Default public members, C-compatible. class: Default private members, C++ features (inheritance, methods). In C, only structs exist.

code.c
// C - only structs struct Point { int x; int y; }; // C++ - struct (public) vs class (private) struct PointStruct { int x; }; // public class PointClass { int x; }; // private
Q35: What is the difference between union and struct?

struct: Each member has separate memory. Size = sum of all members. union: All members share same memory. Size = size of largest member. Only one member active at a time.

code.c
struct S { int a; char b; }; // Size: 8 (with padding) union U { int a; char b; }; // Size: 4 (largest member)
Q36: What is the difference between typedef and #define?

typedef: Creates type alias, type-checked, processed by compiler. #define: Text substitution, no type checking, processed by preprocessor.

code.c
typedef unsigned int uint; // Type alias #define uint unsigned int // Text substitution uint x = 10; // Type-checked #define y 20 // No type checking
Q37: What is the difference between const and #define?

const: Variable with fixed value, type-checked, scoped. #define: Macro, text substitution, no type, global scope. Use const for type safety.

code.c
#define MAX 100 // Global, no type const int max = 100; // Scoped, type-checked void func() { const int local = 50; // Local const // MAX is also available here }
Q38: What is the difference between array and pointer?

array: Fixed size, contiguous memory, sizeof returns total size. pointer: Stores address, can be reassigned, sizeof returns pointer size. Array decays to pointer.

code.c
int arr[5] = {1, 2, 3, 4, 5}; int *ptr = arr; sizeof(arr); // 20 (5 * 4) sizeof(ptr); // 8 (pointer size) arr = ptr; // Error ptr = &arr[0]; // OK
Q39: What is the difference between i++ and ++i in complex expression?

++i: Pre-increment, returns incremented value. i++: Post-increment, returns original value. In complex expressions, order matters for side effects.

code.c
int i = 5, j; j = ++i; // i=6, j=6 j = i++; // j=6, i=7 int arr[] = {1, 2, 3}; int x = arr[i++] + arr[++i]; // Undefined!
Q40: What is the difference between = and ==?

=: Assignment operator, assigns value. ==: Equality operator, compares values. Common mistake: using = in condition instead of ==.

code.c
int x = 10; // Assignment if (x == 10) // Comparison printf("Equal"); if (x = 5) // Wrong! Always true printf("This runs");
Q41: What is the difference between && and &?

&&: Logical AND, short-circuits (stops if first false). &: Bitwise AND, evaluates both operands, operates on bits.

code.c
int a = 5, b = 0; if (a && b) // Logical AND (false) printf("Won't print"); if (a & b) // Bitwise AND (0) printf("Won't print"); if (a && (a = 0)) // Short-circuit, a unchanged printf("Won't print");
Q42: What is the difference between || and |?

||: Logical OR, short-circuits (stops if first true). |: Bitwise OR, evaluates both operands, operates on bits.

code.c
int a = 5, b = 0; if (a || b) // Logical OR (true) printf("Will print"); if (a | b) // Bitwise OR (5) printf("Will print"); if (a || (a = 0)) // Short-circuit, a unchanged printf("Will print");

11. Advanced Pointers

Q43: What is a double pointer (pointer to pointer)?

Double pointer is a pointer that stores the address of another pointer. Used for modifying pointer values in functions, creating dynamic 2D arrays, and implementing data structures like linked lists.

code.c
int a = 10; int *ptr = &a; int **ptr2 = &ptr; printf("%d", **ptr2); // 10
Q44: What is the difference between near, far, and huge pointers?

near pointer: 16-bit pointer for small memory models (64KB segment). far pointer: 32-bit pointer with segment and offset (can access 1MB). huge pointer: 32-bit pointer with normalized segment (can access >1MB). These are specific to DOS/16-bit systems.

Q45: What is pointer to function?

Function pointer stores the address of a function. Used for callbacks, implementing polymorphism, and passing functions as arguments.

code.c
int add(int a, int b) { return a + b; } int (*func_ptr)(int, int) = add; int result = func_ptr(5, 3); // 8
Q46: What is an array of function pointers?

Array of function pointers stores multiple function addresses. Used for implementing state machines, command dispatchers, and switch-like behavior.

code.c
int (*operations[])(int, int) = {add, sub, mul, div}; int result = operations[0](5, 3); // Calls add

12. Bit Manipulation

Q47: What are bitwise operators in C?

Bitwise operators: & (AND), | (OR), ^ (XOR), ~ (NOT), << (left shift), >> (right shift). Used for low-level programming, embedded systems, and optimization.

code.c
int a = 5; // 0101 int b = 3; // 0011 a & b; // 0001 (1) - AND a | b; // 0111 (7) - OR a ^ b; // 0110 (6) - XOR ~a; // 1010 (-6) - NOT a << 1; // 1010 (10) - Left shift a >> 1; // 0010 (2) - Right shift
Q48: How to set, clear, and toggle a bit?

Set bit: num |= (1 << position). Clear bit: num &= ~(1 << position). Toggle bit: num ^= (1 << position). Check bit: (num >> position) & 1.

code.c
int num = 5; // 0101 num |= (1 << 1); // Set bit 1: 0111 (7) num &= ~(1 << 2); // Clear bit 2: 0011 (3) num ^= (1 << 0); // Toggle bit 0: 0010 (2)
Q49: What is the difference between logical and bitwise shift?

Logical shift: Fills with zeros (unsigned). Arithmetic shift: Fills with sign bit (signed). In C, >> is arithmetic for signed, logical for unsigned.

code.c
signed int x = -8; // 11111000 unsigned int y = 8; // 00001000 x >> 2; // Arithmetic: fills with 1s y >> 2; // Logical: fills with 0s

13. Preprocessor Advanced

Q50: What is the difference between #if and #ifdef?

#if evaluates constant expressions. #ifdef checks if macro is defined. #if is more flexible, #ifdef is simpler for existence checks.

code.c
#ifdef DEBUG printf("Debug mode"); #endif #if DEBUG == 1 printf("Debug level 1"); #endif
Q51: What is #pragma once?

#pragma once ensures header file is included only once. Alternative to include guards (#ifndef HEADER_H #define HEADER_H). Non-standard but widely supported.

code.c
#pragma once // Header content
Q52: What are macro operators # and ##?

# (stringizing) converts macro argument to string. ## (token pasting) concatenates tokens.

code.c
#define STR(x) #x #define CONCAT(a, b) a##b STR(hello); // "hello" CONCAT(var, 1); // var1

14. Standard Library

Q53: What is the difference between strcpy and strncpy?

strcpy copies string without length check (unsafe). strncpy copies with length limit (safer but may not null-terminate).

code.c
char dest[10]; strcpy(dest, "hello"); // Unsafe strncpy(dest, "hello", 9); // Safer dest[9] = '\0'; // Ensure null-termination
Q54: What is the difference between strcmp and strncmp?

strcmp compares entire strings. strncmp compares up to n characters.

code.c
strcmp("hello", "world"); // Compare full strings strncmp("hello", "help", 3); // Compare first 3 chars
Q55: What is the difference between sprintf and snprintf?

sprintf writes to buffer without size check (unsafe). snprintf writes with size limit (safer).

code.c
char buffer[10]; sprintf(buffer, "%d", 12345); // Unsafe snprintf(buffer, sizeof(buffer), "%d", 12345); // Safer

15. File Operations

Q56: What is the difference between text and binary mode?

Text mode: Handles line endings (\r\n conversion), may modify data. Binary mode: Raw data, no conversion. Use binary for images, executables.

code.c
FILE *text = fopen("file.txt", "r"); // Text mode FILE *binary = fopen("file.bin", "rb"); // Binary mode
Q57: What is the difference between fseek and rewind?

fseek moves file position to specific offset. rewind moves to beginning (equivalent to fseek(fp, 0, SEEK_SET)).

code.c
fseek(fp, 100, SEEK_SET); // Move to byte 100 rewind(fp); // Move to beginning
Q58: What is the difference between fgets and gets?

gets reads line without size limit (removed in C11, dangerous). fgets reads with size limit (safe).

code.c
char buffer[100]; fgets(buffer, sizeof(buffer), stdin); // Safe // gets(buffer); // Dangerous - removed in C11

16. Memory Management Advanced

Q59: What is memory leak in C?

Memory leak occurs when allocated memory is not freed. Causes: forgetting free(), losing pointer, overwriting pointer. Detected with tools like Valgrind.

code.c
int *ptr = malloc(sizeof(int)); ptr = malloc(sizeof(int)); // Leak - lost first allocation // free(ptr); // Missing free causes leak
Q60: What is the difference between stack and heap?

Stack: Automatic allocation, LIFO, fast, limited size, local variables. Heap: Manual allocation, random access, slower, larger, dynamic memory.

code.c
int stack_var; // Stack int *heap_ptr = malloc(sizeof(int)); // Heap free(heap_ptr); // Must free
Q61: What is the difference between alloca and malloc?

alloca allocates on stack (automatically freed, non-standard). malloc allocates on heap (must free manually, standard).

code.c
void *stack_mem = alloca(100); // Auto-freed void *heap_mem = malloc(100); // Must free free(heap_mem);

17. Structures Advanced

Q62: What is structure padding?

Padding adds bytes to align members for efficient memory access. Can be controlled with #pragma pack. Affects sizeof(struct).

code.c
struct Data { char a; // 1 byte + 3 padding int b; // 4 bytes }; // Total: 8 bytes
Q63: What is a bit field?

Bit field specifies exact number of bits for a member. Used for memory optimization, flags, hardware registers.

code.c
struct Flags { unsigned int flag1 : 1; unsigned int flag2 : 1; unsigned int value : 6; }; // Total: 1 byte
Q64: What is the difference between struct and class in C++?

struct: Default public members, C-compatible. class: Default private members, C++ features. In C, only structs exist.

18. Compiler Directives

Q65: What is static_assert?

static_assert (C11) checks condition at compile-time. Used for compile-time assertions, type checking.

code.c
static_assert(sizeof(int) == 4, "int must be 4 bytes"); static_assert(sizeof(long) >= 8, "long must be at least 8 bytes");
Q66: What is _Generic (C11)?

_Generic selects expression based on type. Used for type-generic programming, overloading-like behavior.

code.c
#define TYPE_NAME(x) _Generic((x), \ int: "int", \ float: "float", \ default: "other") printf("%s", TYPE_NAME(10)); // "int"

19. Security

Q67: What is buffer overflow?

Buffer overflow writes beyond allocated memory. Can crash program, corrupt data, enable exploits. Prevented by bounds checking, safe functions.

code.c
char buffer[5]; strcpy(buffer, "hello world"); // Overflow!
Q68: What is format string vulnerability?

Format string vulnerability occurs when user input is used as format string. Can read/write memory. Always use format specifiers.

code.c
printf(user_input); // Vulnerable! printf("%s", user_input); // Safe
Q69: What is integer overflow?

Integer overflow occurs when value exceeds type range. Signed overflow is undefined behavior. Use larger types or checks.

code.c
int x = INT_MAX; x++; // Undefined behavior (signed overflow) unsigned int y = UINT_MAX; y++; // Defined (wraps to 0)

20. Optimization

Q70: What is inline function?

inline suggests compiler to insert function code at call site. Reduces overhead, may increase code size. Hint only, not guaranteed.

code.c
inline int add(int a, int b) { return a + b; }
Q71: What is the difference between const and volatile?

const value cannot change. volatile value can change externally (hardware, interrupt). Can be both.

code.c
const int x = 10; // Cannot modify volatile int y = 10; // May change externally const volatile int z = 10; // Both
Q72: What is restrict keyword?

restrict tells compiler pointer is only way to access object. Enables optimizations. Promise of no aliasing.

code.c
void copy(int *restrict dest, int *restrict src, int n) { // Compiler knows no overlap for (int i = 0; i < n; i++) dest[i] = src[i]; }

21. Data Structures

Q73: What is a linked list in C?

Linked list is linear data structure with nodes containing data and pointer to next node. Dynamic size, efficient insertion/deletion.

code.c
struct Node { int data; struct Node *next; };
Q74: What is the difference between array and linked list?

Array: Contiguous memory, O(1) access, fixed size. Linked list: Scattered memory, O(n) access, dynamic size.

Q75: What is a binary tree in C?

Binary tree is hierarchical data structure with nodes having at most two children. Used for searching, sorting.

code.c
struct TreeNode { int data; struct TreeNode *left; struct TreeNode *right; };

22. Concurrency

Q76: What is the difference between process and thread?

Process: Separate memory space, heavier. Thread: Shared memory, lighter. C uses pthreads for threading.

Q77: What is a race condition?

Race condition occurs when output depends on timing of concurrent operations. Prevented with mutexes, locks.

Q78: What is a mutex?

Mutex (mutual exclusion) ensures only one thread accesses shared resource. Used with pthread_mutex_t.

23. Embedded Systems

Q79: What is volatile keyword in embedded C?

volatile prevents compiler optimization for variables that can change externally (hardware registers, ISR variables). Ensures each access reads memory.

code.c
volatile int *status_reg = (int*)0x4000; while (*status_reg); // Won't be optimized away
Q80: What is memory-mapped I/O?

Memory-mapped I/O maps hardware registers to memory addresses. Accessed like variables. Use volatile for registers.

code.c
#define PORTA (*(volatile unsigned int*)0x25) PORTA = 0xFF; // Write to hardware register
Q81: What is an interrupt service routine (ISR)?

ISR is function called when interrupt occurs. Must be short, not block, use volatile for shared data. Interrupts can be nested.

code.c
volatile int flag = 0; void ISR_Handler(void) { flag = 1; // Set flag in ISR } int main() { while (!flag); // Wait for interrupt return 0; }

24. Debugging

Q82: What is assert macro?

assert checks condition at runtime. Aborts if false. Disabled with NDEBUG. Used for debugging.

code.c
#include assert(ptr != NULL); // Aborts if NULL
Q83: What is the difference between debugger and profiler?

Debugger: Finds bugs, sets breakpoints, inspects state. Profiler: Measures performance, finds bottlenecks.

code.c
// Debugger (gdb) gdb ./program break main run print variable // Profiler (gprof) gcc -pg program.c -o program ./program gprof program gmon.out > analysis.txt

25. Build System

Q84: What is the difference between static and dynamic linking?

Static linking: Libraries included in executable (larger, self-contained). Dynamic linking: Libraries loaded at runtime (smaller, shared).

code.c
// Static linking gcc program.c -o program -lm -static // Dynamic linking gcc program.c -o program -lm
Q85: What is a makefile?

Makefile defines build rules, dependencies. Automates compilation. Only rebuilds changed files.

code.c
CC = gcc CFLAGS = -Wall -g program: main.o utils.o $(CC) $(CFLAGS) -o program main.o utils.o main.o: main.c $(CC) $(CFLAGS) -c main.c utils.o: utils.c $(CC) $(CFLAGS) -c utils.c

26. Miscellaneous

Q86: What is the difference between exit and return?

return exits function. exit terminates program. exit calls cleanup functions.

code.c
int func() { return 0; // Exits function only } int main() { exit(0); // Terminates program return 0; // Never reached }
Q87: What is the difference between abort and exit?

exit calls cleanup functions, normal termination. abort immediate termination, core dump, no cleanup.

code.c
#include exit(0); // Normal termination, calls atexit functions abort(); // Abnormal termination, no cleanup, core dump
Q88: What is the difference between atexit and on_exit?

atexit registers function to call on normal exit. on_exit passes exit status and argument (non-standard).

code.c
#include void cleanup(void) { printf("Cleanup called\n"); } atexit(cleanup); // Called on normal exit
Q89: What is the difference between setjmp and longjmp?

setjmp saves execution state. longjmp restores state (non-local goto). Used for error handling, coroutines.

code.c
#include jmp_buf env; void error_handler() { longjmp(env, 1); // Jump back } int main() { if (setjmp(env) == 0) { error_handler(); // Will jump back } else { printf("Error recovered\n"); } return 0; }
Q90: What is the difference between signal and exception?

Signal: Asynchronous notification from OS. Exception: Synchronous error (C++). C uses signals for errors.

code.c
#include void sig_handler(int signo) { printf("Received signal %d\n", signo); } signal(SIGINT, sig_handler); // Handle Ctrl+C
Q91: What is the difference between fork and exec?

fork creates child process (duplicate). exec replaces process image (new program). Used together for process creation.

code.c
#include #include pid_t pid = fork(); if (pid == 0) { // Child process execl("/bin/ls", "ls", NULL); } else { // Parent process wait(NULL); }
Q92: What is the difference between pipe and socket?

pipe: Inter-process communication (local). socket: Network communication (remote). Both use file descriptors.

code.c
#include int pipefd[2]; pipe(pipefd); // Create pipe (local IPC) // vs socket for network int sockfd = socket(AF_INET, SOCK_STREAM, 0);
Q93: What is the difference between pipe and shared memory?

pipe: Stream of bytes, unidirectional. shared memory: Shared memory region, bidirectional, faster.

code.c
#include int shmid = shmget(IPC_PRIVATE, 1024, IPC_CREAT | 0666); char *shared_mem = shmat(shmid, NULL, 0); // Attach shared memory
Q94: What is the difference between thread and process?

Thread: Shared address space, lightweight. Process: Separate address space, heavyweight. Threads within process share memory.

code.c
#include void *thread_func(void *arg) { return NULL; } pthread_t tid; pthread_create(&tid, NULL, thread_func, NULL); // Create thread pthread_join(tid, NULL); // Wait for thread
Q95: What is the difference between mutex and semaphore?

mutex: Locking mechanism (binary). semaphore: Counting mechanism (can be >1). Semaphore can signal, mutex cannot.

code.c
#include #include pthread_mutex_t mutex; pthread_mutex_init(&mutex, NULL); pthread_mutex_lock(&mutex); // Critical section pthread_mutex_unlock(&mutex); sem_t sem; sem_init(&sem, 0, 1); sem_wait(&sem); // Critical section sem_post(&sem);
Q96: What is the difference between deadlock and starvation?

deadlock: Processes waiting for each other (circular wait). starvation: Process never gets resource (priority inversion).

code.c
// Deadlock example pthread_mutex_t lock1, lock2; // Thread 1 pthread_mutex_lock(&lock1); pthread_mutex_lock(&lock2); // May deadlock // Thread 2 pthread_mutex_lock(&lock2); pthread_mutex_lock(&lock1); // May deadlock
Q97: What is the difference between little-endian and big-endian?

little-endian: Least significant byte first (x86). big-endian: Most significant byte first (network). Affects byte order in memory.

code.c
int x = 0x12345678; // Little-endian: 78 56 34 12 // Big-endian: 12 34 56 78
Q98: What is the difference between struct padding and packing?

padding: Adds bytes for alignment (default). packing: Removes padding (pragma pack). Packing saves space, may slow access.

code.c
#pragma pack(push, 1) // Pack to 1-byte alignment struct Packed { char a; int b; }; // Size: 5 bytes (no padding) #pragma pack(pop) // Restore default alignment
Q99: What is the difference between const char* and char* const?

const char*: Data is const, pointer can change. char* const: Pointer is const, data can change.

code.c
const char *p1 = "hello"; // Data const char *const p2 = str; // Pointer const p1 = "world"; // OK - pointer can change // p1[0] = 'w'; // Error - data is const // p2 = "world"; // Error - pointer is const p2[0] = 'w'; // OK - data can change

C continues to evolve with C23 standard, improved security, better support for modern hardware. Remains essential for systems programming, embedded systems, and as foundation for other languages.

// C23 features #include #include // Improved safety int x[10] = {0}; // Zero initialization _Static_assert(sizeof(int) == 4, "int size"); // Better bit handling std::uint32_t value = 0x1234; std::countl_zero(value); // Count leading zeros
