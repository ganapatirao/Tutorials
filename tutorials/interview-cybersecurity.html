1. Network Security Fundamentals

Q1: What is the CIA Triad in cybersecurity?

CIA Triad stands for Confidentiality, Integrity, and Availability, the three core principles that guide information security.

  • Confidentiality: Prevent unauthorized disclosure (encryption, access controls)
  • Integrity: Prevent unauthorized modification (hashing, digital signatures)
  • Availability: Ensure reliable access (redundancy, failover, DDoS protection)
Q2: Difference between symmetric and asymmetric encryption?

Symmetric uses a single shared key. Asymmetric uses public/private key pairs.

AspectSymmetricAsymmetric
KeysOne sharedPublic/Private pair
SpeedFastSlower
UseBulk encryptionKey exchange, signatures
ExamplesAES, ChaCha20RSA, ECC

using System; using System.Security.Cryptography; using System.Text; // Symmetric encryption with AES public class AesEncryption { public static byte[] Encrypt(string plainText, byte[] key, byte[] iv) { using (var aes = Aes.Create()) { aes.Key = key; aes.IV = iv; var encryptor = aes.CreateEncryptor(); using (var msEncrypt = new MemoryStream()) { using (var csEncrypt = new CryptoStream(msEncrypt, encryptor, CryptoStreamMode.Write)) { using (var swEncrypt = new StreamWriter(csEncrypt)) { swEncrypt.Write(plainText); } return msEncrypt.ToArray(); } } } } }
Q3: What is a firewall and how does it work?

Firewall monitors and filters network traffic based on ordered security rules and connection state.

Types:

  • Packet filtering: IP/port/protocol checks (L3/L4)
  • Stateful inspection: Tracks connection tables to allow related packets
  • Proxy (application): Terminates connections, app-aware policies
  • NGFW: App-ID, IPS, URL filtering, TLS decryption

Example rules:

firewall_rules.sh
# Allow HTTPS inbound and established connections; drop rest iptables -A INPUT -p tcp --dport 443 -j ACCEPT iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT iptables -A INPUT -j DROP
Q4: What is a VPN and why is it important?

VPN creates an encrypted, authenticated tunnel over untrusted networks to protect data-in-transit and enable secure remote access.

Benefits:

  • Confidentiality (strong encryption)
  • Integrity (message authentication)
  • Access control (least-privilege remote access)

Common protocols:

  • WireGuard: Modern, simple, high performance
  • IPsec/IKEv2: Enterprise standard
  • OpenVPN: Mature and flexible

Q8: What is a Man-in-the-Middle (MitM) attack?

Interception/alteration of communications (e.g., ARP spoofing, SSL stripping). Prevent via TLS, HSTS, certificate pinning, VPN.

TlsSocket.cs
using System; using System.Net.Security; using System.Security.Cryptography.X509Certificates; using System.Net.Sockets; public class SecureTcpClient { public static async Task<SslStream> ConnectWithTls(string host, int port) { var tcpClient = new TcpClient(); await tcpClient.ConnectAsync(host, port); var sslStream = new SslStream( tcpClient.GetStream(), false, new RemoteCertificateValidationCallback((sender, certificate, chain, sslPolicyErrors) => { return sslPolicyErrors == SslPolicyErrors.None; }); await sslStream.AuthenticateAsClientAsync(host); return sslStream; } }
Q9: What is network segmentation and why is it important?

Divides networks into isolated segments (VLANs, subnets, micro-segmentation) to reduce blast radius and enforce least privilege.

Q10: What is a Zero-Trust security model?
  • Never trust, always verify
  • Least privilege access
  • Continuous monitoring and micro-segmentation

Hashing.cs
using System; using System.Security.Cryptography; using System.Text; public class PasswordHasher { public static string HashPassword(string password) { using (var sha256 = SHA256.Create()) { byte[] bytes = Encoding.UTF8.GetBytes(password); byte[] hash = sha256.ComputeHash(bytes); return Convert.ToBase64String(hash); } } // Better: Use PBKDF2, bcrypt, or Argon2 for passwords public static string HashPasswordSecure(string password, string salt) { using (var pbkdf2 = new Rfc2898DeriveBytes(password, Encoding.UTF8.GetBytes(salt), 10000, HashAlgorithmName.SHA256, 32)) { byte[] hash = pbkdf2.GetBytes(32); return Convert.ToBase64String(hash); } } }
Q12: Common cryptographic attacks and mitigations?
  • Brute-force/dictionary → strong keys, rate limiting
  • Rainbow tables → salted hashes
  • Side-channel → constant-time ops, shielding
  • MITM → TLS, cert pinning
Q13: What is PKI and its components?

CA, RA, certificate repository, CRL/OCSP; enables key management, TLS, code signing, email signing.

Q14: What is a digital signature and how does it work?
  1. Hash the message
  2. Sign hash with private key
  3. Verify with public key
Q15: SSL vs TLS?

SSL deprecated; use TLS 1.2/1.3, modern ciphers, forward secrecy.

Q18: What is SQL Injection and how to prevent it?
  • Parameterized queries, stored procedures
  • Least-privilege DB accounts, input validation

using System; using System.Data.SqlClient; public class SafeDatabaseAccess { private readonly string _connectionString; public SafeDatabaseAccess(string connectionString) { _connectionString = connectionString; } // Good: Parameterized query (prevents SQL injection) public User GetUserByUsername(string username) { using (var connection = new SqlConnection(_connectionString)) { string query = "SELECT * FROM users WHERE username = @Username"; using (var command = new SqlCommand(query, connection)) { command.Parameters.AddWithValue("@Username", username); connection.Open(); using (var reader = command.ExecuteReader()) { if (reader.Read()) { return new User { Username = reader["username"].ToString() }; } } } } return null; } // Entity Framework example using Microsoft.EntityFrameworkCore; public class UserRepository { private readonly AppDbContext _context; public UserRepository(AppDbContext context) { _context = context; } // LINQ prevents SQL injection public User GetUser(string username) { return _context.Users.FirstOrDefault(u => u.Username == username); } } }

Q20: What is HSTS?

Forces HTTPS for a domain to prevent downgrade and cookie hijacking.

HstsMiddleware.cs
using Microsoft.AspNetCore.Builder; using Microsoft.AspNetCore.Http; public class HstsMiddleware { private readonly RequestDelegate _next; public HstsMiddleware(RequestDelegate next) { _next = next; } public async Task InvokeAsync(HttpContext context) { // Force HTTPS with HSTS header if (context.Request.IsHttps) { context.Response.Headers.Add("Strict-Transport-Security", "max-age=31536000; includeSubDomains; preload"); } else { // Redirect to HTTPS var httpsUrl = new UriBuilder(context.Request.Url) { Scheme = "https", Port = 443 }.Uri; context.Response.Redirect(httpsUrl.ToString()); return; } await _next(context); } } // In Startup.cs or Program.cs for ASP.NET Core public class Startup { public void Configure(IApplicationBuilder app, IWebHostEnvironment env) { if (env.IsProduction()) { app.UseHsts(); } app.UseHttpsRedirection(); } }

Q22: What is output encoding?

Encode untrusted data before rendering to the browser (HTML/JS/URL/CSS encoders).

Q23: Best practices for authentication and authorization?
  • MFA, password policies, credential rotation
  • Use OIDC/OAuth2, short-lived tokens, refresh tokens
  • Least privilege, RBAC/ABAC
Q24: How to store passwords securely?
  • Use strong, adaptive hash (bcrypt/argon2/scrypt) with per-user salt
  • Never store plaintext or reversible encryption
Q25: Secure session management guidelines?
  • HttpOnly, Secure, SameSite cookies; rotate session IDs
  • Short timeouts, server-side invalidation on logout

5. Identity and Access Management (IAM)

Q26: What is AAA and how do Authentication, Authorization, and Accounting differ?

AAA refers to Authentication (verify identity), Authorization (grant least-privilege access), and Accounting (audit usage and changes).

  • Authentication: Passwords, MFA, certificates, biometrics
  • Authorization: RBAC/ABAC policies, scopes, entitlements
  • Accounting: Logs, audit trails, tamper-evident storage
Q27: OAuth 2.0 vs OpenID Connect (OIDC)?

OAuth 2.0 delegates authorization (access to resources). OIDC is an identity layer on top of OAuth 2.0 that adds authentication with ID Tokens (JWT).

  • OAuth: Access Token for APIs; scopes define permissions
  • OIDC: Adds ID Token with user identity claims
Q28: SAML vs JWT (OIDC) for SSO?
  • SAML: XML-based, enterprise SSO, browser redirects, older
  • JWT (OIDC): JSON-based, mobile/web friendly, modern
  • Choose based on ecosystem, tooling, and partner requirements
Q29: What are MFA factors and good MFA practices?
  • Factors: Something you know (password), have (token), are (biometric)
  • Prefer TOTP/app-based or FIDO2 over SMS; implement step-up MFA for risky actions
Q30: RBAC vs ABAC vs ReBAC?
  • RBAC: Role-based; easy but coarse
  • ABAC: Attribute-based (user/resource/context) for fine-grained control
  • ReBAC: Relationship-based (e.g., Google Zanzibar)
Q31: What is PAM and Just-In-Time (JIT) access?

PAM manages privileged accounts; JIT grants short-lived elevated access via approval and audit to reduce standing privileges.

Q32: Passwordless and WebAuthn?

WebAuthn uses public-key cryptography (FIDO2) for phishing-resistant authentication via platform or roaming authenticators.

6. API Security

Q33: What is the OWASP API Security Top 10?

Common API risks include broken object authorization, broken authentication, excessive data exposure, lack of rate limiting, mass assignment, injection, SSRF, and insufficient logging.

Q34: How to implement API rate limiting?

Use gateway or reverse proxy limits to prevent abuse and DDoS at the edge.

nginx_rate_limit.conf
http { limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s; server { location /api/ { limit_req zone=api burst=20 nodelay; } } }
Q35: API gateway vs service mesh?
  • Gateway: North-south traffic, authn/z, rate limiting, WAF
  • Mesh: East-west traffic, mTLS, retries, policies (e.g., Istio)
Q36: JWT best practices?
  • Short-lived access tokens, refresh tokens, audience/issuer checks
  • Rotate signing keys; validate exp, iat, nbf

using System; using System.IdentityModel.Tokens.Jwt; using Microsoft.IdentityModel.Tokens; public class JwtValidator { public static ClaimsPrincipal ValidateToken(string token, string publicKey) { var tokenHandler = new JwtSecurityTokenHandler(); var key = Encoding.ASCII.GetBytes(publicKey); var validationParameters = new TokenValidationParameters { ValidateIssuerSigningKey = true, IssuerSigningKey = new SymmetricSecurityKey(key), ValidateIssuer = true, ValidIssuer = "https://issuer.example.com/", ValidateAudience = true, ValidAudience = "api://default", ValidateLifetime = true, ClockSkew = TimeSpan.Zero }; return tokenHandler.ValidateToken(token, validationParameters, out SecurityToken validatedToken); } }
Q37: HMAC request signing for APIs?

Compute a signature over canonical request bytes with a shared secret.

HmacSigner.cs
using System; using System.Security.Cryptography; using System.Text; public class HmacSigner { public static string ComputeSignature(string secret, string message) { using (var hmac = new HMACSHA256(Encoding.UTF8.GetBytes(secret))) { byte[] hashBytes = hmac.ComputeHash(Encoding.UTF8.GetBytes(message)); return Convert.ToBase64String(hashBytes); } } public static bool VerifySignature(string secret, string message, string signature) { var computedSignature = ComputeSignature(secret, message); return computedSignature == signature; } }

7. Secure SDLC & DevSecOps

Q38: SAST vs DAST vs SCA vs IAST?
  • SAST: Static code analysis (early, build time)
  • DAST: Black-box testing of running app
  • SCA: Dependency scanning for known CVEs
  • IAST: Hybrid (instrumented runtime)
Q39: What is shift-left security and why?

Embed security earlier to reduce rework and cost; run checks in CI, enforce policy as code, provide developer guardrails.

Q40: How should secrets be managed?
  • Use secret stores (Vault, cloud KMS/SM), not .env in source control
  • Rotate regularly; use workload identity (OIDC) over static keys
SecretManagement.cs
using System; using Azure.Security.KeyVault; using Azure.Identity; public class SecretManager { private readonly SecretClient _secretClient; public SecretManager(string keyVaultUrl) { var credential = new DefaultAzureCredential(); _secretClient = new SecretClient(new Uri(keyVaultUrl), credential); } public async Task<string> GetSecretAsync(string secretName) { try { var secret = await _secretClient.GetSecretAsync(secretName); return secret.Value.Value; } catch (Azure.RequestFailedException ex) { throw new InvalidOperationException("Failed to retrieve secret", ex); } } // Using environment variables for local development public static string GetSecretFromEnvironment(string variableName) { var secret = Environment.GetEnvironmentVariable(variableName); if (string.IsNullOrEmpty(secret)) { throw new InvalidOperationException($"Secret missing: {variableName}"); } return secret; } }
Q41: What is an SBOM and why does it matter?

SBOM (Software Bill of Materials) inventories components to manage supply-chain risk, enable CVE impact analysis, and meet compliance (e.g., Executive Order 14028).

8. Logging, Monitoring & SIEM

Q42: What should be logged, and how long to retain logs?
  • Auth events, admin actions, access denials, data exports, errors
  • Minimize sensitive data, use structured logs, retain per policy/regulation
Q43: Centralized logging architecture?

Ship logs to centralized lake (e.g., ELK/OpenSearch, Loki) via agents; enrich with metadata; index for search and analytics.

9. Vulnerability Management & Threat Modeling

Q45: What is CVSS and how is risk prioritized?

CVSS scores severity; combine with exploitability, exposure, and business impact to prioritize remediation SLAs.

Q46: Patch and exception management?

Track assets, apply risk-based patching; document time-bound exceptions with compensating controls and approvals.

Q47: What is STRIDE threat modeling?
  • STRIDE: Spoofing, Tampering, Repudiation, Information disclosure, Denial of service, Elevation of privilege
  • Use data-flow diagrams to enumerate threats and mitigations
