1. EF Fundamentals

Q1: What is Entity Framework?

Entity Framework (EF) is an Object-Relational Mapper (ORM) for .NET. It enables developers to work with relational data using domain-specific objects, eliminating the need for most data-access code.

Key Benefits:

  • Productivity: Reduces data access code
  • Maintainability: Less code to maintain
  • Abstraction: Works with objects, not database
  • Flexibility: Supports multiple databases
Q2: What is the difference between EF6 and EF Core?

EF6 is for .NET Framework, while EF Core is for .NET Core/.NET 5+. EF Core is cross-platform, faster, and more lightweight.

EF Core Package
// EF Core Installation dotnet add package Microsoft.EntityFrameworkCore.SqlServer
Q3: What is ORM?

ORM (Object-Relational Mapping) is a technique that maps object-oriented models to relational databases. It allows developers to work with objects instead of SQL.

Program.cs
// Without ORM - SQL string sql = "SELECT * FROM Students WHERE Age > 18"; // With ORM - LINQ var students = context.Students.Where(s => s.Age > 18);
Q4: What are the approaches in Entity Framework?

Entity Framework provides three main approaches for working with databases. Each approach offers different workflows depending on your project requirements and existing infrastructure.

EF Approaches:

  • Database First: Generate model from existing database. Best when you have a legacy database and need to build an application around it.
  • Model First: Design model visually using Entity Designer, then generate database. Useful for new projects where you prefer visual modeling.
  • Code First: Write code classes to define your model, then generate database. Most popular approach for new projects with version control support.
Models/Student.cs
// Code First Approach Example public class Student { public int Id { get; set; } public string Name { get; set; } }
Q5: What is Code First approach?

Code First allows you to define your model using C# classes. EF creates the database schema based on these classes.

Models/Student.cs
public class Student { public int Id { get; set; } public string Name { get; set; } public string Email { get; set; } }
Q6: What is Database First approach?

Database First generates entity classes from an existing database. It's useful when you have an existing database schema.

Terminal
dotnet ef dbcontext scaffold "Server=.;Database=MyDb;Trusted_Connection=True;" Microsoft.EntityFrameworkCore.SqlServer
Q7: What is Model First approach?

Model First allows you to design your model visually using the Entity Designer. The database is generated from the model. This approach is not available in EF Core.

This approach was popular in EF6 where developers could drag and drop entities on a visual designer surface. The designer would then generate the database schema and entity classes automatically.

Model First is ideal for teams that prefer visual modeling over code-first development. However, it's been deprecated in EF Core in favor of Code First approach which provides better version control integration.

Note
// Model First is only available in EF6, not EF Core // Use Code First in EF Core instead public class Student { public int Id { get; set; } public string Name { get; set; } }
Q8: What is DbContext?

DbContext is the primary class for interacting with the database. It manages entity objects during runtime and handles database connections.

Data/SchoolContext.cs
public class SchoolContext : DbContext { public DbSet<Student> Students { get; set; } }
Q9: What is DbSet?

DbSet represents a collection of entities that can be queried from the database. It corresponds to a table in the database.

Data/SchoolContext.cs
public DbSet<Student> Students { get; set; } public DbSet<Course> Courses { get; set; }
Q10: What is an Entity?

Entity is a class that represents a table in the database. Each instance of the entity corresponds to a row in the table.

Models/Student.cs
public class Student { public int Id { get; set; } public string Name { get; set; } }

2. EF Core Basics

Q11: What is Entity Framework Core?

Entity Framework Core is a lightweight, extensible, and cross-platform version of Entity Framework. It works with .NET Core and .NET 5+.

Program.cs
using Microsoft.EntityFrameworkCore; public class MyContext : DbContext { public DbSet<User> Users { get; set; } }
Q12: How do you install EF Core?

EF Core is installed via NuGet packages. You need to install the core package along with a database provider package for your specific database.

The main package is Microsoft.EntityFrameworkCore which provides the core functionality. Additionally, you need a database-specific provider like SqlServer, PostgreSQL, or SQLite depending on your database.

For migrations and scaffolding, you also need the Microsoft.EntityFrameworkCore.Tools package. This package provides CLI commands for database operations and model generation.

Terminal
// Install EF Core Core Package dotnet add package Microsoft.EntityFrameworkCore // Install Database Provider (SQL Server) dotnet add package Microsoft.EntityFrameworkCore.SqlServer // Install Tools for Migrations dotnet add package Microsoft.EntityFrameworkCore.Tools
Q13: What is connection string?

Connection String contains information to connect to the database. It includes server name, database name, authentication details.

appsettings.json
"ConnectionStrings": { "DefaultConnection": "Server=(localdb)\\mssqllocaldb;Database=MyDb;Trusted_Connection=True;" }
Q14: How do you configure DbContext in EF Core?

DbContext is configured in the dependency injection container using AddDbContext method. This allows EF Core to manage the DbContext lifecycle automatically.

You specify the database provider using methods like UseSqlServer, UseSqlite, or UseNpgsql. The connection string is typically retrieved from configuration files like appsettings.json.

DbContext can be registered as scoped, transient, or singleton depending on your application needs. Scoped is the default and recommended for web applications.

Program.cs
// Configure DbContext with SQL Server builder.Services.AddDbContext<ApplicationDbContext>(options => options.UseSqlServer( builder.Configuration.GetConnectionString("DefaultConnection"))); // Configure DbContext with SQLite builder.Services.AddDbContext<ApplicationDbContext>(options => options.UseSqlite("Data Source=app.db"));
Q15: What is OnModelCreating?

OnModelCreating is used to configure the model using Fluent API. It's called when the model is being created.

Data/ApplicationDbContext.cs
protected override void OnModelCreating(ModelBuilder modelBuilder) { modelBuilder.Entity<Student>() .Property(s => s.Name) .IsRequired() .HasMaxLength(100); }
Q16: What is Fluent API?

Fluent API provides a way to configure model properties and relationships. It offers more configuration options than data annotations.

Data/ApplicationDbContext.cs
modelBuilder.Entity<Student>() .ToTable("Students") .HasKey(s => s.Id) .Property(s => s.Name) .IsRequired() .HasMaxLength(100);
Q17: What are Data Annotations?

Data Annotations are attributes used to configure model classes. They provide a simple way to configure the model.

Models/Student.cs
public class Student { public int Id { get; set; } [Required] [MaxLength(100)] public string Name { get; set; } }
Q18: What is lazy loading?

Lazy Loading delays loading of related data until it's accessed. It requires virtual navigation properties and proxy creation.

Program.cs
builder.Services.AddDbContext<BloggingContext>(options => options.UseLazyLoadingProxies());
Q19: What is eager loading?

Eager Loading loads related entities as part of the initial query using Include() method.

Controllers/StudentController.cs
var students = context.Students .Include(s => s.Enrollments) .ToList();
Q20: What is explicit loading?

Explicit Loading loads related entities manually using Load() method after the entity is loaded.

Controllers/StudentController.cs
context.Entry(student) .Collection(s => s.Enrollments) .Load();

3. DbContext and Models

Q21: What is the purpose of DbContext?

DbContext serves as the bridge between your domain classes and the database. It handles database connections, change tracking, and query execution.

DbContext manages the entity lifecycle including tracking changes, executing queries, and persisting changes to the database. It also manages database connections and transactions automatically.

Each DbContext instance represents a unit of work and should be short-lived. In web applications, it's typically registered as scoped in the dependency injection container.

Data/ApplicationDbContext.cs
public class ApplicationDbContext : DbContext { public ApplicationDbContext(DbContextOptions<ApplicationDbContext> options) : base(options) { } public DbSet<User> Users { get; set; } public DbSet<Order> Orders { get; set; } }
Q22: How do you create a DbContext?
Data/ApplicationDbContext.cs
public class ApplicationDbContext : DbContext { public ApplicationDbContext(DbContextOptions<ApplicationDbContext> options) : base(options) { } public DbSet<User> Users { get; set; } public DbSet<Order> Orders { get; set; } }
Q23: What is the difference between Add, Attach, and Update?

DbContext Methods:

  • Add: Marks entity as Added (new)
  • Attach: Marks entity as Unchanged (existing)
  • Update: Marks entity as Modified
Controllers/StudentController.cs
context.Students.Add(student); // New entity context.Students.Attach(student); // Existing, unchanged context.Students.Update(student); // Existing, modified
Q24: What is Change Tracking?

Change Tracking tracks changes made to entities. EF Core detects which properties have changed and generates appropriate SQL statements.

Controllers/StudentController.cs
var entry = context.Entry(student); var state = entry.State; // Added, Modified, Deleted, etc. var modifiedProps = entry.Properties.Where(p => p.IsModified);
Q25: What is EntityState?

EntityState represents the state of an entity being tracked by the context. States include Added, Modified, Deleted, Unchanged, and Detached.

Controllers/StudentController.cs
var state = context.Entry(student).State; // Possible values: Added, Modified, Deleted, Unchanged, Detached
Q26: What is AsNoTracking?

AsNoTracking disables change tracking for read-only queries. It improves performance when you don't need to modify entities.

Controllers/StudentController.cs
var students = context.Students .AsNoTracking() .ToList();
Q27: What is Find method?

Find retrieves an entity by its primary key. It first checks the context before querying the database.

Controllers/StudentController.cs
var student = context.Students.Find(1);
Q28: What is the difference between FromSqlRaw and FromSqlInterpolated?

FromSqlRaw accepts raw SQL strings with parameters. FromSqlInterpolated uses string interpolation for safer parameter handling.

Controllers/StudentController.cs
// FromSqlRaw var students = context.Students .FromSqlRaw("EXEC GetStudents @Grade", gradeParam); // FromSqlInterpolated var students = context.Students .FromSqlInterpolated($"EXEC GetStudents {grade}");
Q29: What is the difference between ToListAsync and ToArrayAsync?

ToListAsync returns a List. ToArrayAsync returns an array. Both execute the query asynchronously.

Controllers/StudentController.cs
var list = await context.Students.ToListAsync(); var array = await context.Students.ToArrayAsync();
Q30: What is SaveChanges?

SaveChanges saves all changes made to entities in the context to the database. It returns the number of entities written.

Controllers/StudentController.cs
context.Students.Add(student); await context.SaveChangesAsync();

4. Querying Data

Q31: How do you query data in EF Core?
Controllers/StudentController.cs
var students = context.Students .Where(s => s.Age > 18) .OrderBy(s => s.Name) .ToList();
Q32: What is the difference between IEnumerable and IQueryable?

IEnumerable executes in-memory. IQueryable executes in database with SQL translation. IQueryable is more efficient for database queries.

Controllers/StudentController.cs
// IQueryable - SQL translation var query = context.Students.Where(s => s.Age > 18); // IEnumerable - in-memory var list = query.ToList();
Q33: What is deferred execution?

Deferred Execution means the query is not executed until you iterate over the results or call a method like ToList().

Controllers/StudentController.cs
var query = context.Students.Where(s => s.Age > 18); // Not executed var list = query.ToList(); // Executed now
Q34: What is immediate execution?

Immediate Execution executes the query immediately. Methods like ToList(), Count(), First() trigger immediate execution.

Controllers/StudentController.cs
var count = context.Students.Count(); // Immediate var first = context.Students.First(); // Immediate
Q35: What is the difference between First and FirstOrDefault?

First throws exception if no match. FirstOrDefault returns null if no match. Use FirstOrDefault for safer queries.

Controllers/StudentController.cs
var student = context.Students.FirstOrDefault(s => s.Id == 1);
Q36: What is the difference between Single and SingleOrDefault?

Single expects exactly one match. SingleOrDefault returns null if no match. Both throw if multiple matches found.

Controllers/StudentController.cs
var student = context.Students.SingleOrDefault(s => s.Id == 1);
Q37: What is the difference between Where and Find?

Where filters based on a predicate. Find retrieves by primary key and checks the context cache first.

Controllers/StudentController.cs
// Find - checks cache first var student = context.Students.Find(1); // Where - always queries database var student = context.Students.Where(s => s.Id == 1).FirstOrDefault();
Q38: What is the difference between Take and Skip?

Take returns the first n elements. Skip skips the first n elements. Used together for pagination.

Controllers/StudentController.cs
var page2 = context.Students .Skip(10) .Take(10) .ToList();
Q39: What is the difference between OrderBy and ThenBy?

OrderBy is the primary sort. ThenBy is used for secondary sorting after OrderBy.

Controllers/StudentController.cs
var students = context.Students .OrderBy(s => s.Name) .ThenBy(s => s.Age) .ToList();
Q40: What is GroupBy in LINQ to Entities?

GroupBy groups elements based on a key value. Returns groups where each group has a key and elements.

Controllers/StudentController.cs
var grouped = context.Students .GroupBy(s => s.Grade) .Select(g => new { Grade = g.Key, Count = g.Count() }) .ToList();

5. CRUD Operations

Q41: How do you add a new entity?
Controllers/StudentController.cs
var student = new Student { Name = "John", Email = "john@example.com" }; context.Students.Add(student); await context.SaveChangesAsync();
Q42: How do you update an entity?
Controllers/StudentController.cs
var student = context.Students.Find(1); student.Name = "Updated Name"; await context.SaveChangesAsync();
Q43: How do you delete an entity?
Controllers/StudentController.cs
var student = context.Students.Find(1); context.Students.Remove(student); await context.SaveChangesAsync();
Q44: What is the difference between Remove and Delete?

Remove marks an entity for deletion. Delete is not a direct EF method; you use Remove or ExecuteDelete for bulk operations.

Controllers/StudentController.cs
// Remove - marks for deletion context.Students.Remove(student); // ExecuteDelete - bulk delete await context.Students.Where(s => s.Id == 1).ExecuteDeleteAsync();
Q45: What is ExecuteUpdate?

ExecuteUpdate performs bulk updates directly in SQL without loading entities. More efficient for large updates.

Controllers/StudentController.cs
await context.Students .Where(s => s.Grade == "A") .ExecuteUpdateAsync(s => s.SetProperty(x => x.Status, "Honors"));
Q46: What is ExecuteDelete?

ExecuteDelete performs bulk deletes directly in SQL without loading entities. More efficient for large deletions.

Controllers/StudentController.cs
await context.Students .Where(s => s.Age < 18) .ExecuteDeleteAsync();
Q47: What is the difference between AddRange and Add?

Add adds a single entity. AddRange adds multiple entities in one operation, more efficient for bulk inserts.

Controllers/StudentController.cs
// AddRange - bulk insert context.Students.AddRange(students); await context.SaveChangesAsync();
Q48: What is the difference between UpdateRange and Update?

Update updates a single entity. UpdateRange updates multiple entities in one operation.

Controllers/StudentController.cs
context.Students.UpdateRange(students); await context.SaveChangesAsync();
Q49: What is the difference between RemoveRange and Remove?

Remove removes a single entity. RemoveRange removes multiple entities in one operation.

Controllers/StudentController.cs
context.Students.RemoveRange(students); await context.SaveChangesAsync();
Q50: What is HasData?

HasData is used in migrations to seed initial data into tables. It's configured in OnModelCreating.

Data/ApplicationDbContext.cs
modelBuilder.Entity<Student>().HasData( new Student { Id = 1, Name = "John" }, new Student { Id = 2, Name = "Jane" } );

6. Relationships

Q51: What are the types of relationships in EF Core?

EF Core supports three main types of relationships between entities. These relationships define how entities are related to each other in the database schema.

Relationship Types:

  • One-to-One: Each entity in A relates to exactly one in B. Useful for splitting large tables or optional data.
  • One-to-Many: Each entity in A relates to many in B. Most common relationship type, like a category with many products.
  • Many-to-Many: Entities in A relate to many in B and vice versa. Requires a join table in the database.
Models/Student.cs
// One-to-Many Relationship public class Grade { public int Id { get; set; } public List<Student> Students { get; set; } } public class Student { public int Id { get; set; } public int GradeId { get; set; } public Grade Grade { get; set; } }
Q52: How do you configure a one-to-many relationship?

One-to-Many relationship is configured using navigation properties and foreign keys. EF Core can automatically detect this relationship by convention.

The principal entity has a collection navigation property, while the dependent entity has a reference navigation property and a foreign key property. This is the most common relationship type.

You can configure this relationship using Fluent API for more control over cascade delete behavior, foreign key constraints, and other relationship settings.

Models/Student.cs
public class Student { public int Id { get; set; } public string Name { get; set; } public int GradeId { get; set; } public Grade Grade { get; set; } } // Fluent API Configuration modelBuilder.Entity<Student>() .HasOne(s => s.Grade) .WithMany(g => g.Students) .HasForeignKey(s => s.GradeId);
Q53: How do you configure a many-to-many relationship?

Many-to-Many relationship is configured using collection navigation properties on both entities. EF Core automatically creates a join table to manage this relationship.

In EF Core 5.0 and later, many-to-many relationships are supported natively without requiring an explicit join entity. The join table is created automatically with foreign keys to both entities.

For earlier versions or when you need additional properties on the join table, you must create an explicit join entity and configure two separate one-to-many relationships.

Models/Student.cs
public class Student { public int Id { get; set; } public List<Course> Courses { get; set; } } public class Course { public int Id { get; set; } public List<Student> Students { get; set; } } // Fluent API Configuration (optional) modelBuilder.Entity<Student>() .HasMany(s => s.Courses) .WithMany(c => c.Students);
Q54: What is a navigation property?

Navigation Property defines a relationship between entities. It can be a reference (single entity) or collection (multiple entities).

Models/Student.cs
public class Student { public int Id { get; set; } public Grade Grade { get; set; } // Reference public List<Enrollment> Enrollments { get; set; } // Collection }
Q55: What is foreign key?

Foreign Key is a property that references the primary key of another entity. It establishes the relationship between tables.

Models/Student.cs
public class Student { public int Id { get; set; } public int GradeId { get; set; } // Foreign Key public Grade Grade { get; set; } // Navigation }
Q56: What is the difference between required and optional navigation?

Required navigation means the relationship is mandatory (non-nullable FK). Optional allows null FK values.

Data/ApplicationDbContext.cs
// Required modelBuilder.Entity<Student>() .HasOne(s => s.Grade) .WithOne() .IsRequired(); // Optional modelBuilder.Entity<Student>() .HasOne(s => s.Grade) .WithOne() .IsRequired(false);
Q57: What is cascade delete?

Cascade Delete automatically deletes related entities when the parent entity is deleted. Configured via Fluent API.

Data/ApplicationDbContext.cs
modelBuilder.Entity<Student>() .HasOne(s => s.Grade) .WithMany() .OnDelete(DeleteBehavior.Cascade);
Q58: What is the difference between HasOne and HasMany?

HasOne defines a single relationship. HasMany defines a collection relationship in Fluent API configuration.

Data/ApplicationDbContext.cs
// HasOne - single relationship modelBuilder.Entity<Student>() .HasOne(s => s.Grade); // HasMany - collection relationship modelBuilder.Entity<Grade>() .HasMany(g => g.Students);
Q59: What is WithOne and WithMany?

WithOne specifies the inverse is a single entity. WithMany specifies the inverse is a collection.

Data/ApplicationDbContext.cs
modelBuilder.Entity<Student>() .HasOne(s => s.Grade) .WithMany(g => g.Students);
Q60: What is a shadow property?

Shadow Property is a property that doesn't exist in the entity class but is defined in the EF Core model. Useful for audit fields.

Data/ApplicationDbContext.cs
modelBuilder.Entity<Student>() .Property<DateTime>("LastUpdated");

7. Migrations

Q61: What are migrations in EF Core?

Migrations are a way to incrementally update the database schema to keep it in sync with the EF Core model.

Terminal
dotnet ef migrations add InitialCreate dotnet ef database update
Q62: How do you create a migration?
Terminal
dotnet ef migrations add InitialCreate
Q63: How do you apply migrations?
Terminal
dotnet ef database update
Q64: How do you remove the last migration?
Terminal
dotnet ef migrations remove
Q65: How do you revert a migration?
Terminal
dotnet ef database update PreviousMigrationName
Q66: What is the difference between Add-Migration and Update-Database?

Add-Migration creates a migration file based on model changes. Update-Database applies migrations to the database.

Q67: What is the difference between Drop-Database and Update-Database 0?

Drop-Database completely drops the database. Update-Database 0 reverts all migrations, keeping the database structure.

Q68: What is a migration snapshot?

Migration Snapshot is a file that represents the current state of the model. It's used to generate new migrations by comparing with the current model.

Q69: How do you generate SQL from migrations?
Terminal
dotnet ef migrations script
Q70: What is the difference between automatic migrations and manual migrations?

Automatic migrations (not in EF Core) would auto-generate and apply. Manual migrations require explicit creation and application commands.

8. Performance Optimization

Q71: What is AsNoTracking used for?

AsNoTracking improves performance for read-only queries by disabling change tracking overhead.

Controllers/StudentController.cs
var students = context.Students .AsNoTracking() .ToList();
Q72: What is the difference between eager loading and lazy loading for performance?

Eager loading is generally better for performance as it executes a single query. Lazy loading can cause N+1 query problems.

Controllers/StudentController.cs
// Eager Loading - Single query var students = context.Students .Include(s => s.Enrollments) .ToList();
Q73: What is the N+1 problem?

N+1 problem occurs when lazy loading executes one query for the parent and N queries for related entities. Use eager loading to avoid.

Controllers/StudentController.cs
// N+1 Problem - Bad var students = context.Students.ToList(); foreach (var student in students) { // N additional queries var enrollments = student.Enrollments.ToList(); } // Solution - Eager Loading var students = context.Students .Include(s => s.Enrollments) .ToList();
Q74: What is splitting queries?

Split Queries splits a single LINQ query into multiple SQL queries. Useful for complex queries with many includes.

Controllers/StudentController.cs
var students = context.Students .Include(s => s.Enrollments) .AsSplitQuery() .ToList();
Q75: What is query caching?

Query Caching caches the translation of LINQ queries to SQL. EF Core automatically caches compiled queries for better performance.

Controllers/StudentController.cs
// EF Core automatically caches this query translation var students = context.Students.Where(s => s.Age > 18).ToList();
Q76: What is the difference between FromSql and LINQ?

FromSql executes raw SQL queries. LINQ uses C# syntax translated to SQL. LINQ is type-safe and composable.

Controllers/StudentController.cs
// FromSql - Raw SQL var students = context.Students .FromSqlRaw("SELECT * FROM Students WHERE Age > 18"); // LINQ - Type-safe var students = context.Students.Where(s => s.Age > 18);
Q77: What is the difference between DbContext pooling and standard DbContext?

DbContext Pooling reuses DbContext instances instead of creating new ones. Improves performance in high-throughput scenarios.

Program.cs
builder.Services.AddDbContextPool<ApplicationDbContext>(options => options.UseSqlServer(connectionString), 128);
Q78: What is the difference between AddDbContext and AddDbContextFactory?

AddDbContext registers DbContext as scoped. AddDbContextFactory registers a factory for creating DbContext instances manually.

Program.cs
// AddDbContext - Scoped builder.Services.AddDbContext<ApplicationDbContext>(); // AddDbContextFactory - Factory pattern builder.Services.AddDbContextFactory<ApplicationDbContext>();
Q79: What is the difference between index and clustered index?

Index improves query performance on specific columns. Clustered Index determines physical storage order of data.

Data/ApplicationDbContext.cs
// Non-clustered index modelBuilder.Entity<Student>() .HasIndex(s => s.Name); // Clustered index (default on PK) modelBuilder.Entity<Student>() .HasKey(s => s.Id);
Q80: How do you add an index in EF Core?
Data/ApplicationDbContext.cs
modelBuilder.Entity<Student>() .HasIndex(s => s.Name);

9. Advanced Topics

Q81: What is a complex type?

Complex Type (Owned Entity) is a type that doesn't have its own identity and is mapped to the same table as the owner.

Models/Address.cs
public class Address { public string Street { get; set; } public string City { get; set; } } public class Student { public Address Address { get; set; } }
Q82: What is the difference between owned entity and table splitting?

Owned Entity maps to the same table. Table Splitting maps multiple entities to the same table with different columns.

Data/ApplicationDbContext.cs
// Owned Entity modelBuilder.Entity<Student>() .OwnsOne(s => s.Address); // Table Splitting modelBuilder.Entity<Student>() .ToTable("Students"); modelBuilder.Entity<StudentDetails>() .ToTable("Students");
Q83: What is table-per-hierarchy inheritance?

TPH stores all hierarchy levels in a single table with a discriminator column to identify the type.

Data/ApplicationDbContext.cs
modelBuilder.Entity<Person>() .HasDiscriminator<string>("PersonType") .HasValue<Student>("Student") .HasValue<Teacher>("Teacher");
Q84: What is table-per-type inheritance?

TPT stores each type in its own table, with base properties in the base table and specific properties in derived tables.

Data/ApplicationDbContext.cs
modelBuilder.Entity<Student>() .ToTable("Students"); modelBuilder.Entity<Teacher>() .ToTable("Teachers");
Q85: What is global query filters?

Global Query Filters apply filters automatically to all queries for an entity type. Useful for soft delete or multi-tenancy.

Data/ApplicationDbContext.cs
modelBuilder.Entity<Student>() .HasQueryFilter(s => !s.IsDeleted);
Q86: What is the difference between HasQueryFilter and Where?

HasQueryFilter applies globally to all queries. Where is applied per query and can be overridden.

Data/ApplicationDbContext.cs
// Global filter - applies to all queries modelBuilder.Entity<Student>() .HasQueryFilter(s => !s.IsDeleted); // Where - per query var students = context.Students.Where(s => s.Age > 18);
Q87: What is concurrency in EF Core?

Concurrency handles situations where multiple users try to update the same data simultaneously. Uses RowVersion or Timestamp columns.

Models/Student.cs
public class Student { public byte[] RowVersion { get; set; } }
Q88: How do you handle concurrency conflicts?
Models/Student.cs
public class Student { public byte[] RowVersion { get; set; } } // In DbContext modelBuilder.Entity<Student>() .Property(s => s.RowVersion) .IsRowVersion();
Q89: What is the difference between optimistic and pessimistic concurrency?

Optimistic assumes no conflicts and checks on save. Pessimistic locks records during read (not natively supported in EF Core).

Controllers/StudentController.cs
try { await context.SaveChangesAsync(); } catch (DbUpdateConcurrencyException ex) { // Handle concurrency conflict }
Q90: What is value conversion?

Value Conversion converts property values between the entity and database. Useful for enums, encrypted data, or custom types.

Data/ApplicationDbContext.cs
modelBuilder.Entity<Student>() .Property(s => s.Status) .HasConversion( v => v.ToString(), v => (Status)Enum.Parse(typeof(Status), v));
Q91: What is the difference between HasConversion and ValueConverter?

HasConversion is a shorthand for simple conversions. ValueConverter provides more control for complex conversions.

Data/ApplicationDbContext.cs
// HasConversion - Simple modelBuilder.Entity<Student>() .Property(s => s.Status) .HasConversion(v => v.ToString(), v => (Status)Enum.Parse(typeof(Status), v)); // ValueConverter - Complex var converter = new ValueConverter<Status, string>( v => v.ToString(), v => (Status)Enum.Parse(typeof(Status), v));
Q92: What is the difference between EF Core and Dapper?

EF Core is a full ORM with change tracking and LINQ. Dapper is a micro-ORM that's faster but requires manual SQL and mapping.

Controllers/StudentController.cs
// EF Core - LINQ var students = context.Students.Where(s => s.Age > 18).ToList(); // Dapper - Raw SQL var students = connection.Query<Student>("SELECT * FROM Students WHERE Age > 18");
Q93: What is the difference between EF Core and ADO.NET?

EF Core provides high-level abstraction with LINQ. ADO.NET is low-level with manual SQL and connection management.

Controllers/StudentController.cs
// EF Core - High-level var students = context.Students.ToList(); // ADO.NET - Low-level using (var cmd = new SqlCommand("SELECT * FROM Students", connection)) { using (var reader = await cmd.ExecuteReaderAsync()) { // Manual mapping } }
Q94: What is the difference between DbContext and ObjectContext?

DbContext is the simplified API in EF Core/EF6. ObjectContext is the older, more complex API from EF.

Program.cs
// DbContext - Modern API public class MyContext : DbContext { public DbSet<Student> Students { get; set; } } // ObjectContext - Legacy API var context = new ObjectContext(connectionString);
Q95: What is the difference between LINQ to Entities and LINQ to Objects?

LINQ to Entities translates to SQL and executes in database. LINQ to Objects executes in-memory on collections.

Controllers/StudentController.cs
// LINQ to Entities - Database var students = context.Students.Where(s => s.Age > 18); // LINQ to Objects - In-memory var adults = students.Where(s => s.Name.Length > 5);
Q96: What is the difference between Include and ThenInclude?

Include loads the first level of related entities. ThenInclude loads nested levels of related entities.

Controllers/StudentController.cs
var students = context.Students .Include(s => s.Enrollments) .ThenInclude(e => e.Course) .ToList();
Q97: What is the difference between Include and Select for performance?

Include loads full entities. Select can project only needed fields, which is more efficient for large entities.

Controllers/StudentController.cs
// Include - Loads full entity var students = context.Students .Include(s => s.Enrollments) .ToList(); // Select - Projects only needed fields var studentNames = context.Students .Select(s => new { s.Name, s.Email }) .ToList();
Q98: What is the difference between HasDefaultSchema and ToTable?

HasDefaultSchema sets the default schema for all entities. ToTable specifies the table name for a specific entity.

Data/ApplicationDbContext.cs
// HasDefaultSchema - For all entities modelBuilder.HasDefaultSchema("dbo"); // ToTable - For specific entity modelBuilder.Entity<Student>() .ToTable("Students");
Q99: What is the difference between HasDefaultValue and HasDefaultValueSql?

HasDefaultValue sets a .NET default value. HasDefaultValueSql sets a SQL default value expression.

Data/ApplicationDbContext.cs
// HasDefaultValue - .NET value modelBuilder.Entity<Student>() .Property(s => s.Age) .HasDefaultValue(18); // HasDefaultValueSql - SQL expression modelBuilder.Entity<Student>() .Property(s => s.CreatedDate) .HasDefaultValueSql("GETUTCDATE()");
Q100: What is the difference between ConfigureWarnings and Ignore?

ConfigureWarnings configures how EF Core handles warnings. Ignore prevents a property from being mapped to the database.

Data/ApplicationDbContext.cs
// ConfigureWarnings - Handle warnings modelBuilder.ConfigureWarnings(w => w .Default(WarningBehavior.Log)); // Ignore - Don't map property modelBuilder.Entity<Student>() .Ignore(s => s.ComputedProperty);
