1. Lit Fundamentals

Q1: What is Lit and what are its key features?

Lit is a lightweight library for building fast, lightweight web components. It provides a simple base class for creating custom elements with reactive state and efficient rendering.

Key Features:

  • Lightweight: Small bundle size (~5KB)
  • Fast: Efficient rendering with declarative templates
  • Web Components: Built on Web Components standards
  • Reactive: Automatic updates when properties change
  • TypeScript support: First-class TypeScript support
  • Framework-agnostic: Works with any framework
  • Declarative templates: HTML template literals
Q2: What are Web Components?

Web Components are a set of web platform APIs that allow you to create new custom, reusable, encapsulated HTML tags to use in web pages and web apps.

  • Custom Elements: Define new HTML elements
  • Shadow DOM: Encapsulate DOM and styles
  • HTML Templates: Define reusable markup
  • ES Modules: Import and export JavaScript

2. Lit Components

Q3: How do you create a Lit component?

Lit components extend LitElement and define reactive properties, styles, and a render method that returns a template.

my-element.js
import { LitElement, html, css } from 'lit'; class MyElement extends LitElement { static styles = css`:host { display: block; padding: 16px; }`; static properties = { name: { type: String }, count: { type: Number } }; constructor() { super(); this.name = 'World'; this.count = 0; } render() { return html` <h1>Hello, ${this.name}!</h1> <p>Count: ${this.count}</p> <button @click="${this._increment}">Increment</button> `; } _increment() { this.count++; } } customElements.define('my-element', MyElement);
Q4: What are the different property types in Lit?

Property types in Lit define how properties are converted from attributes and how they trigger updates.

Type Description
String String values
Number Number values
Boolean Boolean values
Array Array values
Object Object values
Function Function callbacks
properties.js
static properties = { name: { type: String }, count: { type: Number }, active: { type: Boolean }, items: { type: Array }, config: { type: Object }, onChange: { type: Function } };

3. Reactive Properties

Q5: How do reactive properties work in Lit?

Reactive properties automatically trigger re-renders when they change. Lit tracks property changes, schedules updates efficiently, batches multiple changes, and updates only changed DOM parts.

reactive.js
static properties = { items: { type: Array }, selected: { type: Number } }; render() { return html` <ul> ${this.items.map((item, index) => html` <li class="${index === this.selected ? 'selected' : ''}">${item.name}</li> `)} </ul> `; }
Q6: What is the difference between internal state and properties?

Properties: Public, reactive, can be set from outside. Triggers re-render when changed.

Internal state: Private, not reactive by default. Use for component-internal state that doesn't need to trigger re-render.

state.js
// Property - reactive, public static properties = { count: { type: Number } }; // Internal state - not reactive private _internalState = { data: null };

4. Templates and Rendering

Q7: How do you use Lit's template system?

Lit templates use the html tag function to create declarative templates with expression interpolation, conditional rendering, looping, and event handling.

template.js
render() { return html` <div> <h1>${this.title}</h1> ${this.items.length > 0 ? html` <ul>${this.items.map(item => html`<li>${item.name}</li>`)}</ul> ` : html`<p>No items</p>`} <button @click="${this.handleClick}">Click</button> </div> `; }
Q8: What are Lit directives?

Directives modify template behavior. Built-in directives include repeat (efficient list rendering), classMap (conditional classes), styleMap (conditional styles), and ifDefined.

directives.js
import { repeat } from 'lit/directives/repeat.js'; import { classMap } from 'lit/directives/class-map.js'; import { styleMap } from 'lit/directives/style-map.js'; render() { return html` <div class="${classMap({ active: this.isActive })}"> ${repeat(this.items, (item) => item.id, (item) => html` <div style="${styleMap({ color: item.color })}">${item.name}</div> `)} </div> `; }

5. Styling

Q9: How do you style Lit components?

Lit styling uses Shadow DOM by default for encapsulation. Styles are defined in the static styles property using the css tag function.

styles.js
import { css } from 'lit'; class MyElement extends LitElement { static styles = css` :host { display: block; padding: 16px; background: #f0f0f0; } h1 { color: #333; font-size: 24px; } button { background: #512bd4; color: white; border: none; padding: 8px 16px; cursor: pointer; } button:hover { background: #4230a3; } `; }
Q10: What is the difference between scoped and global styles?
  • Scoped styles: Defined in static styles property. Encapsulated to component's Shadow DOM. Don't leak out or get affected by external styles.
  • Global styles: Can be applied to light DOM or shared across components. Use constructable stylesheets or global CSS.

6. Lifecycle

Q11: What are the Lit component lifecycle methods?
  • constructor(): Called when element is created. Set initial state.
  • connectedCallback(): Called when element is added to DOM. Set up event listeners, timers.
  • disconnectedCallback(): Called when element is removed from DOM. Clean up resources.
  • attributeChangedCallback(): Called when observed attributes change.
  • willUpdate(): Called before update. Can modify properties before render.
  • updated(): Called after update. Post-render logic.
  • firstUpdated(): Called after first update. One-time setup.

7. Testing

Q12: How do you test Lit components?

Lit testing uses @open-wc/testing for web component testing. Test rendering, property changes, and user interactions with fixture and expect.

test.js
import { fixture, expect } from '@open-wc/testing'; import { MyElement } from './my-element.js'; describe('MyElement', () => { it('renders a greeting', async () => { const el = await fixture(''); const h1 = el.shadowRoot.querySelector('h1'); expect(h1.textContent).to.equal('Hello, World!'); }); });

12. Common Interview Questions

Q14: What is Shadow DOM and why is it important?

Shadow DOM is a web standard that provides encapsulation for DOM and styles. It:

  • Isolates component styles from the page
  • Prevents style conflicts
  • Enables component composition
  • Provides true encapsulation
Q15: What is the difference between @event and .addEventListener?

@event is Lit's declarative event handling with cleaner syntax and automatic binding. addEventListener is the standard DOM API with manual binding but more control.

events.js
// Lit's @event <button @click="${this.handleClick}">Click</button> // addEventListener this.shadowRoot.querySelector('button') .addEventListener('click', this.handleClick.bind(this));
Q16: How do you handle forms in Lit?

Form handling in Lit uses two-way binding with .value for controlled inputs and @input to update properties on change.

form.js
class MyForm extends LitElement { static properties = { username: { type: String }, email: { type: String } }; render() { return html` <form @submit="${this._handleSubmit}"> <input .value="${this.username}" @input="${this._handleInput}" /> <button type="submit">Submit</button> </form> `; } _handleInput(e) { this[e.target.name] = e.target.value; } }
Q17: What is conditional rendering in Lit?

Conditional rendering uses ternary operators, logical AND, or function calls to conditionally include template parts based on state.

conditional.js
render() { return html` ${this.showContent ? html`<div>Content is visible</div>` : html`<div>Content is hidden</div>` } `; }
Q18: How do you work with lists and keys in Lit?

Lists and keys use JavaScript array methods with map. Use the repeat directive for efficient list rendering with unique keys.

lists.js
render() { return html` <ul> ${this.items.map((item, index) => html` <li key="${item.id}">${item.name}</li> `)} </ul> `; }
Q19: How do you optimize Lit performance?

Performance optimization in Lit includes: Reactive properties (only update when properties change), Virtual DOM diffing (efficient DOM updates), Lazy loading (load components on demand), Code splitting (reduce bundle size), Debouncing (limit expensive operations), Memoization (cache computed values).

Q20: What is the difference between LitElement and HTMLElement?

LitElement: Base class is Lit library, features are reactive/templating/scoped styles, complexity is higher-level and easier, use case is production components.

HTMLElement: Base class is native Web Component, features are basic custom element, complexity is lower-level and more manual, use case is simple custom elements.

Q21: What are reactive controllers in Lit?

Reactive controllers are objects that can be attached to Lit elements to share reactive logic across components.

Benefits: Code reuse (share logic between components), Modularity (split complex logic), Lifecycle hooks (respond to element lifecycle), State management (external state integration).

Q22: How do you handle API calls in Lit?

API calls are typically handled in connectedCallback lifecycle method to fetch data when the component is added to DOM.

api.js
class MyElement extends LitElement { @property() data = []; async connectedCallback() { super.connectedCallback(); this.data = await this.fetchData(); } async fetchData() { const response = await fetch('/api/data'); return response.json(); } }
Q23: What is the difference between @property and @state decorators?
Aspect @property @state
Scope Public API Internal state
Attributes Reflects to HTML attributes Does not reflect
Use case Component configuration Internal data
Q24: How do you implement lazy loading in Lit?

Lazy loading uses dynamic imports to load components on demand, reducing initial bundle size and improving performance.

lazy.js
class MyElement extends LitElement { @property() lazyComponent = null; async loadComponent() { const module = await import('./heavy-component.js'); this.lazyComponent = module.default; } }
Q25: What is the role of the render() method in Lit?

The render() method defines the template for a Lit component. It:

  • Returns: A TemplateResult from html tagged template
  • Reactive: Called when properties change
  • Efficient: Only updates changed parts
  • Required: Must be implemented in LitElement subclasses

9. Advanced Lit Patterns

Q26: What are slots in Lit and how do you use them?

Slots allow you to compose components by projecting content from the light DOM into the shadow DOM. Use the slot element to define insertion points.

slots.js
class MyCard extends LitElement { render() { return html` <div class="card"> <slot name="header"></slot> <slot></slot> <slot name="footer"></slot> </div> `; } } // Usage <my-card> <h2 slot="header">Title</h2> <p>Content</p> <button slot="footer">Action</button> </my-card>
Q27: How do you create custom directives in Lit?

Custom directives are functions that return a directive result. They allow you to create reusable template behavior for specific use cases.

custom-directive.js
import { directive, Part } from 'lit/directive.js'; const myDirective = directive((value) => (part: Part) => { part.setValue(value.toUpperCase()); }); // Usage render() { return html`<div>${myDirective(this.text)}</div>`; }
Q28: How do you use CSS custom properties in Lit?

CSS custom properties (CSS variables) allow you to create themeable components. Define them in styles and allow consumers to override them.

css-vars.js
class MyElement extends LitElement { static styles = css` :host { --my-color: #512bd4; --my-padding: 16px; } .box { color: var(--my-color); padding: var(--my-padding); } `; }

10. Lit 3+ Features

Q29: What are the new features in Lit 3?

Lit 3 introduces several improvements: Smaller bundle size, faster rendering, better TypeScript support, improved reactive system, new directives, and enhanced developer experience.

  • Smaller: Reduced bundle size by ~40%
  • Faster: Improved render performance
  • TypeScript: Better type inference and support
  • Reactive: Enhanced property change detection
Q30: How do you use the new reactive system in Lit 3?

Lit 3 reactive system uses the @property and @state decorators with improved change detection and better performance for complex objects.

reactive-v3.js
import { property, state } from 'lit/decorators.js'; class MyElement extends LitElement { @property({ type: String }) name = 'World'; @state() internalCount = 0; }

11. Performance Optimization

Q31: How do you optimize Lit component performance?

Performance optimization in Lit includes using the repeat directive for lists, avoiding unnecessary re-renders, memoizing expensive computations, and lazy loading components.

performance.js
import { repeat } from 'lit/directives/repeat.js'; render() { return html` ${repeat(this.items, (item) => item.id, (item) => html` <div>${item.name}</div> `)} `; }
Q32: How do you implement virtual scrolling in Lit?

Virtual scrolling renders only visible items in a list, improving performance for large datasets. Use libraries like @lit-labs/virtualizer or implement custom logic.

virtual-scroll.js
import { Virtualizer } from '@lit-labs/virtualizer/virtualizer.js'; render() { return html` <virtualizer .items="${this.items}"> ${(item) => html`<div>${item.name}</div>`} </virtualizer> `; }

12. Additional Topics

Q33: How do you use TypeScript with Lit?

TypeScript with Lit provides full type safety for properties, events, and templates. Use decorators and type annotations for better developer experience.

typescript.js
import { LitElement, html } from 'lit'; import { property } from 'lit/decorators.js'; interface Item { id: number; name: string; } class MyElement extends LitElement { @property({ type: Array }) items: Item[] = []; }
Q34: How do you handle accessibility in Lit components?

Accessibility (a11y) in Lit includes using ARIA attributes, keyboard navigation, semantic HTML, and proper focus management. Test with @web/tester/a11y.

a11y.js
render() { return html` <button aria-label="Close dialog" role="button" @click="${this.close}"> × </button> `; }
Q35: How do you implement server-side rendering with Lit?

SSR with Lit uses @lit-labs/ssr to render components on the server. This improves SEO and initial page load performance.

ssr.js
import { render } from '@lit-labs/ssr/lib/render-with-global-dom-shim.js'; import { MyElement } from './my-element.js'; const result = await render(html`<my-element></my-element>`); console.log(result);
Q36: What is the difference between Lit and other web component libraries?

Lit vs others: Lit is lightweight (~5KB), has excellent TypeScript support, uses declarative templates, and provides reactive properties. Alternatives include Stencil (compiler-based), Angular Elements (framework-specific), and Polymer (deprecated predecessor).

Interview Tip: When discussing Lit, emphasize web components standards, Shadow DOM encapsulation, reactive properties, and performance.
