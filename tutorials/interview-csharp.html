C# Interview Guide

Comprehensive guide covering C# language features with detailed explanations and code examples

1. C# Fundamentals

Q1: What is C#?

C# (pronounced "C Sharp") is a modern, object-oriented programming language developed by Microsoft as part of the .NET initiative. It was designed by Anders Hejlsberg and first released in 2000. C# combines the power of C++ with the simplicity of Visual Basic, making it suitable for a wide range of applications from desktop to web to mobile.

Key Features: Type-safe, garbage-collected, component-oriented, supports multiple programming paradigms (imperative, declarative, functional, generic, object-oriented).

Program.cs
using System; class Program { static void Main() { Console.WriteLine("Hello, World!"); } }
Q2: What is the difference between value types and reference types?

Value types store the actual data directly in memory (typically on the stack). They include primitive types like int, float, bool, char, and struct types. When you assign a value type to another variable, the actual value is copied.

Reference types store a reference (memory address) to the actual data, which is stored on the heap. They include classes, interfaces, delegates, arrays, and strings. When you assign a reference type to another variable, only the reference is copied, not the actual data.

ValueReferenceTypes.cs
int a = 10; int b = a; // Copy of value b = 20; // a is still 10 class MyClass { public int Value; } MyClass obj1 = new MyClass { Value = 10 }; MyClass obj2 = obj1; // Copy of reference obj2.Value = 20; // obj1.Value is now 20
Q3: What is the difference between struct and class?

Struct is a value type that is typically used for small, lightweight objects. It's stored on the stack (when not boxed) and is copied by value. Structs cannot inherit from other structs or classes (except they can implement interfaces). They are sealed by default.

Class is a reference type stored on the heap. Classes support inheritance, can be abstract, and are passed by reference. Use classes for larger objects or when inheritance is needed.

StructClass.cs
public struct Point { public int X; public int Y; } public class Person { public string Name; public int Age; } Point p1 = new Point { X = 1, Y = 2 }; Point p2 = p1; // Value copy p2.X = 10; // p1.X is still 1 Person person1 = new Person { Name = "John", Age = 25 }; Person person2 = person1; // Reference copy person2.Age = 30; // person1.Age is now 30
Q4: What is the difference between string and String?

string is an alias for System.String in C#. They are completely identical and interchangeable. The lowercase string is the C# keyword, while String is the .NET Framework type.

Both represent a sequence of Unicode characters and are immutable (once created, cannot be changed). The choice between string and String is purely stylistic - most C# developers prefer the lowercase string.

String.cs
string s1 = "hello"; String s2 = "world"; Console.WriteLine(s1.GetType().Name); // String
Q5: What is the difference between const and readonly?

const fields are compile-time constants. They must be initialized at declaration and cannot be changed thereafter. const values are embedded directly into the IL code at compile time. They can only be primitive types or enums.

readonly fields can be initialized at declaration or in the constructor. They can be assigned only once (either at declaration or in constructor). readonly fields are runtime constants and can be of any type.

ConstReadonly.cs
public class Constants { public const int MaxConnections = 100; public readonly DateTime CreatedDate; public Constants() { CreatedDate = DateTime.Now; } }
Q6: What is the difference between static and instance?

Static members belong to the type itself rather than to a specific instance. They are shared across all instances of the class and can be accessed using the class name.

Instance members belong to a specific object instance. Each object has its own copy of instance members. They can only be accessed through an instance of the class.

StaticInstance.cs
public class Counter { public static int Count = 0; public int InstanceCount = 0; public Counter() { Count++; InstanceCount++; } }
Q7: What is the difference between abstract and virtual?

abstract methods have no implementation and must be overridden in derived classes. A class containing abstract members must itself be marked as abstract and cannot be instantiated.

virtual methods have a default implementation but can be overridden in derived classes. The base class provides a default behavior that derived classes can choose to override or use as-is.

AbstractVirtual.cs
public abstract class Shape { public abstract double CalculateArea(); public virtual void Display() { Console.WriteLine("Shape display"); } } public class Circle : Shape { public double Radius { get; set; } public override double CalculateArea() { return Math.PI * Radius * Radius; } }
Q8: What is the difference between interface and abstract class?

Interface defines a contract without any implementation. It can only contain method signatures, properties, events, and indexers. A class can implement multiple interfaces.

Abstract class can contain both abstract members (no implementation) and concrete members (with implementation). A class can inherit from only one abstract class.

InterfaceAbstract.cs
public interface IShape { double CalculateArea(); } public abstract class Animal { public string Name { get; set; } public abstract void MakeSound(); public void Sleep() { Console.WriteLine($"{Name} is sleeping"); } }
Q9: What is the difference between sealed and override?

sealed prevents a class from being inherited or a method from being overridden further.

override is used to provide a new implementation for a virtual or abstract method from a base class.

SealedOverride.cs
public class BaseClass { public virtual void Method() { Console.WriteLine("Base method"); } } public class DerivedClass : BaseClass { public override void Method() { Console.WriteLine("Derived method"); } } public class FinalClass : DerivedClass { public sealed override void Method() { Console.WriteLine("Final method"); } }
Q10: What is the difference between public, private, protected, internal, and protected internal?

public - Accessible from any code

private - Accessible only within the same class

protected - Accessible within the same class and derived classes

internal - Accessible only within the same assembly

protected internal - Accessible within the same assembly OR from derived classes in any assembly

AccessModifiers.cs
public class AccessModifiers { public int PublicField; private int _privateField; protected int _protectedField; internal int _internalField; protected internal int _protectedInternalField; }

2. C# Data Types and Variables

Q11: What are the built-in data types in C#?

C# has several built-in data types organized into categories:

  • Integral types: sbyte, byte, short, ushort, int, uint, long, ulong, char
  • Floating-point types: float, double, decimal
  • Boolean type: bool
  • String type: string
  • Object type: object (base type for all types)
DataTypes.cs
int age = 25; double price = 19.99; decimal money = 1000.50m; bool isActive = true; char grade = 'A'; string name = "John";
Q12: What is the difference between float, double, and decimal?

float is a 32-bit single-precision floating-point type. It has about 7 digits of precision and is used when memory is a concern.

double is a 64-bit double-precision floating-point type. It has about 15-16 digits of precision and is the default for floating-point operations.

decimal is a 128-bit decimal type with 28-29 digits of precision. It's used for financial calculations where precision is critical.

FloatDoubleDecimal.cs
float f = 3.14f; double d = 3.14159265358979; decimal m = 1000.50m; // Use decimal for financial calculations decimal total = 100.50m + 50.25m;
Q13: What is the difference between implicit and explicit type conversion?

Implicit conversion happens automatically when converting from a smaller type to a larger type without data loss.

Explicit conversion requires a cast operator and is used when converting from a larger type to a smaller type, which may result in data loss.

TypeConversion.cs
// Implicit conversion - no data loss int i = 100; long l = i; // Implicit double d = i; // Implicit // Explicit conversion - possible data loss double d2 = 123.45; int i2 = (int)d2; // Explicit, truncates to 123
Q14: What is the difference between var and explicit type declaration?

var is an implicitly typed local variable. The compiler infers the type from the initialization expression. The type is determined at compile time and cannot be changed.

Explicit type declaration specifies the type explicitly. It makes the code more readable and is preferred when the type isn't obvious from the right side.

VarExplicit.cs
// Using var - type inferred var name = "John"; // string var age = 25; // int var list = new List<int>(); // List // Using explicit type string name2 = "John"; int age2 = 25; List<int> list2 = new List<int>();
Q15: What is the difference between nullable value types and nullable reference types?

Nullable value types (T?) allow value types to have null values. They're represented as Nullable<T> struct and are useful when a value type may not have a value.

Nullable reference types are a compiler feature that makes reference types non-nullable by default. You must explicitly mark them as nullable (T?) to allow null values.

NullableTypes.cs
// Nullable value type int? nullableInt = null; if (nullableInt.HasValue) { int value = nullableInt.Value; } // Nullable reference type string? nullableString = null; string nonNullString = "hello"; // Cannot be null

3. C# Operators and Expressions

Q16: What is the null-coalescing operator (??)?

?? (null-coalescing operator) returns the left-hand operand if it's not null, otherwise it returns the right-hand operand. It's used to provide a default value for nullable types.

NullCoalescing.cs
string? name = null; string result = name ?? "Default"; // "Default" int? number = null; int value = number ?? 0; // 0
Q17: What is the null-conditional operator (?.)?

?. (null-conditional operator) returns null if the left-hand operand is null, otherwise it accesses the member. It's used to safely access members of potentially null objects.

NullConditional.cs
string? name = null; int? length = name?.Length; // null string? name2 = "John"; int? length2 = name2?.Length; // 4
Q18: What is the difference between == and Equals()?

== operator compares values for value types and references for reference types (by default). It can be overloaded to provide custom comparison logic.

Equals() method is inherited from Object and compares values. For reference types, it compares references by default but can be overridden to compare values.

EqualsOperator.cs
string s1 = "hello"; string s2 = "hello"; bool same1 = (s1 == s2); // true bool same2 = s1.Equals(s2); // true // For value types, both compare values int a = 5; int b = 5; bool same3 = (a == b); // true
Q19: What is the difference between is and as operators?

is operator checks if an expression is compatible with a given type. It returns true if the expression can be cast to that type.

as operator attempts to cast an expression to a specified type. If the cast fails, it returns null instead of throwing an exception.

IsAs.cs
object obj = "hello"; // is operator if (obj is string) { Console.WriteLine("It's a string"); } // as operator string? str = obj as string; if (str != null) { Console.WriteLine(str); }
Q20: What is the nameof operator?

nameof operator returns the name of a variable, type, or member as a string. It's evaluated at compile time and is useful for avoiding hard-coded strings in code.

Nameof.cs
string className = nameof(MyClass); // "MyClass" string methodName = nameof(MyMethod); // "MyMethod" public void MyMethod(string name) { if (name == null) throw new ArgumentNullException(nameof(name)); }

4. C# Control Flow

Q21: What is the difference between for, foreach, and while loops?

for loop is used when you know the number of iterations in advance. It has an initialization, condition, and iterator.

foreach loop is used to iterate over collections. It's read-only and cannot modify the collection during iteration.

while loop is used when you don't know the number of iterations and the loop continues as long as the condition is true.

Loops.cs
// for loop for (int i = 0; i < 10; i++) { Console.WriteLine(i); } // foreach loop foreach (var item in collection) { Console.WriteLine(item); } // while loop int count = 0; while (count < 10) { Console.WriteLine(count); count++; }
Q22: What is the difference between break, continue, and return?

break exits the nearest enclosing loop or switch statement immediately.

continue skips the current iteration of the loop and continues with the next iteration.

return exits the current method and optionally returns a value to the caller.

BreakContinueReturn.cs
// break for (int i = 0; i < 10; i++) { if (i == 5) break; // Exits loop } // continue for (int i = 0; i < 10; i++) { if (i % 2 == 0) continue; // Skips even numbers Console.WriteLine(i); } // return public int Add(int a, int b) { return a + b; }
Q23: What is the difference between switch statement and switch expression?

switch statement is the traditional switch construct that uses case labels and requires break statements. It's a statement, not an expression.

switch expression is a more concise syntax introduced in C# 8 that returns a value and uses => syntax. It's an expression that can be assigned to a variable.

SwitchExpression.cs
// Switch statement string result; switch (value) { case 1: result = "one"; break; case 2: result = "two"; break; default: result = "other"; break; } // Switch expression string result = value switch { 1 => "one", 2 => "two", _ => "other" };
Q24: What is the using statement?

using statement ensures that IDisposable objects are properly disposed. It's syntactic sugar for a try-finally block that calls Dispose(). It's essential for managing resources like file handles and database connections.

UsingStatement.cs
// using statement using (FileStream fs = new FileStream("file.txt", FileMode.Open)) { // Use the file } // fs.Dispose() is called automatically // Multiple using statements using (var stream = new FileStream("file.txt", FileMode.Open)) using (var reader = new StreamReader(stream)) { // Use reader }

5. C# Methods and Parameters

Q25: What is the difference between ref and out parameters?

ref parameter passes a reference to a variable. The variable must be initialized before being passed. The method can read and modify the variable.

out parameter passes a reference to a variable that must be assigned by the method before returning. The variable doesn't need to be initialized before being passed.

RefOut.cs
// ref parameter void Square(ref int x) { x = x * x; } int num = 5; Square(ref num); // num is now 25 // out parameter bool TryParse(string s, out int result) { return int.TryParse(s, out result); }
Q26: What is the difference between params and optional parameters?

params allows a method to accept a variable number of arguments of the same type. It must be the last parameter and can accept zero or more arguments.

Optional parameters have default values and can be omitted when calling the method. They must come after required parameters.

ParamsOptional.cs
// params parameter void PrintNumbers(params int[] numbers) { foreach (var num in numbers) Console.WriteLine(num); } PrintNumbers(1, 2, 3); // optional parameters void Greet(string name, string greeting = "Hello") { Console.WriteLine($"{greeting}, {name}!"); } Greet("John"); // Uses default greeting Greet("John", "Hi"); // Uses custom greeting
Q27: What is the difference between named arguments and positional arguments?

Positional arguments are passed in the order defined by the method signature. The order must match exactly.

Named arguments specify the parameter name followed by a colon and the value. They can be passed in any order and improve code readability.

NamedArguments.cs
// Positional arguments CreateUser("John", 25, true); // Named arguments CreateUser(name: "John", age: 25, isActive: true); // Named arguments in any order CreateUser(isActive: true, name: "John", age: 25);
Q28: What is the difference between method overloading and method overriding?

Method overloading is compile-time polymorphism where multiple methods have the same name but different parameters (different number or types of parameters).

Method overriding is runtime polymorphism where a derived class provides a new implementation for a virtual method from a base class.

OverloadingOverriding.cs
// Method overloading public class Calculator { public int Add(int a, int b) => a + b; public double Add(double a, double b) => a + b; } // Method overriding public class BaseClass { public virtual void Display() { Console.WriteLine("Base display"); } } public class DerivedClass : BaseClass { public override void Display() { Console.WriteLine("Derived display"); } }
Q29: What is the difference between extension methods and regular methods?

Extension methods allow you to add methods to existing types without modifying the original type or creating a derived type. They are static methods in a static class with the first parameter prefixed with 'this'.

Regular methods are defined within the class itself and have direct access to private members.

ExtensionMethods.cs
// Extension method public static class StringExtensions { public static bool IsNullOrEmpty(this string value) { return string.IsNullOrEmpty(value); } } // Usage string text = "hello"; bool empty = text.IsNullOrEmpty();
Q30: What is the difference between local functions and lambda expressions?

Local functions are methods defined inside another method. They can access variables in the enclosing scope and can be async, use generics, and have attributes.

Lambda expressions are anonymous functions that can be used to create delegates or expression trees. They're more concise but have some limitations compared to local functions.

LocalFunctionLambda.cs
// Local function public int Calculate(int x) { int Square(int n) => n * n; return Square(x); } // Lambda expression Func<int, int> square = n => n * n; int result = square(5);

6. C# Properties and Indexers

Q31: What is the difference between auto-implemented properties and traditional properties?

Auto-implemented properties use shorthand syntax where the compiler generates the backing field automatically. They're used when no additional logic is needed in the getter/setter.

Traditional properties explicitly define the backing field and can include custom logic in the getter/setter.

Properties.cs
// Auto-implemented property public string Name { get; set; } // Traditional property with backing field private string _name; public string Name { get { return _name; } set { _name = value?.Trim(); } }
Q32: What is the difference between get-only properties and init-only properties?

get-only properties can only be read. They can be set only in the constructor or via a field initializer. They're immutable after construction.

init-only properties can be set during object initialization or with a with expression. They're immutable after initialization but can be set in object initializers.

InitOnly.cs
public class Person { public string Name { get; init; } // init-only public string Address { get; } // get-only public Person(string name) { Name = name; Address = "Unknown"; } } var person = new Person { Name = "John" };
Q33: What is the difference between indexers and properties?

Indexers allow instances of a class or struct to be indexed like arrays. They use this[] syntax and can have multiple parameters.

Properties expose data as public members with get and set accessors. They use named accessors and don't support parameters.

Indexers.cs
public class SampleCollection { private List<string> _items = new List<string>(); // Indexer public string this[int index] { get { return _items[index]; } set { _items[index] = value; } } // Property public int Count => _items.Count; }
Q34: What are required properties?

required properties must be set during object initialization. The compiler enforces this requirement at compile time. They're useful for ensuring that essential properties are always initialized.

RequiredProperties.cs
public class Person { public required string Name { get; init; } // Must be set public string? Address { get; init; } // Optional } // var person = new Person { Address = "123 St" }; // Error - Name is required var person = new Person { Name = "John" }; // OK

7. C# Delegates and Events

Q35: What is a delegate?

Delegate is a type-safe function pointer that references a method. Delegates are used to pass methods as arguments to other methods. They are the foundation for events in C#.

Delegate.cs
// Declare delegate public delegate void MessageDelegate(string message); // Method that matches delegate signature public void DisplayMessage(string message) { Console.WriteLine(message); } // Create delegate instance MessageDelegate del = new MessageDelegate(DisplayMessage); del("Hello");
Q36: What is the difference between delegate and Func/Action?

Delegate is a custom type that you define with a specific signature. You need to declare the delegate type before using it.

Func and Action are predefined generic delegates. Func returns a value, Action doesn't. They eliminate the need to declare custom delegate types.

FuncAction.cs
// Custom delegate public delegate int CalculateDelegate(int a, int b); // Func - returns a value Func<int, int, int> add = (a, b) => a + b; // Action - doesn't return a value Action<string> print = message => Console.WriteLine(message);
Q37: What is the difference between multicast delegate and single-cast delegate?

Multicast delegate can reference multiple methods. When invoked, it calls all the referenced methods in the order they were added. All delegates in C# are multicast by default.

Single-cast delegate references only one method. In C#, all delegates are multicast, but you can treat them as single-cast by only assigning one method.

MulticastDelegate.cs
public delegate void MessageDelegate(string message); MessageDelegate del = Method1; del += Method2; // Add another method del += Method3; // Add another method del("Hello"); // Calls all three methods
Q38: What is the difference between event and delegate?

Event is a special kind of multicast delegate that can only be invoked from within the class that declares it. It provides a publish/subscribe mechanism.

Delegate can be invoked from anywhere it's accessible. Events add a layer of protection by restricting invocation to the declaring class.

Event.cs
public class Publisher { // Event - can only be invoked from within this class public event EventHandler<EventArgs> OnEvent; public void RaiseEvent() { OnEvent?.Invoke(this, EventArgs.Empty); } }
Q39: What is the difference between EventHandler and custom delegate?

EventHandler is a predefined delegate in .NET for events that follow the standard event pattern (sender, EventArgs). It's the recommended way to declare events.

Custom delegate is a delegate you define yourself. Use it when you need a different signature than the standard EventHandler pattern.

EventHandler.cs
// Standard EventHandler public event EventHandler<EventArgs> OnClick; // Custom delegate public delegate void CustomEventHandler(object sender, int value); public event CustomEventHandler OnValueChanged;

8. C# Async/Await

Q40: What is async/await?

async/await is a language feature for writing asynchronous code that looks synchronous. The async keyword marks a method as asynchronous, and await suspends execution until the awaited task completes.

AsyncAwait.cs
public async Task<string> GetDataAsync() { await Task.Delay(1000); return "Data"; } // Calling async method string data = await GetDataAsync();
Q41: What is the difference between Task and Task<T>?

Task represents an asynchronous operation that doesn't return a value. It's used for fire-and-forget operations or when you only need to know when the operation completes.

Task<T> represents an asynchronous operation that returns a value of type T. The Result property contains the returned value after the task completes.

TaskTaskT.cs
// Task - no return value public async Task DoWorkAsync() { await Task.Delay(1000); } // Task - returns a value public async Task<int> CalculateAsync() { await Task.Delay(1000); return 42; }
Q42: What is the difference between ConfigureAwait(false) and ConfigureAwait(true)?

ConfigureAwait(false) tells the awaiter not to capture the current context. This can improve performance and avoid deadlocks in library code.

ConfigureAwait(true) captures the current context and resumes on it. This is the default behavior and is needed when you need to access UI elements or HttpContext.

ConfigureAwait.cs
// Library code - use ConfigureAwait(false) public async Task<string> GetDataAsync() { await Task.Delay(1000).ConfigureAwait(false); return "Data"; } // UI code - use ConfigureAwait(true) or default public async Task UpdateUIAsync() { await LoadDataAsync(); // Captures context Label.Text = "Loaded"; }
Q43: What is the difference between ValueTask and Task?

ValueTask is a struct that can represent either a completed synchronous result or an asynchronous Task. It's more memory-efficient for operations that may complete synchronously.

Task is a class that always represents an asynchronous operation. It has more allocation overhead but is more flexible.

ValueTask.cs
// ValueTask - better for operations that may complete synchronously public ValueTask<int> GetValueAsync() { if (_cached) return new ValueTask<int>(_cachedValue); return new ValueTask<int>(LoadValueAsync()); }
Q44: What is the difference between async void and async Task?

async void should only be used for event handlers. It cannot be awaited and exceptions cannot be caught outside the method.

async Task is the standard way to write async methods. It can be awaited and exceptions can be caught by the caller.

AsyncVoid.cs
// async void - only for event handlers private async void Button_Click(object sender, EventArgs e) { await DoWorkAsync(); } // async Task - standard async method public async Task DoWorkAsync() { await Task.Delay(1000); }

9. C# Generics

Q45: What are generics in C#?

Generics allow you to define type-safe data structures without committing to a specific data type. They provide type safety, performance improvements, and code reuse.

Generics.cs
// Generic class public class Repository<T> { public void Add(T item) { } public T Get(int id) => default(T); } // Generic method public T Max<T>(T a, T b) where T : IComparable<T> { return a.CompareTo(b) > 0 ? a : b; }
Q46: What is the difference between generic class and generic method?

Generic class is a class that is parameterized with one or more type parameters. All methods in the class can use these type parameters.

Generic method is a method that has its own type parameters, independent of the class's type parameters. It can be called with different type arguments each time.

GenericClassMethod.cs
// Generic class public class Container<T> { public T Value { get; set; } } // Generic method in non-generic class public class Helper { public T Convert<T>(object value) { return (T)value; } }
Q47: What is the difference between covariance and contravariance?

Covariance allows you to use a more derived type than originally specified. It works with generic type parameters on interfaces and delegates with the 'out' keyword.

Contravariance allows you to use a more generic (less derived) type than originally specified. It works with the 'in' keyword.

CovarianceContravariance.cs
// Covariance - out keyword interface ICovariant<out T> { } // Contravariance - in keyword interface IContravariant<in T> { } // Example IEnumerable<string> strings = new List<string>(); IEnumerable<object> objects = strings; // Covariant
Q48: What is the difference between List<T> and IEnumerable<T>?

List<T> is a concrete collection that implements IList<T>. It provides methods for adding, removing, and modifying elements. It's stored in memory and supports indexing.

IEnumerable<T> is an interface that represents a sequence. It's read-only and supports deferred execution. It's used for iteration and LINQ queries.

ListIEnumerable.cs
// List - concrete, modifiable List<int> list = new List<int> { 1, 2, 3 }; list.Add(4); int item = list[0]; // IEnumerable - interface, read-only IEnumerable<int> enumerable = list; foreach (var num in enumerable) { Console.WriteLine(num); }
Q49: What is the difference between Dictionary<TKey, TValue> and Hashtable?

Dictionary<TKey, TValue> is a generic collection that stores key-value pairs. It's type-safe and offers better performance than Hashtable.

Hashtable is a non-generic collection that stores key-value pairs as objects. It's not type-safe and requires boxing/unboxing, leading to performance issues.

DictionaryHashtable.cs
// Dictionary - type-safe Dictionary<string, int> dict = new Dictionary<string, int>(); dict["one"] = 1; // Hashtable - non-generic, not type-safe Hashtable hashtable = new Hashtable(); hashtable["one"] = 1;
Q50: What are generic constraints?

Generic constraints restrict the types that can be used as type arguments for a generic type or method. They ensure that the type argument meets certain requirements.

GenericConstraints.cs
// where T : class - reference type public class Container<T> where T : class { } // where T : struct - value type public class ValueContainer<T> where T : struct { } // where T : new() - parameterless constructor public class Factory<T> where T : new() { }

10. C# Exception Handling

Q51: What is exception handling in C#?

Exception handling is a mechanism to handle runtime errors gracefully. It uses try-catch-finally blocks to catch and handle exceptions, preventing application crashes.

ExceptionHandling.cs
try { int result = 10 / 0; } catch (DivideByZeroException ex) { Console.WriteLine($"Error: {ex.Message}"); } finally { Console.WriteLine("Cleanup code"); }
Q52: What is the difference between throw and throw ex?

throw preserves the original stack trace. It re-throws the caught exception without modifying it.

throw ex resets the stack trace to the current location. This loses information about where the original exception occurred.

Throw.cs
try { // Some code that throws } catch (Exception ex) { throw; // Preserves stack trace // throw ex; // Resets stack trace }
Q53: What is the difference between finally and using?

finally block always executes regardless of whether an exception occurred. It's used for cleanup code.

using statement ensures that IDisposable objects are disposed properly. It's syntactic sugar for try-finally with Dispose.

FinallyUsing.cs
// Finally block try { // Code that might throw } finally { // Cleanup code } // Using statement using (FileStream fs = new FileStream("file.txt", FileMode.Open)) { // fs is automatically disposed }
Q54: What is the difference between custom exception and built-in exception?

Built-in exceptions are predefined exceptions in .NET like ArgumentException, InvalidOperationException, etc.

Custom exceptions are user-defined exceptions that inherit from Exception. They provide domain-specific error information.

CustomException.cs
public class InvalidUserException : Exception { public InvalidUserException(string message) : base(message) { } } throw new InvalidUserException("User not found");
Q55: What is the difference between checked and unchecked?

checked causes arithmetic overflow to throw an OverflowException.

unchecked allows arithmetic overflow to wrap around without throwing an exception (default behavior).

CheckedUnchecked.cs
checked { int x = int.MaxValue; int y = x + 1; // Throws OverflowException } unchecked { int x = int.MaxValue; int y = x + 1; // Wraps to int.MinValue }

11. C# Memory Management

Q56: What is garbage collection in C#?

Garbage Collection (GC) is an automatic memory management feature that automatically reclaims memory occupied by objects that are no longer in use. It prevents memory leaks and simplifies development.

GarbageCollection.cs
public class MyClass { private byte[] _data = new byte[1000000]; } // GC automatically collects when object is no longer referenced void CreateObject() { var obj = new MyClass(); } // obj is eligible for GC here // Force GC (rarely needed) GC.Collect(); GC.WaitForPendingFinalizers();
Q57: What is the difference between IDisposable and IAsyncDisposable?

IDisposable provides a Dispose method for releasing unmanaged resources synchronously.

IAsyncDisposable provides a DisposeAsync method for releasing unmanaged resources asynchronously, which is useful for I/O-bound cleanup operations.

IDisposableAsync.cs
public class Resource : IDisposable { public void Dispose() => // Cleanup } public class AsyncResource : IAsyncDisposable { public async ValueTask DisposeAsync() => await // Async cleanup }
Q58: What is the difference between WeakReference and strong reference?

Strong reference is a normal reference that prevents the object from being garbage collected.

WeakReference allows the object to be garbage collected even if it's still referenced. It's useful for caching.

WeakReference.cs
// Strong reference - prevents GC var strong = new MyClass(); // Weak reference - allows GC var weak = new WeakReference<MyClass>(new MyClass()); if (weak.TryGetTarget(out MyClass target)) { // Object is still alive }
Q59: What is the difference between boxing and unboxing?

Boxing is the process of converting a value type to a reference type (object). It copies the value to the heap.

Unboxing is the process of converting a reference type back to a value type. It copies the value from the heap to the stack.

BoxingUnboxing.cs
int value = 42; // Boxing - value type to reference type object boxed = value; // Unboxing - reference type to value type int unboxed = (int)boxed; // Avoid boxing/unboxing for performance List<int> list = new List<int>(); // No boxing ArrayList arrayList = new ArrayList(); // Boxing occurs
Q60: What is the difference between Stack and Heap?

Stack is a LIFO data structure that stores value types and reference pointers. It's fast and automatically managed.

Heap is a memory area that stores reference type objects. It's managed by the garbage collector and is slower than the stack.

StackHeap.cs
// Stack - value types int x = 10; bool flag = true; // Heap - reference types string text = "Hello"; var obj = new MyClass(); // Reference on stack, object on heap var list = new List<int>();

12. Advanced C# Features

Q61: What is the difference between record and class?

record is a reference type introduced in C# 9 that provides built-in value-based equality, immutability, and with expressions. It's ideal for data models.

class is a traditional reference type with reference-based equality. It's mutable by default and requires manual implementation of equality.

RecordClass.cs
public record Person(string Name, int Age); var p1 = new Person("John", 30); var p2 = new Person("John", 30); bool equal = p1 == p2; // true (value-based equality) var p3 = p1 with { Age = 31 }; // with expression
Q62: What is pattern matching?

Pattern matching is a feature that allows you to test if a value matches a certain pattern and extract information from it. It includes type patterns, property patterns, relational patterns, and more.

PatternMatching.cs
object obj = "hello"; // Type pattern if (obj is string s) { Console.WriteLine(s.Length); } // Relational pattern in switch expression int x = 5; string result = x switch { > 10 => "large", < 5 => "small", _ => "medium" };
Q63: What is the difference between Span and Memory?

Span<T> is a ref struct that can only be used on the stack. It provides high-performance access to contiguous memory.

Memory<T> can be stored on the heap and used in async scenarios. It's a stack-only wrapper for Memory<T>.

SpanMemory.cs
// Span - stack-only, cannot be used in async void Process(Span<byte> buffer) { // High-performance processing } // Memory - can be stored on heap Memory<byte> memory = new byte[1000]; await ProcessAsync(memory);
Q64: What is the difference between ref struct and struct?

struct is a value type that can be boxed and stored on the heap. It can implement interfaces.

ref struct is a stack-only type that cannot be boxed, stored on the heap, or implement interfaces. It's used for high-performance scenarios.

RefStruct.cs
// Regular struct public struct Point { public int X, Y; } // Ref struct - stack only public ref struct SpanWrapper { public Span<int> Data; }
Q65: What is the difference between default and default(T)?

default returns the default value of a type. In C# 7.1+, it can infer the type from context.

default(T) explicitly specifies the type. It's the older syntax but still useful when type inference isn't possible.

Default.cs
// With type inference (C# 7.1+) int x = default; // 0 string s = default; // null // Explicit type specification int x = default(int); // 0 string s = default(string); // null // Generic method T GetDefault<T>() => default(T);
Q66: What is the difference between ref return and ref local?

ref return allows a method to return a reference to a variable instead of a value. The caller can modify the original variable.

ref local is a variable that stores a reference to another variable. Changes to the ref local affect the original variable.

RefReturnLocal.cs
private int[] _numbers = { 1, 2, 3 }; // ref return public ref int GetNumber(int index) => ref _numbers[index]; // ref local ref int num = ref GetNumber(0); num = 10; // Modifies _numbers[0]
Q67: What is the difference between readonly struct and struct?

readonly struct is immutable - all its fields are readonly. It provides better performance and thread safety.

struct is mutable by default. Its fields can be modified after creation.

ReadonlyStruct.cs
// readonly struct - immutable public readonly struct Point { public readonly int X; public readonly int Y; } // Regular struct - mutable public struct MutablePoint { public int X; public int Y; }
Q68: What is the difference between unsafe code and safe code?

Safe code is code that runs under the supervision of the CLR with type safety and bounds checking. It's the default in C#.

Unsafe code can use pointers and direct memory manipulation. It requires the unsafe keyword and can bypass CLR safety checks.

UnsafeCode.cs
// Safe code int[] numbers = new int[10]; numbers[0] = 1; // Unsafe code - requires unsafe context unsafe { int* ptr = stackalloc int[10]; ptr[0] = 1; }
Q69: What is the difference between stackalloc and heap allocation?

stackalloc allocates memory on the stack. It's very fast but limited in size and lifetime (freed when the method returns). It can only be used in unsafe context.

Heap allocation allocates memory on the managed heap. It's garbage-collected and can hold larger objects. It's the default allocation method in C#.

Stackalloc.cs
// stackalloc - stack allocation (unsafe) unsafe { int* buffer = stackalloc int[100]; buffer[0] = 1; } // Heap allocation (safe) int[] buffer = new int[100]; buffer[0] = 1;
Q70: What is the difference between caller argument expression and caller member name?

Caller argument expression (CallerArgumentExpression) attribute captures the expression used as an argument. It's useful for logging and validation.

Caller member name (CallerMemberName) attribute captures the name of the calling method or property. It's useful for logging and INotifyPropertyChanged.

CallerInfo.cs
using System.Runtime.CompilerServices; public void Log(string message, [CallerMemberName] string memberName = null, [CallerArgumentExpression(nameof(message))] string expression = null) { Console.WriteLine($"{memberName}: {expression} = {message}"); }

13. C# Security

Q71: What is the difference between SecureString and string?

SecureString is a secure string that stores text in memory in an encrypted format. It's designed to hold sensitive data like passwords and is automatically disposed when no longer needed.

string is immutable and stored in plain text in memory. It stays in memory until garbage collected, making it unsuitable for sensitive data.

SecureString.cs
using System.Security; // SecureString - encrypted in memory using (SecureString securePwd = new SecureString()) { foreach (char c in password) securePwd.AppendChar(c); } // Regular string - plain text in memory string password = "secret"; // Not secure
Q72: What is the difference between hashing and encryption?

Hashing is a one-way function that converts data to a fixed-size string. It cannot be reversed. Used for password storage and data integrity verification.

Encryption is a two-way function that converts data to unreadable format using a key. It can be decrypted with the correct key. Used for data protection in transit and at rest.

HashingEncryption.cs
using System.Security.Cryptography; // Hashing - one-way using (SHA256 sha256 = SHA256.Create()) { byte[] hash = sha256.ComputeHash(Encoding.UTF8.GetBytes(data)); } // Encryption - two-way using (Aes aes = Aes.Create()) { byte[] encrypted = aes.EncryptCbc(plainText, aes.IV); byte[] decrypted = aes.DecryptCbc(encrypted, aes.IV); }
Q73: What is the difference between symmetric and asymmetric encryption?

Symmetric encryption uses the same key for encryption and decryption. It's faster but requires secure key exchange. Examples: AES, DES.

Asymmetric encryption uses a public key for encryption and a private key for decryption. It's slower but doesn't require secure key exchange. Examples: RSA, ECC.

EncryptionTypes.cs
// Symmetric encryption (AES) using (Aes aes = Aes.Create()) { aes.Key = key; // Same key for encrypt/decrypt byte[] encrypted = aes.EncryptCbc(data, aes.IV); } // Asymmetric encryption (RSA) using (RSA rsa = RSA.Create()) { byte[] encrypted = rsa.Encrypt(data, RSAEncryptionPadding.OaepSHA256); byte[] decrypted = rsa.Decrypt(encrypted, RSAEncryptionPadding.OaepSHA256); }
Q74: What is the difference between Code Access Security (CAS) and Security Transparency?

Code Access Security (CAS) is deprecated security model that restricts what code can do based on its origin and permissions. It's obsolete in .NET Core and later.

Security Transparency is a security model that distinguishes between security-critical code (can perform privileged operations) and transparent code (cannot). It's the modern approach in .NET.

SecurityTransparency.cs
// Security-critical code [SecurityCritical] public class SecureOperations { // Can perform privileged operations } // Security-transparent code [SecuritySafeCritical] public class SafeOperations { // Limited operations }
Q75: What is the difference between PrincipalPermission and IPrincipal?

PrincipalPermission is a declarative security attribute that checks if the current user has specific roles or is authenticated. It's used for role-based security.

IPrincipal is an interface that represents the security context of a user, including their identity and roles. It's used for programmatic security checks.

PrincipalPermission.cs
using System.Security.Permissions; // Declarative security [PrincipalPermission(SecurityAction.Demand, Role = "Admin")] public void AdminOnlyMethod() { // Only admins can execute } // Programmatic security if (Thread.CurrentPrincipal.IsInRole("Admin")) { // User is admin }

14. C# Reflection

Q76: What is reflection in C#?

Reflection is the ability to inspect and manipulate types, methods, properties, and other members at runtime. It allows you to dynamically create instances, invoke methods, and access fields without knowing them at compile time.

Reflection.cs
using System.Reflection; Type type = typeof(MyClass); // Get all methods MethodInfo[] methods = type.GetMethods(); // Create instance dynamically object instance = Activator.CreateInstance(type); // Invoke method MethodInfo method = type.GetMethod("MyMethod"); method.Invoke(instance, null);
Q77: What is the difference between Type and TypeInfo?

Type is the fundamental class for reflection in .NET. It represents type declarations and provides methods to get information about types.

TypeInfo is part of the Reflection API in .NET Core/5+. It provides richer type information and is obtained through GetTypeInfo() on Type.

TypeInfo.cs
// Type (classic reflection) Type type = typeof(MyClass); MethodInfo[] methods = type.GetMethods(); // TypeInfo (modern reflection) TypeInfo typeInfo = type.GetTypeInfo(); IEnumerable<MethodInfo> methods = typeInfo.DeclaredMethods;
Q78: What is the difference between BindingFlags and default binding?

BindingFlags control how reflection searches for members. You can specify flags like Public, NonPublic, Instance, Static, etc., to filter the search.

Default binding returns only public instance members. BindingFlags allows you to get private, static, and other non-public members.

BindingFlags.cs
// Default - public instance only MethodInfo[] methods = type.GetMethods(); // With BindingFlags - includes private and static MethodInfo[] allMethods = type.GetMethods( BindingFlags.Public | BindingFlags.NonPublic | BindingFlags.Instance | BindingFlags.Static );
Q79: What is the difference between Activator.CreateInstance and new keyword?

Activator.CreateInstance creates an instance of a type at runtime using reflection. It can create instances when the type is not known at compile time.

new keyword creates an instance at compile time. It's type-safe and faster but requires the type to be known at compile time.

Activator.cs
// Compile-time creation (new) MyClass obj = new MyClass(); // Runtime creation (Activator) Type type = Type.GetType("MyNamespace.MyClass"); object obj = Activator.CreateInstance(type); // With parameters object obj = Activator.CreateInstance(type, new object[] { param1, param2 });
Q80: What is the difference between dynamic and reflection?

dynamic bypasses compile-time type checking. Operations on dynamic types are resolved at runtime using the DLR (Dynamic Language Runtime). It's simpler but less type-safe.

Reflection explicitly inspects and invokes members at runtime. It's more verbose but provides more control and information about types.

DynamicReflection.cs
// Dynamic - simple but runtime errors dynamic obj = new MyClass(); obj.SomeMethod(); // Resolved at runtime // Reflection - verbose but explicit Type type = typeof(MyClass); MethodInfo method = type.GetMethod("SomeMethod"); method.Invoke(obj, null);

15. C# Attributes

Q81: What are attributes in C#?

Attributes are declarative tags that provide additional information about program elements (classes, methods, properties, etc.). They can be used by the compiler, runtime, or custom tools to modify behavior or provide metadata.

Attributes.cs
[Serializable] public class Person { [Obsolete("Use NewMethod instead")] public void OldMethod() { } [Conditional("DEBUG")] public void DebugMethod() { } }
Q82: What is the difference between Obsolete and Conditional attributes?

Obsolete marks a program element as deprecated. The compiler can issue warnings or errors when the element is used. It's used to phase out old code.

Conditional makes method execution dependent on a preprocessor symbol. The method is only compiled if the symbol is defined. It's used for debug-specific code.

ObsoleteConditional.cs
[Obsolete("Use NewMethod instead", true)] public void OldMethod() { } // Compiler error if used [Conditional("DEBUG")] public void DebugOnly() { } // Only in DEBUG builds [Conditional("RELEASE")] public void ReleaseOnly() { } // Only in RELEASE builds
Q83: What is the difference between custom attributes and built-in attributes?

Built-in attributes are predefined in .NET like Serializable, Obsolete, Conditional, etc. They provide common functionality.

Custom attributes are user-defined by inheriting from Attribute. They allow you to add custom metadata and behavior to your code.

CustomAttribute.cs
// Custom attribute [AttributeUsage(AttributeTargets.Class | AttributeTargets.Method)] public class AuthorAttribute : Attribute { public string Name { get; } public AuthorAttribute(string name) { Name = name; } } // Usage [Author("John Doe")] public class MyClass { }
Q84: What is the difference between AttributeUsage and AttributeTargets?

AttributeUsage is an attribute applied to custom attributes to specify where they can be used (on classes, methods, properties, etc.) and whether they can be used multiple times.

AttributeTargets is an enum that specifies the valid targets for an attribute (Class, Method, Property, Field, etc.).

AttributeUsage.cs
// Allow on classes and methods, multiple times [AttributeUsage(AttributeTargets.Class | AttributeTargets.Method, AllowMultiple = true)] public class MyAttribute : Attribute { } // Allow on properties only, single use [AttributeUsage(AttributeTargets.Property, AllowMultiple = false, Inherited = true)] public class SingleAttribute : Attribute { }
Q85: What is the difference between reflection on attributes and attribute inheritance?

Reflection on attributes uses GetCustomAttributes to retrieve attributes applied to a type or member at runtime.

Attribute inheritance controls whether attributes are inherited by derived classes. The Inherited property in AttributeUsage specifies this behavior.

AttributeReflection.cs
// Get attributes via reflection Type type = typeof(MyClass); AuthorAttribute[] attrs = (AuthorAttribute[])type.GetCustomAttributes(typeof(AuthorAttribute), false); // Inherited attribute [AttributeUsage(AttributeTargets.Class, Inherited = true)] public class InheritedAttribute : Attribute { }

16. C# Serialization

Q86: What is serialization in C#?

Serialization is the process of converting an object into a stream of bytes to store it or transmit it. Deserialization is the reverse process of reconstructing the object from the stream.

Serialization.cs
[Serializable] public class Person { public string Name; public int Age; } // Binary serialization Person person = new Person { Name = "John", Age = 30 }; // Serialize to bytes, then deserialize back
Q87: What is the difference between binary serialization and JSON serialization?

Binary serialization converts objects to binary format. It's compact and fast but not human-readable and .NET-specific. Used for internal storage.

JSON serialization converts objects to JSON text format. It's human-readable, language-agnostic, and widely used for web APIs. Used for cross-platform communication.

SerializationTypes.cs
using System.Text.Json; // JSON serialization string json = JsonSerializer.Serialize(person); Person deserialized = JsonSerializer.Deserialize<Person>(json); // Binary serialization (with BinaryFormatter - deprecated) // Use newer libraries like MessagePack or protobuf instead
Q88: What is the difference between ISerializable and IXmlSerializable?

ISerializable interface allows custom control over serialization and deserialization. It's used with binary serialization and provides GetObjectData and a special constructor.

IXmlSerializable interface allows custom XML serialization control. It provides ReadXml and WriteXml methods for custom XML format.

SerializableInterfaces.cs
// ISerializable [Serializable] public class MyClass : ISerializable { public void GetObjectData(SerializationInfo info, StreamingContext context) { info.AddValue("Name", Name); } protected MyClass(SerializationInfo info, StreamingContext context) { } }
Q89: What is the difference between DataContract and Serializable?

Serializable is a simple attribute that marks a class as serializable. It uses the default serialization mechanism and serializes all fields.

DataContract is part of WCF and provides fine-grained control over serialization. You explicitly mark members with DataMember and can control names and order.

DataContract.cs
// Serializable - all fields [Serializable] public class Person { public string Name; } // DataContract - explicit control [DataContract] public class Person { [DataMember(Name = "fullName", Order = 1)] public string Name { get; set; } [IgnoreDataMember] public string InternalId { get; set; } }
Q90: What is the difference between JSON and XML serialization?

JSON serialization produces lightweight, human-readable text. It's widely used for web APIs and modern applications. It's more compact than XML.

XML serialization produces structured XML documents. It's more verbose but supports schemas and namespaces. Used in legacy systems and SOAP services.

JsonXml.cs
using System.Text.Json; using System.Xml.Serialization; // JSON string json = JsonSerializer.Serialize(obj); // Result: {"Name":"John","Age":30} // XML XmlSerializer serializer = new XmlSerializer(typeof(Person)); // Result: John30

17. C# Threading and Parallel Programming

Q91: What is the difference between Thread and Task?

Thread is a lower-level abstraction representing an OS thread. Creating threads is expensive and requires manual management. It's rarely used directly in modern C#.

Task is a higher-level abstraction representing an asynchronous operation. It uses the thread pool efficiently and is the recommended way to write async/parallel code.

ThreadTask.cs
// Thread - low-level, expensive new Thread(() => DoWork()).Start(); // Task - high-level, efficient Task.Run(() => DoWork()); // Task with async/await await Task.Run(() => DoWork());
Q92: What is the difference between ThreadPool and Thread?

Thread creates a new OS thread for each request. Creating and destroying threads is expensive and can lead to performance issues.

ThreadPool maintains a pool of worker threads that can be reused. It's more efficient as it reuses threads and manages concurrency automatically.

ThreadPool.cs
// Thread - creates new thread new Thread(() => DoWork()).Start(); // ThreadPool - reuses thread from pool ThreadPool.QueueUserWorkItem(state => DoWork()); // Task uses ThreadPool internally Task.Run(() => DoWork);
Q93: What is the difference between Parallel.For and Task.WhenAll?

Parallel.For is used for data parallelism - executing the same operation on a collection of data in parallel. It blocks until all iterations complete.

Task.WhenAll is used to await multiple independent tasks to complete. It's non-blocking and used with async/await.

ParallelWhenAll.cs
// Parallel.For - data parallelism, blocking Parallel.For(0, 100, i => Process(i)); // Task.WhenAll - await multiple tasks, non-blocking Task[] tasks = { Task1(), Task2(), Task3() }; await Task.WhenAll(tasks);
Q94: What is the difference between lock and Monitor?

lock is a syntactic sugar for Monitor.Enter and Monitor.Exit. It provides a simple way to ensure thread safety for a critical section.

Monitor provides more control with Enter, Exit, TryEnter, Wait, and Pulse methods. It's used when you need more advanced synchronization.

LockMonitor.cs
private readonly object _lock = new object(); // lock - simple syntax lock (_lock) { // Critical section } // Monitor - more control Monitor.Enter(_lock); try { // Critical section } finally { Monitor.Exit(_lock); }
Q95: What is the difference between Mutex and Semaphore?

Mutex is a synchronization primitive that allows only one thread to access a resource at a time. It can be named and used across processes.

Semaphore allows a specified number of threads to access a resource concurrently. It's used to limit concurrent access to a resource pool.

MutexSemaphore.cs
// Mutex - one thread at a time using (Mutex mutex = new Mutex(false, "MyMutex")) { mutex.WaitOne(); // Only one thread here mutex.ReleaseMutex(); } // Semaphore - limited concurrent access using (Semaphore semaphore = new Semaphore(3, 3)) { semaphore.WaitOne(); // Max 3 threads here semaphore.Release(); }

18. C# File I/O and Streams

Q96: What is the difference between File and FileInfo?

File is a static class with static methods for file operations. It's simpler but performs security checks on each call.

FileInfo is an instance class that provides file information and operations. It's more efficient for multiple operations as it caches file information.

FileFileInfo.cs
// File - static methods string text = File.ReadAllText("file.txt"); File.WriteAllText("file.txt", "content"); // FileInfo - instance methods FileInfo file = new FileInfo("file.txt"); string text = file.OpenText().ReadToEnd(); file.Delete();
Q97: What is the difference between Stream and FileStream?

Stream is an abstract base class for all streams. It provides a generic view of a sequence of bytes.

FileStream is a concrete implementation for file I/O. It provides file-specific features like file handles, sharing modes, and file attributes.

StreamFileStream.cs
// Stream - abstract base void ProcessStream(Stream stream) { // Read from any stream } // FileStream - file-specific using (FileStream fs = new FileStream("file.txt", FileMode.Open)) { // File-specific operations }
Q98: What is the difference between StreamReader and StringReader?

StreamReader reads characters from a stream (like a file or network stream). It's used for reading text from external sources.

StringReader reads characters from a string. It's used for parsing or processing in-memory strings.

ReaderTypes.cs
// StreamReader - from stream/file using (StreamReader reader = new StreamReader("file.txt")) { string line = reader.ReadLine(); } // StringReader - from string using (StringReader reader = new StringReader("text")) { string line = reader.ReadLine(); }
Q99: What is the difference between MemoryStream and BufferedStream?

MemoryStream stores data in memory as a byte array. It's used for in-memory operations and doesn't involve disk I/O.

BufferedStream wraps another stream and adds a buffer to reduce the number of I/O operations. It improves performance for small, frequent reads/writes.

MemoryBufferedStream.cs
// MemoryStream - in-memory using (MemoryStream ms = new MemoryStream()) { ms.Write(data, 0, data.Length); } // BufferedStream - buffered I/O using (FileStream fs = new FileStream("file.txt", FileMode.Open)) using (BufferedStream bs = new BufferedStream(fs)) { bs.Read(buffer, 0, buffer.Length); }
Q100: What is the difference between BinaryReader/BinaryWriter and TextReader/TextWriter?

BinaryReader/BinaryWriter read and write primitive types in binary format. They're more compact and faster but not human-readable.

TextReader/TextWriter read and write text data. They're human-readable and handle encoding but are less compact.

BinaryTextWriter.cs
// BinaryWriter - binary format using (BinaryWriter writer = new BinaryWriter(File.OpenWrite("data.bin"))) { writer.Write(42); // Binary writer.Write("hello"); } // TextWriter - text format using (TextWriter writer = new StreamWriter("data.txt")) { writer.Write(42); // Text "42" writer.Write("hello"); }
