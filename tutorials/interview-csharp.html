1. C# Fundamentals

Q1: What is C# and what are its key features?

C# (pronounced "C-sharp") is a modern, object-oriented programming language developed by Microsoft as part of the .NET initiative. Key features include:

  • Object-Oriented: Supports encapsulation, inheritance, and polymorphism
  • Type-Safe: Strongly typed language with compile-time type checking
  • Component-Oriented: Supports properties, methods, events, and attributes
  • Garbage Collection: Automatic memory management
  • Modern Language Features: LINQ, async/await, pattern matching, records
  • Cross-Platform: Runs on Windows, Linux, and macOS via .NET Core/.NET 5+
  • Interoperability: Can work with other .NET languages and COM components
Q2: What is the difference between value types and reference types in C#?

Value Types store the actual data directly on the stack:

  • Primitive types (int, float, bool, char, etc.)
  • Struct types
  • Enum types
  • Directly contain their data
  • Copied by value
  • Cannot be null (unless nullable)

Reference Types store a reference to the data on the heap:

  • Class types
  • Interface types
  • Array types
  • Delegate types
  • String types
  • Contain a reference to the actual data
  • Copied by reference
  • Can be null
// Value type example
int x = 10;
int y = x; // Creates a copy
y = 20;
Console.WriteLine(x); // Output: 10 (unchanged)

// Reference type example
class MyClass { public int Value; }
MyClass obj1 = new MyClass { Value = 10 };
MyClass obj2 = obj1; // Copies reference
obj2.Value = 20;
Console.WriteLine(obj1.Value); // Output: 20 (changed)

2. Data Types and Variables

Q4: What are nullable value types in C#?

Nullable value types allow value types to represent null values. They are useful when you need to represent the absence of a value.

// Nullable syntax using ?
int? nullableInt = null;
bool? nullableBool = null;
DateTime? nullableDate = null;

// Using Nullable syntax
Nullable nullableInt2 = null;

// Checking for null
if (nullableInt.HasValue) {
    Console.WriteLine(nullableInt.Value);
}

// Null-coalescing operator
int value = nullableInt ?? 0;

// Null-conditional operator
int? result = nullableInt?.GetValueOrDefault();
Tip: Nullable types are commonly used with database operations where a value might be missing, or when working with optional parameters.

Q6: What is the difference between var and explicit type declaration?

var is type inference where the compiler determines the type from the initializer:

// Using var - type inferred
var name = "John"; // Compiler infers string
var age = 25; // Compiler infers int
var list = new List(); // Compiler infers List

// Explicit type declaration
string name2 = "John";
int age2 = 25;
List list2 = new List();

// var cannot be used without initialization
// var x; // Error: Implicitly-typed variables must be initialized

// var cannot be used for class-level fields
class MyClass {
    // var field = 10; // Error: var cannot be used at class level
    private int field = 10; // Correct
}
Tip: Use var when the type is obvious from the right side. Use explicit types when the type is not obvious or when you want to make the type explicit for clarity.

3. Object-Oriented Programming in C#

Q7: What are the four pillars of OOP in C#?

1. Encapsulation: Bundling data and methods that operate on the data within a single unit (class), and restricting access to the internal state.

public class BankAccount {
    private decimal balance; // Private field (encapsulated)
    
    public void Deposit(decimal amount) {
        if (amount > 0)
            balance += amount;
    }
    
    public decimal GetBalance() {
        return balance;
    }
}

2. Inheritance: Creating new classes based on existing classes, inheriting their properties and methods.

public class Animal {
    public void Eat() {
        Console.WriteLine("Animal is eating");
    }
}

public class Dog : Animal {
    public void Bark() {
        Console.WriteLine("Dog is barking");
    }
}

// Dog inherits Eat() method from Animal

3. Polymorphism: Objects can take many forms. Method overloading (compile-time) and method overriding (runtime).

// Method Overloading (compile-time polymorphism)
public class Calculator {
    public int Add(int a, int b) => a + b;
    public double Add(double a, double b) => a + b;
}

// Method Overriding (runtime polymorphism)
public class Shape {
    public virtual void Draw() {
        Console.WriteLine("Drawing shape");
    }
}

public class Circle : Shape {
    public override void Draw() {
        Console.WriteLine("Drawing circle");
    }
}

4. Abstraction: Hiding implementation details and showing only functionality to the user.

// Abstract class
public abstract class Vehicle {
    public abstract void Start(); // Abstract method
    public void Stop() {
        Console.WriteLine("Vehicle stopped");
    }
}

// Interface
public interface IShape {
    void Draw();
    double GetArea();
}

Q9: What is the difference between virtual, override, and new keywords?

virtual: Indicates a method can be overridden in derived classes.

override: Provides a new implementation for a virtual method in a derived class.

new: Hides a base class method (method hiding, not overriding).

public class Base {
    public virtual void Method1() {
        Console.WriteLine("Base Method1");
    }
    
    public virtual void Method2() {
        Console.WriteLine("Base Method2");
    }
}

public class Derived : Base {
    public override void Method1() {
        Console.WriteLine("Derived Method1 (override)");
    }
    
    public new void Method2() {
        Console.WriteLine("Derived Method2 (new)");
    }
}

// Usage
Base obj = new Derived();
obj.Method1(); // Output: Derived Method1 (polymorphism)
obj.Method2(); // Output: Base Method2 (new doesn't affect polymorphism)

Derived obj2 = new Derived();
obj2.Method1(); // Output: Derived Method1
obj2.Method2(); // Output: Derived Method2
Important: Use override when you want polymorphism. Use new when you want to hide the base method intentionally (usually with a warning).

4. Advanced C# Features

Q10: What are generics in C#?

Generics allow you to define type-safe data structures without committing to specific data types. They provide type safety, performance, and code reusability.

// Generic class
public class Repository {
    private List items = new List();
    
    public void Add(T item) {
        items.Add(item);
    }
    
    public T GetById(int id) {
        return items[id];
    }
}

// Generic method
public T Max(T a, T b) where T : IComparable {
    return a.CompareTo(b) > 0 ? a : b;
}

// Generic interface
public interface IRepository {
    void Add(T item);
    T GetById(int id);
}

// Usage
var intRepo = new Repository();
intRepo.Add(10);

var stringRepo = new Repository();
stringRepo.Add("Hello");

Q12: What are extension methods in C#?

Extension methods allow you to add new methods to existing types without modifying the original type or creating a derived type.

// Extension method for string
public static class StringExtensions {
    public static bool IsNullOrEmpty(this string value) {
        return string.IsNullOrEmpty(value);
    }
    
    public static string ToTitleCase(this string value) {
        return CultureInfo.CurrentCulture.TextInfo
            .ToTitleCase(value.ToLower());
    }
}

// Usage

bool isEmpty = name.IsNullOrEmpty(); // Extension method
string title = name.ToTitleCase(); // "John Doe"

// Extension method for IEnumerable
public static class EnumerableExtensions {
    public static IEnumerable Shuffle(this IEnumerable source) {
        var random = new Random();
        return source.OrderBy(x => random.Next());
    }
}

// Usage
var numbers = new[] { 1, 2, 3, 4, 5 };
var shuffled = numbers.Shuffle();

5. Exception Handling

Q13: What are the different ways to handle exceptions in C#? 
// 1. try-catch-finally
try {
    int result = 10 / 0;
}
catch (DivideByZeroException ex) {
    Console.WriteLine("Division by zero");
}
catch (Exception ex) {
    Console.WriteLine($"General error: {ex.Message}");
}
finally {
    Console.WriteLine("Cleanup code");
}

// 2. throw - re-throw exception
try {
    ProcessData();
}
catch (Exception ex) {
    LogError(ex);
    throw; // Re-throw with original stack trace
}

// 3. throw new - throw new exception
try {
    ProcessData();
}
catch (Exception ex) {
    throw new CustomException("Processing failed", ex);
}

// 4. using - for IDisposable objects
using (var file = new FileStream("test.txt", FileMode.Open)) {
    // Process file
} // Automatically disposes

// 5. try-catch with when (exception filtering)
try {
    ProcessData();
}
catch (Exception ex) when (ex is InvalidOperationException) {
    Console.WriteLine("Invalid operation");
}
Q14: What is the difference between throw and throw ex?

throw: Preserves the original stack trace.

throw ex: Resets the stack trace to the current location.

try {
    MethodThatThrows();
}
catch (Exception ex) {
    throw; // Preserves original stack trace
}

try {
    MethodThatThrows();
}
catch (Exception ex) {
    throw ex; // Resets stack trace - loses original location
}
Tip: Always use throw instead of throw ex to preserve the original stack trace for debugging.

6. Async/Await

Q15: What is async/await and how does it work?

async/await is a pattern for writing asynchronous code that looks and behaves like synchronous code. It prevents blocking the UI thread and improves application responsiveness.

// Synchronous (blocking)
public string GetData() {
    Thread.Sleep(1000); // Blocks thread
    return "Data";
}

// Asynchronous (non-blocking)
public async Task GetDataAsync() {
    await Task.Delay(1000); // Doesn't block thread
    return "Data";
}

// Usage
public async Task ProcessDataAsync() {
    string data = await GetDataAsync(); // Waits without blocking
    Console.WriteLine(data);
}

// Multiple async operations
public async Task ProcessMultipleAsync() {
    var task1 = GetDataAsync();
    var task2 = GetDataAsync();
    
    await Task.WhenAll(task1, task2); // Wait for both
}
Important: Never use async void (except for event handlers). Always return Task or Task for async methods.
Q16: What is the difference between Task and ValueTask?

Task: Reference type, always allocates memory on the heap.

ValueTask: Value type, can avoid heap allocation for synchronous results or already-completed tasks.

// Task - always allocates
public async Task GetValueAsync() {
    await Task.Delay(100);
    return 42;
}

// ValueTask - may avoid allocation
public async ValueTask GetValueOptimizedAsync() {
    // If result is cached, no allocation needed
    if (_cached) {
        return _cachedValue; // Returns without allocation
    }
    
    await Task.Delay(100);
    _cachedValue = 42;
    _cached = true;
    return _cachedValue;
}
Tip: Use ValueTask for high-performance scenarios where methods are called frequently and often return synchronously.

7. LINQ

Q17: What are the different types of LINQ?

1. LINQ to Objects: Query in-memory collections.

var numbers = new[] { 1, 2, 3, 4, 5 };
var evenNumbers = numbers.Where(n => n % 2 == 0);

2. LINQ to XML: Query XML documents.

XDocument doc = XDocument.Load("data.xml");
var elements = doc.Descendants("Item")
    .Where(e => (int)e.Element("Value") > 10);

3. LINQ to SQL: Query SQL databases (Entity Framework).

var users = dbContext.Users
    .Where(u => u.Age > 18)
    .OrderBy(u => u.Name)
    .ToList();

4. LINQ to Entities: Query Entity Framework models.

var orders = context.Orders
    .Include(o => o.Customer)
    .Where(o => o.Total > 1000);
Q18: What is the difference between deferred and immediate execution in LINQ?

Deferred Execution: Query is executed when enumerated (ToList, foreach, etc.).

var query = numbers.Where(n => n > 5); // Not executed yet
numbers.Add(10); // Adding after query definition
var result = query.ToList(); // Executes now, includes 10

Immediate Execution: Query is executed immediately.

var result = numbers.Count(n => n > 5); // Executes immediately
numbers.Add(10); // Adding after execution
// result doesn't include 10
Tip: Methods like ToList(), ToArray(), Count(), First(), Single() cause immediate execution. Where(), Select(), OrderBy() are deferred.

8. Memory Management

Q19: How does garbage collection work in C#?

Garbage Collection (GC) automatically manages memory allocation and deallocation. It works in generations:

  • Generation 0: Short-lived objects (newly allocated)
  • Generation 1: Objects that survived Gen 0 collection
  • Generation 2: Long-lived objects (survived Gen 1 collection)
// GC methods
GC.Collect(); // Force garbage collection (usually not needed)
GC.WaitForPendingFinalizers(); // Wait for finalizers
GC.GetTotalMemory(false); // Get memory usage

// IDisposable pattern for manual cleanup
public class ResourceHolder : IDisposable {
    private IntPtr unmanagedResource;
    private bool disposed = false;
    
    public void Dispose() {
        Dispose(true);
        GC.SuppressFinalize(this);
    }
    
    protected virtual void Dispose(bool disposing) {
        if (!disposed) {
            if (disposing) {
                // Dispose managed resources
            }
            // Free unmanaged resources
            CloseHandle(unmanagedResource);
            disposed = true;
        }
    }
    
    ~ResourceHolder() {
        Dispose(false);
    }
}
Q20: What is the using statement and why is it important?

The using statement ensures that IDisposable objects are properly disposed, even if an exception occurs.

// Traditional way
FileStream file = null;
try {
    file = new FileStream("test.txt", FileMode.Open);
    // Process file
}
finally {
    file?.Dispose();
}

// Using statement (recommended)
using (var file = new FileStream("test.txt", FileMode.Open)) {
    // Process file
} // Automatically calls Dispose()

// Multiple using statements (C# 8+)
using var file1 = new FileStream("file1.txt", FileMode.Open);
using var file2 = new FileStream("file2.txt", FileMode.Open);
// Both disposed at end of scope
Important: Always use using statements for objects that implement IDisposable to prevent resource leaks.

Advanced C# Concepts

Q21: What are delegates and events in C#?

Delegates are type-safe function pointers that reference methods with a specific signature. Events are a way for a class to provide notifications to clients when something happens.

// Delegates
public delegate void NotifyDelegate(string message);

// Event
public class Publisher {
    public event NotifyDelegate OnNotify;
    public void Notify(string message) => OnNotify?.Invoke(message);
}
Q22: What is Dependency Injection (DI) in C#?

Dependency Injection is a design pattern that implements Inversion of Control (IoC). It allows creating dependent objects outside of a class and provides those objects to a class through constructor injection.

// Constructor Injection

    private readonly IRepository _repository;
    public UserService(IRepository repository) => _repository = repository;
}

// Using DI Container
builder.Services.AddScoped();

Q45: What is the difference between Task and ValueTask?

Q46: What is the difference between lock and Monitor?

}

// Monitor (more control)

try {
    // Critical section
} finally {
    Monitor.Exit(_lock);
}

Q48: What is the difference between static class and regular class?

Static class: Cannot be instantiated, all members static, cannot inherit or be inherited. Regular class: Can be instantiated, can have instance and static members, supports inheritance.

    public static int Add(int a, int b) => a + b;
}

Q49: What is the difference between is and as operators?

is: Checks if object is of specific type, returns bool. as: Attempts to cast, returns null if fails instead of exception.

Q50: What is the difference between property and field?

Field: Variable in class, can be public/private. Property: Encapsulated field with get/set accessors, can add logic, supports data binding.

Q53: What is the difference between Dispose and Finalize?

Dispose: Called explicitly by user, deterministically releases resources. Finalize: Called by GC, non-deterministic, for unmanaged resources.

public class Resource : IDisposable {
    public void Dispose() {
        // Release managed and unmanaged resources
    }
    
    ~Resource() {
        // Finalizer - called by GC
        Dispose(false);
    }
}
Q54: What is the difference between struct and record?

struct: Value type, mutable by default, no built-in equality comparison. record: Reference type (or record struct), immutable, built-in equality comparison, with expressions.

Q55: What is the difference between nameof and typeof?

Q56: What is the difference between null-coalescing (??) and null-conditional (?.)?

Q57: What is the difference between Tuple and ValueTuple?
