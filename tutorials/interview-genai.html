1. Generative AI Fundamentals

Q1: What is Generative AI?

Generative AI is a type of artificial intelligence that can create new content, including text, images, audio, and code. Unlike traditional AI that classifies or predicts, generative AI generates novel outputs based on learned patterns from training data.

Key Characteristics:

  • Creation: Generates new, original content
  • Pattern-based: Learns patterns from training data
  • Probabilistic: Uses probability distributions for generation
  • Multi-modal: Can handle text, images, audio, video
Q2: What is the difference between discriminative and generative models?
Aspect Discriminative Generative
Goal Distinguish between classes Generate new samples
Output P(Y|X) P(X,Y) or P(X)
Examples Classification, regression GANs, VAEs, LLMs
Q3: What are the main types of generative models?

Types of Generative Models:

  • Autoregressive Models: GPT, generate token by token
  • VAEs: Variational Autoencoders, encode-decode architecture
  • GANs: Generative Adversarial Networks, generator vs discriminator
  • Diffusion Models: Gradual noise addition and removal
  • Flow-based Models: Transform simple distributions to complex ones
Q4: What is a GAN?

Generative Adversarial Network (GAN) consists of two neural networks: a generator that creates fake data and a discriminator that distinguishes real from fake. They compete in a game-theoretic framework.

gan.py
# Simple GAN structure with PyTorch import torch import torch.nn as nn class Generator(nn.Module): def __init__(self): super().__init__() self.model = nn.Sequential( nn.Linear(100, 256), nn.ReLU(), nn.Linear(256, 784), nn.Tanh() ) def forward(self, z): return self.model(z) class Discriminator(nn.Module): def __init__(self): super().__init__() self.model = nn.Sequential( nn.Linear(784, 256), nn.ReLU(), nn.Linear(256, 1), nn.Sigmoid() ) def forward(self, x): return self.model(x)
Q5: What is a VAE?

Variational Autoencoder (VAE) is a generative model that learns to encode data into a latent space and decode back. It uses probabilistic encoding to enable smooth interpolation and generation.

vae.py
# VAE encoder class VAEEncoder(nn.Module): def __init__(self): super().__init__() self.fc1 = nn.Linear(784, 400) self.fc_mu = nn.Linear(400, 20) self.fc_logvar = nn.Linear(400, 20) def forward(self, x): h = torch.relu(self.fc1(x)) mu = self.fc_mu(h) logvar = self.fc_logvar(h) return mu, logvar
Q6: What is a diffusion model?

Diffusion Model generates data by gradually adding noise to training data and learning to reverse the process. It's the foundation for models like Stable Diffusion and DALL-E.

Key Components:

  • Forward Process: Gradually add Gaussian noise
  • Reverse Process: Learn to denoise step by step
  • Training: Predict noise at each timestep
  • Inference: Generate from pure noise

2. Large Language Models

Q7: What is a Large Language Model (LLM)?

Large Language Model (LLM) is a neural network trained on vast amounts of text data to understand and generate human-like text. Examples include GPT, BERT, LLaMA, and Claude.

Characteristics:

  • Scale: Billions of parameters
  • Training Data: Trillions of tokens
  • Capabilities: Translation, summarization, code generation
  • Few-shot Learning: Learn from few examples

Q9: What is the transformer architecture?

Transformer is a neural network architecture introduced in "Attention Is All You Need" (2017). It uses self-attention mechanisms to process sequential data in parallel, enabling efficient training of large models.

Key Components:

  • Self-Attention: Captures relationships between all tokens
  • Multi-head Attention: Multiple attention heads
  • Position Encoding: Adds positional information
  • Feed-forward Networks: Non-linear transformations
Q10: What is self-attention?

Self-Attention allows each token to attend to all other tokens in the sequence, computing weighted sums based on similarity scores.

self_attention.py
# Self-attention mechanism import torch import torch.nn.functional as F def self_attention(query, key, value): # Q, K, V: (batch, seq_len, d_model) scores = torch.matmul(query, key.transpose(-2, -1)) scores = scores / torch.sqrt(torch.tensor(key.size(-1), dtype=torch.float32)) attention_weights = F.softmax(scores, dim=-1) output = torch.matmul(attention_weights, value) return output
Q11: What is multi-head attention?

Multi-head Attention runs multiple self-attention operations in parallel, each learning different aspects of relationships. The outputs are concatenated and linearly transformed.

multi_head_attention.py
# Multi-head attention class MultiHeadAttention(nn.Module): def __init__(self, d_model, num_heads): super().__init__() self.num_heads = num_heads self.d_model = d_model self.d_k = d_model // num_heads self.W_q = nn.Linear(d_model, d_model) self.W_k = nn.Linear(d_model, d_model) self.W_v = nn.Linear(d_model, d_model) self.W_o = nn.Linear(d_model, d_model)
Q12: What is position encoding?

Position Encoding adds positional information to tokens since transformers process all positions in parallel. It can be sinusoidal (fixed) or learned embeddings.

positional_encoding.py
# Sinusoidal position encoding import math def positional_encoding(seq_len, d_model): pe = torch.zeros(seq_len, d_model) position = torch.arange(0, seq_len).unsqueeze(1).float() div_term = torch.exp(torch.arange(0, d_model, 2).float() * -(math.log(10000.0) / d_model)) pe[:, 0::2] = torch.sin(position * div_term) pe[:, 1::2] = torch.cos(position * div_term) return pe

3. Training and Fine-tuning

Q14: What is pre-training?

Pre-training is the initial training phase where a model learns from large, unlabeled datasets. For LLMs, this involves next-token prediction or masked language modeling on massive text corpora.

Q15: What is fine-tuning?

Fine-tuning is the process of adapting a pre-trained model to a specific task or domain by training on a smaller, task-specific dataset.

finetuning.py

Q16: What is instruction tuning?

Instruction Tuning fine-tunes models on instruction-response pairs to improve their ability to follow natural language instructions. It's crucial for building helpful AI assistants.

Q17: What is RLHF?

Reinforcement Learning from Human Feedback (RLHF) uses human preferences to align model outputs with human values. It involves training a reward model from human feedback and using it to optimize the LLM.

RLHF Steps:

  1. Collect human comparisons of model outputs
  2. Train a reward model to predict human preferences
  3. Use PPO to optimize the LLM with the reward model
Q18: What is LoRA?

LoRA (Low-Rank Adaptation) is a parameter-efficient fine-tuning method that adds trainable low-rank matrices to existing weights, reducing the number of trainable parameters significantly.

lora.py
# LoRA with PEFT from peft import LoraConfig, get_peft_model lora_config = LoraConfig( r=8, lora_alpha=32, target_modules=["q_proj", "v_proj"], lora_dropout=0.05 ) model = get_peft_model(model, lora_config) model.print_trainable_parameters()

Q20: What is quantization?

Quantization reduces model size and memory usage by using lower precision (e.g., 8-bit instead of 32-bit) for weights and activations. It enables running large models on consumer hardware.

# Quantization with bitsandbytes

    bnb_8bit_compute_dtype=torch.float16
)

model = AutoModelForCausalLM.from_pretrained(
    "model_name",
    quantization_config=quantization_config
)

4. Prompt Engineering

Q21: What is prompt engineering?

Prompt Engineering is the art of crafting effective prompts to elicit desired outputs from language models. It involves techniques like few-shot learning, chain-of-thought, and role-playing.

Q22: What is zero-shot learning?

Zero-shot Learning prompts the model to perform a task without providing any examples in the prompt. The model relies on its pre-training knowledge.

# Zero-shot example
prompt = "Classify the sentiment: 'I love this product!'"
response = model.generate(prompt)
Q23: What is few-shot learning?

Few-shot Learning provides a few examples in the prompt to guide the model's behavior. It's more effective than zero-shot for complex tasks.

# Few-shot example
prompt = """
Classify sentiment:
"I hate this" -> Negative
"I love this" -> Positive
"This is okay" -> Neutral
"This product is amazing" -> 
"""
Q24: What is chain-of-thought prompting?

Chain-of-Thought (CoT) prompting encourages the model to show its reasoning process step-by-step, improving performance on complex reasoning tasks.

# Chain-of-thought example
prompt = """
Solve: 23 * 17
Let's think step by step:
23 * 17 = 23 * (10 + 7)
= 23 * 10 + 23 * 7
= 230 + 161
= 391
"""
Q25: What is role prompting?

Role Prompting assigns a specific role or persona to the model, which influences its response style and content. Examples include "You are a helpful assistant" or "You are a Python expert."

Q26: What is temperature in LLM generation?

Temperature controls the randomness of generation. Lower temperature (0.1-0.3) produces more deterministic outputs, while higher temperature (0.7-1.0) produces more diverse and creative outputs.

# Temperature control
response = model.generate(
    prompt,
    temperature=0.7,
    max_tokens=100
)
Q27: What is top-p sampling?

Top-p (Nucleus) Sampling samples from the smallest set of tokens whose cumulative probability exceeds a threshold p. It balances quality and diversity better than pure sampling.

# Top-p sampling
response = model.generate(
    prompt,
    top_p=0.9,
    do_sample=True
)

5. RAG and Retrieval

Q28: What is RAG?

Retrieval-Augmented Generation (RAG) combines retrieval of relevant documents with generation. It retrieves context from a knowledge base and uses it to generate more accurate and factual responses.

RAG Benefits:

  • Accuracy: Reduces hallucinations
  • Up-to-date: Can access current information
  • Explainable: Sources can be cited
  • Domain-specific: Can use custom knowledge bases
Q29: What is vector embedding?

Vector Embedding represents text, images, or other data as dense vectors in a high-dimensional space. Similar items have similar embeddings, enabling semantic search.

# Text embedding with sentence-transformers
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')
embeddings = model.encode(["Hello world", "Hi there"])
similarity = model.similarity(embeddings[0], embeddings[1])
Q30: What is a vector database?

Vector Database stores and indexes vector embeddings for efficient similarity search. Examples include Pinecone, Weaviate, Chroma, and FAISS.

# Chroma vector database
import chromadb

client = chromadb.Client()
collection = client.create_collection("documents")

collection.add(
    documents=["Hello world", "Hi there"],
    embeddings=[[0.1, 0.2], [0.3, 0.4]],
    ids=["doc1", "doc2"]
)

results = collection.query(
    query_embeddings=[[0.15, 0.25]],
    n_results=1
)
Q31: What is semantic search?

Semantic Search finds documents based on meaning rather than keyword matching. It uses vector embeddings to measure semantic similarity between queries and documents.

Q32: What is the difference between keyword search and semantic search?
Aspect Keyword Search Semantic Search
Matching Exact word matches Meaning similarity
Flexibility Rigid Flexible
Understanding Synonyms not recognized Understands context
Q33: What is chunking in RAG?

Chunking splits large documents into smaller pieces for embedding and retrieval. Good chunking balances context preservation with retrieval precision.

# Text chunking
def chunk_text(text, chunk_size=500, overlap=50):
    chunks = []
    for i in range(0, len(text), chunk_size - overlap):
        chunk = text[i:i + chunk_size]
        chunks.append(chunk)
    return chunks
Q34: What is reranking in RAG?

Reranking refines initial retrieval results by scoring retrieved documents more precisely, often using cross-encoders or more sophisticated models to improve relevance.

6. GenAI Applications

Q35: What are common GenAI applications?

GenAI Applications:

  • Content Creation: Writing, coding, image generation
  • Customer Service: Chatbots, virtual assistants
  • Education: Tutoring, explanation generation
  • Healthcare: Medical report generation, diagnosis assistance
  • Finance: Report generation, analysis
Q36: What is text generation?

Text Generation creates coherent text based on a prompt. Applications include article writing, story generation, code generation, and summarization.

# Text generation with Hugging Face
from transformers import pipeline

generator = pipeline("text-generation", model="gpt2")
output = generator("Once upon a time", max_length=100)
Q37: What is image generation?

Image Generation creates images from text descriptions using diffusion models like Stable Diffusion or DALL-E.

# Image generation with Stable Diffusion
from diffusers import StableDiffusionPipeline

pipe = StableDiffusionPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5"
)
image = pipe("A beautiful sunset over mountains").images[0]
Q38: What is code generation?

Code Generation creates code from natural language descriptions. Models like GitHub Copilot and CodeLlama are trained on vast code repositories.

# Code generation
from transformers import AutoModelForCausalLM, AutoTokenizer

model = AutoModelForCausalLM.from_pretrained("bigcode/santacoder")
prompt = "def fibonacci(n):"
output = model.generate(prompt, max_tokens=100)
Q39: What is summarization?

Summarization condenses long documents into concise summaries while preserving key information. It can be extractive (selecting sentences) or abstractive (generating new text).

# Summarization
summarizer = pipeline("summarization")
summary = summarizer(long_text, max_length=100)
Q40: What is translation?

Translation converts text from one language to another. Modern translation uses sequence-to-sequence models like T5 or mBART.

# Translation
translator = pipeline("translation", model="Helsinki-NLP/opus-mt-en-fr")
result = translator("Hello world", src_lang="en", tgt_lang="fr")

7. GenAI Ethics and Safety

Q41: What is hallucination in LLMs?

Hallucination occurs when LLMs generate plausible but factually incorrect information. It's a significant challenge for reliable AI systems.

Mitigation Strategies:

  • RAG: Provide relevant context
  • Few-shot: Provide accurate examples
  • Fact-checking: Verify outputs
  • Temperature: Use lower temperature
Q42: What is AI alignment?

AI Alignment ensures AI systems behave in accordance with human values and intentions. It's crucial for safe and beneficial AI development.

Q43: What is content moderation?

Content Moderation filters harmful content from AI outputs. It involves detecting and blocking hate speech, violence, self-harm, and other inappropriate content.

# Content moderation with OpenAI
from openai import Moderation

moderation = Moderation()
result = moderation.create(input="Your text here")
if result.results[0].flagged:
    print("Content flagged as inappropriate")
Q44: What is data privacy in GenAI?

Data Privacy in GenAI involves protecting sensitive information in training data and outputs. Techniques include differential privacy, data anonymization, and secure training environments.

Q45: What is bias in GenAI?

Bias in GenAI occurs when models reflect and amplify biases present in training data. It can lead to unfair or discriminatory outputs.

Bias Mitigation:

  • Diverse Training Data: Ensure representation
  • Fairness Metrics: Measure and monitor bias
  • Debiasing Techniques: Algorithmic interventions
  • Human Review: Regular audits
Q46: What is responsible AI?

Responsible AI is an approach to AI development that considers ethical, social, and environmental impacts. It includes fairness, transparency, accountability, and privacy.
