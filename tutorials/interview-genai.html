1. LLM Fundamentals

Q1: What is a Large Language Model (LLM)?

LLM: Neural network trained on massive text data to understand and generate human-like text.

Key Characteristics:

  • Billions of parameters
  • Trained on internet-scale data
  • Can perform various NLP tasks
  • Uses transformer architecture

Examples: GPT-4, Claude, Gemini, LLaMA.

Q2: What is the difference between Generative AI and Traditional AI?
Aspect Generative AI Traditional AI
Output Creates new content Classifies/predicts existing data
Training Self-supervised on unlabeled data Supervised on labeled data
Use Cases Content creation, synthesis Classification, prediction
Examples GPT, DALL-E, Midjourney Random Forest, SVM, CNN
Q3: What are embeddings and why are they useful?

Embeddings: Dense vector representations that capture semantic meaning of text.

Benefits:

  • Capture semantic relationships
  • Enable similarity search
  • Reduce dimensionality
  • Transfer learning across tasks

Applications: RAG, semantic search, recommendation systems, clustering.

Q4: What is a vector database?

Vector Database: Database optimized for storing and searching high-dimensional vectors.

Key Features:

  • Approximate nearest neighbor search
  • High-dimensional indexing
  • Scalable storage
  • Real-time similarity search

Examples: Pinecone, Weaviate, Chroma, Milvus.

2. Transformers & Architecture

Q5: What is the transformer architecture?

Transformer: Neural network architecture using self-attention mechanism to process sequential data.

Key Components:

  • Self-attention layers
  • Positional encoding
  • Multi-head attention
  • Feed-forward networks

Advantages: Parallel processing, long-range dependencies, no recurrence.

Q6: What is the attention mechanism?

Attention: Mechanism that allows model to focus on relevant parts of input when producing output.

How it works:

  1. Compute query, key, value vectors
  2. Calculate attention scores (query · key)
  3. Apply softmax to get weights
  4. Weighted sum of values

Types: Self-attention, cross-attention, multi-head attention.

Q7: What is the difference between encoder-only, decoder-only, and encoder-decoder models?
Type Architecture Use Cases Examples
Encoder-only Transformer encoder Understanding, classification BERT, RoBERTa
Decoder-only Transformer decoder Generation, completion GPT, LLaMA
Encoder-decoder Both encoder and decoder Translation, summarization T5, BART
Q8: What is positional encoding?

Positional Encoding: Technique to inject position information into transformer models.

Why needed: Transformers process all tokens in parallel, losing sequential order.

Types:

  • Sinusoidal (original transformer)
  • Learned embeddings
  • Relative position encoding
  • RoPE (Rotary Position Embedding)

3. Prompt Engineering

Q9: What is prompt engineering?

Prompt Engineering: Art of crafting effective prompts to get desired outputs from LLMs.

Techniques:

  • Zero-shot prompting
  • Few-shot prompting
  • Chain-of-thought prompting
  • Role-playing
  • Context injection
Q10: What is the difference between zero-shot, one-shot, and few-shot prompting?

Zero-shot: No examples provided, only task description.

One-shot: One example provided to guide the model.

Few-shot: Multiple examples provided (typically 3-5) for better guidance.

Tradeoff: More examples = better performance but higher token cost and latency.

Q11: What is chain-of-thought prompting?

Chain-of-Thought (CoT): Prompting technique that encourages models to show reasoning steps.

Benefits:

  • Improves complex reasoning
  • Reduces errors
  • Provides interpretability

Example: "Let's think step by step to solve this problem."

Q12: What is temperature in LLM generation?

Temperature: Hyperparameter controlling randomness in token selection.

Low temperature (0.1-0.3): More deterministic, focused outputs.

High temperature (0.7-1.0): More random, creative outputs.

Use cases: Low for factual answers, high for creative writing.

4. RAG & Vector Databases

Q13: What is RAG (Retrieval-Augmented Generation)?

RAG: Framework that combines retrieval systems with LLMs to improve accuracy and reduce hallucinations.

Components:

  1. Document ingestion and chunking
  2. Embedding generation
  3. Vector storage
  4. Retrieval (similarity search)
  5. Generation with retrieved context
Q14: When should you use RAG vs fine-tuning?

Use RAG when:

  • Need access to dynamic information
  • Want to reduce hallucinations
  • Need to cite sources
  • Data changes frequently

Use fine-tuning when:

  • Need specific domain knowledge
  • Want to change model behavior/style
  • Have large domain-specific dataset
  • Need lower latency
Q15: What is semantic search vs keyword search?

Keyword Search: Matches exact words or phrases.

Semantic Search: Matches meaning and intent using embeddings.

Advantages of semantic:

  • Handles synonyms
  • Understands context
  • Better for natural language queries

5. Fine-Tuning & RLHF

Q16: What is fine-tuning?

Fine-tuning: Process of adapting a pre-trained model to specific task or domain.

Types:

  • Full fine-tuning (all parameters)
  • LoRA (Low-Rank Adaptation)
  • QLoRA (Quantized LoRA)
  • Adapter layers

Benefits: Improved performance on specific tasks, reduced latency, smaller model size.

Q17: What is LoRA?

LoRA (Low-Rank Adaptation): Parameter-efficient fine-tuning technique.

How it works:

  • Freezes pre-trained weights
  • Adds trainable low-rank decomposition matrices
  • Dramatically reduces trainable parameters

Benefits: Faster training, less memory, easier to deploy multiple adapters.

Q18: What is RLHF (Reinforcement Learning from Human Feedback)?

RLHF: Technique to align LLMs with human preferences using reinforcement learning.

Three stages:

  1. SFT: Supervised fine-tuning on human demonstrations
  2. Reward Model: Train model to predict human preferences
  3. PPO: Optimize policy using reward model

Goal: Make models helpful, harmless, and honest.

Q19: What is hallucination in LLMs?

Hallucination: When LLM generates false or fabricated information confidently.

Causes:

  • Training data limitations
  • Model overconfidence
  • Lack of grounding

Mitigation: RAG, fact-checking, uncertainty quantification, human oversight.

6. AI Safety & Ethics

Q20: What is prompt injection?

Prompt Injection: Attack where users manipulate prompts to bypass safety measures.

Examples:

  • Jailbreaking (bypassing restrictions)
  • DAN (Do Anything Now) prompts
  • Role-playing attacks

Defense: Input validation, output filtering, adversarial training.

Q21: What are the ethical considerations in generative AI?

Key Ethical Issues:

  • Copyright and data ownership
  • Deepfakes and misinformation
  • Bias and fairness
  • Privacy concerns
  • Environmental impact
  • Job displacement
Q22: What is AI alignment?

AI Alignment: Field focused on ensuring AI systems act in accordance with human values and intentions.

Challenges:

  • Defining human values
  • Specification problems
  • Robustness to distribution shift
  • Outer alignment vs inner alignment

Q29: What is the difference between instruction tuning and RLHF?

Instruction Tuning: Fine-tuning model on instruction-response pairs to follow instructions better.

RLHF (Reinforcement Learning from Human Feedback): Using human feedback to train reward model, then optimizing policy with RL.

Comparison:

  • Instruction tuning: Supervised learning, faster, simpler
  • RLHF: Reinforcement learning, slower, more complex, better alignment

Use case: Often used together - instruction tuning first, then RLHF for fine alignment.

Q30: What is the difference between zero-shot, one-shot, and few-shot learning?
Learning Type Examples Description
Zero-shot 0 Model uses pretraining knowledge without examples
One-shot 1 Model learns from single example
Few-shot Few (2-10) Model learns from multiple examples

Use case: Few-shot learning typically works best for LLMs due to their pretraining.

Q31: What is the difference between chain-of-thought and standard prompting?

Standard Prompting: Direct question-answer format without intermediate reasoning.

Chain-of-Thought (CoT): Prompting model to show step-by-step reasoning before final answer.

Benefits of CoT:

  • Improves reasoning on complex tasks
  • Reduces errors
  • More interpretable

Use case: Math, logic, multi-step reasoning tasks.

Q32: What is the difference between self-consistency and chain-of-thought?

Chain-of-Thought: Single reasoning path to answer.

Self-Consistency: Generate multiple reasoning paths, choose most consistent answer.

Process:

  1. Generate multiple CoT responses
  2. Aggregate answers (majority vote)
  3. Select most consistent answer

Benefits: Reduces randomness, improves accuracy on reasoning tasks.

Q33: What is the difference between tree-of-thought and chain-of-thought?

Chain-of-Thought: Linear reasoning path.

Tree-of-Thought (ToT): Explores multiple reasoning branches like a tree, can backtrack.

Benefits of ToT:

  • Can explore alternative solutions
  • Can backtrack from wrong paths
  • Better for complex problem-solving

Tradeoff: More computationally expensive than CoT.

Q34: What is the difference between ReAct and chain-of-thought?

Chain-of-Thought: Pure reasoning without external actions.

ReAct (Reasoning + Acting): Interleaves reasoning with external tool use/actions.

Process:

  1. Reason about task
  2. Take action (use tool)
  3. Observe result
  4. Repeat until done

Use case: Agents that need to interact with external systems (APIs, databases).

Q37: What is the difference between hybrid retrieval and dense retrieval?

Dense Retrieval: Uses embeddings for semantic similarity search.

Hybrid Retrieval: Combines sparse (keyword) and dense (semantic) retrieval.

Benefits of Hybrid:

  • Best of both worlds - keyword precision + semantic understanding
  • Handles rare terms better
  • More robust to different query types

Methods: Score fusion, reciprocal rank fusion (RRF).

Q40: What is the difference between chunking strategies in RAG?

Fixed-size chunking: Split text into fixed-size chunks (e.g., 512 tokens). Simple but may break context.

Semantic chunking: Split at semantic boundaries (paragraphs, sections). Better context preservation.

Recursive chunking: Hierarchical chunking for different granularity levels.

Use case: Semantic chunking generally performs better for RAG as it preserves context better.

Q42: What is the difference between LoRA and QLoRA?

LoRA: Low-Rank Adaptation - adds trainable low-rank matrices to model.

QLoRA: Quantized LoRA - combines LoRA with 4-bit quantization.

Benefits of QLoRA:

  • Even lower memory requirements
  • Can fine-tune large models on single GPU
  • Maintains performance close to full fine-tuning

Use case: Fine-tuning very large models (70B+) with limited resources.

Q44: What is the difference between P-tuning and prompt tuning?

Prompt Tuning: Adds trainable prompt embeddings at input layer.

P-tuning: Uses LSTM to generate prompt embeddings (more flexible).

P-tuning v2: Improved version with deep prompt tuning across all layers.

Comparison: P-tuning v2 generally outperforms prompt tuning, especially for small models.

Q48: What is the difference between PagedAttention and standard attention?

Standard Attention: Stores entire KV cache in memory, limits batch size and sequence length.

PagedAttention: Pages KV cache like virtual memory, allows dynamic allocation.

Benefits of PagedAttention:

  • Higher batch sizes
  • Longer sequences
  • Better memory utilization
  • Reduced memory fragmentation

Use case: vLLM for high-throughput LLM serving.

Q49: What is the difference between Flash Attention and standard attention?

Standard Attention: Materializes attention matrix, O(n²) memory and time complexity.

Flash Attention: IO-aware exact attention, avoids materializing full matrix.

Benefits of Flash Attention:

  • Linear memory complexity
  • Faster computation (2-4x)
  • Exact attention (no approximation)

Use case: Training and inference of long-context models.

Q50: What is the difference between speculative decoding and standard decoding?

Standard Decoding: Generate tokens one at a time sequentially.

Speculative Decoding: Use small model to draft tokens, large model to verify in parallel.

Benefits:

  • Faster generation (2-3x speedup)
  • Same quality as original model
  • No accuracy loss

Use case: High-throughput text generation serving.

Q52: What is the difference between top-k and top-p sampling?

Top-k Sampling: Sample from top-k most likely tokens.

Top-p (Nucleus) Sampling: Sample from smallest set of tokens with cumulative probability ≥ p.

Comparison:

  • Top-k: Fixed number of candidates, simpler
  • Top-p: Dynamic number, adapts to distribution

Use case: Often used together (top-k + top-p) for better diversity.

Q53: What is the difference between temperature and top-p sampling?

Temperature: Controls randomness by scaling logits. Higher = more random, lower = more deterministic.

Top-p: Controls diversity by limiting token set.

Comparison:

  • Temperature: Affects probability distribution shape
  • Top-p: Affects which tokens are considered

Use case: Often used together for controlled generation.

Q54: What is the difference between repetition penalty and presence penalty?

Repetition Penalty: Penalizes repeated tokens (including consecutive and non-consecutive).

Presence Penalty: Penalizes tokens that have already appeared (regardless of repetition).

Use case:

  • Repetition penalty: Reduce word/phrase repetition
  • Presence penalty: Encourage topic diversity

Q57: What is the difference between function calling and tool use?

Function Calling: Structured way for LLM to call external functions (OpenAI format).

Tool Use: General concept of LLM using external tools/APIs.

Relationship: Function calling is a specific implementation of tool use.

Benefits of Function Calling:

  • Structured output
  • Type safety
  • Better error handling
Q58: What is the difference between agents and chains?

Chains: Fixed sequence of operations. Deterministic flow.

Agents: Autonomous decision-making. Can choose which tools/actions to take.

Comparison:

  • Chains: Simpler, predictable, less flexible
  • Agents: More complex, adaptive, can handle dynamic tasks

Use case: Chains for fixed workflows, agents for dynamic problem-solving.

Q59: What is the difference between ReAct agent and Plan-and-Solve agent?

ReAct Agent: Interleaves reasoning and actions iteratively.

Plan-and-Solve Agent: Plans entire sequence first, then executes.

Comparison:

  • ReAct: More flexible, can adapt during execution
  • Plan-and-Solve: More efficient for known tasks, less adaptive

Use case: ReAct for exploration, Plan-and-Solve for structured tasks.
