1. AI Fundamentals

Q1: What is Artificial Intelligence?

Artificial Intelligence (AI) is the simulation of human intelligence processes by machines, especially computer systems. These processes include learning, reasoning, problem-solving, perception, and self-correction. AI systems can perform tasks that typically require human intelligence such as visual perception, speech recognition, decision-making, and translation between languages.

Types of AI:

  • Narrow AI (Weak AI): Designed for specific tasks (e.g., Siri, chess AI, image recognition). It excels in its domain but cannot perform tasks outside its specialization.
  • General AI (Strong AI):strong> Human-level intelligence across all tasks. It would have the ability to understand, learn, and apply knowledge across different domains, similar to human intelligence.
  • Super AI: Surpasses human intelligence in all aspects. It would be self-aware, conscious, and capable of solving problems that humans cannot.

AI Approaches:

  • Symbolic AI: Uses logic rules and knowledge representation
  • Connectionist AI: Uses neural networks to learn from data
  • Evolutionary AI: Uses genetic algorithms to evolve solutions
Q2: What is the difference between AI, ML, and Deep Learning?

AI is the broad field of making machines smart. ML is a subset of AI that uses data to learn patterns. Deep Learning is a subset of ML that uses neural networks with multiple layers to learn complex patterns automatically.

Concept Scope Example
AI Broad field of making machines smart Robotics, expert systems, game playing
ML Subset of AI using data to learn Regression, classification, clustering
Deep Learning Subset of ML using neural networks CNNs, RNNs, transformers, GANs

Key Differences:

  • AI includes any technique enabling computers to mimic human intelligence, including rule-based systems
  • ML specifically focuses on algorithms that learn from data without explicit programming
  • Deep Learning uses multi-layered neural networks to automatically discover features from raw data

Q4: What is the Turing Test?

Turing Test is a test of a machine's ability to exhibit intelligent behavior equivalent to, or indistinguishable from, that of a human. Proposed by Alan Turing in 1950 in his paper "Computing Machinery and Intelligence," a machine passes the test if it can converse with humans without being identified as a machine. The test involves a human evaluator who engages in natural language conversations with both humans and machines without knowing which is which. If the evaluator cannot reliably tell the machine from the human, the machine is said to have passed the test.

Key Aspects:

  • Imitation Game: The original test setup where a judge decides between human and machine
  • Conversation: The test relies on natural language conversation as the interface
  • Limitations: Doesn't test physical capabilities, only linguistic intelligence
  • Modern Relevance: Inspired chatbots and conversational AI development

RL is inspired by behavioral psychology and is used in game playing, robotics, autonomous vehicles, and recommendation systems.

import numpy as np # Q-table initialization Q = np.zeros((state_space, action_space)) # Q-learning update formula # Q(s,a) = Q(s,a) + α * [r + γ * max(Q(s',a')) - Q(s,a)] def update_q_table(state, action, reward, next_state, alpha=0.1, gamma=0.95): best_next_action = np.argmax(Q[next_state]) td_target = reward + gamma * Q[next_state, best_next_action] td_error = td_target - Q[state, action] Q[state, action] += alpha * td_error return td_error

2. Machine Learning Basics

Q7: What is overfitting and how to prevent it?

Overfitting occurs when a model learns the training data too well, including noise and random fluctuations, and performs poorly on new, unseen data. It happens when the model is too complex relative to the amount of training data. The model memorizes the training data instead of learning generalizable patterns.

Signs of Overfitting:

  • High training accuracy but low test accuracy
  • Large gap between training and validation performance
  • Model performs well on training data but fails on new data

Prevention Methods:

  • Cross-validation: Use k-fold cross-validation to get a more accurate estimate of model performance
  • Regularization: Add L1 (Lasso) or L2 (Ridge) regularization to penalize complex models
  • Early stopping: Stop training when validation error starts increasing
  • Dropout: Randomly drop neurons during training to prevent co-adaptation
  • Data augmentation: Increase training data diversity through transformations
  • Reduce model complexity: Use fewer layers or neurons
  • Ensemble methods: Combine multiple models to reduce variance

Q8: What is underfitting?

Underfitting occurs when a model is too simple to capture the underlying pattern in the data. It performs poorly on both training and test data because it cannot learn the complexity of the problem. Underfitting is the opposite of overfitting and is often caused by using an overly simple model, insufficient training time, or inadequate features.

Signs of Underfitting:

  • Low training accuracy and low test accuracy
  • Model cannot learn the training data patterns
  • High bias and low variance
  • Model fails to capture important relationships in data

Solutions:

  • Increase model complexity: Add more layers, neurons, or features
  • Reduce regularization: Lower regularization strength to allow more flexibility
  • Train longer: Increase training epochs or iterations
  • Feature engineering: Add more relevant features or better transformations
  • Use a more complex algorithm: Switch from linear to non-linear models
  • Hyperparameter tuning: Optimize learning rate, batch size, and other parameters

Q9: What is bias-variance tradeoff?

Bias-Variance Tradeoff is the fundamental problem in supervised learning: we want to minimize both bias (error from erroneous assumptions) and variance (error from sensitivity to small fluctuations in the training set). There's an inherent tension between these two types of errors - reducing one typically increases the other.

Understanding the Tradeoff:

  • High Bias (Underfitting): Model is too simple, fails to capture underlying patterns. Solutions: increase model complexity, add more features, reduce regularization.
  • High Variance (Overfitting): Model is too complex, captures noise in training data. Solutions: increase training data, regularization, simplify model, ensemble methods.
  • Optimal Balance: Sweet spot where model complexity is just right to capture patterns without memorizing noise.

Q10: What is cross-validation?

Cross-validation is a technique to assess model performance by splitting data into multiple subsets (folds) and training/evaluating on different combinations. It provides a more accurate estimate of model performance on unseen data compared to a simple train-test split. The most common method is k-fold cross-validation, where data is divided into k folds, and each fold is used once as validation while the remaining k-1 folds form the training set.

Benefits of Cross-Validation:

  • Better Performance Estimate: Reduces variance in performance estimate by averaging over multiple splits
  • Efficient Data Usage: Every observation is used for both training and validation
  • Hyperparameter Tuning: Provides robust evaluation for model selection
  • Overfitting Detection: Helps identify when model is overfitting to specific data splits
cross_validation.py
from sklearn.model_selection import cross_val_score, KFold from sklearn.ensemble import RandomForestClassifier # K-fold cross-validation model = RandomForestClassifier(n_estimators=100, random_state=42) kfold = KFold(n_splits=5, shuffle=True, random_state=42) scores = cross_val_score(model, X, y, cv=kfold, scoring='accuracy') print(f"Cross-validation scores: {scores}") print(f"Mean accuracy: {scores.mean():.3f} (+/- {scores.std():.3f})") # Stratified K-fold for classification from sklearn.model_selection import StratifiedKFold skfold = StratifiedKFold(n_splits=5, shuffle=True)

Q14: What is backpropagation?

Backpropagation is an algorithm for training neural networks by calculating the gradient of the loss function with respect to each weight by propagating errors backward through the network. It uses the chain rule of calculus to compute gradients efficiently, allowing the network to learn by adjusting weights to minimize the error.

How Backpropagation Works:

  • Forward Pass: Input data flows through the network, producing predictions
  • Loss Calculation: Compute error between predictions and actual values
  • Backward Pass: Calculate gradients of loss with respect to each weight using chain rule
  • Weight Update: Adjust weights using gradient descent or other optimizers
backprop.py
import numpy as np def sigmoid(x): return 1 / (1 + np.exp(-x)) def sigmoid_derivative(x): return x * (1 - x) # Simple backpropagation example class NeuralNetwork: def train(self, X, y, epochs): for epoch in range(epochs): # Forward pass hidden = sigmoid(np.dot(X, self.w1)) output = sigmoid(np.dot(hidden, self.w2)) # Backward pass error = y - output d_output = error * sigmoid_derivative(output) d_hidden = sigmoid_derivative(hidden) * np.dot(d_output, self.w2.T) # Update weights self.w2 += np.dot(hidden.T, d_output) self.w1 += np.dot(X.T, d_hidden)

CNN has no memory of past inputs, while RNN has internal memory that allows it to process sequences of data.

Q17: What is an RNN?

RNN (Recurrent Neural Network) is designed for sequential data with loops that allow information to persist, making it suitable for time series and text data. Unlike traditional neural networks, RNNs have internal memory that allows them to process sequences of data by maintaining a hidden state that captures information from previous time steps.

RNN Characteristics:

  • Sequential Processing: Processes data in sequence, maintaining context
  • Hidden State: Internal memory that carries information across time steps
  • Parameter Sharing: Same weights used at each time step, reducing parameters
  • Variants: Vanilla RNN, LSTM, GRU for different use cases
  • Applications: Language modeling, speech recognition, time series forecasting

Q18: What is LSTM?

LSTM (Long Short-Term Memory) is a type of RNN that can learn long-term dependencies using gates to control information flow, solving the vanishing gradient problem. LSTMs use a complex architecture with input gate, forget gate, and output gate to regulate the flow of information, allowing them to remember information over long sequences while selectively forgetting irrelevant information.

LSTM Gates:

  • Forget Gate: Decides what information to discard from the cell state
  • Input Gate: Decides what new information to store in the cell state
  • Cell State: Carries information across time steps with minimal modification
  • Output Gate: Decides what information to output based on the cell state

from tensorflow import keras model = keras.Sequential([ keras.layers.LSTM(128, input_shape=(10, 32), return_sequences=True), keras.layers.Dropout(0.2), keras.layers.LSTM(64), keras.layers.Dense(32, activation='relu'), keras.layers.Dense(1, activation='sigmoid') ]) model.compile(optimizer='adam', loss='binary_crossentropy') model.fit(X_train, y_train, epochs=30, batch_size=32)
Q19: What is GRU?

GRU (Gated Recurrent Unit) is a simpler variant of LSTM with fewer gates, making it computationally more efficient while still capturing long-term dependencies. GRUs combine the forget and input gates of LSTM into a single update gate, and merge the cell state and hidden state, reducing the number of parameters and training time while maintaining similar performance on many tasks.

GRU vs LSTM:

  • Fewer Gates: GRU has 2 gates (update, reset) vs LSTM's 3 gates (input, forget, output)
  • Simpler Architecture: GRU merges cell state and hidden state
  • Faster Training: Fewer parameters lead to faster convergence
  • Better for Small Datasets: Less prone to overfitting on limited data
  • LSTM Advantage: Better for very long sequences and complex dependencies

Q20: What is the vanishing gradient problem?

Vanishing Gradient Problem occurs when gradients become extremely small during backpropagation, making it difficult for deep networks to learn. This happens when using activation functions like sigmoid or tanh that squash outputs to a small range, causing gradients to shrink exponentially as they propagate backward through many layers. As a result, early layers learn very slowly or not at all.

Solutions:

  • ReLU Activation: Uses max(0, x) which doesn't saturate for positive inputs
  • LSTM/GRU: Use gates to control gradient flow through time
  • Batch Normalization: Normalizes layer inputs to prevent saturation
  • Residual Connections: Skip connections allow gradients to flow directly
  • Proper Initialization: Xavier/He initialization helps maintain gradient scale
vanishing_gradient_fix.py
from tensorflow import keras # Solution 1: Use ReLU activation model = keras.Sequential([ keras.layers.Dense(128, activation='relu', input_shape=(10,)), keras.layers.Dense(64, activation='relu'), keras.layers.Dense(1) ]) # Solution 2: Residual connections inputs = keras.layers.Input(shape=(10,)) x = keras.layers.Dense(64, activation='relu')(inputs) residual = x x = keras.layers.Dense(64, activation='relu')(x) x = keras.layers.Add()([x, residual]) # Skip connection outputs = keras.layers.Dense(1)(x)
Q21: What is batch normalization?

Batch Normalization normalizes layer inputs to have zero mean and unit variance, stabilizing training and allowing higher learning rates. It reduces internal covariate shift by normalizing the inputs of each layer across a mini-batch, which allows the network to learn faster and be less sensitive to initialization. It also has a slight regularization effect.

Benefits:

  • Faster Training: Allows higher learning rates and reduces vanishing gradients
  • Stability: Reduces sensitivity to weight initialization
  • Regularization: Adds noise through batch statistics, reducing overfitting
  • Better Gradient Flow: Helps gradients propagate through deep networks

Q22: What is dropout?

Dropout is a regularization technique that randomly drops neurons during training to prevent overfitting. During each training iteration, a random subset of neurons is temporarily "dropped out" (set to zero), which forces the network to learn more robust features that don't rely on specific neurons. At test time, all neurons are used but their outputs are scaled by the dropout rate.

How Dropout Works:

  • Training Phase: Randomly deactivates neurons with probability p (dropout rate)
  • Test Phase: All neurons active but outputs scaled by (1-p)
  • Effect: Prevents co-adaptation of neurons, acts like ensemble training
  • Common Rates: 0.2-0.5 for hidden layers, 0.1-0.2 for input layers

Q24: What is tokenization?

Tokenization is breaking text into smaller units called tokens (words, subwords, or characters). It's a fundamental preprocessing step in NLP that converts raw text into a format suitable for machine learning models. Different tokenization strategies include word-level (splitting by whitespace), character-level (splitting by characters), and subword-level (like BPE or WordPiece) which balances vocabulary size and meaning representation.

Tokenization Methods:

  • Word Tokenization: Splits text by whitespace, handles punctuation separately
  • Character Tokenization: Splits text into individual characters, handles unknown words
  • Subword Tokenization: BPE, WordPiece, SentencePiece - balances vocabulary and meaning
  • Sentence Tokenization: Splits text into sentences based on punctuation and grammar

Q25: What is word embedding?

Word Embedding represents words as dense vectors where similar words have similar representations. Unlike one-hot encoding which creates sparse vectors, word embeddings capture semantic relationships between words in a continuous vector space. Words with similar meanings are close together in this space, enabling models to understand word relationships and analogies.

Popular Word Embeddings:

  • Word2Vec: Uses CBOW or skip-gram to learn word vectors from context
  • GloVe: Uses global word co-occurrence statistics
  • FastText: Uses subword information to handle rare words
  • Contextual Embeddings: BERT, GPT provide context-dependent representations

Q26: What is BERT?

BERT (Bidirectional Encoder Representations from Transformers) is a pre-trained transformer model that learns contextual text representations using masked language modeling and next sentence prediction. Unlike previous models that processed text left-to-right, BERT uses bidirectional context, allowing it to understand the full context of a word from both directions. It's trained on large corpora and can be fine-tuned for specific NLP tasks.

BERT Features:

  • Bidirectional: Uses masked attention to see context from both directions
  • Pre-training Tasks: Masked Language Modeling (MLM) and Next Sentence Prediction (NSP)
  • Versions: BERT-base, BERT-large, RoBERTa, DistilBERT

Q38: What is semantic vs instance segmentation?

Semantic Segmentation classifies each pixel in an image into a category (e.g., road, car, pedestrian) but doesn't distinguish between different objects of the same class. Instance Segmentation goes further by detecting and delineating individual objects of the same class separately, providing unique masks for each object instance. Key Differences:

  • Instance: Separate labels for each object instance, even if same class
  • Complexity: Instance segmentation is more complex as it requires object detection + segmentation
  • Use Cases: Semantic for scene understanding, Instance for counting/individual object analysis

  • Composition: How privacy budget accumulates across multiple queries
  • Applications: Census data, healthcare analytics, private ML training

Q57: What is collaborative filtering?

Collaborative Filtering uses user behavior to recommend items through user-based or item-based similarity. It relies on the assumption that users who agreed in the past will agree in the future, and that similar items will be liked by similar users. There are two main approaches: user-based (find similar users and recommend what they liked) and item-based (find items similar to what the user liked).

Collaborative Filtering Types:

  • User-Based: Find users with similar preferences and recommend items they liked
  • Item-Based: Find items similar to what the user has liked
  • Matrix Factorization: Decompose user-item matrix to learn latent factors
  • Deep Learning: Use neural networks to learn complex patterns
collaborative_filtering.py
from sklearn.metrics.pairwise import cosine_similarity import numpy as np # User-based collaborative filtering def user_based_recommend(user_item_matrix, user_id, k=3): # Calculate user similarity user_similarity = cosine_similarity(user_item_matrix) # Find similar users similar_users = np.argsort(user_similarity[user_id])[k+1:] # Recommend items liked by similar users recommendations = user_item_matrix.iloc[similar_users].mean(axis=0) return recommendations.nlargest(5)
