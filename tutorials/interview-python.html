1. Python Fundamentals

Q1: What is Python?

Python is a high-level, interpreted programming language known for its simplicity and readability. Created by Guido van Rossum and first released in 1991, Python emphasizes code readability with its notable use of significant whitespace. It supports multiple programming paradigms including procedural, object-oriented, and functional programming.

Key Features: Dynamic typing, automatic memory management, extensive standard library, cross-platform compatibility, and a large ecosystem of third-party packages. Python is widely used for web development, data science, machine learning, automation, scripting, and more.

main.py
print("Hello, World!") # Variables and basic types name = "Python" version = 3.11 is_dynamic = True # Lists and dictionaries languages = ["Python", "JavaScript", "Go"] data = {"name": "Python", "type": "interpreted"}
Q2: What is the difference between Python 2 and Python 3?

Python 2 is the legacy version (end-of-life in 2020) with different print statement syntax, integer division behavior, and Unicode handling. It uses print as a statement and has separate str and unicode types.

Python 3 is the current and actively maintained version with modern features. It uses print as a function, has true division by default, unified string type (Unicode), better error messages, and many language improvements. Python 3 is not backward compatible with Python 2.

main.py
# Python 2 print "Hello" # Statement print 3 / 2 # Output: 1 (integer division) type(u"text") # unicode # Python 3 print("Hello") # Function print(3 / 2) # Output: 1.5 (true division) type("text") # str (Unicode by default)
Q3: What is the difference between list and tuple?

List is a mutable, ordered collection that can be modified after creation. Lists are defined with square brackets [] and support operations like append, insert, remove, and sort. They use more memory due to their dynamic nature.

Tuple is an immutable, ordered collection that cannot be modified after creation. Tuples are defined with parentheses () and are hashable, making them suitable as dictionary keys. They use less memory and are generally faster than lists.

main.py
my_list = [1, 2, 3] my_list[0] = 10 # OK - mutable my_list.append(4) # OK my_list.sort() # OK my_tuple = (1, 2, 3) # my_tuple[0] = 10 # Error - immutable # my_tuple.append(4) # Error - no append method # Tuples can be dictionary keys point = (10, 20) locations = {point: "Home"} # OK # Lists cannot be dictionary keys # point_list = [10, 20] # locations = {point_list: "Home"} # Error - unhashable type
Q4: What is the difference between set and frozenset?

Set is a mutable, unordered collection of unique elements. Sets support operations like add, remove, and update. They are defined with curly braces {} or the set() constructor. Sets are not hashable and cannot be used as dictionary keys.

Frozenset is an immutable, unordered collection of unique elements. Once created, frozensets cannot be modified. They are hashable and can be used as dictionary keys or elements of other sets.

main.py
my_set = {1, 2, 3} my_set.add(4) # OK - mutable my_set.remove(1) # OK my_set.update({5, 6}) # OK my_fset = frozenset([1, 2, 3]) # my_fset.add(4) # Error - immutable # my_fset.remove(1) # Error - no remove method # Frozenset can be dictionary key cache = {frozenset([1, 2]): "value"} # OK # Set cannot be dictionary key # cache = {set([1, 2]): "value"} # Error - unhashable
Q5: What is the difference between dict and OrderedDict?

Dict is the standard dictionary type. In Python 3.7+, regular dicts maintain insertion order as an implementation detail, and in Python 3.7+ it's guaranteed. They are highly optimized for performance.

OrderedDict is a specialized dictionary from collections module that explicitly maintains insertion order regardless of Python version. It provides additional methods like move_to_end() and popitem(last=True/False). Use OrderedDict when you need order-dependent operations or compatibility with older Python versions.

main.py
from collections import OrderedDict # Regular dict (Python 3.7+ maintains order) regular_dict = {'a': 1, 'b': 2, 'c': 3} regular_dict['d'] = 4 print(list(regular_dict.keys())) # ['a', 'b', 'c', 'd'] # OrderedDict (explicit order guarantee) od = OrderedDict([('a', 1), ('b', 2), ('c', 3)]) od['d'] = 4 print(list(od.keys())) # ['a', 'b', 'c', 'd'] # OrderedDict-specific methods od.move_to_end('a') # Move 'a' to end od.popitem(last=False) # Remove first item
Q6: What is the difference between is and ==?

is checks for identity - whether two variables reference the same object in memory. It compares object IDs using id(). Use is for None, True, False, and singleton comparisons.

== checks for equality - whether two objects have the same value. It uses the __eq__() method to compare values. Use == for comparing values of strings, numbers, lists, etc.

main.py
a = [1, 2, 3] b = [1, 2, 3] c = a print(a == b) # True - same values print(a is b) # False - different objects print(a is c) # True - same object # Use is for None x = None if x is None: # Correct print("x is None") # Use is for singletons y = True if y is True: # Correct print("y is True") # String interning s1 = "hello" s2 = "hello" print(s1 is s2) # True - interned strings s3 = "hello world" s4 = "hello world" print(s3 is s4) # May be True or False - depends on implementation
Q7: What is the difference between None and null?

None is Python's representation of null or absence of value. It's a singleton object of type NoneType. There's only one None instance in Python, so you should always use 'is' to compare with None.

null doesn't exist in Python. This is a concept from other languages like Java, C#, JavaScript. In Python, you use None instead of null.

main.py
x = None print(type(x)) # # Correct comparison if x is None: print("x is None") # Also works but not recommended if x == None: print("x equals None") # Function returning None def find_item(items, target): for item in items: if item == target: return item return None # Explicitly return None result = find_item([1, 2, 3], 5) if result is None: print("Item not found") # None as default argument def process(data=None): if data is None: data = [] return data
Q8: What is the difference between pass and continue?

pass is a null operation - it does nothing when executed. It's used as a placeholder where syntactically some code is required but you don't want any action. Commonly used in empty function bodies, class definitions, or if statements.

continue is used within loops to skip the rest of the current iteration and move to the next iteration of the loop. It only works inside for or while loops.

main.py
# pass - placeholder def empty_function(): pass # Function body required, but does nothing class EmptyClass: pass # Class body required, but does nothing if condition: pass # TODO: implement later # continue - skip iteration for i in range(5): if i == 2: continue # Skip when i is 2 print(i) # Prints: 0, 1, 3, 4 # Using continue with while count = 0 while count < 5: count += 1 if count % 2 == 0: continue # Skip even numbers print(count) # Prints: 1, 3, 5
Q9: What is the difference between break and continue?

break terminates the loop entirely and execution continues with the statement after the loop. It's used when you want to exit a loop early based on a condition.

continue skips the rest of the current iteration and moves to the next iteration of the loop. The loop continues running, just the current iteration is skipped.

main.py
# break - exit loop for i in range(10): if i == 5: break # Exit loop when i is 5 print(i) # Prints: 0, 1, 2, 3, 4 # continue - skip iteration for i in range(10): if i == 5: continue # Skip when i is 5 print(i) # Prints: 0, 1, 2, 3, 4, 6, 7, 8, 9 # break with while loop count = 0 while True: count += 1 if count > 5: break # Exit infinite loop print(count) # Prints: 1, 2, 3, 4, 5 # continue with while loop count = 0 while count < 10: count += 1 if count % 2 == 0: continue # Skip even numbers print(count) # Prints: 1, 3, 5, 7, 9
Q10: What is the difference between return and yield?

return exits a function and returns a value to the caller. Once return is executed, the function terminates completely. All local variables are destroyed. A function can only return once.

yield turns a function into a generator. It returns a value but pauses the function's execution, saving its state. The function can be resumed later to continue from where it left off. Generators are memory-efficient for large sequences.

main.py
# return - regular function def get_numbers(): numbers = [1, 2, 3, 4, 5] return numbers # Returns entire list at once result = get_numbers() print(result) # [1, 2, 3, 4, 5] # yield - generator function def generate_numbers(): for i in range(1, 6): yield i # Yields one value at a time gen = generate_numbers() print(next(gen)) # 1 print(next(gen)) # 2 print(next(gen)) # 3 # Using generator in loop for num in generate_numbers(): print(num) # 1, 2, 3, 4, 5 # Memory efficiency comparison def large_list(): return [i for i in range(1000000)] # Creates entire list in memory def large_generator(): for i in range(1000000): yield i # Generates one at a time, low memory

2. Data Structures

Q11: What is a list?

List is an ordered, mutable collection that can hold elements of different types. Lists are defined with square brackets [] and support indexing, slicing, and various methods for manipulation. They are dynamic in size and can grow or shrink as needed.

main.py
# Creating lists my_list = [1, 2, 3, 4, 5] mixed_list = [1, "hello", 3.14, True] nested_list = [[1, 2], [3, 4]] # Indexing and slicing print(my_list[0]) # 1 print(my_list[-1]) # 5 print(my_list[1:4]) # [2, 3, 4] # List methods my_list.append(6) # Add to end my_list.insert(0, 0) # Insert at index my_list.remove(3) # Remove by value my_list.pop() # Remove and return last my_list.extend([7, 8]) # Add multiple elements my_list.sort() # Sort in place my_list.reverse() # Reverse in place
Q12: What is a tuple?

Tuple is an ordered, immutable collection that can hold elements of different types. Tuples are defined with parentheses () and are hashable, making them suitable as dictionary keys. They are more memory-efficient than lists and generally faster for fixed-size collections.

main.py
# Creating tuples my_tuple = (1, 2, 3, 4, 5) single_tuple = (1,) # Note the comma empty_tuple = () # Tuple unpacking a, b, c = (1, 2, 3) x, y = y, x # Swap values # Named tuples from collections import namedtuple Point = namedtuple('Point', ['x', 'y']) p = Point(10, 20) print(p.x, p.y) # 10 20 # Tuple methods print(my_tuple.count(3)) # Count occurrences print(my_tuple.index(2)) # Find index
Q13: What is a set?

Set is an unordered collection of unique elements. Sets are defined with curly braces {} or the set() constructor. They support mathematical set operations like union, intersection, difference, and symmetric difference. Sets are mutable and not hashable.

main.py
# Creating sets my_set = {1, 2, 3, 4, 5} empty_set = set() # Not {} - that's a dict from_list = set([1, 2, 2, 3]) # {1, 2, 3} - duplicates removed # Set operations set1 = {1, 2, 3, 4} set2 = {3, 4, 5, 6} print(set1 | set2) # Union: {1, 2, 3, 4, 5, 6} print(set1 & set2) # Intersection: {3, 4} print(set1 - set2) # Difference: {1, 2} print(set1 ^ set2) # Symmetric difference: {1, 2, 5, 6} # Set methods my_set.add(6) # Add element my_set.remove(1) # Remove element (raises error if not found) my_set.discard(2) # Remove element (no error if not found) my_set.pop() # Remove and return arbitrary element my_set.clear() # Remove all elements
Q14: What is a dictionary?

Dictionary is an unordered collection of key-value pairs (in Python 3.7+, insertion order is preserved). Dictionaries are defined with curly braces {} and provide fast O(1) average time complexity for lookups. Keys must be hashable (immutable), while values can be any type.

main.py
# Creating dictionaries my_dict = {'a': 1, 'b': 2, 'c': 3} empty_dict = {} from_tuples = dict([('a', 1), ('b', 2)]) # Accessing values print(my_dict['a']) # 1 print(my_dict.get('d', 'default')) # 'default' - safe access # Modifying dictionaries my_dict['d'] = 4 # Add/Update key del my_dict['a'] # Delete key value = my_dict.pop('b') # Remove and return value keys = my_dict.keys() # View of keys values = my_dict.values() # View of values items = my_dict.items() # View of key-value pairs # Dictionary comprehension squares = {x: x**2 for x in range(5)} print(squares) # {0: 0, 1: 1, 2: 4, 3: 9, 4: 16} # Merging dictionaries (Python 3.5+) dict1 = {'a': 1, 'b': 2} dict2 = {'c': 3, 'd': 4} merged = {**dict1, **dict2} # {'a': 1, 'b': 2, 'c': 3, 'd': 4}
Q15: What is the difference between shallow copy and deep copy?

Shallow copy creates a new object but inserts references to the objects found in the original. Changes to mutable objects in the copy will affect the original. Use copy.copy() or the copy() method for shallow copies.

Deep copy creates a new object and recursively copies all objects found in the original. Changes to the copy will not affect the original. Use copy.deepcopy() for deep copies.

main.py
import copy # Shallow copy original = [[1, 2], [3, 4]] shallow = original.copy() # or copy.copy(original) shallow[0][0] = 99 print(original) # [[99, 2], [3, 4]] - affected! # Deep copy original = [[1, 2], [3, 4]] deep = copy.deepcopy(original) deep[0][0] = 99 print(original) # [[1, 2], [3, 4]] - not affected
Q16: What is the difference between append and extend?

append adds a single element to the end of a list. If you append another list, it adds the entire list as a single element (nested list).

extend adds multiple elements to the end of a list. It takes an iterable and adds each element individually. If you extend with another list, it adds each element of that list separately.

main.py
my_list = [1, 2] # append - adds single element my_list.append(3) print(my_list) # [1, 2, 3] # append with list - adds as nested list my_list.append([4, 5]) print(my_list) # [1, 2, 3, [4, 5]] # extend - adds multiple elements my_list = [1, 2] my_list.extend([3, 4]) print(my_list) # [1, 2, 3, 4] # extend with string - adds each character my_list.extend("hi") print(my_list) # [1, 2, 3, 4, 'h', 'i']
Q17: What is list comprehension?

List comprehension is a concise way to create lists using a single line of code. It consists of brackets containing an expression followed by a for clause, then zero or more for or if clauses. It's more readable and often faster than using traditional for loops with append.

main.py
# Traditional approach squares = [] for x in range(10): squares.append(x ** 2) # List comprehension squares = [x ** 2 for x in range(10)] # With condition evens = [x for x in range(10) if x % 2 == 0] # Nested comprehension matrix = [[i * j for j in range(3)] for i in range(3)]
Q18: What is the difference between remove and pop?

remove removes the first occurrence of a specified value from the list. It searches for the value and removes it. Raises ValueError if the value is not found.

pop removes and returns an element at a specified index (or the last element if no index is specified). It's useful when you need both to remove an element and get its value.

main.py
my_list = [1, 2, 3, 2, 4] # remove - by value my_list.remove(2) # Removes first 2 print(my_list) # [1, 3, 2, 4] # pop - by index (default: last) value = my_list.pop() # Removes and returns last print(value) # 4 print(my_list) # [1, 3, 2] value = my_list.pop(0) # Removes and returns first print(value) # 1 print(my_list) # [3, 2]

3. Object-Oriented Programming

Q19: What is a class in Python?

Class is a blueprint for creating objects. It defines the properties (attributes) and behaviors (methods) that objects of the class will have. Classes support inheritance, encapsulation, and polymorphism - the key principles of object-oriented programming.

main.py
class Person: def __init__(self, name, age): self.name = name self.age = age def greet(self): return f"Hello, I'm {self.name}" def celebrate_birthday(self): self.age += 1 # Creating an object person = Person("Alice", 30) print(person.greet()) # Hello, I'm Alice person.celebrate_birthday() print(person.age) # 31
Q20: What is the difference between __init__ and __new__?

__new__ is a static method that creates and returns a new instance of the class. It's called first when creating an object, before __init__. It's responsible for actually allocating memory for the object.

__init__ is an instance method that initializes the newly created object. It's called after __new__ and receives the already-created instance. It sets up the initial state of the object by setting attributes.

main.py
class MyClass: def __new__(cls, *args, **kwargs): print("__new__ called") instance = super().__new__(cls) return instance def __init__(self, value): print("__init__ called") self.value = value # Creating an instance obj = MyClass(42) # Output: __new__ called # __init__ called
Q21: What is inheritance in Python?

Inheritance is a mechanism where a new class (child/derived class) inherits attributes and methods from an existing class (parent/base class). It promotes code reuse and establishes a hierarchical relationship between classes. Python supports single inheritance and multiple inheritance.

main.py
class Animal: def __init__(self, name): self.name = name def speak(self): pass class Dog(Animal): def speak(self): return f"{self.name} says Woof!" class Cat(Animal): def speak(self): return f"{self.name} says Meow!" dog = Dog("Buddy") cat = Cat("Whiskers") print(dog.speak()) # Buddy says Woof! print(cat.speak()) # Whiskers says Meow!
Q22: What is the difference between sort and sorted?

sort is a list method that sorts the list in-place. It modifies the original list and returns None. It's more memory-efficient as it doesn't create a new list.

sorted is a built-in function that returns a new sorted list from any iterable. It doesn't modify the original. It works on any iterable, not just lists.

main.py
# sort - in-place my_list = [3, 1, 4, 1, 5, 9, 2, 6] my_list.sort() print(my_list) # [1, 1, 2, 3, 4, 5, 6, 9] # sorted - returns new list my_list = [3, 1, 4, 1, 5, 9, 2, 6] sorted_list = sorted(my_list) print(my_list) # [3, 1, 4, 1, 5, 9, 2, 6] - unchanged print(sorted_list) # [1, 1, 2, 3, 4, 5, 6, 9] # sorted with any iterable sorted_tuple = sorted((3, 1, 4)) # Returns list sorted_string = sorted("hello") # ['e', 'h', 'l', 'l', 'o']
Q23: What is polymorphism in Python?

Polymorphism allows objects of different classes to be treated as objects of a common superclass. It enables different classes to implement methods with the same name but different behaviors. Python achieves polymorphism through method overriding and duck typing.

main.py
class Shape: def area(self): pass class Rectangle(Shape): def __init__(self, width, height): self.width = width self.height = height def area(self): return self.width * self.height class Circle(Shape): def __init__(self, radius): self.radius = radius def area(self): return 3.14 * self.radius ** 2 # Polymorphism in action shapes = [Rectangle(5, 3), Circle(4)] for shape in shapes: print(shape.area()) # 15, 50.24
Q24: What is encapsulation in Python?

Encapsulation is the practice of hiding internal details of how an object works and exposing only what's necessary. In Python, encapsulation is achieved through naming conventions (single underscore for protected, double underscore for private) and properties.

main.py
class BankAccount: def __init__(self, balance): self._balance = balance # Protected attribute self.__pin = "1234" # Private attribute def deposit(self, amount): if amount > 0: self._balance += amount def get_balance(self): return self._balance account = BankAccount(1000) account.deposit(500) print(account.get_balance()) # 1500
Q25: What is method overriding in Python?

Method overriding occurs when a child class provides a specific implementation of a method that is already defined in its parent class. The child class's method overrides the parent's method, allowing the child to customize behavior while maintaining the same interface.

main.py
class Animal: def make_sound(self): return "Some sound" class Dog(Animal): def make_sound(self): return "Woof!" # Overriding parent method class Cat(Animal): def make_sound(self): return "Meow!" # Overriding parent method dog = Dog() cat = Cat() print(dog.make_sound()) # Woof! print(cat.make_sound()) # Meow!
Q26: What is the difference between @staticmethod and @classmethod?

@staticmethod defines a method that doesn't receive the instance (self) or class (cls) as the first argument. It behaves like a regular function but belongs to the class namespace. It can't access or modify class or instance state.

defines a method that receives the class (cls) as the first argument. It can access and modify class state but not instance state. It's often used for factory methods or alternative constructors.

main.py
class Math: @staticmethod def add(a, b): return a + b @classmethod def from_string(cls, string): return cls(int(string)) # Using static method print(Math.add(5, 3)) # 8 # Using class method obj = Math.from_string("42")
Q27: What is the difference between __str__ and __repr__?

__str__ returns a string representation of an object that is meant to be readable and user-friendly. It's called by str() and print(). It should be easy to understand for end users.

__repr__ returns a string representation that is meant to be unambiguous and developer-friendly. It's called by repr() and in the interactive interpreter. If possible, it should return a string that could be used to recreate the object.

main.py
class Point: def __init__(self, x, y): self.x = x self.y = y def __str__(self): return f"Point({self.x}, {self.y})" def __repr__(self): return f"Point(x={self.x}, y={self.y})" p = Point(10, 20) print(str(p)) # Point(10, 20) print(repr(p)) # Point(x=10, y=20)
Q28: What is the difference between @property and getter/setter methods?

@property is a decorator that allows you to define a method that can be accessed like an attribute. It provides a clean, Pythonic way to implement getters without changing the class interface. It enables computed properties and lazy evaluation.

Getter/setter methods are traditional methods (get_x(), set_x()) used to access and modify private attributes. While they work, they're less Pythonic and require changing the interface when adding validation or computation.

main.py
class Temperature: def __init__(self, celsius): self._celsius = celsius @property def celsius(self): return self._celsius @celsius.setter def celsius(self, value): if value < -273.15: raise ValueError("Temperature below absolute zero") self._celsius = value @property def fahrenheit(self): return self._celsius * 9/5 + 32 temp = Temperature(25) print(temp.celsius) # 25 print(temp.fahrenheit) # 77.0
Q29: What is multiple inheritance in Python?

Multiple inheritance allows a class to inherit from more than one parent class. Python supports this feature, which enables a child class to inherit attributes and methods from multiple parents. Python uses the C3 linearization algorithm (MRO - Method Resolution Order) to determine the order in which methods are called.

main.py
class Flyer: def fly(self): return "Flying" class Swimmer: def swim(self): return "Swimming" class Duck(Flyer, Swimmer): def quack(self): return "Quacking" duck = Duck() print(duck.fly()) # Flying print(duck.swim()) # Swimming print(duck.quack()) # Quacking

4. Functions

Q30: What is a function in Python?

Function is a reusable block of code that performs a specific task. Functions help organize code, promote reusability, and make programs easier to understand and maintain. In Python, functions are defined using the def keyword and can accept parameters and return values.

main.py
# Basic function def greet(name): return f"Hello, {name}!" # Function with default parameter def greet_with_default(name="World"): return f"Hello, {name}!" # Function with multiple parameters def add(a, b): return a + b # Calling functions print(greet("Alice")) # Hello, Alice! print(greet_with_default()) # Hello, World! print(add(5, 3)) # 8
Q31: What is the difference between *args and **kwargs?

*args allows a function to accept any number of positional arguments. It collects them into a tuple. The asterisk (*) unpacks the arguments.

**kwargs allows a function to accept any number of keyword arguments. It collects them into a dictionary. The double asterisk (**) unpacks the keyword arguments.

main.py
# *args - variable positional arguments def sum_all(*args): return sum(args) print(sum_all(1, 2, 3, 4)) # 10 print(sum_all(5, 10)) # 15 # **kwargs - variable keyword arguments def print_info(**kwargs): for key, value in kwargs.items(): print(f"{key}: {value}") print_info(name="Alice", age=30, city="NYC") # Combining both def func(*args, **kwargs): print("args:", args) print("kwargs:", kwargs)
Q32: What is a lambda function?

Lambda function is an anonymous function defined using the lambda keyword. It can have any number of arguments but only one expression. Lambda functions are typically used for short, simple operations where a full function definition would be overkill.

main.py
# Regular function def square(x): return x ** 2 # Lambda function square_lambda = lambda x: x ** 2 print(square(5)) # 25 print(square_lambda(5)) # 25 # Lambda with multiple arguments add = lambda a, b: a + b print(add(3, 4)) # 7 # Lambda with built-in functions numbers = [1, 2, 3, 4, 5] evens = list(filter(lambda x: x % 2 == 0, numbers)) print(evens) # [2, 4]
Q33: What is the difference between map, filter, and reduce?

map applies a function to all items in an iterable and returns a map object (iterator). It transforms each element.

filter applies a predicate function to all items and returns only those for which the predicate returns True. It filters elements based on a condition.

reduce applies a function of two arguments cumulatively to the items of an iterable, from left to right, to reduce the iterable to a single value. It's in the functools module in Python 3.

main.py
from functools import reduce numbers = [1, 2, 3, 4, 5] # map - transform each element squared = list(map(lambda x: x**2, numbers)) print(squared) # [1, 4, 9, 16, 25] # filter - keep elements that match condition evens = list(filter(lambda x: x % 2 == 0, numbers)) print(evens) # [2, 4] # reduce - reduce to single value sum_all = reduce(lambda x, y: x + y, numbers) print(sum_all) # 15
Q34: What is a closure in Python?

Closure is a function object that remembers values in enclosing scopes even if they are not present in memory. Closures allow functions to access variables from an outer function that has already completed execution. They're used for data hiding, decorators, and function factories.

main.py
def outer_function(x): def inner_function(y): return x + y return inner_function add_five = outer_function(5) print(add_five(3)) # 8 print(add_five(10)) # 15 # Closure for function factory def make_multiplier(n): def multiplier(x): return x * n return multiplier times_three = make_multiplier(3) print(times_three(5)) # 15
Q35: What is a decorator in Python?

Decorator is a design pattern that allows you to modify the behavior of a function or class without permanently modifying it. Decorators are functions that take another function as an argument and return a new function. They're commonly used for logging, timing, authentication, and memoization.

main.py
@timer def slow_function(): import time time.sleep(1) return "Done" # Custom decorator def timer(func): def wrapper(*args, **kwargs): import time start = time.time() result = func(*args, **kwargs) end = time.time() print(f"Time: {end - start:.2f}s") return result return wrapper
Q36: What is the difference between global and nonlocal?

global keyword allows you to modify a global variable from within a function. Without it, you can only read global variables inside functions. Use global when you need to change a variable defined at the module level.

nonlocal keyword allows you to modify a variable from an enclosing (non-global) scope. It's used in nested functions to modify variables from the outer function's scope. Use nonlocal when working with closures.

main.py
x = 10 # Global variable def modify_global(): global x x = 20 # Modifies global x def outer(): y = 10 def inner(): nonlocal y y = 20 # Modifies outer's y inner() print(y) # 20

5. Exception Handling

Q37: What is exception handling in Python?

Exception handling is a mechanism to handle runtime errors gracefully, preventing programs from crashing. Python uses try, except, else, and finally blocks to manage exceptions. It allows you to catch and respond to errors, or clean up resources regardless of whether an error occurred.

main.py
try: result = 10 / 0 except ZeroDivisionError: print("Cannot divide by zero") except Exception as e: print(f"Error: {e}") else: print("No error occurred") finally: print("This always runs")
Q38: What is the difference between try-except and try-finally?

try-except is used to catch and handle exceptions. The except block executes only if an exception occurs in the try block. It allows you to provide alternative logic or error messages when something goes wrong.

try-finally is used to ensure cleanup code always executes, regardless of whether an exception occurred. The finally block runs after the try block completes, even if an exception was raised and not caught. It's used for resource cleanup.

main.py
# try-except - handle errors try: result = 10 / 0 except ZeroDivisionError: print("Handled division by zero") # try-finally - always cleanup file = None try: file = open("data.txt", "r") data = file.read() finally: if file: file.close() # Always runs
Q39: What is the difference between raise and assert?

raise is used to explicitly raise an exception. You can raise built-in exceptions or custom exceptions. It's used when you detect an error condition and want to signal it to the calling code.

assert is used for debugging purposes to check if a condition is true. If the condition is false, it raises an AssertionError. Assertions can be disabled with the -O flag, so they shouldn't be used for runtime error handling.

main.py
# raise - explicitly raise exception def divide(a, b): if b == 0: raise ValueError("Cannot divide by zero") return a / b # assert - debugging check def calculate_age(birth_year): current_year = 2024 age = current_year - birth_year assert age >= 0, "Invalid birth year" return age
Q40: What is a custom exception in Python?

Custom exception is a user-defined exception class that inherits from the built-in Exception class or its subclasses. Custom exceptions allow you to create specific error types for your application, making error handling more precise and code more readable.

main.py

6. File Handling

Q41: How do you read a file in Python?

Reading a file in Python is typically done using the open() function with the 'r' mode. The recommended approach is to use a with statement, which automatically closes the file even if an exception occurs. You can read the entire file, read line by line, or read specific amounts of data.

main.py
# Read entire file with open("file.txt", "r") as file: content = file.read() print(content) # Read line by line with open("file.txt", "r") as file: for line in file: print(line.strip()) # Read all lines into list with open("file.txt", "r") as file: lines = file.readlines()
Q42: How do you write to a file in Python?

Writing to a file in Python is done using the open() function with 'w' mode (write) or 'a' mode (append). The 'w' mode overwrites existing content, while 'a' mode appends to the end. Always use the with statement to ensure the file is properly closed.

main.py
# Write (overwrites existing content) with open("file.txt", "w") as file: file.write("Hello, World!\n") file.write("This is a new line.") # Append (adds to end of file) with open("file.txt", "a") as file: file.write("\nAppended line") # Write multiple lines lines = ["Line 1\n", "Line 2\n", "Line 3\n"] with open("file.txt", "w") as file: file.writelines(lines)
Q43: What is the difference between 'r', 'w', and 'a' file modes?

'r' (read) mode opens a file for reading only. The file pointer is placed at the beginning of the file. This is the default mode. Raises FileNotFoundError if the file doesn't exist.

'w' (write) mode opens a file for writing only. If the file exists, it truncates the file to zero length (overwrites). If the file doesn't exist, it creates a new file.

'a' (append) mode opens a file for appending. The file pointer is at the end of the file if the file exists. If the file doesn't exist, it creates a new file for writing.

main.py
# 'r' - read mode (default) with open("file.txt", "r") as file: content = file.read() # Read only # 'w' - write mode (overwrites) with open("file.txt", "w") as file: file.write("New content") # Overwrites existing # 'a' - append mode (adds to end) with open("file.txt", "a") as file: file.write("\nMore content") # Adds to end
Q44: What is the with statement in Python?

with statement is used for resource management, particularly with file operations. It ensures that resources are properly cleaned up after use, even if an exception occurs. The with statement automatically calls the __enter__ and __exit__ methods of context managers.

main.py
# with statement for file handling with open("file.txt", "r") as file: content = file.read() # File is automatically closed here # Equivalent without with (not recommended) file = open("file.txt", "r") try: content = file.read() finally: file.close()

8. Advanced Python

Q49: What is a generator in Python?

Generator is a special type of function that returns an iterator object. Instead of returning all values at once, it yields one value at a time, pausing execution between each yield. Generators are memory-efficient for large datasets and support lazy evaluation.

main.py
# Generator function def count_up_to(n): count = 1 while count <= n: yield count count += 1 # Using generator counter = count_up_to(5) print(next(counter)) # 1 print(next(counter)) # 2 # Generator expression squares = (x**2 for x in range(5)) print(list(squares)) # [0, 1, 4, 9, 16]
Q50: What is the difference between yield and return?

return exits a function and returns a single value to the caller. The function's local state is lost, and subsequent calls start fresh. Return is used in regular functions.

yield produces a value but pauses the function's execution, preserving its state. The function can be resumed later to produce more values. Yield is used in generator functions for lazy evaluation and memory efficiency.

main.py
# return - returns all values at once def get_squares_return(n): return [x**2 for x in range(n)] # yield - yields one value at a time def get_squares_yield(n): for x in range(n): yield x**2 # return creates entire list in memory squares_list = get_squares_return(1000000) # Uses lots of memory # yield generates values on demand squares_gen = get_squares_yield(1000000) # Uses minimal memory
Q51: What is the difference between iterator and generator?

Iterator is an object that implements the __iter__ and __next__ methods. It represents a stream of data and can be iterated over using a for loop. Iterators can be created from any iterable using the iter() function.

Generator is a special type of iterator created using functions with the yield keyword or generator expressions. Generators automatically implement the iterator protocol and are more concise to write. They also support lazy evaluation.

main.py
# Custom iterator class CountIterator: def __init__(self, n): self.n = n self.count = 0 def __iter__(self): return self def __next__(self): if self.count >= self.n: raise StopIteration self.count += 1 return self.count # Generator (simpler) def count_generator(n): for i in range(1, n + 1): yield i
Q52: What is the difference between list comprehension and generator expression?

List comprehension creates a complete list in memory immediately. It's enclosed in square brackets [] and returns a list. Useful when you need to access elements multiple times or need random access.

Generator expression creates a generator object that yields values lazily. It's enclosed in parentheses () and returns an iterator. Memory-efficient for large datasets and when you only need to iterate once.

main.py
# List comprehension - creates full list squares_list = [x**2 for x in range(1000000)] # Uses lots of memory print(squares_list[0]) # Can access by index # Generator expression - lazy evaluation squares_gen = (x**2 for x in range(1000000)) # Minimal memory # squares_gen[0] # Error: cannot index # Iterating through generator for sq in squares_gen: pass # Process each value
Q53: What is contextlib in Python?

contextlib is a Python module that provides utilities for working with context managers and the with statement. It includes decorators and functions like @contextmanager for creating context managers easily, and closing() for automatically closing objects.

main.py

Q54: What is threading in Python?

Threading is a technique for concurrent execution where multiple threads run within a single process. Python's threading module allows you to create and manage threads. Due to the GIL (Global Interpreter Lock), Python threads are best for I/O-bound tasks rather than CPU-bound tasks.

main.py
import threading import time def worker(name): print(f"Thread {name} starting") time.sleep(1) print(f"Thread {name} done") # Create and start threads t1 = threading.Thread(target=worker, args=("A",)) t2 = threading.Thread(target=worker, args=("B",)) t1.start() t2.start() t1.join() t2.join()
Q55: What is multiprocessing in Python?

Multiprocessing is a technique for parallel execution where multiple processes run independently, each with its own Python interpreter and memory space. Python's multiprocessing module bypasses the GIL, making it suitable for CPU-bound tasks. Each process has its own memory, so they don't share data directly.

main.py
from multiprocessing import Process def worker(name): print(f"Process {name}") # Create and start processes p1 = Process(target=worker, args=("A",)) p2 = Process(target=worker, args=("B",)) p1.start() p2.start() p1.join() p2.join()
Q56: What is the GIL (Global Interpreter Lock) in Python?

GIL (Global Interpreter Lock) is a mutex that protects access to Python objects, preventing multiple native threads from executing Python bytecodes at once. This means that in CPython, only one thread can execute Python code at a time, even on multi-core systems. The GIL exists because CPython's memory management is not thread-safe.

main.py
import threading import time # CPU-bound task - GIL limits performance def count_down(n): while n > 0: n -= 1 # I/O-bound task - GIL released during I/O def io_task(n): for i in range(n): time.sleep(0.01) # GIL released here # Use multiprocessing for CPU-bound tasks # Use threading for I/O-bound tasks
Q57: What is the difference between threading and multiprocessing?

Threading runs multiple threads within a single process, sharing the same memory space. Due to the GIL, only one thread executes Python bytecode at a time. Best for I/O-bound tasks where threads spend time waiting for external operations.

Multiprocessing runs multiple processes, each with its own Python interpreter and memory space. Bypasses the GIL, allowing true parallel execution. Best for CPU-bound tasks that require heavy computation. Processes don't share memory directly.

main.py
# Threading - same memory, limited by GIL import threading t = threading.Thread(target=func) t.start() # Multiprocessing - separate memory, true parallel from multiprocessing import Process p = Process(target=func) p.start()
Q58: What is asyncio in Python?

asyncio is a library for writing asynchronous code using the async/await syntax. It provides a framework for managing concurrent I/O-bound tasks using coroutines, event loops, and futures. Asyncio is single-threaded and cooperative, making it efficient for I/O operations.

main.py
import asyncio async def fetch_data(delay): await asyncio.sleep(delay) return f"Data after {delay}s" async def main(): # Run tasks concurrently task1 = asyncio.create_task(fetch_data(1)) task2 = asyncio.create_task(fetch_data(2)) result1 = await task1 result2 = await task2 print(result1, result2) asyncio.run(main())
Q59: What is the difference between async and sync?

Synchronous (sync) code executes sequentially, blocking on each operation. Each statement waits for the previous one to complete before executing. Simple to understand but can be inefficient for I/O-bound tasks.

Asynchronous (async) code uses coroutines and the event loop to execute tasks concurrently without blocking. Operations that would block yield control back to the event loop, allowing other tasks to run. More complex but efficient for I/O-bound operations.

main.py
# Synchronous - blocks on each call import time def sync_task(name): time.sleep(1) print(name) # Asynchronous - non-blocking import asyncio async def async_task(name): await asyncio.sleep(1) print(name)
Q60: What is a metaclass in Python?

Metaclass is a class of a class that defines how a class behaves. It's used to customize class creation, allowing you to modify class attributes, methods, or inheritance before the class is fully created. Metaclasses are specified using the metaclass keyword in class definition.

9. Data Structures

Q61: What is the difference between list and tuple?

List is a mutable, ordered collection that can be modified after creation. Lists are defined with square brackets [] and support operations like append, remove, and sort. They use more memory due to their dynamic nature.

Tuple is an immutable, ordered collection that cannot be modified after creation. Tuples are defined with parentheses () and are hashable, making them usable as dictionary keys. They use less memory and are faster than lists.

Q62: What is the difference between list and set?

List is an ordered, mutable collection that allows duplicate elements. Lists maintain insertion order and support indexing. Defined with square brackets [].

Set is an unordered, mutable collection that only allows unique elements. Sets don't maintain order and don't support indexing. They provide O(1) membership testing. Defined with curly braces {} or set().

main.py
# List - ordered, allows duplicates my_list = [1, 2, 2, 3] print(my_list[0]) # 1 (indexing works) # Set - unordered, unique elements my_set = {1, 2, 2, 3} print(my_set) # {1, 2, 3} (duplicates removed) # my_set[0] # Error: no indexing # Fast membership test print(2 in my_set) # True (O(1))
Q63: What is the difference between set and frozenset?

Set is a mutable collection that can be modified after creation. You can add, remove, and update elements. Sets are not hashable and cannot be used as dictionary keys or elements of other sets.

Frozenset is an immutable collection that cannot be modified after creation. Frozensets are hashable and can be used as dictionary keys or elements of other sets. Useful when you need a set that won't change.

Q64: What is the difference between dict and OrderedDict?

dict is a standard dictionary that maintains insertion order in Python 3.7+. It's optimized for fast lookups and is the default mapping type in Python.

OrderedDict is a dictionary subclass that explicitly remembers insertion order and provides additional methods like move_to_end() and popitem(). In Python 3.7+, regular dicts maintain order, but OrderedDict is still useful for its extra methods and when you need to guarantee order across Python versions.

main.py

Q65: What is the difference between defaultdict and dict?

dict raises a KeyError when you try to access a non-existent key. You need to check if a key exists or use get() with a default value to avoid errors.

defaultdict is a dictionary subclass that automatically creates a default value for missing keys. You specify a default factory function (like list, int, or a custom function) that's called when a missing key is accessed. This simplifies code that accumulates values.

main.py

Q66: What is the difference between deque and list?

List is optimized for random access and operations at the end (append, pop). Operations at the beginning (insert, pop(0)) are O(n) because all elements need to be shifted.

deque (double-ended queue) is optimized for operations at both ends. Adding/removing from either end is O(1). Deques are implemented as doubly-linked lists internally. Use deque when you need frequent insertions/deletions at both ends.

main.py
from collections import deque # List - slow at beginning my_list = [1, 2, 3] my_list.insert(0, 0) # O(n) - shifts all elements # Deque - fast at both ends my_deque = deque([1, 2, 3]) my_deque.appendleft(0) # O(1) - fast my_deque.popleft() # O(1) - fast
Q67: What is the difference between Counter and dict?

dict is a general-purpose mapping that stores key-value pairs. To count occurrences, you need to manually check and increment values.

Counter is a dict subclass specifically designed for counting hashable objects. It automatically initializes missing keys to 0 and provides useful methods like most_common(), elements(), and arithmetic operations (+, -, &, |). Perfect for frequency counting.

main.py

List can be used as a heap but requires manual implementation of heap operations. It's not optimized for heap operations and doesn't maintain the heap property automatically.

heapq is a module that provides heap queue algorithm operations on lists. It maintains the heap property (min-heap by default) and provides efficient O(log n) operations for push, pop, and peek. Use heapq when you need priority queue functionality.

main.py

Q84: What is the difference between == and is in Python?

== is the equality operator that compares the values of two objects. It checks if the objects have the same content or value. For custom objects, it calls the __eq__ method.

is is the identity operator that checks if two variables point to the same object in memory. It compares object IDs using id(). Two objects can be equal (==) but not identical (is).

main.py
a = [1, 2, 3] b = [1, 2, 3] c = a print(a == b) # True (same values) print(a is b) # False (different objects) print(a is c) # True (same object) # None comparison - use 'is' if x is None: pass
Q85: What is the difference between deep copy and shallow copy?

Shallow copy creates a new object but inserts references to the objects found in the original. Changes to mutable objects in the copy affect the original. Use copy.copy() or the copy() method.

Deep copy creates a new object and recursively copies all objects found in the original. Changes to the copy don't affect the original. Use copy.deepcopy() for complete independence.

main.py
import copy original = [[1, 2], [3, 4]] # Shallow copy - nested lists are shared shallow = copy.copy(original) shallow[0][0] = 99 # Affects original # Deep copy - complete independence deep = copy.deepcopy(original) deep[0][0] = 99 # Doesn't affect original
Q86: What is the difference between __str__ and __repr__?

__str__ returns a string representation of an object that is readable and user-friendly. It's called by str() and print(). The goal is to be readable for end users.

__repr__ returns a string representation that is unambiguous and, if possible, a valid Python expression that could recreate the object. It's called by repr() and in the interactive interpreter. The goal is to be useful for developers and debugging.

main.py
class Point: def __init__(self, x, y): self.x = x self.y = y def __str__(self): return f"Point({self.x}, {self.y})" def __repr__(self): return f"Point(x={self.x}, y={self.y})" p = Point(1, 2) print(str(p)) # Point(1, 2) print(repr(p)) # Point(x=1, y=2)
Q87: What is the difference between @staticmethod and @classmethod?

@staticmethod defines a method that doesn't receive any special first argument (no self or cls). It behaves like a regular function but belongs to the class namespace. Can't access or modify class or instance state.

@classmethod defines a method that receives the class (cls) as the first argument. It can access and modify class state but not instance state. Often used for factory methods that create instances.

main.py
class Math: pi = 3.14159 @staticmethod def add(a, b): return a + b @classmethod def from_degrees(cls, degrees): return cls(degrees * cls.pi / 180) # Static method - called on class or instance Math.add(1, 2) # Class method - receives class Math.from_degrees(90)
Q88: What is the difference between __init__ and __new__?

__new__ is a static method that creates and returns a new instance of the class. It's called first, before __init__. __new__ is responsible for actually creating the object and is used in singleton patterns or when subclassing immutable types.

__init__ is an instance method that initializes the newly created object. It's called after __new__ and receives the already-created instance (self). __init__ sets up the object's initial state.

main.py
class Singleton: _instance = None def __new__(cls): if not cls._instance: cls._instance = super().__new__(cls) return cls._instance def __init__(self): if not hasattr(self, 'initialized'): self.initialized = True
Q89: What is the difference between __eq__ and __hash__?

__eq__ defines how objects are compared for equality using the == operator. It should return True if two objects are considered equal based on their content or attributes.

__hash__ defines the hash value of an object, used in hash-based collections like sets and dict keys. If you define __eq__, you should also define __hash__ to maintain the invariant that equal objects have equal hashes. If __eq__ is defined, the default __hash__ is set to None.

main.py
class Person: def __init__(self, name, age): self.name = name self.age = age def __eq__(self, other): return self.name == other.name and self.age == other.age def __hash__(self): return hash((self.name, self.age)) # Now Person can be used in sets and as dict keys people_set = {Person('John', 30), Person('Jane', 25)}
Q90: What is the difference between @property and getter/setter methods?

@property decorator allows you to define methods that are accessed like attributes. It provides a clean, Pythonic way to implement getters without changing the calling syntax. You can also use @setter and @deleter decorators.

Getter/setter methods are traditional Java-style methods like get_name() and set_name(). They're more verbose and require changing the calling syntax when adding logic to attribute access. @property is preferred in Python.

main.py
class Circle: def __init__(self, radius): self._radius = radius @property def radius(self): return self._radius @radius.setter def radius(self, value): if value < 0: raise ValueError("Radius must be positive") self._radius = value c = Circle(5) print(c.radius) # Access like attribute c.radius = 10 # Set like attribute

Q91: What is a decorator in Python?

Decorator is a design pattern that allows you to modify the behavior of a function or class without changing its source code. Decorators are implemented as functions that take a callable and return a new callable. The @ syntax is syntactic sugar for applying decorators.

main.py
def timer(func): def wrapper(*args, **kwargs): import time start = time.time() result = func(*args, **kwargs) end = time.time() print(f"{func.__name__} took {end - start:.2f}s") return result return wrapper @timer def slow_function(): time.sleep(1) slow_function() # Prints execution time
Q92: What is a context manager in Python?

Context manager is an object that defines the runtime context for a block of code, using the with statement. It implements __enter__ and __exit__ methods to set up and tear down resources. Context managers are commonly used for file handling, database connections, and locks.

main.py
class FileManager: def __init__(self, filename, mode): self.filename = filename self.mode = mode self.file = None def __enter__(self): self.file = open(self.filename, self.mode) return self.file def __exit__(self, exc_type, exc_val, exc_tb): if self.file: self.file.close() return False with FileManager('test.txt', 'w') as f: f.write('Hello')
Q93: What is the difference between __call__ and regular method?

__call__ is a special method that makes an object callable like a function. When you define __call__ in a class, instances of that class can be invoked using parentheses. This is useful for creating callable objects, functors, or implementing strategy patterns.

Regular method requires explicit method call syntax (object.method()). __call__ allows more natural function-like syntax while maintaining state in the object.

main.py
class Multiplier: def __init__(self, factor): self.factor = factor def __call__(self, x): return x * self.factor double = Multiplier(2) triple = Multiplier(3) # Call like a function print(double(5)) # 10 print(triple(5)) # 15
Q94: What is the difference between __getattr__ and __getattribute__?

__getattr__ is called only when an attribute is not found through normal means (not in __dict__ or class hierarchy). It's used as a fallback for missing attributes and is commonly used for dynamic attribute access or delegation.

__getattribute__ is called for every attribute access, regardless of whether the attribute exists. It intercepts all attribute lookups and provides complete control. Must be implemented carefully to avoid infinite recursion.

main.py
class DynamicAttributes: def __getattr__(self, name): # Called only for missing attributes return f"Dynamic: {name}" obj = DynamicAttributes() print(obj.anything) # Dynamic: anything # __getattribute__ intercepts ALL attribute access # Must use super().__getattribute__ to avoid recursion
Q95: What is the difference between __len__ and __bool__?

__len__ defines the behavior of the len() built-in function. It should return the length of the object. For containers, this typically returns the number of items. If __len__ is not defined, Python falls back to __bool__ for truthiness testing.

__bool__ defines the truth value of an object in boolean contexts (if statements, bool(), not). It should return True or False. If __bool__ is not defined, Python uses __len__ (objects with length > 0 are True) or falls back to True for non-None objects.

main.py
class MyList: def __init__(self, items): self.items = items def __len__(self): return len(self.items) def __bool__(self): return len(self.items) > 0 my_list = MyList([1, 2, 3]) print(len(my_list)) # 3 if my_list: print("Not empty")
Q96: What is the difference between __iter__ and __next__?

__iter__ should return an iterator object. It's called when you start iterating over an object (for loop, iter()). For self-iterating objects, __iter__ typically returns self.

__next__ returns the next item in the iteration. It's called repeatedly by the iteration protocol. When there are no more items, it should raise StopIteration. Together, __iter__ and __next__ implement the iterator protocol.

main.py
class CountDown: def __init__(self, start): self.current = start def __iter__(self): return self def __next__(self): if self.current <= 0: raise StopIteration value = self.current self.current -= 1 return value for num in CountDown(3): print(num) # 3, 2, 1
Q97: What is the difference between __contains__ and __eq__ for membership testing?

__contains__ defines the behavior of the in operator. It should return True if the item is in the container. Implementing __contains__ provides efficient membership testing without requiring iteration.

__eq__ is used for equality comparison with ==. While you can use == for membership testing by iterating, __contains__ is the proper way to implement membership and is more efficient for large collections.

main.py
class MySet: def __init__(self, items): self.items = set(items) def __contains__(self, item): return item in self.items my_set = MySet([1, 2, 3]) print(2 in my_set) # True print(4 in my_set) # False
Q98: What is the difference between __getitem__ and __iter__?

__getitem__ defines how objects respond to indexing (obj[key]) and slicing. It's used for sequence-like and mapping-like objects. If __iter__ is not defined, Python can use __getitem__ with integer indices starting from 0 for iteration.

__iter__ is the proper way to define iteration. It should return an iterator object. __iter__ is more flexible and efficient for iteration, while __getitem__ is primarily for indexing and slicing operations.

main.py
class MySequence: def __init__(self, data): self.data = data def __getitem__(self, index): return self.data[index] def __iter__(self): return iter(self.data) seq = MySequence([1, 2, 3]) print(seq[0]) # 1 (indexing) for item in seq: # iteration print(item)
Q99: What is the difference between __add__ and __radd__?

__add__ defines the behavior of the + operator when the object is on the left side (obj + other). It's called for expressions like a + b where a is an instance of your class.

__radd__ (right add) is called when the object is on the right side and the left operand doesn't know how to handle it (other + obj). It's used to ensure commutative operations work correctly when your class is on the right side.

main.py
class Vector: def __init__(self, x, y): self.x = x self.y = y def __add__(self, other): return Vector(self.x + other.x, self.y + other.y) def __radd__(self, other): return Vector(other + self.x, other + self.y) v1 = Vector(1, 2) v2 = Vector(3, 4) result = v1 + v2 # Calls __add__
Q100: What is the difference between __enter__ and __exit__ in context managers?

__enter__ is called when entering the with block. It should set up the context and return the resource that will be assigned to the variable after as. This is where you acquire resources like opening files or database connections.
