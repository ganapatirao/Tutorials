Python is a high-level, interpreted programming language known for its simplicity and readability. It was created by Guido van Rossum and first released in 1991.

Key Features: Easy to learn and use, Interpreted, Dynamically typed, Object-oriented, Extensive libraries, Cross-platform, Garbage collection, Large community.

Q3: What is the difference between is and == in Python?

== compares values (equality check). is compares object identity (memory location check). Use == for comparing values, is for checking if two variables reference the same object.

a = [1, 2, 3]
b = [1, 2, 3]

print(a == b)  # True (same values)
print(a is b)  # False (different objects)

c = a
print(c is a)  # True (same object)

# Integer interning
x = 256
y = 256
print(x is y)  # True
Q4: What are *args and **kwargs?

*args accepts variable number of positional arguments as a tuple. **kwargs accepts variable number of keyword arguments as a dictionary. They allow functions to accept flexible arguments.

def function(*args, **kwargs):
    print("Positional:", args)
    print("Keyword:", kwargs)

function(1, 2, 3, name="Alice", age=25)
# Positional: (1, 2, 3)
# Keyword: {"name": "Alice", "age": 25}

Q6: What is the difference between list and tuple?

List: Mutable, can be modified, uses more memory, syntax [1,2,3]. Tuple: Immutable, cannot be modified, uses less memory, faster, syntax (1,2,3). Use lists for dynamic data, tuples for fixed data and dictionary keys.

# List (mutable)

# Tuple (immutable)

Q7: What is the difference between shallow copy and deep copy?

Shallow copy: Creates new object but references same nested objects. Changes to nested objects affect both copies. Deep copy: Creates completely independent copy with new nested objects. Use copy.copy() for shallow, copy.deepcopy() for deep copy.

import copy

original = [[1, 2], [3, 4]]

# Shallow copy - nested objects shared
shallow = copy.copy(original)
shallow[0][0] = 99
print(original[0][0])  # 99 (affected)

# Deep copy - completely independent
deep = copy.deepcopy(original)
deep[0][0] = 100
print(original[0][0])  # 99 (not affected)
Q8: What are the advanced data structures in Python's collections module?

deque: Double-ended queue with O(1) append/pop. defaultdict: Dictionary with default values. Counter: Dictionary for counting. OrderedDict: Dictionary with insertion order. namedtuple: Tuple with named fields. ChainMap: Group multiple dictionaries.

from collections import deque, defaultdict, Counter, namedtuple

# deque - efficient append/pop from both ends
d = deque([1, 2, 3])
d.appendleft(0)
d.pop()

# defaultdict - automatic default values
dd = defaultdict(int)
dd['a'] += 1

# Counter - counting elements
c = Counter([1, 2, 2, 3, 3, 3])
print(c.most_common(2))  # [(3, 3), (2, 2)]

# namedtuple - readable tuples
Point = namedtuple('Point', ['x', 'y'])
p = Point(10, 20)
print(p.x, p.y)  # 10 20

3. Object-Oriented Programming

Q9: What are the four pillars of OOP in Python?

Encapsulation: Bundling data and methods, hiding internal state. Inheritance: Creating new classes based on existing ones. Polymorphism: Same function/method for different types. Abstraction: Hiding implementation details, showing only essential features.

# Four Pillars of OOP

# 1. Encapsulation - private attributes
class BankAccount:
    def __init__(self, balance):
        self.__balance = balance
    
    def deposit(self, amount):
        self.__balance += amount
    
    def get_balance(self):
        return self.__balance

# 2. Inheritance - extending classes
class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def speak(self):
        return f"{self.name} says Woof!"

# 3. Polymorphism - same method, different behavior
dog = Dog("Buddy")
print(dog.speak())  # Buddy says Woof!

# 4. Abstraction - hiding implementation
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

class Rectangle(Shape):
    def area(self):
        return 100

    def __init__(self):
        print("Initializing object")

# Initializing object
Q11: Explain the difference between class and instance variables.

Class variables: Shared across all instances, defined outside methods. Instance variables: Unique to each instance, defined using self. Changes to class variables affect all instances.

class Dog:
    species = "Canis familiaris"  # Class variable
    
    def __init__(self, name):
        self.name = name  # Instance variable

dog1 = Dog("Buddy")
dog2 = Dog("Max")

print(dog1.species)  # Canis familiaris (shared)
print(dog2.species)  # Canis familiaris (shared)
print(dog1.name)     # Buddy (unique)
print(dog2.name)     # Max (unique)
Q12: What is Method Resolution Order (MRO) in Python?

MRO determines the order Python searches for methods in multiple inheritance. Python uses C3 linearization algorithm. You can check MRO with ClassName.mro() or help(ClassName).

class A:
    def method(self):
        print("A")

class B(A):
    def method(self):
        print("B")

class C(A):
    def method(self):
        print("C")

class D(B, C):
    pass

d = D()
d.method()  # B (B comes before C in MRO)

print(D.mro())
# [, , , , ]
Q13: What are magic methods (dunder methods) in Python?

Magic methods are special methods with double underscores that enable operator overloading. Common ones: __init__ (constructor), __str__ (user-friendly string), __repr__ (developer-friendly string), __eq__ (equality), __hash__ (hash), __len__ (length), __getitem__ (index access), __call__ (callable), __iter__ (iterator), __enter__/__exit__ (context manager).

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __str__(self):
        return f"Vector({self.x}, {self.y})"
    
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)
    
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

v1 = Vector(1, 2)
v2 = Vector(3, 4)
v3 = v1 + v2
print(v3)  # Vector(4, 6)

4. Advanced Python Features

Q14: What are decorators in Python?

Decorators are functions that modify behavior of other functions without changing source code. They wrap functions to extend functionality. Use @decorator syntax. Common uses: timing, logging, caching, authentication, validation.

import time
import functools

def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} took {end-start:.2f}s")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(1)
    return "Done"

slow_function()  # slow_function took 1.00s
Q15: What are generators and how do they work?

Generators are functions that produce sequences lazily using yield instead of return. Benefits: memory efficient, can represent infinite sequences, pipeline data processing. Use generator expressions for simple cases, generator functions for complex logic.

# Generator function
def fibonacci(n):
    a, b = 0, 1
    for _ in range(n):
        yield a
        a, b = b, a + b

# Generator expression
squares = (x**2 for x in range(10))

# Using generators
for num in fibonacci(10):
    print(num)

# Memory efficient
large_gen = (x for x in range(1000000))
Q16: What are context managers and how do you use them?

Context managers manage resources ensuring proper setup and cleanup. Used with 'with' statement. Implement __enter__ and __exit__ methods or use @contextmanager decorator from contextlib module. Used for files, database connections, locks.

# Context manager using class
class FileManager:
    def __init__(self, filename, mode):
        self.filename = filename
        self.mode = mode
        self.file = None
    
    def __enter__(self):
        self.file = open(self.filename, self.mode)
        return self.file
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        if self.file:
            self.file.close()
        return True

# Using context manager
with FileManager("example.txt", "w") as f:
    f.write("Hello, World!")

# Using @contextmanager decorator
from contextlib import contextmanager

@contextmanager
def timer():
    import time
    start = time.time()
    yield
    end = time.time()
    print(f"Time: {end-start:.2f}s")

with timer():
    time.sleep(1)
Q17: What is the difference between list comprehension and generator expression?

List comprehension: Creates entire list in memory, uses [], can be indexed, reusable. Generator expression: Produces values lazily, uses (), memory efficient, single-use. Use list comprehension for small datasets, generator for large or infinite sequences.

# List comprehension - stores all in memory
list_comp = [x**2 for x in range(10)]
print(list_comp)  # [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

# Generator expression - lazy evaluation
gen_expr = (x**2 for x in range(10))
print(list(gen_expr))  # [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

# Memory difference
import sys
list_size = sys.getsize([x for x in range(1000)])
gen_size = sys.getsize((x for x in range(1000)))
print(f"List: {list_size}, Gen: {gen_size}")
Q18: What are closures in Python?

Closures are function objects that remember values in enclosing scopes even if not present in memory. Used for function factories, callbacks, maintaining state. Inner function has access to variables from outer function's scope.

# Closure example - function factory
def make_multiplier(factor):
    def multiplier(number):
        return number * factor
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(5))  # 10
print(triple(5))  # 15

# Closure for maintaining state
def counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment

my_counter = counter()
print(my_counter())  # 1
print(my_counter())  # 2
Q19: What is the difference between map, filter, and reduce?

map: Applies function to all items in iterable, returns iterator. filter: Filters items based on condition, returns iterator. reduce: Applies function cumulatively to items, returns single value. Use list comprehensions for better readability when possible.

from functools import reduce

# map - apply function to all items
numbers = [1, 2, 3, 4, 5]
squared = list(map(lambda x: x**2, numbers))
print(squared)  # [1, 4, 9, 16, 25]

# filter - keep items matching condition
evens = list(filter(lambda x: x % 2 == 0, numbers))
print(evens)  # [2, 4]

# reduce - reduce to single value
total = reduce(lambda x, y: x + y, numbers)
print(total)  # 15

5. File Handling and I/O

Q20: What are the different file modes in Python?

'r': Read (default), file must exist. 'w': Write, creates new or overwrites. 'a': Append, adds to end, creates if not exists. 'r+': Read and write, file must exist. 'w+': Read and write, creates new or overwrites. 'a+': Read and append. Add 'b' for binary mode: 'rb', 'wb', 'ab'.

# Read mode (default)
with open("file.txt", "r") as f:
    content = f.read()

# Write mode (overwrites)
with open("file.txt", "w") as f:
    f.write("Hello")

# Append mode (adds to end)
with open("file.txt", "a") as f:
    f.write(" World")

# Binary mode
with open("image.jpg", "rb") as f:
    data = f.read()

# Read and write
with open("file.txt", "r+") as f:
    content = f.read()
    f.write("Updated")
Q21: How do you handle exceptions in Python?

Use try-except blocks to handle exceptions. Specify exception types for precise handling. Use else for code that runs if no exception occurs. Use finally for cleanup code that always runs. Use raise to raise exceptions, assert for debugging.

# Basic exception handling
try:
    result = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero")

# Multiple exception types
try:
    x = int("not a number")
except ValueError:
    print("Invalid number")
except TypeError:
    print("Type error")

# else and finally
try:
    file = open("file.txt", "r")
except FileNotFoundError:
    print("File not found")
else:
    content = file.read()
    file.close()
finally:
    print("Cleanup code")
Q22: What is the difference between __str__ and __repr__?

__str__: User-friendly string representation, used by str() and print(). Should be readable. __repr__: Developer-friendly string representation, used by repr() and debugging. Should be unambiguous and ideally recreate the object. If __str__ is not defined, Python uses __repr__.

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def __str__(self):
        return f"Person: {self.name}"
    
    def __repr__(self):
        return f"Person(name='{self.name}', age={self.age})"

p = Person("Alice", 25)
print(str(p))   # Person: Alice
print(repr(p))  # Person(name='Alice', age=25)
Q23: What is the difference between append and extend in lists?

append: Adds single element to end of list. If element is iterable, adds it as one element. extend: Adds all elements from iterable to end of list. Use append for single items, extend for adding multiple items from another iterable.

# append - adds single element
my_list = [1, 2, 3]
my_list.append(4)
print(my_list)  # [1, 2, 3, 4]

# append with list - adds as single element
my_list.append([5, 6])
print(my_list)  # [1, 2, 3, 4, [5, 6]]

# extend - adds all elements
my_list = [1, 2, 3]
my_list.extend([4, 5])
print(my_list)  # [1, 2, 3, 4, 5]
Q24: What is the difference between pop and remove in lists?

pop: Removes and returns element at specified index (default last). Raises IndexError if index out of range. remove: Removes first occurrence of specified value. Raises ValueError if value not found. Use pop when you know index, remove when you know value.

# pop - removes by index, returns value
my_list = [1, 2, 3, 4, 5]
value = my_list.pop(2)
print(value)    # 3
print(my_list)  # [1, 2, 4, 5]

# pop last element (default)
my_list.pop()
print(my_list)  # [1, 2, 4]

# remove - removes by value
my_list = [1, 2, 3, 2, 4]
my_list.remove(2)
print(my_list)  # [1, 3, 2, 4] (first 2 removed)
Q25: What is the difference between sort and sorted in Python?

sort: Method that sorts list in-place, returns None, modifies original list. sorted: Function that returns new sorted list, doesn't modify original. Use sort for in-place sorting (memory efficient), sorted when you need to keep original list.

# sort - modifies list in-place
my_list = [3, 1, 4, 1, 5]
my_list.sort()
print(my_list)  # [1, 1, 3, 4, 5]

# sorted - returns new list
my_list = [3, 1, 4, 1, 5]
new_list = sorted(my_list)
print(my_list)   # [3, 1, 4, 1, 5] (unchanged)
print(new_list)  # [1, 1, 3, 4, 5]

# reverse sorting
my_list.sort(reverse=True)
print(my_list)  # [5, 4, 3, 1, 1]

Q29: What are fixtures in pytest?

Fixtures are functions that provide fixed baseline setup for tests. Defined using @pytest.fixture decorator. Can set up database connections, test data, mock objects. Support scope (function, class, module, session) and dependency injection. Use fixtures to avoid code duplication in tests.

import pytest

@pytest.fixture
def sample_data():
    return {"name": "Alice", "age": 25}

def test_with_fixture(sample_data):
    assert sample_data["name"] == "Alice"
    assert sample_data["age"] == 25

# Fixture with setup/teardown
@pytest.fixture
def database():
    db = setup_database()
    yield db
    teardown_database(db)

def test_database(database):
    result = database.query("SELECT * FROM users")
    assert len(result) > 0

Q31: What is the difference between NumPy array and Python list?

NumPy array: Homogeneous (same data type), memory efficient, fast mathematical operations, fixed size, supports vectorization. Python list: Heterogeneous (mixed types), less memory efficient, slower for math operations, dynamic size, no vectorization. Use NumPy for numerical computing, lists for general purpose.

import numpy as np

# NumPy array - homogeneous, fast

# Python list - heterogeneous, slower
py_list = [1, 2, 3, 4]
print([x * 2 for x in py_list])  # [2, 4, 6, 8]

Q32: What is the difference between loc and iloc in Pandas?

loc: Label-based indexing, uses row/column labels, inclusive of end. iloc: Integer position-based indexing, uses integer positions, exclusive of end. Use loc when you know labels, iloc when you know positions.

import pandas as pd

df = pd.DataFrame({'A': [1, 2, 3], 'B': [4, 5, 6]}, 
                   index=['x', 'y', 'z'])

# loc - label-based
print(df.loc['x'])      # Row with label 'x'
print(df.loc['x':'y'])  # Rows from 'x' to 'y' (inclusive)
print(df.loc['x', 'A']) # Specific value

# iloc - position-based
print(df.iloc[0])       # First row
print(df.iloc[0:2])     # First two rows (exclusive)
print(df.iloc[0, 0])    # First value

10. Python Performance

Q35: How do you optimize Python code performance?

Use built-in functions and libraries instead of custom implementations. Use list comprehensions instead of loops. Use generators for large datasets. Use caching with functools.lru_cache. Use proper data structures (set for lookups). Use Cython or numba for critical sections. Profile with cProfile to find bottlenecks.

# Use built-in functions
sum_list = sum([1, 2, 3, 4])  # Fast
# vs manual loop
total = 0
for x in [1, 2, 3, 4]:
    total += x  # Slower

# List comprehension
squares = [x**2 for x in range(1000)]

# Generator for memory efficiency
large_gen = (x**2 for x in range(1000000))

# Use set for O(1) lookups
my_set = {1, 2, 3, 4, 5}
print(3 in my_set)  # Fast

# Caching

Q37: What is the GIL (Global Interpreter Lock) in Python?

GIL is a mutex that prevents multiple native threads from executing Python bytecodes simultaneously. Only one thread runs Python code at a time. Exists due to CPython memory management not being thread-safe. Affects CPU-bound multithreading but not I/O-bound. Use multiprocessing to bypass GIL for CPU tasks.

import time

def cpu_task():
    count = 0
    for _ in range(10000000):
        count += 1

# Multiple threads won't run in parallel due to GIL
threads = [threading.Thread(target=cpu_task) for _ in range(4)]
for t in threads:
    t.start()
for t in threads:
    t.join()

# Solution: Use multiprocessing for CPU-bound tasks
from multiprocessing import Pool

with Pool(4) as p:
    p.map(cpu_task, [None]*4)

print(len(v3))  # Uses __len__
Q63: What is the difference between __str__ and __repr__?

__str__: User-friendly string representation, used by str() and print(). Should be readable. __repr__: Developer-friendly representation, used by repr() and interactive interpreter. Should be unambiguous and ideally valid Python code. If __str__ not defined, __repr__ is used as fallback.

class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __str__(self):
        return f"Point({self.x}, {self.y})"
    
    def __repr__(self):
        return f"Point(x={self.x}, y={self.y})"

p = Point(3, 4)
print(str(p))    # Point(3, 4) - user-friendly
print(repr(p))   # Point(x=3, y=4) - developer-friendly

# In interactive interpreter
>>> p
Point(x=3, y=4)  # Uses __repr__

# If only __repr__ is defined
class Point2:
    def __repr__(self):
        return f"Point2(x={self.x}, y={self.y})"
    
    def __init__(self, x, y):
        self.x = x
        self.y = y

p2 = Point2(1, 2)
print(p2)  # Uses __repr__ as fallback

27. Python Dataclasses

Q64: What is a dataclass in Python?

dataclass decorator automatically generates __init__, __repr__, __eq__, and other methods. Reduces boilerplate for classes that primarily store data. Supports type hints, default values, frozen instances, field customization. Available in Python 3.7+. Use for DTOs, config objects, simple data containers.

from dataclasses import dataclass, field

@dataclass
class Person:
    name: str
    age: int
    email: str = "unknown@example.com"  # Default value
    active: bool = True

# Automatically generates __init__, __repr__, __eq__
person = Person("Alice", 25)
print(person)  # Person(name='Alice', age=25, email='unknown@example.com', active=True)

# With field customization
@dataclass
class InventoryItem:
    name: str
    price: float
    quantity: int = field(default=0)
    total_price: float = field(init=False)

    def __post_init__(self):
        self.total_price = self.price * self.quantity

item = InventoryItem("Widget", 10.0, 5)
print(item.total_price)  # 50.0
Q65: What is the difference between dataclass and namedtuple?

namedtuple: Immutable, lightweight, tuple subclass, faster, less flexible. dataclass: Mutable by default, more features (methods, inheritance), type hints, field customization. Use namedtuple for simple immutable records, dataclass for more complex data objects with behavior.

from dataclasses import dataclass
from collections import namedtuple

# namedtuple - immutable, simple
Point = namedtuple('Point', ['x', 'y'])
p1 = Point(1, 2)
print(p1.x, p1.y)  # 1 2
# p1.x = 3  # AttributeError: can't set attribute

# dataclass - mutable, more features
@dataclass
class Point2:
    x: int
    y: int

p2 = Point2(1, 2)
print(p2.x, p2.y)  # 1 2
p2.x = 3  # Works - mutable

# Frozen dataclass (immutable like namedtuple)
@dataclass(frozen=True)
class ImmutablePoint:
    x: int
    y: int

p3 = ImmutablePoint(1, 2)
# p3.x = 3  # FrozenInstanceError

28. Python Type Hints

Q66: What are type hints in Python?

Type hints annotate variables and function signatures with expected types. Don't affect runtime but enable static analysis with mypy. Use typing module for complex types: List, Dict, Tuple, Optional, Union. Improve code documentation and IDE support.

from typing import List, Dict, Tuple, Optional, Union

# Basic type hints
def greet(name: str) -> str:
    return f"Hello, {name}!"

# Complex types
def process_items(items: List[int]) -> Dict[str, int]:
    return {"count": len(items), "sum": sum(items)}

# Optional type (can be None)
def find_user(user_id: int) -> Optional[str]:
    return "Alice" if user_id == 1 else None

# Union type (multiple possible types)
def process_value(value: Union[int, str]) -> str:
    return str(value)

# Type aliases
UserId = int
UserName = str
def get_user_name(user_id: UserId) -> UserName:
    return "User" + str(user_id)
Q67: What is the typing module in Python?

typing module provides support for type hints. Includes generic types: List, Dict, Set, Tuple. Special types: Optional, Union, Any, Callable. Use for complex type annotations, generics, protocols. Python 3.9+ uses built-in types (list, dict) instead of typing module for simple cases.

from typing import List, Dict, Set, Tuple, Optional, Union, Any, Callable

# Generic types
def process_list(items: List[int]) -> int:
    return sum(items)

def get_mapping() -> Dict[str, int]:
    return {"a": 1, "b": 2}

# Callable (function type)
def apply(func: Callable[[int], int], value: int) -> int:
    return func(value)

result = apply(lambda x: x * 2, 5)

# Any (any type)
def process_any(value: Any) -> str:
    return str(value)

# Protocol (structural typing)
from typing import Protocol

class Drawable(Protocol):
    def draw(self) -> None: ...

def render(obj: Drawable) -> None:
    obj.draw()

29. Python Testing

Q68: How do you write unit tests in Python?

Use unittest module (built-in) or pytest (third-party). unittest provides TestCase class with assert methods. pytest uses simpler syntax with plain assert statements. Use fixtures for setup/teardown. Mock external dependencies with unittest.mock. Test edge cases and error conditions.

# Using unittest
import unittest

def add(a, b):
    return a + b

class TestMath(unittest.TestCase):
    def test_add_positive(self):
        self.assertEqual(add(2, 3), 5)
    
    def test_add_negative(self):
        self.assertEqual(add(-1, 1), 0)
    
    def test_add_strings(self):
        self.assertEqual(add("a", "b"), "ab")

# Using pytest
def test_add_pytest():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0

# Run with: python -m unittest test_module.py
# Or with pytest: pytest test_module.py
Q69: What is mocking in Python testing?

Mocking replaces real objects with test doubles to isolate code under test. Use unittest.mock for mocking functions, classes, external services. Mock objects record calls, can configure return values. Use patch to temporarily replace objects. Essential for testing code with side effects, external APIs, databases.

from unittest.mock import Mock, patch

# Basic mock
mock_func = Mock(return_value=42)
result = mock_func()
print(result)  # 42
print(mock_func.called)  # True

# Mocking external API
def get_user(user_id):
    # Simulates API call
    return {"id": user_id, "name": "Alice"}

@patch('__main__.get_user')
def test_get_user(mock_get_user):
    mock_get_user.return_value = {"id": 1, "name": "Bob"}
    result = get_user(1)
    assert result["name"] == "Bob"
    mock_get_user.assert_called_once_with(1)

# Mocking class methods
class Database:
    def query(self, sql):
        return [{"id": 1}]

mock_db = Mock(spec=Database)
mock_db.query.return_value = [{"id": 2}]
result = mock_db.query("SELECT *")

30. Python Best Practices

Q70: What are Python best practices for writing clean code?

PEP 8 style guide: 4 spaces indentation, max 79 chars per line, snake_case for variables/functions. Use docstrings for documentation. Keep functions small and focused. Use list/dict comprehensions when appropriate. Avoid global variables. Use type hints. Follow DRY principle. Use meaningful names.

# Good: PEP 8 compliant
def calculate_average(numbers: list[float]) -> float:
    """Calculate the average of a list of numbers."""
    if not numbers:
        return 0.0
    return sum(numbers) / len(numbers)

# Good: List comprehension
squares = [x**2 for x in range(10)]

# Good: Meaningful names
user_authentication_token = get_auth_token()

# Good: Small, focused function
def validate_email(email: str) -> bool:
    """Validate email format."""
    return "@" in email and "." in email

# Avoid: Too long line
# result = some_very_long_function_name(with_many_arguments_that_make_this_line_too_long)

# Good: Break long lines
result = some_very_long_function_name(
    with_many_arguments,
    that_make_this_line_too_long
)
Q71: What is the Zen of Python?

Zen of Python is a collection of 19 guiding principles for Python design. Access via `import this`. Key principles: Beautiful is better than ugly. Explicit is better than implicit. Simple is better than complex. Complex is better than complicated. Flat is better than nested. Sparse is better than dense. Readability counts.

# View the Zen of Python
import this

# Key principles demonstrated:
# Explicit is better than implicit
# Bad: Implicit behavior
def process(data):
    return data * 2  # What does this do?

# Good: Explicit
def multiply_by_two(data: int) -> int:
    return data * 2

# Simple is better than complex
# Bad: Complex one-liner
result = sum([x for x in range(100) if x % 2 == 0])

# Good: Simple, readable
even_numbers = [x for x in range(100) if x % 2 == 0]
result = sum(even_numbers)

# Readability counts
# Bad: cryptic variable names
def f(x, y, z):
    return x + y + z

# Good: descriptive names
def calculate_total(price: float, tax: float, shipping: float) -> float:
    return price + tax + shipping
Q72: What is Python's EAFP vs LBYL philosophy?

EAFP: Easier to Ask Forgiveness than Permission - try operation, handle exceptions. Pythonic approach. LBYL: Look Before You Leap - check conditions before acting. More common in other languages. EAFP is generally preferred in Python for its simplicity and efficiency.

# EAFP (Pythonic) - try first, handle exception
def get_item_eafp(dictionary: dict, key: str):
    try:
        return dictionary[key]
    except KeyError:
        return None

# LBYL (Less Pythonic) - check first
def get_item_lbyl(dictionary: dict, key: str):
    if key in dictionary:
        return dictionary[key]
    return None

# EAFP example with file handling
def read_file_eafp(filename: str) -> str:
    try:
        with open(filename, 'r') as f:
            return f.read()
    except FileNotFoundError:
        return "File not found"
    except IOError:
        return "Error reading file"

# LBYL example with file handling
def read_file_lbyl(filename: str) -> str:
    if os.path.exists(filename) and os.path.isfile(filename):
        with open(filename, 'r') as f:
            return f.read()
    return "File not found"
Q73: What is pattern matching in Python 3.10+?

Pattern matching (match/case) provides powerful structural pattern matching. Similar to switch/case but more powerful. Can match patterns, structures, guard conditions. Simplifies complex conditional logic, data parsing, AST processing. Available in Python 3.10+.

# Pattern matching
def describe(value):
    match value:
        case 0:
            return "Zero"
        case 1:
            return "One"
        case _:
            return "Other"

# Match structures
def process(data):
    match data:
        case {"type": "user", "id": uid}:
            return f"User {uid}"
        case _:
            return "Unknown"

# Match with guards
def classify(n):
    match n:
        case x if x < 0:
            return "negative"
        case _:
            return "positive"
