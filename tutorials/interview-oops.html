1. OOP Fundamentals

Q1: What is Object-Oriented Programming (OOP)?

OOP is a programming paradigm based on the concept of "objects", which can contain data and code. The data is in the form of fields (attributes), and the code is in the form of procedures (methods).

Key Features:

  • Encapsulation: Bundling data and methods together
  • Inheritance: Deriving new classes from existing ones
  • Polymorphism: Same interface, different implementations
  • Abstraction: Hiding implementation details
Car.cs
// Basic OOP example in C# public class Car { public string Brand { get; set; } public string Model { get; set; } public void Drive() { Console.WriteLine($"{Brand} {Model} is driving"); } } // Create object Car myCar = new Car { Brand = "Toyota", Model = "Camry" }; myCar.Drive(); // Output: Toyota Camry is driving

Q3: What are the four pillars of OOP?

Four Pillars of OOP:

  1. Encapsulation: Bundling data and methods that operate on the data within a single unit (class). Hides internal state and requires interaction through methods.
  2. Inheritance: Mechanism where a new class derives properties and behavior from an existing class. Promotes code reusability.
  3. Polymorphism: Ability of objects to take many forms. Same method name can behave differently based on the object.
  4. Abstraction: Hiding complex implementation details and showing only essential features to the user.

2. Encapsulation

Q4: What is encapsulation and why is it important?

Encapsulation is the bundling of data and methods that operate on that data within a single unit (class). It restricts direct access to some of an object's components.

Benefits:

  • Data hiding and protection
  • Control over data access
  • Improved maintainability
  • Flexibility to change implementation
BankAccount.cs
// Encapsulation example in C# public class BankAccount { private decimal _balance; // Private field public BankAccount(decimal initialBalance) { _balance = initialBalance; } public void Deposit(decimal amount) { if (amount > 0) _balance += amount; } public decimal GetBalance() { return _balance; } } BankAccount account = new BankAccount(1000); account.Deposit(500); Console.WriteLine(account.GetBalance()); // Output: 1500 // account._balance = 0; // Error: inaccessible
Q5: What are access modifiers?

Access modifiers control the visibility of class members (fields, methods, constructors).

Types:

  • Public: Accessible from anywhere
  • Private: Accessible only within the class
  • Protected: Accessible within class and subclasses
  • Internal: Accessible within same assembly
  • Protected Internal: Accessible within same assembly or subclasses
Person.cs
// Access modifiers in C# public class Person { public string Name; // Public private int _age; // Private protected string Address; // Protected internal string Email; // Internal public void SetAge(int age) { _age = age; } private void ValidateAge() { // Validation logic } }

3. Inheritance

Q6: What is inheritance and its types?

Inheritance allows a class to derive properties and behavior from another class, promoting code reusability.

Types of Inheritance:

  • Single Inheritance: One class inherits from one parent
  • Multiple Inheritance: One class inherits from multiple parents (C# supports via interfaces)
  • Multilevel Inheritance: Chain of inheritance (A → B → C)
  • Hierarchical Inheritance: Multiple classes inherit from one parent
Animal.cs
// Single inheritance in C# public class Animal { public void Speak() { Console.WriteLine("Animal speaks"); } } public class Dog : Animal { public void Bark() { Console.WriteLine("Dog barks"); } } Dog dog = new Dog(); dog.Speak(); // Output: Animal speaks dog.Bark(); // Output: Dog barks

4. Polymorphism

Q8: What is polymorphism and its types?

Polymorphism allows objects of different classes to be treated as objects of a common superclass. It enables one interface to have multiple implementations.

Types of Polymorphism:

  • Compile-time (Static): Method overloading, operator overloading
  • Runtime (Dynamic): Method overriding, virtual functions
Animal.cs
// Runtime polymorphism in C# public abstract class Animal { public abstract string Speak(); } public class Dog : Animal { public override string Speak() { return "Woof"; } } public class Cat : Animal { public override string Speak() { return "Meow"; } } List<Animal> animals = new List<Animal> { new Dog(), new Cat() }; foreach (var animal in animals) { Console.WriteLine(animal.Speak()); // Output: Woof, Meow }
Q9: What is method overloading vs method overriding?
Aspect Method Overloading Method Overriding

Scope Same class Parent and child class
Resolution Compile-time Runtime
Keywords No special keyword virtual/override
Parent.cs
// Method overriding in C# public class Parent { public virtual void Show() { Console.WriteLine("Parent method"); } } public class Child : Parent { public override void Show() { Console.WriteLine("Child method"); } } Child child = new Child(); child.Show(); // Output: Child method

5. Abstraction

Q10: What is abstraction and how is it implemented?

Abstraction is the concept of hiding complex implementation details and showing only essential features to the user. It focuses on what an object does rather than how it does it.

Implementation:

  • Abstract Classes: Cannot be instantiated, may have abstract methods
  • Interfaces: Pure abstraction, only method signatures
  • Concrete Methods: Provide implementation details
Shape.cs
// Abstraction in C# using abstract class public abstract class Shape { public abstract double Area(); public abstract double Perimeter(); } public class Rectangle : Shape { public double Width { get; set; } public double Height { get; set; } public Rectangle(double width, double height) { Width = width; Height = height; } public override double Area() { return Width * Height; } public override double Perimeter() { return 2 * (Width + Height); } } Rectangle rect = new Rectangle(5, 3); Console.WriteLine(rect.Area()); // Output: 15

6. SOLID Principles

Q12: What are the SOLID principles?

SOLID Principles:

  1. S - Single Responsibility: A class should have one reason to change
  2. O - Open/Closed: Open for extension, closed for modification
  3. L - Liskov Substitution: Subtypes must be substitutable for base types
  4. I - Interface Segregation: Clients shouldn't depend on unused interfaces
  5. D - Dependency Inversion: Depend on abstractions, not concretions
SRP.cs
// Single Responsibility Principle in C# public class User { public string Name { get; set; } public string Email { get; set; } } public class UserRepository { public void Save(User user) { // Save user to database } } public class EmailService { public void Send(User user, string message) { // Send email to user } }
Q13: Explain the Single Responsibility Principle with example.

Single Responsibility Principle (SRP) states that a class should have only one reason to change, meaning it should have only one job.

SRP.cs
// Violating SRP public class User { public string Name { get; set; } public string Email { get; set; } public void SaveToDatabase() { // Database logic } public void SendEmail(string message) { // Email logic } } // Following SRP public class User { public string Name { get; set; } public string Email { get; set; } } public class UserRepository { public void Save(User user) { // Database logic } } public class EmailService { public void Send(User user, string message) { // Email logic } }
Q14: Explain the Open/Closed Principle with example.

Open/Closed Principle (OCP) states that software entities should be open for extension but closed for modification.

OCP.cs
// Violating OCP public class DiscountCalculator { public double Calculate(string productType, double price) { if (productType == "regular") return price * 0.9; else if (productType == "premium") return price * 0.8; return price; } } // Following OCP public interface IDiscount { double Calculate(double price); } public class RegularDiscount : IDiscount { public double Calculate(double price) { return price * 0.9; } } public class PremiumDiscount : IDiscount { public double Calculate(double price) { return price * 0.8; } }

7. Design Patterns

Q15: What are design patterns and their types?

Design Patterns are reusable solutions to common problems in software design. They are best practices that developers can use to solve common problems.

Types of Design Patterns:

  • Creational: Object creation mechanisms (Singleton, Factory, Builder)
  • Structural: Class and object composition (Adapter, Decorator, Facade)
  • Behavioral: Communication between objects (Observer, Strategy, Command)
Q16: What is the Singleton pattern?

Singleton Pattern ensures a class has only one instance and provides a global point of access to it.

Singleton.cs
// Singleton in C# public sealed class Singleton { private static readonly Lazy<Singleton> _instance = new Lazy<Singleton>(() => new Singleton()); public static Singleton Instance => _instance.Value; private Singleton() { } public void DoSomething() { Console.WriteLine("Singleton instance working"); } } Singleton s1 = Singleton.Instance; Singleton s2 = Singleton.Instance; Console.WriteLine(ReferenceEquals(s1, s2)); // Output: True
Q17: What is the Factory pattern?

Factory Pattern provides an interface for creating objects in a superclass, but allows subclasses to alter the type of objects that will be created.

Factory.cs
// Factory Pattern in C# public interface IAnimal { string Speak(); } public class Dog : IAnimal { public string Speak() => "Woof"; } public class Cat : IAnimal { public string Speak() => "Meow"; } public static class AnimalFactory { public static IAnimal CreateAnimal(string type) { return type.ToLower() switch { "dog" => new Dog(), "cat" => new Cat(), _ => throw new ArgumentException("Invalid animal type") }; } } IAnimal dog = AnimalFactory.CreateAnimal("dog"); Console.WriteLine(dog.Speak()); // Output: Woof
Q18: What is the Observer pattern?

Observer Pattern defines a one-to-many dependency between objects so that when one object changes state, all its dependents are notified.

Observer.cs
// Observer Pattern in C# public interface IObserver { void Update(string message); } public class Subject { private readonly List<IObserver> _observers = new List<IObserver>(); public void Attach(IObserver observer) { _observers.Add(observer); } public void Notify(string message) { foreach (var observer in _observers) { observer.Update(message); } } } public class ConcreteObserver : IObserver { public void Update(string message) { Console.WriteLine($"Received: {message}"); } }

8. Advanced OOP Concepts

Q19: What is a virtual function?

Virtual Function is a member function in the base class that can be redefined (overridden) in the derived class. It enables runtime polymorphism.

Virtual.cs
// Virtual function in C# public class Animal { public virtual void Speak() { Console.WriteLine("Animal speaks"); } } public class Dog : Animal { public override void Speak() { Console.WriteLine("Dog barks"); } } Animal animal = new Dog(); animal.Speak(); // Output: Dog barks

Q21: What is method chaining?

Method Chaining is a technique where multiple methods are called on the same object in a single statement. Each method returns the object, allowing the next method to be called.

StringBuilder.cs
// Method chaining in C# public class StringBuilder { private string _string = ""; public StringBuilder Add(string text) { _string += text; return this; } public StringBuilder ToUpper() { _string = _string.ToUpper(); return this; } public string Build() { return _string; } } var result = new StringBuilder() .Add("hello") .Add(" ") .Add("world") .ToUpper() .Build(); Console.WriteLine(result); // Output: HELLO WORLD

Q24: Explain the Liskov Substitution Principle with example.

Liskov Substitution Principle (LSP) states that objects of a superclass should be replaceable with objects of a subclass without breaking the application.

LSP.cs
// Violating LSP public class Rectangle { public virtual double Width { get; set; } public virtual double Height { get; set; } public double Area() => Width * Height; } public class Square : Rectangle { public override double Width { set { base.Width = base.Height = value; } } public override double Height { set { base.Width = base.Height = value; } } } // Following LSP public abstract class Shape { public abstract double Area(); } public class Rectangle : Shape { public double Width { get; set; } public double Height { get; set; } public override double Area() => Width * Height; }
Q25: Explain the Interface Segregation Principle with example.

Interface Segregation Principle (ISP) states that clients should not be forced to depend on interfaces they don't use. Split large interfaces into smaller, specific ones.

ISP.cs
// Violating ISP public interface IWorker { void Work(); void Eat(); } public class Robot : IWorker { public void Work() { } public void Eat() { } // Robot doesn't eat } // Following ISP public interface IWorkable { void Work(); } public interface IEatable { void Eat(); } public class Human : IWorkable, IEatable { public void Work() { } public void Eat() { } } public class Robot : IWorkable { public void Work() { } }
Q26: Explain the Dependency Inversion Principle with example.

Dependency Inversion Principle (DIP) states that high-level modules should not depend on low-level modules. Both should depend on abstractions.

DIP.cs
// Violating DIP public class LightBulb { public void TurnOn() { Console.WriteLine("Bulb on"); } } public class Switch { private LightBulb _bulb = new LightBulb(); public void Operate() { _bulb.TurnOn(); } } // Following DIP public interface ISwitchable { void TurnOn(); void TurnOff(); } public class LightBulb : ISwitchable { public void TurnOn() => Console.WriteLine("Bulb on"); public void TurnOff() => Console.WriteLine("Bulb off"); } public class Switch { private readonly ISwitchable _device; public Switch(ISwitchable device) { _device = device; } public void Operate() => _device.TurnOn(); }
Q27: What is the Adapter pattern?

Adapter Pattern allows incompatible interfaces to work together. It acts as a bridge between two incompatible interfaces.

Adapter.cs
// Adapter Pattern in C# public interface IMediaPlayer { void Play(string audioType, string fileName); } public class AdvancedMediaPlayer { public void PlayVlc(string fileName) { Console.WriteLine($"Playing VLC: {fileName}"); } public void PlayMp4(string fileName) { Console.WriteLine($"Playing MP4: {fileName}"); } } public class MediaAdapter : IMediaPlayer { private readonly AdvancedMediaPlayer _advancedPlayer; public MediaAdapter() { _advancedPlayer = new AdvancedMediaPlayer(); } public void Play(string audioType, string fileName) { if (audioType == "vlc") _advancedPlayer.PlayVlc(fileName); else if (audioType == "mp4") _advancedPlayer.PlayMp4(fileName); } }
Q28: What is the Decorator pattern?

Decorator Pattern allows adding behavior to individual objects dynamically without affecting other objects of the same class.

Decorator.cs
// Decorator Pattern in C# public abstract class Coffee { public abstract double Cost(); } public class SimpleCoffee : Coffee { public override double Cost() => 5; } public abstract class CoffeeDecorator : Coffee { protected Coffee _coffee; public CoffeeDecorator(Coffee coffee) { _coffee = coffee; } public override double Cost() => _coffee.Cost(); } public class MilkDecorator : CoffeeDecorator { public MilkDecorator(Coffee coffee) : base(coffee) { } public override double Cost() => base.Cost() + 2; } public class SugarDecorator : CoffeeDecorator { public SugarDecorator(Coffee coffee) : base(coffee) { } public override double Cost() => base.Cost() + 1; } Coffee coffee = new SimpleCoffee(); coffee = new MilkDecorator(coffee); coffee = new SugarDecorator(coffee); Console.WriteLine(coffee.Cost()); // Output: 8
Q29: What is the Strategy pattern?

Strategy Pattern defines a family of algorithms, encapsulates each one, and makes them interchangeable. Strategy lets the algorithm vary independently from clients.

Strategy.cs
// Strategy Pattern in C# public interface IPaymentStrategy { void Pay(decimal amount); } public class CreditCardPayment : IPaymentStrategy { public void Pay(decimal amount) { Console.WriteLine($"Paid ${amount} with Credit Card"); } } public class PayPalPayment : IPaymentStrategy { public void Pay(decimal amount) { Console.WriteLine($"Paid ${amount} with PayPal"); } } public class ShoppingCart { private readonly IPaymentStrategy _paymentStrategy; public ShoppingCart(IPaymentStrategy paymentStrategy) { _paymentStrategy = paymentStrategy; } public void Checkout(decimal amount) { _paymentStrategy.Pay(amount); } } var cart = new ShoppingCart(new CreditCardPayment()); cart.Checkout(100); // Output: Paid $100 with Credit Card

Q31: What is a constructor and its types?

Constructor is a special method called when an object is created. It initializes the object's state.

Types of Constructors:

  • Default Constructor: No parameters
  • Parameterized Constructor: With parameters
  • Copy Constructor: Creates object from another object
  • Static Constructor: Called once for the class

Q32: What is coupling and cohesion?

Coupling measures how closely connected classes are. Cohesion measures how closely related the responsibilities of a single class are.

Goal:

  • Low Coupling: Classes are independent and loosely connected
  • High Cohesion: Class has focused, single responsibility

Q34: What is the Command pattern?

Command Pattern encapsulates a request as an object, thereby allowing parameterization of clients with different requests, queuing of requests, and logging of operations.

Q35: What is the Facade pattern?

Facade Pattern provides a simplified interface to a complex subsystem. It hides the complexities of the system and provides a simple interface to the client.

Facade.cs
// Facade Pattern in C# public class CPU { public void Freeze() { Console.WriteLine("CPU frozen"); } public void Jump(long position) { Console.WriteLine($"Jumping to {position}"); } } public class Memory { public void Load(long position, byte[] data) { Console.WriteLine($"Loading data at {position}"); } } public class ComputerFacade { private readonly CPU _cpu; private readonly Memory _memory; public ComputerFacade() { _cpu = new CPU(); _memory = new Memory(); } public void Start() { _cpu.Freeze(); _memory.Load(0, new byte[] { 0x01, 0x02 }); _cpu.Jump(0); } }

Q37: What is the DRY principle?

DRY (Don't Repeat Yourself) principle states that every piece of knowledge must have a single, unambiguous, authoritative representation within a system.

Benefits:

  • Reduces code duplication
  • Easier maintenance
  • Consistent behavior
  • Reduced bug count
DRY.cs
// Violating DRY public double CalculateAreaCircle(double radius) { return 3.14 * radius * radius; } public double CalculateCircumferenceCircle(double radius) { return 2 * 3.14 * radius; } // Following DRY public static class MathConstants { public const double PI = 3.14; } public double CalculateAreaCircle(double radius) { return MathConstants.PI * radius * radius; } public double CalculateCircumferenceCircle(double radius) { return 2 * MathConstants.PI * radius; }

Guidelines:

5. Advanced OOP Concepts

Q41: What is a virtual method?

Virtual Method is a method that can be overridden in derived classes. It allows polymorphic behavior where the method implementation is determined at runtime based on the object type.

Virtual Method Characteristics:

  • Can be overridden in derived classes
  • Enables runtime polymorphism
  • Uses dynamic binding
  • Must have implementation in base class
VirtualMethod.cs
public class Animal { public virtual void MakeSound() { Console.WriteLine("Generic animal sound"); } } public class Dog : Animal { public override void MakeSound() { Console.WriteLine("Bark"); } } // Runtime polymorphism Animal animal = new Dog(); animal.MakeSound(); // Output: Bark
Q42: What is an abstract method?

Abstract Method is a method without implementation that must be implemented in derived classes. It can only exist in abstract classes and enforces a contract for derived classes.

Abstract Method Characteristics:

  • No implementation in base class
  • Must be implemented in derived classes
  • Can only be declared in abstract classes
  • Enforces implementation contract
AbstractMethod.cs
public abstract class Shape { public abstract double CalculateArea(); public abstract double CalculatePerimeter(); } public class Circle : Shape { private readonly double _radius; public Circle(double radius) { _radius = radius; } public override double CalculateArea() { return Math.PI * _radius * _radius; } public override double CalculatePerimeter() { return 2 * Math.PI * _radius; } }
Q43: What is the difference between virtual and abstract methods?

Modifier virtual abstract
Q44: What is method hiding?

Method Hiding occurs when a derived class has a method with the same name as a base class method but doesn't override it. It uses the `new` keyword to hide the base method.

Method Hiding Characteristics:

  • Uses `new` keyword
  • Doesn't override base method
  • Static binding (compile-time)
  • Hides base implementation
MethodHiding.cs
public class BaseClass { public void Display() { Console.WriteLine("Base class"); } } public class DerivedClass : BaseClass { public new void Display() { Console.WriteLine("Derived class"); } } // Method hiding - compile-time binding BaseClass obj = new DerivedClass(); obj.Display(); // Output: Base class

Polymorphism Supports polymorphism Doesn't support polymorphism
Q46: What is a sealed class?

Sealed Class is a class that cannot be inherited. It prevents other classes from deriving from it, ensuring the class cannot be modified through inheritance.

Sealed Class Use Cases:

  • Security: Prevent inheritance for security reasons
  • Performance: Enables compiler optimizations
  • Design: Enforces final implementation
  • Constants: Classes that should never be extended
SealedClass.cs
public sealed class Configuration { private static readonly Configuration _instance = new Configuration(); private Configuration() { } public static Configuration Instance => _instance; public string GetSetting(string key) { } } // Cannot inherit from sealed class // public class CustomConfig : Configuration { } // Error
Q47: What is a sealed method?

Sealed Method is a method that cannot be overridden in derived classes. It prevents further modification of the method implementation through inheritance.

Sealed Method Use Cases:

  • Prevent overriding critical methods
  • Enforce final implementation
  • Security for sensitive operations
  • Performance optimizations
SealedMethod.cs
public class BaseClass { public virtual void Process() { } public sealed override void Calculate() { // Cannot be overridden further } } public class DerivedClass : BaseClass { public override void Process() { } // Cannot override sealed method // public override void Calculate() { } // Error }
Q48: What is a static class?

Static Class is a class that cannot be instantiated and can only contain static members. It's commonly used for utility methods and extension methods.

Static Class Characteristics:

  • Cannot be instantiated
  • Can only contain static members
  • Cannot inherit from or be inherited
  • Can have static constructor
StaticClass.cs
public static class MathHelper { public static int Add(int a, int b) { return a + b; } public static int Multiply(int a, int b) { return a * b; } static MathHelper() { // Static constructor } } // Usage int sum = MathHelper.Add(5, 3);
Q49: What is a static constructor?

Static Constructor is a special constructor that runs once when the class is first loaded. It's used to initialize static members of the class.

Static Constructor Characteristics:

  • Runs once per class
  • No access modifier or parameters
  • Called automatically by CLR
  • Cannot be called directly

Q50: What is a partial class?

Partial Class allows splitting a class definition across multiple files. It's useful for separating auto-generated code from user-written code.

Partial Class Benefits:

  • Separate large classes into smaller files
  • Separate generated code from custom code
  • Multiple developers can work on same class
  • Organize code logically

Q51: What is a partial method?

Partial Method is a method declared in a partial class that can optionally be implemented in another part of the class. It's commonly used in code generation scenarios.

Partial Method Characteristics:

  • Must be declared in partial class
  • Can have implementation or not
  • Private by default
  • Cannot return value (void only)
Q52: What is an extension method?

Extension Method allows adding methods to existing types without modifying the original type. It's a static method with `this` parameter.

Extension Method Benefits:

  • Add functionality to sealed classes
  • Add methods to interfaces
  • Improve code readability
