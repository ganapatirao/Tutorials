1. Azure Fundamentals

Q1: What is Microsoft Azure?

Microsoft Azure is a cloud computing platform with services including analytics, storage, and networking. It provides IaaS, PaaS, and SaaS solutions.

Key Features:

  • Hybrid Cloud: Seamless on-premises and cloud integration
  • Global Reach: Data centers in 60+ regions
  • Security: Enterprise-grade security and compliance
  • Scalability: Auto-scaling and load balancing
  • Services: 200+ cloud services
Q2: What is the difference between IaaS, PaaS, and SaaS?
Service Model Provider Manages You Manage Example
IaaS Hardware, networking OS, applications, data Azure VMs
PaaS Hardware, OS, runtime Applications, data Azure App Service
SaaS Everything except data Data and configuration Office 365
Q3: What are Azure regions?

Azure Regions are geographical locations with data centers. Each region is paired with another region for disaster recovery. Examples include East US, West Europe, Southeast Asia. Azure regions are sets of data centers deployed within a latency-defined perimeter and connected through a dedicated regional low-latency network. Regions provide redundancy and availability for applications and data.

Azure Regions Features:

  • Global Reach: 60+ regions worldwide with more coming
  • Region Pairs: Paired regions for disaster recovery and data residency
  • Availability Zones: Physically separate data centers within a region
  • Compliance: Data residency and sovereignty requirements
  • Low Latency: Regions close to users reduce network latency
  • Resilience: Built-in redundancy and fault tolerance
list_regions.sh
# List all Azure regions az account list-locations \ --query "[].{Name:name, DisplayName:displayName}" \ --output table
Q4: What is Azure Resource Manager?

Azure Resource Manager (ARM) is the deployment and management service for Azure. It provides a consistent management layer for resources through templates, REST APIs, and CLI. ARM enables you to deploy, manage, and monitor all resources in your Azure account as a group, rather than individually. It uses JSON templates to define infrastructure, enabling Infrastructure as Code practices.

ARM Features:

  • Declarative Templates: Define infrastructure in JSON or Bicep
  • Dependency Management: Automatically handles resource dependencies
  • Access Control: Integrated with Azure RBAC for authorization
  • Tagging: Organize resources with metadata tags
  • Rollback: Automatic rollback on deployment failure
  • Extensibility: Supports custom providers and extensions
azure_cli.sh
# Create resource group using Azure CLI az group create \ --name myResourceGroup \ --location eastus # Deploy template az deployment group create \ --resource-group myResourceGroup \ --template-file azuredeploy.json \ --parameters storageAccountName=mystorage
Q5: What is an Azure subscription?

Azure Subscription is a logical container for Azure resources. It provides billing boundaries and access control. Each resource must belong to a subscription. Subscriptions are used to organize and manage costs, access, and resources. You can have multiple subscriptions to separate environments (dev, test, production) or departments, and apply different billing and management policies to each.

Azure Subscription Features:

  • Billing Boundary: Separate billing for each subscription
  • Access Control: Apply RBAC policies at subscription level
  • Resource Limits: Quotas and limits per subscription
  • Management: Organize resources by department or project
  • Cost Management: Track and manage costs per subscription
  • Isolation: Separate environments for security and compliance
list_subscriptions.sh
# List all subscriptions az account list \ --query "[].{Name:name, Id:id, TenantId:tenantId}" \ --output table # Set default subscription az account set \ --subscription "My Subscription"
Q6: What is Azure Resource Group?

Azure Resource Group is a container that holds related resources for an Azure solution. It includes all resources that should be managed as a group. Resource groups enable you to manage the lifecycle of all resources in your solution together. When you delete a resource group, all resources within it are deleted. Resources in a resource group can be located in different regions but must belong to the same subscription.

Resource Group Features:

  • Logical Grouping: Group related resources together
  • Lifecycle Management: Deploy, update, and delete resources as a group
  • Access Control: Apply RBAC at resource group level
  • Tagging: Apply tags to all resources in the group
  • Cost Tracking: Monitor costs by resource group
  • Deployment: Deploy ARM templates to resource groups
resource_group.py
from azure.mgmt.resource import ResourceManagementClient resource_client = ResourceManagementClient(credential, subscription_id) # Create resource group rg_result = resource_client.resource_groups.create_or_update( resource_group_name='myResourceGroup', parameters={'location': 'eastus'} )

2. Azure Compute Services

Q7: What is Azure Virtual Machine?

Azure VM is an IaaS service providing on-demand, scalable computing resources. It runs Windows or Linux in the cloud. Azure VMs give you the flexibility of virtualization without the need to buy and maintain physical hardware. You can choose from various VM sizes and configurations to match your workload requirements, and pay only for what you use.

Azure VM Features:

  • Flexible Sizing: Choose from various VM sizes and types
  • Multiple OS: Support for Windows, Linux, and custom images
  • Auto Scaling: Scale VMs based on demand
  • Availability Sets: High availability with 99.95% SLA
  • Availability Zones: Zone-level isolation for critical workloads
  • Spot Instances: Cost-effective unused capacity
create_vm.sh
# Create Azure VM az vm create \ --resource-group myResourceGroup \ --name myVM \ --image UbuntuLTS \ --admin-username azureuser \ --generate-ssh-keys \ --size Standard_B2s \ --location eastus # List VMs az vm list \ --resource-group myResourceGroup \ --output table
Q8: What is Azure App Service?

Azure App Service is a PaaS service for building, deploying, and scaling web apps and APIs. It supports multiple languages and frameworks. App Service enables you to build and host web applications, RESTful APIs, and mobile backends in the programming language of your choice without managing infrastructure. It provides built-in auto-scaling, load balancing, and deployment slots for zero-downtime deployments.

App Service Features:

  • Multi-Language: Supports .NET, Java, Node.js, Python, PHP, and more
  • Auto Scaling: Automatically scale based on CPU, memory, or custom metrics
  • Deployment Slots: Zero-downtime deployments with staging slots
  • Custom Domains: Use custom domains with SSL certificates
  • Backups: Automated backups and restore functionality
  • Integration: Works with Azure DevOps, GitHub, and CI/CD pipelines
create_app.sh
# Create App Service Plan az appservice plan create \ --name myAppServicePlan \ --resource-group myResourceGroup \ --sku B1 \ --location eastus # Create Web App az webapp create \ --name myWebApp \ --resource-group myResourceGroup \ --plan myAppServicePlan \ --runtime "PYTHON:3.8"
Q9: What is Azure Functions?

Azure Functions is a serverless compute service that runs event-triggered code without managing infrastructure. It's ideal for event-driven scenarios. Functions enables you to run code on-demand without provisioning or managing servers. You pay only for the time your code runs, and it scales automatically based on the number of incoming events. Functions supports multiple programming languages and integrates with various Azure services.

Azure Functions Features:

  • Serverless: No infrastructure to manage or provision
  • Event-Driven: Triggered by HTTP, timers, queues, blobs, and more
  • Multiple Languages: Supports C#, Java, JavaScript, Python, PowerShell, and TypeScript
  • Auto Scaling: Automatically scales based on demand
  • Pay-per-Use: Pay only for execution time and resources consumed
  • Durable Functions: Stateful functions for complex workflows
function.py
import azure.functions as func @app.route("hello") @app.trigger(types.HttpTrigger) def hello(req): name = req.params.get('name', 'World') return func.HttpResponse( f"Hello, {name}!", status_code=200 ) # Timer trigger example @app.schedule("0 */5 * * * *") def timer_trigger(timer): print("Timer function executed")
Q10: What is Azure Kubernetes Service (AKS)?

AKS is a managed Kubernetes service for container orchestration. It simplifies deployment, management, and scaling of containerized applications. AKS reduces the complexity and operational overhead of managing Kubernetes by offloading responsibility to Azure. It provides automated upgrades, self-healing, and integrated monitoring. AKS integrates with Azure DevOps, GitHub, and other CI/CD tools for seamless deployment.

AKS Features:

  • Managed Control Plane: Azure manages the Kubernetes master nodes
  • Auto Scaling: Cluster autoscaler and horizontal pod autoscaler
  • Upgrades: Automated Kubernetes version upgrades
  • Security: Integrated with Azure AD for authentication
  • Monitoring: Built-in integration with Azure Monitor
  • DevOps Integration: Works with Azure DevOps and GitHub Actions
create_aks.sh
# Create AKS cluster az aks create \ --resource-group myResourceGroup \ --name myAKSCluster \ --node-count 2 \ --node-vm-size Standard_B2s \ --enable-cluster-autoscaler \ --min-count 1 \ --max-count 3 # Get credentials az aks get-credentials \ --resource-group myResourceGroup \ --name myAKSCluster
Q11: What is Azure Container Instances?

ACI provides serverless containers that run on demand. It's ideal for burst scenarios and containerized applications. ACI offers the fastest and simplest way to run a container in Azure without having to manage any virtual machines or adopt any orchestration tools. It's perfect for scenarios where you just need to run a container without the complexity of Kubernetes.

ACI Features:

  • Serverless: No infrastructure to manage
  • Fast Deployment: Deploy containers in seconds
  • Per-Second Billing: Pay only for the seconds your container runs
  • Flexible Sizing: Choose CPU and memory configurations
  • Virtual Network: Deploy containers in a VNet for isolation
  • Integration: Works with Azure DevOps and CI/CD pipelines

Q12: What is Azure Batch?

Azure Batch is a cloud service for large-scale parallel and high-performance computing applications. It schedules jobs and manages compute resources. Batch enables you to run large-scale parallel and high-performance computing (HPC) applications efficiently in the cloud. It automatically provisions compute resources, schedules jobs, and manages task execution, allowing you to focus on your application logic rather than infrastructure management.

Azure Batch Features:

  • Job Scheduling: Automatic job and task scheduling
  • Auto Scaling: Scale compute pools based on workload
  • Task Dependencies: Define task dependencies and execution order
  • Low-Priority VMs: Cost-effective compute with spot instances

  • Monitoring: Built-in monitoring and logging

3. Azure Storage

Q13: What are Azure Storage types?

Azure Storage provides multiple storage services to meet different data storage needs. Each storage type is optimized for specific use cases and data access patterns.

Storage Types:

  • Blob Storage: Unstructured data (images, documents, videos)
  • Queue Storage: Message queuing for reliable communication
  • Table Storage: NoSQL key-value store for structured data
  • File Storage: Fully managed file shares accessible via SMB
  • Disk Storage: Persistent block storage for VMs
  • Archive Storage: Long-term data archival at low cost
Q14: What is Azure Blob Storage?

Azure Blob Storage is object storage for unstructured data. It supports three tiers: Hot, Cool, and Archive for different access patterns and costs. Blob Storage is optimized for storing massive amounts of unstructured data such as text or binary data. It's ideal for serving images or documents directly to browsers, storing files for distributed access, streaming video and audio, writing to log files, and storing data for backup and restore, disaster recovery, and archiving.

Blob Storage Features:

  • Storage Tiers: Hot, Cool, and Archive tiers for cost optimization
  • Scalability: Virtually unlimited storage capacity
  • Security: Encryption at rest and in transit
  • Access Tiers: Automatic tiering based on access patterns
  • CORS Support: Cross-origin resource sharing for web apps
  • Lifecycle Management: Automated data lifecycle policies
blob_storage.py
from azure.storage.blob import BlobServiceClient blob_service = BlobServiceClient.from_connection_string(conn_string) container_client = blob_service.get_container_client("mycontainer") # Create container container_client.create_container() # Upload blob blob_client = container_client.get_blob_client("myfile.txt") blob_client.upload_blob(data) # Download blob downloaded_blob = blob_client.download_blob() content = downloaded_blob.readall()
Q15: What is Azure Disk Storage?

Azure Disk Storage provides block-level storage for Azure VMs. Types include Ultra Disk, Premium SSD, Standard SSD, and Standard HDD. Azure Disk Storage offers high-performance, durable block storage designed to be used with Azure Virtual Machines. It provides different disk types to meet various performance and cost requirements, from ultra-low latency workloads to cost-effective storage for less demanding applications.

Disk Storage Types:

  • Ultra Disk: Highest performance with sub-millisecond latency
  • Premium SSD: High-performance storage for production workloads
  • Standard SSD: Cost-effective storage for web servers and dev/test
  • Standard HDD: Lowest cost for backup and non-critical workloads
  • Shared Disks: Multiple VMs can attach to the same disk
  • Snapshots: Point-in-time backups of disks

Q16: What is Azure Files?

Azure Files offers fully managed file shares accessible via SMB protocol. It's ideal for lift-and-shift scenarios and legacy applications. Azure Files provides fully managed file shares in the cloud that are accessible via the industry-standard Server Message Block (SMB) protocol. It's perfect for scenarios where you need to share files across multiple VMs, on-premises servers, or cloud applications without the need to manage file servers.

Azure Files Features:

  • SMB Protocol: Access via SMB 3.0+ with encryption
  • Managed Service: No need to manage file servers
  • Integration: Mount from Windows, Linux, and macOS
  • Backup: Azure Backup integration for file shares
  • Sync: Azure File Sync for on-premises caching
  • Scalability: Up to 100 TiB per file share

Q17: What is Azure Queue Storage?

Azure Queue Storage provides messaging for reliable communication between application components. It supports asynchronous processing. Queue Storage is a service for storing large numbers of messages that can be accessed from anywhere in the world via authenticated calls using HTTP or HTTPS. It's ideal for building asynchronous workflows, decoupling application components, and creating message queues for distributed applications.

Queue Storage Features:

  • Asynchronous Messaging: Decouple application components
  • Reliable Delivery: At-least-once delivery guarantee
  • Scalability: Store millions of messages
  • Access Control: Shared Access Signatures for secure access
  • Visibility Timeout: Control message visibility during processing
  • Integration: Works with Azure Functions and WebJobs
queue_storage.py
from azure.storage.queue import QueueServiceClient queue_service = QueueServiceClient.from_connection_string(conn_string) queue_client = queue_service.get_queue_client("myqueue") # Create queue queue_client.create_queue() # Send message queue_client.send_message("Hello, Queue!") # Receive message messages = queue_client.receive_messages() for message in messages: print(message.content) queue_client.delete_message(message)
Q18: What is Azure Table Storage?

Azure Table Storage is a NoSQL key-value store for structured data. It's ideal for storing large amounts of data with minimal latency. Table Storage is a NoSQL key-value store for storing structured non-relational data. It provides a schemaless design, making it easy to adapt to changing data requirements. It's ideal for storing massive amounts of structured data with fast access, such as user profiles, session data, and telemetry data.

Table Storage Features:

Q19: What is Azure Storage Account?

Azure Storage Account is a unique namespace for all storage objects. It provides access to Azure Blob, File, Queue, and Table storage. A storage account provides a unique namespace for your Azure Storage data that is accessible from anywhere in the world over HTTP or HTTPS. It contains all your Azure Storage data objects: blobs, file shares, queues, tables, and disks. The storage account provides the management endpoint for accessing all these services.

Storage Account Features:

  • Unique Namespace: Unique DNS name for your storage
  • Access Control: Azure AD authentication and shared access signatures
  • Redundancy: LRS, ZRS, GRS, and GZRS options
  • Performance: Standard and Premium performance tiers
  • Encryption: Server-side and client-side encryption
  • Integration: Works with all Azure services

Q20: What are storage tiers?

Azure Storage Tiers allow you to optimize costs by storing data in the most cost-effective tier based on access patterns. Blob storage offers three access tiers: Hot, Cool, and Archive. Each tier is designed for different use cases and has different pricing models based on access frequency and data retrieval costs.

Tier Use Case Cost Access
Hot Frequently accessed data Higher storage cost Fast access (milliseconds)
Cool Infrequently accessed data Lower storage cost 30-day latency
Archive Rarely accessed data Lowest storage cost 180-day latency

Storage Tier Features:

  • Cost Optimization: Reduce costs by choosing the right tier
  • Automatic Tiering: Lifecycle policies to move data between tiers
  • Flexible Access: Change tiers as needed
  • Performance: Hot tier for high-performance workloads
  • Compliance: Archive tier for long-term retention
  • Retrieval Costs: Consider retrieval costs when choosing tiers

4. Azure Networking

Q21: What is Azure Virtual Network?

Azure VNet is a private network in Azure. It enables Azure resources to communicate with each other and the internet securely. Azure Virtual Network (VNet) is the fundamental building block for your private network in Azure. VNet enables many types of Azure resources, such as Azure Virtual Machines (VM), to securely communicate with each other, the internet, and on-premises networks. It provides isolation and segmentation of your cloud resources.

VNet Features:

  • Isolation: Private network isolation for your resources
  • Subnets: Segment your VNet into multiple subnets
  • Peering: Connect VNets across regions
  • VPN Gateway: Connect to on-premises networks
  • ExpressRoute: Private connection to Azure
  • Service Endpoints: Secure access to Azure services
create_vnet.sh
# Create virtual network az network vnet create \ --resource-group myResourceGroup \ --name myVNet \ --address-prefix 10.0.0.0/16 \ --subnet-name mySubnet \ --subnet-prefix 10.0.0.0/24 # List VNets az network vnet list \ --resource-group myResourceGroup \ --output table
Q22: What is Azure VPN Gateway?

Azure VPN Gateway provides encrypted cross-premises connectivity between Azure and on-premises networks or between Azure VNets. VPN Gateway is a type of virtual network gateway that is used to send encrypted traffic between an Azure virtual network and an on-premises location over the public internet. It also provides secure connectivity between Azure VNets.

VPN Gateway Features:

  • Site-to-Site VPN: Connect on-premises networks to Azure
  • Point-to-Site VPN: Connect individual users to Azure
  • VNet-to-VNet: Connect Azure VNets across regions
  • Encryption: IKEv2/IPsec encryption for secure tunnels
  • High Availability: Active-active and active-passive modes
  • BGP Support: Dynamic routing with BGP
create_vpn.sh
# Create VPN Gateway az network vnet-gateway create \ --resource-group myResourceGroup \ --name myVPNGateway \ --vnet myVNet \ --public-ip-address myPublicIP \ --gateway-type Vpn \ --vpn-type RouteBased \ --sku VpnGw1
Q23: What is Azure ExpressRoute?

ExpressRoute provides private, dedicated connections between on-premises networks and Azure data centers, bypassing the public internet. ExpressRoute connections don't go over the public internet, offering more reliability, faster speeds, lower latencies, and higher security than typical connections over the internet. It's ideal for scenarios requiring high security, reliability, and performance.

ExpressRoute Features:

  • Private Connection: Dedicated fiber connection to Azure
  • Higher Security: Traffic doesn't traverse the public internet
  • Lower Latency: Consistent and predictable network performance
  • Higher Bandwidth: Up to 100 Gbps bandwidth
  • Redundancy: Built-in redundancy with multiple circuits
  • Global Reach: Connect to Azure regions worldwide
Q24: What is Azure Load Balancer?

Azure Load Balancer distributes inbound traffic across healthy VMs. It provides high availability and scalability for applications. Azure Load Balancer operates at layer 4 (Transport layer) and distributes incoming network traffic across healthy instances of services defined in a load-balanced set. It provides low latency and high throughput, and scales out to millions of flows for all TCP and UDP applications.

Load Balancer Features:

  • Layer 4 Load Balancing: TCP and UDP traffic distribution
  • Health Probes: Automatic health checks for backend pools
  • High Availability: Zone-redundant and regional deployments
  • Outbound SNAT: Provides outbound internet access
  • HA Ports: Load balance all TCP and UDP flows
  • Secure by Default: Zone-redundant with automatic failover
create_lb.sh
# Create public IP az network public-ip create \ --resource-group myResourceGroup \ --name myPublicIP \ --sku Standard # Create load balancer az network lb create \ --resource-group myResourceGroup \ --name myLoadBalancer \ --public-ip-address myPublicIP \ --sku Standard
Q25: What is Azure Application Gateway?

Azure Application Gateway is a web traffic load balancer with layer 7 routing, SSL termination, and web application firewall capabilities. Application Gateway is a web traffic load balancer that enables you to manage traffic to your web applications. It operates at layer 7 (application layer) and provides advanced features like SSL termination, cookie-based session affinity, URL path-based routing, and integrated web application firewall (WAF).

Application Gateway Features:

  • Layer 7 Load Balancing: HTTP/HTTPS traffic routing
  • SSL Termination: Offload SSL processing to the gateway
  • WAF: Built-in web application firewall for security
  • Path-based Routing: Route based on URL paths
  • Cookie Affinity: Session affinity for sticky sessions
  • Auto Scaling: Automatically scale based on traffic
create_appgw.sh
# Create Application Gateway az network application-gateway create \ --resource-group myResourceGroup \ --name myAppGateway \ --capacity 2 \ --sku Standard_v2 \ --http-settings-cookie-based-affinity Enabled \ --public-ip-address myPublicIP
Q26: What is Azure Front Door?

Azure Front Door is a global layer 7 load balancer and web application firewall that provides fast, reliable, and secure web applications. Front Door is a global Application Delivery Network (ADN) as a service that provides a highly available and globally distributed web application accelerator platform. It offers dynamic site acceleration, SSL offloading, caching, and web application firewall capabilities.

Front Door Features:

  • Global Distribution: Edge locations worldwide for low latency
  • Layer 7 Load Balancing: HTTP/HTTPS traffic routing
  • WAF: Integrated web application firewall
  • Caching: Static content caching at edge
  • SSL Termination: End-to-end SSL with custom certificates
  • Health Probes: Automatic health checks for backend pools
Q27: What is Azure Traffic Manager?

Traffic Manager is a DNS-based traffic load balancer that distributes traffic across global Azure regions based on performance, priority, or geography. Traffic Manager uses DNS to direct client traffic to the closest or most appropriate endpoint based on traffic routing methods such as priority, weighted, performance, geographic, and multi-value routing. It doesn't see the traffic it routes, only the DNS requests from clients.

Traffic Manager Features:

  • DNS-based: DNS-level traffic routing
  • Performance Routing: Route to the closest endpoint
  • Priority Routing: Primary and backup endpoints
  • Weighted Routing: Distribute traffic based on weights
  • Geographic Routing: Route based on user location
  • Health Monitoring: Automatic endpoint health checks

Azure DNS provides a DNS hosting service using Azure infrastructure. It offers high availability, security, and performance. Azure DNS is a hosting service for DNS domains that provides name resolution by using Microsoft Azure infrastructure. By hosting your domains in Azure, you can manage your DNS records using the same credentials, APIs, tools, and billing as your other Azure services.

Azure DNS Features:

Q29: What is Azure Content Delivery Network?

Azure CDN delivers content to users globally with high bandwidth and low latency by caching content at edge locations. Azure Content Delivery Network (CDN) is a global CDN solution for delivering high-bandwidth content. It can be hosted in Azure or any other cloud provider, using edge locations close to users to reduce latency and improve performance for static and dynamic content.

Azure CDN Features:

Azure Private Link provides private connectivity to Azure services without exposing them to the public internet. Private Link enables you to access Azure services like Azure Storage, Azure SQL Database, and Azure Cosmos DB through a private endpoint in your VNet. Private Link Features:

Azure SQL Database is a fully managed relational database service based on SQL Server. It offers high availability, scalability, and security. Azure SQL Database is a fully managed platform as a service (PaaS) database engine that handles most of the database management functions such as upgrading, patching, backups, and monitoring without user involvement. Azure SQL Database Features:

Cosmos DB Features:

Performance Guaranteed performance Burstable performance
Management Individual database management Pool-level management
