Q6: What is the difference between Blob storage tiers?
  • Hot tier: Frequently accessed data. Higher cost, lowest latency.
  • Cool tier: Infrequently accessed data (30+ days). Lower cost, higher latency.
  • Archive tier: Rarely accessed data (180+ days). Lowest cost, highest latency.
Important: Choose the appropriate tier based on access patterns to optimize costs. Data can be moved between tiers.

7. Azure Functions

Q12: What is Azure Functions?

Azure Functions is a serverless compute service that enables you to run event-triggered code without managing infrastructure.

Trigger Types:

  • HTTP triggers: HTTP endpoints
  • Timer triggers: Scheduled execution
  • Blob triggers: Blob storage events
  • Queue triggers: Queue messages
  • Service Bus triggers: Service Bus messages
  • Event Grid triggers: Event-driven
  • Cosmos DB triggers: Database changes
// C# Azure Function
[FunctionName("HttpTrigger")]
public static async Task<IActionResult> Run(
    [HttpTrigger(AuthorizationLevel.Function, "get", "post")] HttpRequest req,
    ILogger log)
{
    log.LogInformation("C# HTTP trigger function processed a request.");
    
    string name = req.Query["name"];
    
    return name != null
        ? (ActionResult)new OkObjectResult($"Hello, {name}")
        : new BadRequestObjectResult("Please pass a name on the query string");
}

11. Azure Architecture Patterns (All Levels Level)

Q31: What are common Azure architecture patterns?
  • Web-Queue-Worker: Decouple web tier from background processing using queues
  • CQRS: Command Query Responsibility Segregation for read/write separation
  • Event Sourcing: Store state as sequence of events
  • Microservices: Small, independent services communicating via APIs
  • N-Tier: Separation into presentation, application, and data layers
  • Space-Based: Distributed caching for high scalability
  • Leader Election: Coordinate distributed tasks
Q32: How do you implement a serverless architecture in Azure? 
 Azure Functions + Azure Cosmos DB + Azure Service Bus
// Function triggered by HTTP request
module.exports = async function (context, req) {
    const message = req.body;
    
    // Add message to Service Bus queue
    context.bindings.outputQueue = message;
    
    context.res = {
        status: 202,
        body: "Message queued"
    };
};

// Function triggered by Service Bus queue
module.exports = async function (context, myQueueItem) {
    // Process message and store in Cosmos DB
    context.bindings.outputDocument = {
        id: myQueueItem.id,
        data: myQueueItem,
        timestamp: new Date().toISOString()
    };
};

// function.json bindings
{
  "bindings": [
    {
      "authLevel": "function",
      "type": "httpTrigger",
      "direction": "in",
      "name": "req"
    },
    {
      "type": "http",
      "direction": "out",
      "name": "res"
    },
    {
      "type": "serviceBus",
      "direction": "out",
      "name": "outputQueue",
      "queueName": "messages"
    }
  ]
}

Q51: How do you implement Azure Management Groups and Azure Policy for governance?

Management Groups enable hierarchical organization and policy enforcement.

# Create management group
az account management-group create --name "Production"

# Create subscription under management group
az account management-group subscription add --name "Production" --subscription-id "12345678-1234-1234-1234-123456789012"

# Create Azure Policy definition
az policy definition create --name "DenyPublicIP" \
  --rules policy.json \
  --params parameters.json \
  --display-name "Deny Public IP Assignment"

# Policy rule
{
  "if": {
    "field": "type",
    "equals": "Microsoft.Network/publicIPAddresses"
  },
  "then": {
    "effect": "deny"
  }
}

# Assign policy to management group
az policy assignment create --name "DenyPublicIPAssignment" \
  --policy "/subscriptions/.../providers/Microsoft.Authorization/policyDefinitions/DenyPublicIP" \
  --scope "/providers/Microsoft.Management/managementGroups/Production"

# Create Azure Blueprint
az blueprint create --name "EnterpriseLandingZone" \
  --management-group-id "Production" \
  --description "Enterprise landing zone blueprint"
Q52: How do you implement Azure Virtual WAN for hybrid networking?

Virtual WAN provides optimized and secure branch-to-Azure connectivity.

# Create Virtual WAN
az network vwan create --name "MyVirtualWAN" \
  --resource-group "MyResourceGroup" \
  --location "EastUS" \
  --type "Standard"

# Create virtual hub
az network vhub create --name "MyHub" \
  --resource-group "MyResourceGroup" \
  --vwan "MyVirtualWAN" \
  --location "EastUS" \
  --address-prefix "10.0.0.0/24"

# Create VPN site
az network vpn-site create --name "BranchOffice" \
  --resource-group "MyResourceGroup" \
  --virtual-wan "MyVirtualWAN" \
  --ip-addresses "203.0.113.1"

# Create VPN gateway
az network vpn-gateway create --name "MyVPNGateway" \
  --resource-group "MyResourceGroup" \
  --vhub "MyHub" \
  --location "EastUS" \
  --sku "VpnGw1"

# Create ExpressRoute gateway
az network express-route-gateway create --name "MyERGateway" \
  --resource-group "MyResourceGroup" \
  --vhub "MyHub" \
  --location "EastUS" \
  --min-vpn-gateway-scale-units 1

Q54: How do you implement Azure Event Grid for event-driven architecture?

Event Grid provides event routing for event-driven applications.

# Create Event Grid topic
az eventgrid topic create --name "OrderEvents" \
  --resource-group "MyResourceGroup" \
  --location "EastUS"

# Create event subscription
az eventgrid event-subscription create \
  --source-resource-id "/subscriptions/.../providers/Microsoft.EventGrid/topics/OrderEvents" \
  --name "ProcessOrderSubscription" \
  --endpoint "https://myapp.azurewebsites.net/api/orders" \
  --event-delivery-schema "eventgridschema"

# Event Grid domain
az eventgrid domain create --name "MyDomain" \
  --resource-group "MyResourceGroup" \
  --location "EastUS"

# Publish event to Event Grid
az eventgrid event trigger --resource-group "MyResourceGroup" \
  --source "/subscriptions/.../providers/Microsoft.EventGrid/topics/OrderEvents" \
  --event-types "OrderCreated" \
  --data '{"orderId":"12345","status":"pending"}'

Q56: How do you implement Azure AKS for Kubernetes orchestration?

AKS (Azure Kubernetes Service) provides managed Kubernetes.

# Create AKS cluster
az aks create --name "MyCluster" \
  --resource-group "MyResourceGroup" \
  --location "EastUS" \
  --node-count 3 \
  --node-vm-size Standard_DS2_v2 \
  --enable-cluster-autoscaler \
  --min-count 2 \
  --max-count 10 \
  --enable-managed-identity \
  --network-plugin azure

# Enable Azure AD integration
az aks update --name "MyCluster" \
  --resource-group "MyResourceGroup" \
  --enable-aad

# Create node pool
az aks nodepool add --name "mynodepool" \
  --cluster-name "MyCluster" \
  --resource-group "MyResourceGroup" \
  --node-count 3 \
  --node-vm-size Standard_DS3_v2

# Scale node pool
az aks nodepool scale --name "mynodepool" \
  --cluster-name "MyCluster" \
  --resource-group "MyResourceGroup" \
  --node-count 5
