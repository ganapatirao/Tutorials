Angular Interview Guide

Comprehensive guide covering Angular framework with detailed explanations and code examples for all experience levels

1. Angular Fundamentals

Q1: What is Angular?

Angular is a TypeScript-based open-source web application framework led by Google. It's a complete rewrite of AngularJS (Angular 1.x) and provides a comprehensive platform for building single-page applications (SPAs). Angular uses TypeScript for type safety, provides built-in dependency injection, two-way data binding, and a component-based architecture.

app.component.ts
import { Component } from '@angular/core'; @Component({ selector: 'app-root', template: '<h1>Hello {{ name }}</h1>' }) export class AppComponent { name = 'Angular'; }
Q2: What is the difference between AngularJS and Angular?

AngularJS (1.x) uses JavaScript and follows MVC pattern. It has two-way data binding by default but lacks TypeScript support and has performance issues with large applications.

Angular (2+) uses TypeScript and follows component-based architecture. It provides better performance, mobile support, improved dependency injection, and a more modular structure. Angular is a complete rewrite, not an upgrade.

comparison.ts
// AngularJS (JavaScript) var app = angular.module('myApp', []); app.controller('MyCtrl', function($scope) { $scope.name = 'AngularJS'; }); // Angular (TypeScript) @Component({ selector: 'app-root' }) export class AppComponent { name = 'Angular'; }
Q3: What is TypeScript and why does Angular use it?

TypeScript is a superset of JavaScript that adds static typing, interfaces, classes, and other features. Angular uses TypeScript because it provides better tooling, early error detection, improved code readability, and enhanced IDE support with IntelliSense.

types.ts
// TypeScript with type annotations interface User { id: number; name: string; email: string; } function getUser(id: number): User { return { id, name: 'John', email: 'john@example.com' }; } // Type checking catches errors at compile time const user: User = getUser(1);
Q4: What are the key features of Angular?

Key Features: Component-based architecture, TypeScript support, Two-way data binding, Dependency Injection, Directives, Pipes, Routing, Forms (Template-driven and Reactive), HTTP Client, Observables (RxJS), Lazy loading, and comprehensive testing support.

Q5: What is the Angular CLI?

Angular CLI (Command Line Interface) is a tool to initialize, develop, scaffold, and maintain Angular applications. It provides commands for creating components, services, modules, and running the development server with hot module replacement.

terminal
# Create new Angular project ng new my-app # Generate component ng g component product # Generate service ng g service product # Run development server ng serve # Build for production ng build --prod

2. Components and Templates

Q6: What is an Angular component?

Component is the fundamental building block of Angular applications. A component consists of a TypeScript class, HTML template, CSS styles, and metadata decorator. Components control a patch of screen called a view.

product.component.ts
import { Component, OnInit } from '@angular/core'; @Component({ selector: 'app-product', templateUrl: './product.component.html', styleUrls: ['./product.component.css'] }) export class ProductComponent implements OnInit { products: any[] = []; ngOnInit() { // Initialize component } }
Q7: What is the component lifecycle?

Component Lifecycle hooks allow you to tap into key moments in a component's life: ngOnChanges (input changes), ngOnInit (after constructor), ngDoCheck (custom change detection), ngAfterContentInit (content projection), ngAfterContentChecked, ngAfterViewInit (view initialization), ngAfterViewChecked, and ngOnDestroy (cleanup).

lifecycle.ts
import { Component, OnInit, OnDestroy } from '@angular/core'; export class MyComponent implements OnInit, OnDestroy { ngOnInit() { // Called after constructor console.log('Component initialized'); } ngOnDestroy() { // Cleanup before destruction console.log('Component destroyed'); } }
Q8: What is the difference between constructor and ngOnInit?

Constructor is a TypeScript feature used for dependency injection and initializing class fields. It's called when the class is instantiated.

ngOnInit is an Angular lifecycle hook called after the constructor and after Angular has set input properties. Use ngOnInit for initialization logic that depends on @Input bindings.

Q9: What are @Input and @Output decorators?

@Input decorator marks a property as an input that receives data from a parent component. @Output decorator marks a property as an output that emits events to a parent component using EventEmitter.

child.component.ts
import { Component, Input, Output, EventEmitter } from '@angular/core'; @Component({ selector: 'app-child' }) export class ChildComponent { @Input() data: string; @Output() dataChanged = new EventEmitter<string>(); onChange(newValue: string) { this.dataChanged.emit(newValue); } }
Q10: What is content projection?

Content projection allows you to project content from a parent component into a child component using ng-content. It's useful for creating reusable components with flexible content.

card.component.ts
// Child component template <div class="card"> <ng-content></ng-content> </div> // Parent component usage <app-card> <h1>Card Title</h1> <p>Card content</p> </app-card>

3. Data Binding

Q11: What are the types of data binding in Angular?

Data Binding types: Interpolation {{}} (one-way), Property binding [] (one-way), Event binding () (one-way), Two-way binding [()] (bidirectional), and Attribute binding. Data binding synchronizes data between component and template.

binding.html
// Interpolation <p>{{ name }}</p> // Property binding <img [src]="imageUrl"> <div [class.active]="isActive"></div> // Event binding <button (click)="onClick()">Click</button> // Two-way binding <input [(ngModel)]="name">
Q12: What is ngModel and how does it work?

ngModel is a directive that implements two-way data binding. It combines property binding [value] and event binding (input) into a single syntax. Requires FormsModule to be imported in the module.

app.module.ts
import { FormsModule } from '@angular/forms'; @NgModule({ imports: [FormsModule], // ... })
Q13: What is the difference between property binding and attribute binding?

Property binding [] sets DOM properties like src, disabled, hidden. It's the preferred way as it works with DOM properties directly.

Attribute binding [attr.] sets HTML attributes like colspan, aria-label. Use it when there's no corresponding DOM property.

binding.html
// Property binding <input [disabled]="isDisabled"> // Attribute binding <td [attr.colspan]="colSpan">
Q14: What is class and style binding?

Class binding [class.className] conditionally adds/removes CSS classes. Style binding [style.styleProperty] dynamically sets inline styles. Both support object and string syntax.

styles.html
// Class binding <div [class.active]="isActive"></div> <div [class]="{'active': isActive, 'disabled': isDisabled}"></div> // Style binding <div [style.color]="textColor"></div> <div [style]="{'color': textColor, 'font-size': fontSize + 'px'}"></div>
Q15: What is safe navigation operator (?)?

Safe navigation operator (?) prevents null reference errors. It returns null if the object is null instead of throwing an error. Useful when properties might not be initialized.

safe.html
// Without safe navigation - throws error if user is null {{ user.name }} // With safe navigation - returns null if user is null {{ user?.name }}

4. Directives

Q16: What are the types of directives in Angular?

Directives are classes that add additional behavior to elements. Types: Components (directives with template), Attribute directives (change appearance/behavior), Structural directives (change DOM layout).

directive-types.ts
// Component - directive with template @Component({ selector: 'app-product', template: '...' }) // Attribute directive - change appearance @Directive({ selector: ''[appHighlight]' }) // Structural directive - change DOM layout @Directive({ selector: ''[appIf]' })
Q17: What are built-in structural directives?

Built-in structural directives: *ngIf (conditional rendering), *ngFor (loop through arrays), *ngSwitch (switch statement), and *ngSwitchCase/*ngSwitchDefault. They add/remove elements from the DOM.

directives.html
// *ngIf - conditional rendering <div *ngIf="isVisible">Visible</div> // *ngFor - loop <li *ngFor="let item of items; let i = index"> {{ i }}: {{ item }} </li> // *ngSwitch - switch statement <div [ngSwitch]="status"> <div *ngSwitchCase="'active'">Active</div> <div *ngSwitchDefault>Inactive</div> </div>
Q18: What are built-in attribute directives?

Built-in attribute directives: ngClass (add/remove classes), ngStyle (set inline styles), ngModel (two-way binding). They modify the appearance or behavior of elements without changing the DOM structure.

attribute.html
// ngClass - dynamic classes <div [ngClass]="{'active': isActive, 'error': hasError}"></div> // ngStyle - dynamic styles <div [ngStyle]="{'color': textColor, 'font-size': fontSize + 'px'}"></div>
Q19: How do you create a custom attribute directive?

Custom attribute directives are created using the @Directive decorator. They can listen to events using @HostListener and manipulate the DOM using ElementRef.

highlight.directive.ts
import { Directive, ElementRef, HostListener } from '@angular/core'; @Directive({ selector: '[appHighlight]' }) export class HighlightDirective { constructor(private el: ElementRef) {} @HostListener('mouseenter') onMouseEnter() { this.el.nativeElement.style.backgroundColor = 'yellow'; } @HostListener('mouseleave') onMouseLeave() { this.el.nativeElement.style.backgroundColor = ''; } }

5. Pipes

Q20: What are pipes and how do you use them?

Pipes transform data in templates. Angular provides built-in pipes and allows custom pipes. Pipes are used with the pipe operator (|).

pipes.html
// Built-in pipes {{ birthday | date:'shortDate' }} {{ name | uppercase }} {{ price | currency:'USD' }} {{ value | number:'1.2-2' }} {{ object | json }} {{ items | slice:0:5 }} {{ observable$ | async }}
Q21: How do you create a custom pipe?

Custom pipes are created using the @Pipe decorator and implementing PipeTransform interface.

truncate.pipe.ts
import { Pipe, PipeTransform } from '@angular/core'; @Pipe({ name: 'truncate' }) export class TruncatePipe implements PipeTransform { transform(value: string, limit: number): string { return value.length > limit ? value.substring(0, limit) + '...' : value; } } // Usage {{ description | truncate:50 }}
Q22: What is the difference between pure and impure pipes?

Pure pipe (default) only re-evaluates when input reference changes. Impure pipe re-evaluates on every change detection cycle. Use impure pipes with caution as they can impact performance.

6. Services and Dependency Injection

Q23: What is dependency injection in Angular?

Dependency Injection (DI) is a design pattern where dependencies are provided to a class rather than the class creating them. Angular has a built-in hierarchical DI system.

product.service.ts
import { Injectable } from '@angular/core'; @Injectable({ providedIn: 'root' }) export class ProductService { constructor(private http: HttpClient) {} getProducts(): Observable<Product[]> { return this.http.get<Product[]>('/api/products'); } }
Q24: What are the different injectors in Angular?

Injectors: Root injector (app-level), Module injector (NgModule-level), Element injector (component-level), and Platform injector (platform-level). Angular's DI system follows a hierarchical injector tree.

injectors.ts
// Root injector - app level @Injectable({ providedIn: 'root' }) // Module injector - module level @NgModule({ providers: [MyService] }) // Component injector - component level @Component({ providers: [MyService] })
Q25: What is providedIn: 'root'?

providedIn: 'root' registers the service as a singleton at the root level. This creates a single instance shared across the entire application and enables tree-shaking for better bundle optimization.

service.ts
@Injectable({ providedIn: 'root' // Singleton at root level }) export class DataService {}

7. Modules

Q26: What is an NgModule?

NgModule is a class decorated with @NgModule that groups related components, directives, pipes, and services. It defines the compilation context for a set of components.

app.module.ts
@NgModule({ declarations: [AppComponent, ProductComponent], imports: [BrowserModule, HttpClientModule, AppRoutingModule], providers: [ProductService], bootstrap: [AppComponent] }) export class AppModule {}
Q27: What is the difference between declarations and imports?

declarations: Components, directives, and pipes that belong to this module. They must be declared in exactly one module.

imports: Other modules whose exported components, directives, and pipes are needed in this module's templates.

module.ts
@NgModule({ declarations: [MyComponent], // Belongs to this module imports: [CommonModule], // From other module exports: [MyComponent] // Exported for other modules })
Q28: What is lazy loading of modules?

Lazy loading loads feature modules on demand when the user navigates to their route. This improves initial load time by only loading the necessary code.

app-routing.module.ts
const routes: Routes = [ { path: 'products', loadChildren: () => import('./products/products.module') .then(m => m.ProductsModule) } ];

8. Routing

Q29: How does Angular routing work?

Angular Router enables navigation from one view to another as users perform application tasks. It maps URL paths to components and manages browser history.

app-routing.module.ts
const routes: Routes = [ { path: '', component: HomeComponent }, { path: 'products', component: ProductsComponent }, { path: 'products/:id', component: ProductDetailComponent }, { path: '**', component: NotFoundComponent } ]; @NgModule({ imports: [RouterModule.forRoot(routes)], exports: [RouterModule] }) export class AppRoutingModule {}
Q30: What are route guards?

Route guards control navigation to and from routes. Types: CanActivate (can navigate to route), CanActivateChild (can navigate to child routes), CanDeactivate (can navigate away), CanLoad (can module be loaded), CanMatch (can route match).

auth.guard.ts
@Injectable({ providedIn: 'root' }) export class AuthGuard implements CanActivate { constructor(private authService: AuthService, private router: Router) {} canActivate(): boolean { if (this.authService.isLoggedIn()) return true; this.router.navigate(['/login']); return false; } }
Q31: What are route resolvers?

Route resolvers fetch data before a route is activated. They ensure that data is available when the component loads, preventing the component from rendering with incomplete data.

resolver.ts
@Injectable({ providedIn: 'root' }) export class ProductResolver implements Resolve<Product> { constructor(private service: ProductService) {} resolve(route: ActivatedRouteSnapshot): Observable<Product> { return this.service.getProduct(route.params['id']); } }

9. Forms

Q32: What is the difference between template-driven and reactive forms?

Template-driven forms use directives in the template and are simpler for basic forms. Reactive forms use FormControl, FormGroup, and FormArray in the component class and are better for complex forms and testing.

Aspect Template-driven Reactive
Setup Template-based Component class
Data model Implicit Explicit
Validation Directives Functions
Scalability Simple forms Complex forms
Testing Harder Easier
Q33: How do you create reactive forms?

Reactive forms are created using FormGroup, FormControl, and Validators from @angular/forms. They provide explicit control over form state and validation.

reactive-form.component.ts
import { FormGroup, FormControl, Validators } from '@angular/forms'; export class ProductFormComponent { productForm = new FormGroup({ name: new FormControl('', [Validators.required]), price: new FormControl('', [Validators.required, Validators.min(0)]) }); onSubmit() { if (this.productForm.valid) { console.log(this.productForm.value); } } }
Q34: How do you implement custom validators?

Custom validators are functions that take a control and return ValidationErrors or null. They can be synchronous or async validators.

validators.ts
export function positiveNumber(control: AbstractControl): ValidationErrors | null { const value = control.value; if (value <= 0) { return { positiveNumber: true }; } return null; } // Usage price: new FormControl('', [Validators.required, positiveNumber])

10. HTTP Client

Q35: How do you make HTTP requests in Angular?

HTTP Client from @angular/common/http provides methods for GET, POST, PUT, DELETE, and other HTTP operations. It returns Observables for handling responses asynchronously.

api.service.ts
import { HttpClient } from '@angular/common/http'; @Injectable({ providedIn: 'root' }) export class ApiService { constructor(private http: HttpClient) {} getItems(): Observable<Item[]> { return this.http.get<Item[]>('/api/items'); } createItem(item: Item): Observable<Item> { return this.http.post<Item>('/api/items', item); } }
Q36: What are HTTP interceptors?

HTTP interceptors intercept HTTP requests and responses to add common logic like authentication, logging, or error handling. They transform or handle HTTP operations before they reach the server or after the response is received.

auth.interceptor.ts
@Injectable() export class AuthInterceptor implements HttpInterceptor { constructor(private authService: AuthService) {} intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> { const token = this.authService.getToken(); const authReq = req.clone({ setHeaders: { Authorization: `Bearer ${token}` } }); return next.handle(authReq); } }

11. Observables and RxJS

Q37: What is an Observable?

Observable is a representation of any set of values over any amount of time. It's the primary data structure in Angular for handling asynchronous operations. Observables are lazy and can be cancelled.

observable.ts
import { Observable } from 'rxjs'; const data$ = new Observable(observer => { observer.next(1); observer.next(2); observer.complete(); }); data$.subscribe(value => console.log(value));
Q38: What is the difference between Observable and Promise?

Observable can emit multiple values over time, can be cancelled, and has powerful operators. Promise returns a single value and cannot be cancelled.

Aspect Observable Promise
Values Multiple over time Single
Cancellation Can be cancelled Cannot be cancelled
Operators Rich operators Limited
Laziness Lazy (executed on subscribe) Eager (executed immediately)
Q39: What are common RxJS operators?

Common operators: map (transform), filter (filter values), switchMap (cancel previous), mergeMap (merge observables), debounceTime (delay), distinctUntilChanged (distinct values), catchError (handle errors), tap (side effects), take (limit emissions), retry (retry on error).

operators.ts
import { map, filter, catchError } from 'rxjs/operators'; data$.pipe( filter(x => x > 0), map(x => x * 2), catchError(err => handleError(err)) )
Q40: What is the difference between Subject and BehaviorSubject?

Subject: Basic multicast Observable. Subscribers only receive values emitted after subscription.

BehaviorSubject: Has initial value. New subscribers receive the latest value immediately upon subscription.

subjects.ts
// Subject const subject = new Subject<string>(); subject.next('hello'); subject.subscribe(value => console.log(value)); // Won't receive 'hello' // BehaviorSubject const behavior = new BehaviorSubject<string>('initial'); behavior.next('hello'); behavior.subscribe(value => console.log(value)); // Receives 'hello'
Q41: What is the difference between switchMap and mergeMap?

switchMap: Cancels previous observable when new value arrives. Good for search, type-ahead.

mergeMap: Keeps all observables active. Good for parallel operations.

operators.ts
// switchMap - cancel previous search$.pipe(switchMap(term => this.search(term))) // mergeMap - parallel operations ids$.pipe(mergeMap(id => this.fetch(id)))

12. Change Detection

Q42: How does Angular change detection work?

Angular uses zone.js to detect changes. When an event occurs (click, HTTP response, timer), Zone triggers change detection. Angular then checks all components for changes by comparing current state with previous state.

component.ts
@Component({ changeDetection: ChangeDetectionStrategy.OnPush }) export class ProductComponent { constructor(private cdr: ChangeDetectorRef) {} updateData() { this.data = new Data(); this.cdr.markForCheck(); } }
Q43: What is OnPush change detection strategy?

OnPush change detection only checks a component when its input references change or events occur within the component. This improves performance by reducing unnecessary change detection cycles.

onpush.component.ts
@Component({ selector: 'app-product', changeDetection: ChangeDetectionStrategy.OnPush }) export class ProductComponent { @Input() product: Product; }
Q44: How do you handle memory leaks with Observables?

Memory leaks occur when subscriptions are not cleaned up when components are destroyed. Use ngOnDestroy and operators like takeUntil, take, or async pipe to prevent leaks.

cleanup.ts
export class ProductComponent implements OnDestroy { private destroy$ = new Subject<void>(); ngOnInit() { this.data$.pipe( takeUntil(this.destroy$) ).subscribe(); } ngOnDestroy() { this.destroy$.next(); this.destroy$.complete(); } }
Q45: What is trackBy in ngFor?

trackBy improves performance by telling Angular how to track items in a list. It prevents unnecessary DOM re-creation when items are reordered.

trackby.html
<div *ngFor="let item of items; trackBy: trackById"> {{ item.name }} </div> trackById(index: number, item: any): number { return item.id; }

13. State Management

Q46: What is NgRx?

NgRx is a state management library for Angular inspired by Redux. It provides a single source of truth, predictable state updates using actions and reducers, and side effects using effects.

actions.ts
import { createAction, createReducer, on } from '@ngrx/store'; export const loadProducts = createAction('[Product] Load Products'); export const productReducer = createReducer( initialState, on(loadProducts, state => ({ ...state, loading: true })) );
Q47: What are Signals in Angular?

Signals are a new feature in Angular for reactive state management. They are wrappers around values that notify consumers when the value changes. Signals enable fine-grained reactivity without zone.js.

signals.ts
import { signal, computed, effect } from '@angular/core'; export class ProductComponent { count = signal(0); doubleCount = computed(() => this.count() * 2); constructor() { effect(() => { console.log('Count changed:', this.count()); }); } increment() { this.count.update(c => c + 1); } }
Q48: What is the difference between NgRx and Signals?

NgRx is a full-featured state management solution with actions, reducers, effects, and selectors. It's suitable for large-scale applications with complex state.

Signals are a simpler, built-in reactive primitive for local component state. They're easier to use and don't require additional libraries.

comparison.ts
// NgRx - complex state management this.store.dispatch(loadProducts()); this.products$ = this.store.select(selectProducts); // Signals - simple local state products = signal([]); filteredProducts = computed(() => this.products().filter(...));

14. Testing

Q49: How do you test Angular components?

Component testing uses TestBed to create component instances, ComponentFixture to interact with the component, and Jasmine/Karma for assertions.

product.component.spec.ts
describe('ProductComponent', () => { let component: ProductComponent; let fixture: ComponentFixture<ProductComponent>; beforeEach(async () => { await TestBed.configureTestingModule({ declarations: [ProductComponent] }).compileComponents(); fixture = TestBed.createComponent(ProductComponent); component = fixture.componentInstance; fixture.detectChanges(); }); it('should create', () => { expect(component).toBeTruthy(); }); });
Q50: What is TestBed?

TestBed is Angular's primary utility for configuring and creating component instances for testing. It sets up a module environment similar to the application runtime.

testbed.ts
beforeEach(async () => { await TestBed.configureTestingModule({ declarations: [MyComponent], imports: [HttpClientModule], providers: [MyService] }).compileComponents(); });
Q51: How do you test services?

Service testing involves creating service instances, mocking dependencies, and testing methods with Jasmine/Karma.

product.service.spec.ts
describe('ProductService', () => { let service: ProductService; beforeEach(() => { TestBed.configureTestingModule({}); service = TestBed.inject(ProductService); }); it('should be created', () => { expect(service).toBeTruthy(); }); });

15. Performance Optimization

Q52: How do you optimize Angular application performance?

Performance optimization: Use OnPush change detection, implement trackBy for ngFor, lazy load modules, use virtual scrolling for large lists, enable AOT compilation, optimize bundle size with tree-shaking, and use server-side rendering.

optimization.ts
@Component({ changeDetection: ChangeDetectionStrategy.OnPush, standalone: true }) export class ProductComponent { trackById(index: number, item: Product): number { return item.id; } }
Q53: What is virtual scrolling?

Virtual scrolling renders only visible items in a list, improving performance for large datasets. Angular CDK provides the ScrollingModule for this.

virtual-scroll.html
<cdk-virtual-scroll-viewport itemSize="50"> <div *cdkVirtualFor="let item of items"> {{ item.name }} </div> </cdk-virtual-scroll-viewport>
Q54: What is AOT compilation?

AOT (Ahead-of-Time) compilation compiles Angular templates during the build process, resulting in faster rendering and smaller bundle sizes. JIT (Just-in-Time) compiles templates in the browser at runtime.

angular.json
"build": { "configurations": { "production": { "aot": true, "buildOptimizer": true } } }

16. Advanced Topics

Q55: What are Angular Elements?

Angular Elements allows you to package Angular components as custom web components that can be used in any framework or vanilla JavaScript application.

elements.ts
import { createCustomElement } from '@angular/elements'; @NgModule({ declarations: [ProductComponent], schemas: [CUSTOM_ELEMENTS_SCHEMA] }) export class ProductModule { constructor(private injector: Injector) { const customElement = createCustomElement(ProductComponent, { injector: this.injector }); customElements.define('app-product', customElement); } }
Q56: What is Angular Universal (SSR)?

Angular Universal enables server-side rendering (SSR) for Angular applications. It improves SEO and initial page load performance by rendering pages on the server before sending them to the client.

Q57: What is TransferState?

TransferState transfers state from server to client in Angular Universal applications. It avoids re-fetching data that was already loaded on the server.

Q58: What are Standalone Components?

Standalone Components are components that don't need to be declared in NgModule. They can directly import other components, directives, and pipes. This simplifies the module system and makes components more reusable.

standalone.ts
@Component({ selector: 'app-product', standalone: true, imports: [CommonModule, FormsModule], template: '...' }) export class ProductComponent {}
Q59: What is the difference between ViewChild and ContentChild?

ViewChild queries elements within the component's template. ContentChild queries content projected into the component using ng-content.

queries.ts
@ViewChild('childElement') child: ElementRef; @ContentChild('projectedContent') content: ElementRef;
Q60: What are the best practices for Angular development?
  • Use TypeScript features: Strict mode, interfaces, type safety

17. Advanced RxJS Patterns

Q61: How do you implement advanced RxJS patterns for complex data flows?

Advanced RxJS patterns include combining multiple streams, error handling strategies, and complex transformations using operators like combineLatest, forkJoin, withLatestFrom, switchMap, catchError, retry, debounceTime, and distinctUntilChanged.

advanced-rxjs.ts
import { combineLatest, forkJoin, withLatestFrom } from 'rxjs'; import { switchMap, catchError, debounceTime } from 'rxjs/operators'; // Combine multiple streams const combined$ = combineLatest([source1$, source2$]).pipe( switchMap(([a, b]) => this.fetchData(a, b)), catchError(error => handleError(error)) );
Q62: How do you optimize Angular bundle size and performance?

Bundle optimization involves analyzing and reducing the size of your JavaScript bundles. Use ng build --stats-json to analyze bundle size, implement lazy loading, use tree-shakeable providers, and enable AOT compilation.

Q64: How do you implement advanced security measures in Angular?

Advanced security includes CSRF protection, XSS prevention, secure authentication with JWT, and implementing security headers via HTTP interceptors.

18. Common Interview Questions (All Levels)

Q71: What is the difference between Promise and Observable?

Promise returns a single value and cannot be cancelled. Observable can emit multiple values over time, can be cancelled, and has powerful operators for transformation and combination.

Q72: What is the difference between BehaviorSubject and ReplaySubject?

BehaviorSubject holds the latest value and emits it to new subscribers. ReplaySubject can replay a specified number of previous emissions to new subscribers.

subjects.ts
// BehaviorSubject - emits latest value const behavior = new BehaviorSubject<string>('initial'); // ReplaySubject - replays last N values const replay = new ReplaySubject<string>(3);
Q73: What is the difference between switchMap, concatMap, mergeMap, and exhaustMap?

switchMap: Cancels previous when new arrives. concatMap: Processes sequentially. mergeMap: Processes in parallel. exhaustMap: Ignores new while current is active.

operators.ts
// switchMap - cancel previous search$.pipe(switchMap(term => this.search(term))) // concatMap - sequential requests$.pipe(concatMap(req => this.execute(req))) // mergeMap - parallel ids$.pipe(mergeMap(id => this.fetch(id))) // exhaustMap - ignore new while active clicks$.pipe(exhaustMap(() => this.submit()))
Q74: What is the difference between take and takeUntil?

take(n) takes first n emissions then completes. takeUntil completes when a notifier observable emits, commonly used for cleanup in ngOnDestroy.

Q75: What is the difference between debounceTime and throttleTime?

debounceTime waits for a pause in emissions before emitting the latest. throttleTime emits at most once per time period.

timing.ts
// debounceTime - wait for pause searchInput$.pipe(debounceTime(300)) // throttleTime - emit at most once per period scroll$.pipe(throttleTime(200))
Q76: What is the difference between map and tap?

map transforms values in the stream. tap performs side effects without changing the stream, useful for logging or debugging.

operators.ts
// map - transform values data$.pipe(map(x => x * 2)) // tap - side effects data$.pipe(tap(x => console.log(x)))
Q77: What is the difference between catchError and throwError?

catchError handles errors and returns a fallback observable. throwError creates an observable that immediately errors, used within catchError to re-throw.

Q78: What is the difference between filter and distinctUntilChanged?

filter removes values based on a predicate. distinctUntilChanged removes consecutive duplicate values.

operators.ts
// filter - remove based on predicate numbers$.pipe(filter(x => x > 5)) // distinctUntilChanged - remove consecutive duplicates search$.pipe(distinctUntilChanged())
Q79: What is the difference between scan and reduce?

scan emits accumulated value for each source emission. reduce only emits final accumulated value when source completes.

operators.ts
// scan - emit for each emission numbers$.pipe(scan((acc, val) => acc + val, 0)) // reduce - emit only final value numbers$.pipe(reduce((acc, val) => acc + val, 0))
Q80: What is the difference between share and shareReplay?

share multicasts to multiple subscribers. shareReplay multicasts and replays a specified number of emissions to late subscribers.

operators.ts
// share - multicast without replay data$.pipe(share()) // shareReplay - multicast with replay data$.pipe(shareReplay(1))
Q81: What is the difference between first and take(1)?

first takes first value and errors if source is empty. take(1) takes first value and completes even if source is empty.

Q82: What is the difference between delay and delayWhen?

delay delays all emissions by a fixed duration. delayWhen delays each emission based on a notifier observable.

operators.ts
// delay - fixed duration data$.pipe(delay(1000)) // delayWhen - based on notifier data$.pipe(delayWhen(x => timer(100)))
Q83: What is the difference between retry and retryWhen?

retry retries a fixed number of times immediately. retryWhen retries based on a notifier observable, allowing custom retry logic.

operators.ts
// retry - fixed number of retries data$.pipe(retry(3)) // retryWhen - custom retry logic data$.pipe(retryWhen(errors => errors.pipe(delay(1000))))
Q84: What is the difference between startWith and of?

startWith prepends a value to an existing observable. of creates a new observable that emits the specified values.

Q85: What is the difference from forkJoin and combineLatest?

forkJoin waits for all observables to complete then emits last values. combineLatest emits whenever any observable emits with latest values from all.

operators.ts
// forkJoin - wait for all to complete forkJoin([obs1$, obs2$]) // combineLatest - emit on any change combineLatest([obs1$, obs2$])
Q86: What is the difference between zip and combineLatest?

zip emits when all have emitted at the same index. combineLatest emits whenever any emits with latest values from all.

operators.ts
// zip - emit at same index zip([obs1$, obs2$]) // combineLatest - emit on any change combineLatest([obs1$, obs2$])
Q87: What is the difference between race and combineLatest?

race emits from the first observable to emit. combineLatest combines latest values from all observables.

Q88: What is the difference between interval and timer?

interval emits sequential numbers at regular intervals. timer emits after a delay then optionally at intervals.

operators.ts
// interval - emit at regular intervals interval(1000) // timer - emit after delay timer(1000)
Q89: What is the difference between from and of?

from creates observable from array, promise, or iterable. of creates observable that emits specified values then completes.

operators.ts
// from - from array/iterable from([1, 2, 3]) // of - emit specified values of(1, 2, 3)
Q90: What is the difference between fromEvent and fromEventPattern?

fromEvent creates observable from DOM events. fromEventPattern creates observable from add/remove handler pattern for custom event APIs.

events.ts
// fromEvent - DOM events fromEvent(button, 'click') // fromEventPattern - custom API fromEventPattern( handler => api.on('event', handler), handler => api.off('event', handler) )
