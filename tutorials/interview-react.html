1. React Fundamentals

Q1: What is React and what are its key features?

React is a JavaScript library for building user interfaces, maintained by Facebook. It focuses on building reusable UI components.

Key Features:

  • Component-based: Build encapsulated components that manage their own state
  • Virtual DOM: Efficient updates through virtual DOM diffing
  • Declarative: Describe what the UI should look like
  • Unidirectional data flow: Data flows down from parent to child
  • JSX: HTML-like syntax for writing components
  • Hooks: Use state and other React features without classes
  • React Native: Build native mobile apps with React
Q2: What is the Virtual DOM?

The Virtual DOM is a lightweight JavaScript representation of the actual DOM. React uses it to:

  • Minimize direct DOM manipulation (expensive)
  • Compare the virtual DOM with the actual DOM (diffing)
  • Calculate the minimum number of changes needed
  • Batch updates for better performance
Tip: The Virtual DOM is not always faster than direct DOM manipulation. Its benefit comes from minimizing reflows and repaints through efficient diffing.

Q4: What are props in React?

Props (properties) are read-only data passed from parent to child components. They enable component composition and data flow.

// Parent component
function App() {
  return <ProductCard name="Laptop" price={999.99} inStock={true} />;
}

// Child component
function ProductCard({ name, price, inStock }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>Price: ${price}</p>
      {inStock ? <p>In Stock</p> : <p>Out of Stock</p>}
    </div>
  );
}

3. State and Lifecycle

Q5: What is state in React?

State is an object that holds component-specific data that can change over time. When state changes, React re-renders the component.

4. Hooks

Q7: What are React Hooks and why were they introduced?

Hooks are functions that let you use state and other React features in functional components. They were introduced to:

  • Reuse stateful logic between components
  • Combine related logic in one place
  • Eliminate class components complexity
  • Make code more readable and maintainable

Common Hooks:

  • useState: Manage component state
  • useEffect: Handle side effects
  • useContext: Consume context
  • useReducer: Manage complex state
  • useCallback: Memoize functions
  • useMemo: Memoize values
  • useRef: Persist values across renders
  • useCustom: Create custom hooks
Q8: How does useEffect work?

useEffect handles side effects in functional components. It runs after every render by default, but can be configured with dependencies.

// Run on every render
useEffect(() => {
  console.log('Component rendered');
});

// Run only on mount
useEffect(() => {
  console.log('Component mounted');
}, []);

// Run when count changes
useEffect(() => {
  console.log('Count changed:', count);
}, [count]);

// Cleanup on unmount
useEffect(() => {
  const timer = setInterval(() => console.log('Tick'), 1000);
  return () => clearInterval(timer);
}, []);
Q9: What is the difference between useEffect and useLayoutEffect?
  • useEffect: Runs asynchronously after paint. Non-blocking. Use for most side effects.
  • useLayoutEffect: Runs synchronously before paint. Blocking. Use for DOM measurements or mutations.
Important: Prefer useEffect over useLayoutEffect unless you need to measure or mutate the DOM before the browser paints.

5. Context API

Q10: What is the Context API?

The Context API provides a way to pass data through the component tree without having to pass props manually at every level. It's designed for sharing data that can be considered "global" for a tree of components.

// Create context
const ThemeContext = createContext('light');

// Provider
function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Toolbar />
    </ThemeContext.Provider>
  );
}

// Consumer
function ThemedButton() {
  const theme = useContext(ThemeContext);
  return <button className={theme}>Click me</button>;
}
Q11: When should you use Context vs Redux?
  • Context: Use for low-frequency updates like theme, language, or user authentication. Not optimized for high-frequency updates.
  • Redux: Use for complex state management, high-frequency updates, middleware support, and time-travel debugging.

6. Redux

Q12: What is Redux and how does it work?

Redux is a predictable state container for JavaScript apps. It follows three principles:

  • Single source of truth: All state is stored in a single store
  • State is read-only: State can only be changed by emitting actions
  • Changes are made with pure functions: Reducers specify how state changes
// Action
const increment = () => ({ type: 'INCREMENT' });

// Reducer
function counter(state = 0, action) {
  switch (action.type) {
    case 'INCREMENT':
      return state + 1;
    default:
      return state;
  }
}

// Store
const store = createStore(counter);

// Dispatch
store.dispatch(increment());
Q13: What is the difference between Redux Toolkit and Redux?

Redux Toolkit is the official recommended way to write Redux logic. It simplifies Redux development by:

  • Providing simplified configuration
  • Immutability helpers (Immer)
  • Thunks included by default
  • Simplified action creators
  • Better TypeScript support
// Redux Toolkit
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',
  initialState: 0,
  reducers: {
    increment: (state) => state + 1,
    decrement: (state) => state - 1
  }
});

export const { increment, decrement } = counterSlice.actions;
export default counterSlice.reducer;

7. Routing

Q14: How does React Router work?

React Router enables client-side routing in React applications. It uses the HTML5 History API to keep the UI in sync with the URL.

// Basic routing
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>
      <Routes>
        <Route path="/" element=<Home /> />
        <Route path="/about" element=<About /> />
      </Routes>
    </BrowserRouter>
  );
}
Q15: What is the difference between BrowserRouter and HashRouter?
  • BrowserRouter: Uses HTML5 History API. Cleaner URLs. Requires server configuration for SPA support.
  • HashRouter: Uses URL hash (#). Works without server configuration. Less clean URLs.

9. Performance Optimization

Q17: How do you optimize React performance?
  • React.memo: Memoize components to prevent unnecessary re-renders
  • useMemo: Memoize expensive calculations
  • useCallback: Memoize functions to prevent recreation
  • Code splitting: Lazy load components with React.lazy
  • Virtualization: Render only visible items for long lists
  • Avoid inline functions: Use useCallback or define outside render
  • Key prop: Provide stable keys for lists
// React.memo
const ExpensiveComponent = React.memo(function ExpensiveComponent({ data }) {
  return <div>{data}</div>;
});

// useMemo
const expensiveValue = useMemo(() => {
  return computeExpensiveValue(a, b);
}, [a, b]);

// useCallback
const handleClick = useCallback(() => {
  doSomething(dependency);
}, [dependency]);

// Code splitting
const LazyComponent = React.lazy(() => import('./LazyComponent'));

// Keys
{items.map(item => (
  <Item key={item.id} item={item} />
))}
Q18: What is the key prop and why is it important?

The key prop helps React identify which items have changed, been added, or been removed. Keys should be:

  • Stable across re-renders
  • Unique among siblings
  • Not array index (unless list is static)
Important: Using array indices as keys can cause issues with reordering, leading to incorrect component state and performance problems.

11. Common Interview Questions

Q20: What is the difference between createElement and JSX?
  • createElement: React's built-in method to create elements. Verbose and less readable.
  • JSX: Syntax extension that looks like HTML. Transpiled to createElement by Babel. More readable and maintainable.
// createElement
React.createElement('h1', null, 'Hello World');

// JSX
<h1>Hello World</h1>
Q21: What is the difference between state and refs?
  • State: Triggers re-render when changed. Use for data that affects rendering.
  • Refs: Does not trigger re-render. Use for DOM access, focus management, or storing values that don't affect rendering.
// State - triggers re-render
const [count, setCount] = useState(0);

// Ref - doesn't trigger re-render
const inputRef = useRef(null);
inputRef.current.focus();
Q22: What is the difference between useCallback and useMemo?
  • useCallback: Memoizes a function. Returns the same function instance unless dependencies change.
  • useMemo: Memoizes a value. Returns the cached value unless dependencies change.
// useCallback - memoizes function
const memoizedCallback = useCallback(() => {
  doSomething(a, b);
}, [a, b]);

// useMemo - memoizes value
const memoizedValue = useMemo(() => {
  return computeExpensiveValue(a, b);
}, [a, b]);
Q23: What is the difference between shallow rendering and full rendering?
  • Shallow rendering: Renders only the component one level deep. Doesn't render child components. Faster, isolated testing.
  • Full rendering: Renders the entire component tree including children. More realistic integration testing.
Q24: What is the difference between useEffect and componentDidMount?
  • componentDidMount: Class lifecycle method. Runs once after initial render.
  • useEffect: Hook. Can run on mount, update, or unmount depending on dependencies. More flexible.
// componentDidMount equivalent
useEffect(() => {
  // Runs on mount
}, []);
Q25: What is the difference between React.PureComponent and React.Component?
  • React.Component: Doesn't implement shouldComponentUpdate. Re-renders on every state/prop change.
  • React.PureComponent: Implements shallow prop and state comparison. Only re-renders if props or state change shallowly.
Tip: Use React.PureComponent or React.memo for performance optimization when components receive the same props frequently.
Q26: What is the difference between synthetic events and native events?
  • Synthetic events: React's cross-browser wrapper around native events. Consistent API across browsers. Pooled for performance.
  • Native events: Browser's native event objects. Browser-specific APIs.
// Access native event
function handleClick(event) {
  const nativeEvent = event.nativeEvent;
  console.log(nativeEvent);
}
Q27: What is the difference between useReducer and useState?
  • useState: Simple state management. Good for independent state pieces.
  • useReducer: Complex state management with reducer pattern. Good for related state pieces or complex state logic.
// useState - simple
const [count, setCount] = useState(0);

// useReducer - complex
const [state, dispatch] = useReducer(reducer, initialState);
Q28: What is the difference between forwardRef and useRef?
  • useRef: Create a ref within a component to access DOM elements or persist values.
  • forwardRef: Forward a ref from parent to child component. Used when parent needs to access child's DOM element.
// forwardRef
const FancyButton = forwardRef((props, ref) => (
  <button ref={ref}>{props.children}</button>
));

// Usage
const buttonRef = useRef();
<FancyButton ref={buttonRef}>Click me</FancyButton>;
Q29: What is the difference between React.lazy and Suspense?
  • React.lazy: Dynamically import a component. Returns a promise that resolves to the component.
  • Suspense: Displays a fallback while waiting for lazy-loaded components or data fetching.
const LazyComponent = React.lazy(() => import('./LazyComponent'));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <LazyComponent />
    </Suspense>
  );
}

    e.preventDefault();
    mutation.mutate({ title: 'New Post' });

Q38: What are custom hooks? 
 Custom hook for localStorage
function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      return initialValue;
    }
  });

  const setValue = (value) => {
    try {
      const valueToStore = value instanceof Function ? value(storedValue) : value;
      setStoredValue(valueToStore);
      window.localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.error(error);
    }
  };

  return [storedValue, setValue];
}

// Usage
const [name, setName] = useLocalStorage('name', 'John');

// Custom hook for debouncing
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

// Usage
const [searchTerm, setSearchTerm] = useState('');
const debouncedSearchTerm = useDebounce(searchTerm, 500);

Q51: How do you implement infinite scroll? 
import { useEffect, useState, useRef } from 'react';

function InfiniteScroll() {
  const [items, setItems] = useState([]);
  const [loading, setLoading] = useState(false);
  const [page, setPage] = useState(1);
  const observerRef = useRef();

  const loadMore = async () => {
    setLoading(true);
    const newItems = await fetchItems(page);
    setItems(prev => [...prev, ...newItems]);
    setPage(prev => prev + 1);
    setLoading(false);
  };

  useEffect(() => {
    const observer = new IntersectionObserver(
      (entries) => {
        if (entries[0].isIntersecting && !loading) {
          loadMore();
        }
      },
      { threshold: 1 }
    );

    if (observerRef.current) {
      observer.observe(observerRef.current);
    }

    return () => observer.disconnect();
  }, [loading, page]);

  return (
    <div>
      {items.map(item => <Item key={item.id} item={item} />)}
      {loading && <div>Loading...</div>}
      <div ref={observerRef} />
    </div>
  );
}

function FileUpload() {
  const [file, setFile] = useState(null);
  const [uploading, setUploading] = useState(false);
  const [progress, setProgress] = useState(0);

  const handleFileChange = (e) => {
    setFile(e.target.files[0]);
  };

  const handleUpload = async () => {
    if (!file) return;

    setUploading(true);
    const formData = new FormData();
    formData.append('file', file);

    try {
      const xhr = new XMLHttpRequest();
      
      xhr.upload.onprogress = (e) => {
        if (e.lengthComputable) {
          setProgress((e.loaded / e.total) * 100);
        }
      };

      xhr.onload = () => {
        setUploading(false);
        setProgress(0);
      };

      xhr.open('POST', '/api/upload');
      xhr.send(formData);
    } catch (error) {
      console.error('Upload failed:', error);
      setUploading(false);
    }
  };

  return (
    <div>
      <input type="file" onChange={handleFileChange} />
      <button onClick={handleUpload} disabled={uploading}>
        {uploading ? 'Uploading...' : 'Upload'}
      </button>
      {uploading && <progress value={progress} max={100} />}
    </div>
  );
}

Q59: How do you implement drag and drop in React? 
 Using react-beautiful-dnd
import { DragDropContext, Droppable, Draggable } from 'react-beautiful-dnd';

function KanbanBoard() {
  const [columns, setColumns] = useState(initialData);

  const onDragEnd = (result) => {
    const { source, destination } = result;

    if (!destination) return;

    const sourceCol = columns[source.droppableId];
    const destCol = columns[destination.droppableId];

    if (source.droppableId === destination.droppableId) {
      // Reorder within same column
      const newItems = Array.from(sourceCol.items);
      const [removed] = newItems.splice(source.index, 1);
      newItems.splice(destination.index, 0, removed);

      setColumns({
        ...columns,
        [source.droppableId]: {
          ...sourceCol,
          items: newItems
        }
      });
    } else {
      // Move between columns
      const sourceItems = [...sourceCol.items];
      const destItems = [...destCol.items];
      const [removed] = sourceItems.splice(source.index, 1);
      destItems.splice(destination.index, 0, removed);

      setColumns({
        ...columns,
        [source.droppableId]: {
          ...sourceCol,
          items: sourceItems
        },
        [destination.droppableId]: {
          ...destCol,
          items: destItems
        }
      });
    }
  };

  return (
    <DragDropContext onDragEnd={onDragEnd}>
      {Object.entries(columns).map(([colId, col]) => (
        <Droppable key={colId} droppableId={colId}>
          {(provided) => (
            <div ref={provided.innerRef} {...provided.droppableProps}>
              {col.items.map((item, index) => (
                <Draggable key={item.id} draggableId={item.id} index={index}>
                  {(provided) => (
                    <div
                      ref={provided.innerRef}
                      {...provided.draggableProps}
                      {...provided.dragHandleProps}
                    >
                      {item.content}
                    </div>
                  )}
                </Draggable>
              ))}
              {provided.placeholder}
            </div>
          )}
        </Droppable>
      ))}
    </DragDropContext>
  );
}
