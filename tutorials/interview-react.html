1. React Fundamentals

Q1: What is React and what are its key features?

React is a JavaScript library for building user interfaces, maintained by Facebook. It focuses on building reusable UI components.

Key Features:

  • Component-based: Build encapsulated components that manage their own state
  • Virtual DOM: Efficient updates through virtual DOM diffing
  • Declarative: Describe what the UI should look like
  • Unidirectional data flow: Data flows down from parent to child
  • JSX: HTML-like syntax for writing components
  • Hooks: Use state and other React features without classes
  • React Native: Build native mobile apps with React
Q2: What is the Virtual DOM?

The Virtual DOM is a lightweight JavaScript representation of the actual DOM. React uses it to:

  • Minimize direct DOM manipulation (expensive)
  • Compare the virtual DOM with the actual DOM (diffing)
  • Calculate the minimum number of changes needed
  • Batch updates for better performance
Tip: The Virtual DOM is not always faster than direct DOM manipulation. Its benefit comes from minimizing reflows and repaints through efficient diffing.

4. Hooks

Q7: What are React Hooks and why were they introduced?

Hooks are functions that let you use state and other React features in functional components. They were introduced to:

  • Reuse stateful logic between components
  • Combine related logic in one place
  • Eliminate class components complexity
  • Make code more readable and maintainable

Common Hooks:

  • useState: Manage component state
  • useEffect: Handle side effects
  • useContext: Consume context
  • useReducer: Manage complex state
  • useCallback: Memoize functions
  • useMemo: Memoize values
  • useRef: Persist values across renders
  • useCustom: Create custom hooks
Q8: How does useEffect work?

useEffect handles side effects in functional components. It runs after every render by default, but can be configured with dependencies.

useeffect.js
// Run on every render useEffect(() => { console.log('Component rendered'); }); // Run only on mount useEffect(() => { console.log('Component mounted'); }, []); // Run when count changes useEffect(() => { console.log('Count changed:', count); }, [count]); // Cleanup on unmount useEffect(() => { const timer = setInterval(() => console.log('Tick'), 1000); return () => clearInterval(timer); }, []);
Q9: What is the difference between useEffect and useLayoutEffect?
  • useEffect: Runs asynchronously after paint. Non-blocking. Use for most side effects.
  • useLayoutEffect: Runs synchronously before paint. Blocking. Use for DOM measurements or mutations.
Important: Prefer useEffect over useLayoutEffect unless you need to measure or mutate the DOM before the browser paints.

5. Context API

Q10: What is the Context API?

The Context API provides a way to pass data through the component tree without having to pass props manually at every level. It's designed for sharing data that can be considered "global" for a tree of components.

context.js
// Create context const ThemeContext = createContext('light'); // Provider function App() { return ( <ThemeContext.Provider value="dark"> <Toolbar /> </ThemeContext.Provider> ); } // Consumer function ThemedButton() { const theme = useContext(ThemeContext); return <button className={theme}>Click me</button>; }
Q11: When should you use Context vs Redux?
  • Context: Use for low-frequency updates like theme, language, or user authentication. Not optimized for high-frequency updates.
  • Redux: Use for complex state management, high-frequency updates, middleware support, and time-travel debugging.

6. Redux

Q12: What is Redux and how does it work?

Redux is a predictable state container for JavaScript apps. It follows three principles:

  • Single source of truth: All state is stored in a single store
  • State is read-only: State can only be changed by emitting actions
  • Changes are made with pure functions: Reducers specify how state changes
redux.js
// Action const increment = () => ({ type: 'INCREMENT' }); // Reducer function counter(state = 0, action) { switch (action.type) { case 'INCREMENT': return state + 1; default: return state; } } // Store const store = createStore(counter); // Dispatch store.dispatch(increment());
Q13: What is the difference between Redux Toolkit and Redux?

Redux Toolkit is the official recommended way to write Redux logic. It simplifies Redux development by:

  • Providing simplified configuration
  • Immutability helpers (Immer)
  • Thunks included by default
  • Simplified action creators
  • Better TypeScript support
redux-toolkit.js
// Redux Toolkit import { createSlice } from '@reduxjs/toolkit'; const counterSlice = createSlice({ name: 'counter', initialState: 0, reducers: { increment: (state) => state + 1, decrement: (state) => state - 1 } }); export const { increment, decrement } = counterSlice.actions; export default counterSlice.reducer;

7. Routing

Q14: How does React Router work?

React Router enables client-side routing in React applications. It uses the HTML5 History API to keep the UI in sync with the URL.

router.js
// Basic routing import { BrowserRouter, Routes, Route, Link } from 'react-router-dom'; function App() { return ( <BrowserRouter> <nav> <Link to="/">Home</Link> <Link to="/about">About</Link> </nav> <Routes> <Route path="/" element=<Home /> /> <Route path="/about" element=<About /> /> </Routes> </BrowserRouter> ); }
Q15: What is the difference between BrowserRouter and HashRouter?
  • BrowserRouter: Uses HTML5 History API. Cleaner URLs. Requires server configuration for SPA support.
  • HashRouter: Uses URL hash (#). Works without server configuration. Less clean URLs.

9. Performance Optimization

Q17: How do you optimize React performance?
  • React.memo: Memoize components to prevent unnecessary re-renders
  • useMemo: Memoize expensive calculations
  • useCallback: Memoize functions to prevent recreation
  • Code splitting: Lazy load components with React.lazy
  • Virtualization: Render only visible items for long lists
  • Avoid inline functions: Use useCallback or define outside render
  • Key prop: Provide stable keys for lists
performance.js
// React.memo const ExpensiveComponent = React.memo(({ data }) => <div>{data}</div>); // useMemo const expensiveValue = useMemo(() => computeExpensiveValue(a, b), [a, b]); // useCallback const handleClick = useCallback(() => doSomething(dependency), [dependency]); // Code splitting const LazyComponent = React.lazy(() => import('./LazyComponent')); // Keys {items.map(item => <Item key={item.id} item={item} />)}
Q18: What is the key prop and why is it important?

The key prop helps React identify which items have changed, been added, or been removed. Keys should be:

  • Stable across re-renders
  • Unique among siblings
  • Not array index (unless list is static)
Important: Using array indices as keys can cause issues with reordering, leading to incorrect component state and performance problems.

11. Common Interview Questions

Q20: What is the difference between createElement and JSX?
  • createElement: React's built-in method to create elements. Verbose and less readable.
  • JSX: Syntax extension that looks like HTML. Transpiled to createElement by Babel. More readable and maintainable.
jsx.js
// createElement React.createElement('h1', null, 'Hello World'); // JSX <h1>Hello World</h1>
Q21: What is the difference between state and refs?
  • State: Triggers re-render when changed. Use for data that affects rendering.
  • Refs: Does not trigger re-render. Use for DOM access, focus management, or storing values that don't affect rendering.
state-refs.js
// State - triggers re-render const [count, setCount] = useState(0); // Ref - doesn't trigger re-render const inputRef = useRef(null); inputRef.current.focus();
Q22: What is the difference between useCallback and useMemo?
  • useCallback: Memoizes a function. Returns the same function instance unless dependencies change.
  • useMemo: Memoizes a value. Returns the cached value unless dependencies change.
hooks.js
// useCallback - memoizes function const memoizedCallback = useCallback(() => doSomething(a, b), [a, b]); // useMemo - memoizes value const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
Q23: What is the difference between shallow rendering and full rendering?
  • Shallow rendering: Renders only the component one level deep. Doesn't render child components. Faster, isolated testing.
  • Full rendering: Renders the entire component tree including children. More realistic integration testing.
Q24: What is the difference between useEffect and componentDidMount?
  • componentDidMount: Class lifecycle method. Runs once after initial render.
  • useEffect: Hook. Can run on mount, update, or unmount depending on dependencies. More flexible.
lifecycle.js
// componentDidMount equivalent useEffect(() => { // Runs on mount }, []);
Q25: What is the difference between React.PureComponent and React.Component?
  • React.Component: Doesn't implement shouldComponentUpdate. Re-renders on every state/prop change.
  • React.PureComponent: Implements shallow prop and state comparison. Only re-renders if props or state change shallowly.
Tip: Use React.PureComponent or React.memo for performance optimization when components receive the same props frequently.
Q26: What is the difference between synthetic events and native events?
  • Synthetic events: React's cross-browser wrapper around native events. Consistent API across browsers. Pooled for performance.
  • Native events: Browser's native event objects. Browser-specific APIs.
events.js
// Access native event function handleClick(event) { const nativeEvent = event.nativeEvent; console.log(nativeEvent); }
Q27: What is the difference between useReducer and useState?
  • useState: Simple state management. Good for independent state pieces.
  • useReducer: Complex state management with reducer pattern. Good for related state pieces or complex state logic.
hooks.js
// useState - simple const [count, setCount] = useState(0); // useReducer - complex const [state, dispatch] = useReducer(reducer, initialState);
Q28: What is the difference between forwardRef and useRef?
  • useRef: Create a ref within a component to access DOM elements or persist values.
  • forwardRef: Forward a ref from parent to child component. Used when parent needs to access child's DOM element.
forwardref.js
// forwardRef const FancyButton = forwardRef((props, ref) => ( <button ref={ref}>{props.children}</button> )); // Usage const buttonRef = useRef(); <FancyButton ref={buttonRef}>Click me</FancyButton>;
Q29: What is the difference between React.lazy and Suspense?
  • React.lazy: Dynamically import a component. Returns a promise that resolves to the component.
  • Suspense: Displays a fallback while waiting for lazy-loaded components or data fetching.

Q31: What are the different state management approaches in React?
  • Component State: Local state using useState/useReducer
  • Context API: Global state for component tree

import { configureStore, createSlice } from '@reduxjs/toolkit'; const counterSlice = createSlice({ name: 'counter', initialState: { value: 0 }, reducers: { increment: (state) => { state.value += 1; }, decrement: (state) => { state.value -= 1; }, incrementByAmount: (state, action) => { state.value += action.payload; } } }); export const { increment, decrement, incrementByAmount } = counterSlice.actions; export const store = configureStore({ reducer: { counter: counterSlice.reducer } }); // Component import { useSelector, useDispatch } from 'react-redux'; import { increment, decrement } from './store'; function Counter() { const count = useSelector((state) => state.counter.value); const dispatch = useDispatch(); return ( <div> <button onClick={() => dispatch(decrement())}>-</button> <span>{count}</span> <button onClick={() => dispatch(increment())}>+</button> </div> ); }
Q33: How do you use React Query for server state?

import { useQuery, useMutation, useQueryClient, QueryClient, QueryClientProvider } from '@tanstack/react-query'; const queryClient = new QueryClient(); function App() { return ( <QueryClientProvider client={queryClient}> <Posts /> </QueryClientProvider> ); } function Posts() { const { data, isLoading, error } = useQuery({ queryKey: ['posts'], queryFn: () => fetch('https://api.example.com/posts').then(res => res.json()) }); if (isLoading) return <div>Loading...</div>; if (error) return <div>Error: {error.message}</div>; return ( <ul> {data.map(post => ( <li key={post.id}>{post.title}</li> ))} </ul> ); } function CreatePost() { const queryClient = useQueryClient(); const mutation = useMutation({ mutationFn: (newPost) => fetch('https://api.example.com/posts', { method: 'POST', body: JSON.stringify(newPost) }).then(res => res.json()), onSuccess: () => { queryClient.invalidateQueries({ queryKey: ['posts'] }); } }); const handleSubmit = (e) => { e.preventDefault(); mutation.mutate({ title: 'New Post' }); }; return <form onSubmit={handleSubmit}><button type="submit">Create</button></form>; }

12. Advanced React Patterns

Q35: What are compound components?

Compound components are components that work together to form a complete UI. They share state implicitly through context, allowing flexible composition of child components.

compound-components.js
// Compound components pattern const Tabs = ({ children }) => { const [activeTab, setActiveTab] = useState(0); return ( <TabsContext.Provider value={{ activeTab, setActiveTab }}> {children} </TabsContext.Provider> ); }; const TabList = ({ children }) => { return <div className="tab-list">{children}</div>; }; const Tab = ({ children, index }) => { const { activeTab, setActiveTab } = useContext(TabsContext); return ( <button className={activeTab === index ? 'active' : ''} onClick={() => setActiveTab(index)} > {children} </button> ); }; const TabPanels = ({ children }) => { return <div className="tab-panels">{children}</div>; }; const TabPanel = ({ children, index }) => { const { activeTab } = useContext(TabsContext); return activeTab === index ? <div>{children}</div> : null; }; // Usage <Tabs> <TabList> <Tab index={0}>Tab 1</Tab> <Tab index={1}>Tab 2</Tab> </TabList> <TabPanels> <TabPanel index={0}>Content 1</TabPanel> <TabPanel index={1}>Content 2</TabPanel> </TabPanels> </Tabs>
Q36: What are render props?

Render props is a technique where a component receives a function as a prop that returns a React element. This allows sharing code between components using a prop whose value is a function.

render-props.js
// Render props pattern const MouseTracker = ({ render }) => { const [position, setPosition] = useState({ x: 0, y: 0 }); const handleMouseMove = (e) => { setPosition({ x: e.clientX, y: e.clientY }); }; return ( <div onMouseMove={handleMouseMove}> {render(position)} </div> ); }; // Usage <MouseTracker render={({ x, y }) => ( <p>Mouse position: {x}, {y}</p> )} /> // Using children as render prop <MouseTracker> {({ x, y }) => <p>Mouse: {x}, {y}</p>} </MouseTracker>
Q37: What are higher-order components (HOCs)?

Higher-order components (HOCs) are functions that take a component and return a new component. They're used to reuse component logic like authentication, data fetching, or loading states.

hoc.js
// HOC pattern const withLoading = (WrappedComponent) => { return (props) => { const [loading, setLoading] = useState(true); const [data, setData] = useState(null); useEffect(() => { fetchData().then(result => { setData(result); setLoading(false); }); }, []); if (loading) return <div>Loading...</div>; return <WrappedComponent data={data} {...props} />; }; }; // Usage const UserProfile = ({ data }) => { return <div>{data.name}</div>; }; const UserProfileWithLoading = withLoading(UserProfile); // HOC with authentication const withAuth = (WrappedComponent) => { return (props) => { const isAuthenticated = useAuth(); if (!isAuthenticated) { return <Navigate to="/login" />; } return <WrappedComponent {...props} />; }; };
Q38: What are custom hooks?

Custom hooks are JavaScript functions that start with "use" and can call other hooks. They allow you to extract and reuse stateful logic between components without changing hierarchy.

custom-hooks.js
// Custom hook for localStorage function useLocalStorage(key, initialValue) { const [storedValue, setStoredValue] = useState(() => { try { const item = window.localStorage.getItem(key); return item ? JSON.parse(item) : initialValue; } catch (error) { return initialValue; } }); const setValue = (value) => { try { const valueToStore = value instanceof Function ? value(storedValue) : value; setStoredValue(valueToStore); window.localStorage.setItem(key, JSON.stringify(valueToStore)); } catch (error) { console.error(error); } }; return [storedValue, setValue]; } // Usage const [name, setName] = useLocalStorage('name', 'John'); // Custom hook for debouncing function useDebounce(value, delay) { const [debouncedValue, setDebouncedValue] = useState(value); useEffect(() => { const handler = setTimeout(() => { setDebouncedValue(value); }, delay); return () => { clearTimeout(handler); }; }, [value, delay]); return debouncedValue; } // Usage const [searchTerm, setSearchTerm] = useState(''); const debouncedSearchTerm = useDebounce(searchTerm, 500);
Q39: What is the difference between controlled and uncontrolled components?

Controlled components have their form data controlled by React state. Uncontrolled components store their own state internally and are accessed using refs.

controlled-uncontrolled.js
// Controlled component function ControlledForm() { const [value, setValue] = useState(''); return ( <input value={value} onChange={(e) => setValue(e.target.value)} /> ); } // Uncontrolled component function UncontrolledForm() { const inputRef = useRef(); const handleSubmit = (e) => { e.preventDefault(); console.log(inputRef.current.value); }; return ( <form onSubmit={handleSubmit}> <input ref={inputRef} defaultValue="initial" /> <button type="submit">Submit</button> </form> ); } // When to use each: // Controlled: Form validation, dynamic inputs, real-time formatting // Uncontrolled: Simple forms, integrating with non-React code, performance

13. Server-Side Rendering

Q40: How do you implement SSR with Next.js?

Server-Side Rendering (SSR) in Next.js uses getServerSideProps to fetch data on each request. Static Generation (SSG) uses getStaticProps to generate pages at build time.

pages/index.js
// pages/index.js (SSR) export async function getServerSideProps(context) { const res = await fetch('https://api.example.com/data'); const data = await res.json(); return { props: { data } }; } function HomePage({ data }) { return <div>{data.title}</div>; } export default HomePage; // Static generation (SSG) export async function getStaticProps() { const res = await fetch('https://api.example.com/data'); const data = await res.json(); return { props: { data }, revalidate: 60 // Revalidate every 60 seconds }; } // Incremental Static Regeneration (ISR) export async function getStaticPaths() { const res = await fetch('https://api.example.com/posts'); const posts = await res.json(); return { paths: posts.map(post => ({ params: { id: post.id } })), fallback: 'blocking' }; }
Q41: How do you implement SSR with React 18?

React 18 SSR uses renderToPipeableStream for streaming rendering and hydrateRoot for hydration. This enables progressive rendering and better performance.

server.js
import express from 'express'; import { renderToPipeableStream } from 'react-dom/server'; import App from './App'; const app = express(); app.get('/', (req, res) => { const { pipe } = renderToPipeableStream( <App />, { bootstrapScripts: ['/client.js'], onShellReady() { res.statusCode = 200; res.setHeader('Content-type', 'text/html'); pipe(res); }, onShellError(error) { res.statusCode = 500; res.send('Error!'); } } ); }); app.listen(3000); // client.js import { hydrateRoot } from 'react-dom/client'; import App from './App'; hydrateRoot(document.getElementById('root'), <App />);

14. React 18+ Features

Q43: What are the new features in React 18?
  • Concurrent Rendering: React can prepare multiple versions of UI simultaneously
  • Automatic Batching: Multiple state updates are batched automatically
  • Transitions: Mark non-urgent updates with useTransition
  • Suspense on Server: Suspense works with SSR
  • New Hooks: useId, useDeferredValue, useSyncExternalStore
  • New Root API: createRoot replaces ReactDOM.render
  • Strict Mode Changes: Components mount twice in development
Q44: How do you use concurrent features in React 18?

Concurrent features include useTransition for non-urgent updates and useDeferredValue for expensive UI. These allow React to interrupt rendering for more urgent updates.

concurrent.js
// useTransition for non-urgent updates function SearchResults() { const [query, setQuery] = useState(''); const [results, setResults] = useState([]); const [isPending, startTransition] = useTransition(); const handleChange = (e) => { // Urgent update setQuery(e.target.value); // Non-urgent update startTransition(() => { setResults(search(e.target.value)); }); }; return ( <div> <input value={query} onChange={handleChange} /> {isPending ? <Spinner /> : <ResultsList results={results} />} </div> ); } // useDeferredValue for expensive UI function ProductList({ products }) { const deferredProducts = useDeferredValue(products); return ( <div> {deferredProducts.map(product => ( <ProductCard key={product.id} product={product} /> ))} </div> ); }
Q45: How do you use the new root API?

New root API uses createRoot instead of ReactDOM.render. This enables concurrent features and provides better control over hydration and unmounting.

root-api.js
// Old way (React 17) import ReactDOM from 'react-dom'; import App from './App'; ReactDOM.render(<App />, document.getElementById('root')); // New way (React 18+) import { createRoot } from 'react-dom/client'; import App from './App'; const root = createRoot(document.getElementById('root')); root.render(<App />); // Unmounting root.unmount(); // Concurrent features enabled by default root.render( <StrictMode> <App /> </StrictMode> );
Q46: What is automatic batching in React 18?

Automatic batching groups multiple state updates into a single re-render for better performance. In React 18, batching works in async operations, promises, and timeouts, not just in event handlers.

batching.js
// React 18 automatically batches state updates // even in async operations, promises, timeouts function handleClick() { setCount(c => c + 1); setFlag(f => !f); // Both updates are batched - only one re-render } // In async operations async function fetchData() { const data = await fetch('/api/data'); setLoading(false); setData(data); setError(null); // All three updates are batched } // In event handlers setTimeout(() => { setCount(c => c + 1); setFlag(f => !f); // Batched in React 18, not in React 17 }, 1000); // To opt out of batching import { flushSync } from 'react-dom'; function handleClick() { flushSync(() => { setCount(c => c + 1); }); // Re-renders immediately setFlag(f => !f); // Re-renders again }

Q51: How do you implement infinite scroll?

Infinite scroll loads more content as the user scrolls down. Use IntersectionObserver to detect when the user reaches the bottom and trigger data fetching.

infinite-scroll.js
import { useEffect, useState, useRef } from 'react'; function InfiniteScroll() { const [items, setItems] = useState([]); const [loading, setLoading] = useState(false); const [page, setPage] = useState(1); const observerRef = useRef(); const loadMore = async () => { setLoading(true); const newItems = await fetchItems(page); setItems(prev => [...prev, ...newItems]); setPage(prev => prev + 1); setLoading(false); }; useEffect(() => { const observer = new IntersectionObserver( (entries) => { if (entries[0].isIntersecting && !loading) { loadMore(); } }, { threshold: 1 } ); if (observerRef.current) { observer.observe(observerRef.current); } return () => observer.disconnect(); }, [loading, page]); return ( <div> {items.map(item => <Item key={item.id} item={item} />)} {loading && <div>Loading...</div>} <div ref={observerRef} /> </div> ); }

// Using react-beautiful-dnd import { DragDropContext, Droppable, Draggable } from 'react-beautiful-dnd'; function KanbanBoard() { const [columns, setColumns] = useState(initialData); const onDragEnd = (result) => { const { source, destination } = result; if (!destination) return; const sourceCol = columns[source.droppableId]; const destCol = columns[destination.droppableId]; if (source.droppableId === destination.droppableId) { // Reorder within same column const newItems = Array.from(sourceCol.items); const [removed] = newItems.splice(source.index, 1); newItems.splice(destination.index, 0, removed); setColumns({ ...columns, [source.droppableId]: { ...sourceCol, items: newItems } }); } else { // Move between columns const sourceItems = [...sourceCol.items]; const destItems = [...destCol.items]; const [removed] = sourceItems.splice(source.index, 1); destItems.splice(destination.index, 0, removed); setColumns({ ...columns, [source.droppableId]: { ...sourceCol, items: sourceItems }, [destination.droppableId]: { ...destCol, items: destItems } }); } }; return ( <DragDropContext onDragEnd={onDragEnd}> {Object.entries(columns).map(([colId, col]) => ( <Droppable key={colId} droppableId={colId}> {(provided) => ( <div ref={provided.innerRef} {...provided.droppableProps}> {col.items.map((item, index) => ( <Draggable key={item.id} draggableId={item.id} index={index}> {(provided) => ( <div ref={provided.innerRef} {...provided.draggableProps} {...provided.dragHandleProps} > {item.content} </div> )} </Draggable> ))} {provided.placeholder} </div> )} </Droppable> ))} </DragDropContext> ); }
Q61: What is JSX and why can't browsers read it directly?

JSX (JavaScript XML) is a syntax extension for JavaScript that allows you to write HTML-like code within JavaScript files. It makes it easier to write and understand React components.

Why browsers can't read JSX directly:

  • Browsers only understand standard JavaScript, not JSX syntax
  • JSX needs to be transpiled (converted) to regular JavaScript using tools like Babel
  • The transpilation process converts JSX to React.createElement() calls
jsx.js
// JSX (what you write) const element = <h1>Hello, World!</h1>; // Transpiled to JavaScript (what browsers execute) const element = React.createElement('h1', null, 'Hello, World!');

Q66: How do you handle events in React components?

Q67: What is unidirectional data flow in React?

Unidirectional data flow means data flows in one direction - from parent to child components. This makes it easier to understand and debug applications.

  • Children communicate back via callbacks
  • State is managed at the component level where it's needed

Q69: What is prop drilling and how can you avoid it?

Q70: What is React Fiber?

React Fiber is the new reconciliation algorithm in React 16+. It enables:

  • Incremental rendering (render work can be split into chunks)
  • Prioritization of updates (render high-priority updates first)
  • Ability to pause, abort, or reuse work
  • Better support for concurrent rendering
  • Improved error handling and recovery
Key concept: Fiber made React's reconciliation algorithm incremental, allowing React to interrupt rendering work to handle higher-priority updates.
