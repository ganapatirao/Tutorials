1. LINQ Fundamentals

Q1: What is LINQ and what are its benefits?

LINQ (Language Integrated Query) is a set of technologies that integrates query capabilities directly into C# language. It provides a consistent query experience for objects (LINQ to Objects), relational databases (LINQ to SQL/Entity Framework), and XML (LINQ to XML).

Benefits:

  • Type Safety: Compile-time checking of queries
  • IntelliSense: Full IDE support
  • Consistency: Same syntax for different data sources
  • Declarative: Describe what you want, not how to get it
  • Composable: Queries can be built incrementally
  • Readable: More readable than traditional loops
Q2: What are the different types of LINQ?
  1. LINQ to Objects: Query in-memory collections
  2. LINQ to SQL: Query SQL Server databases (legacy)
  3. LINQ to Entities (Entity Framework): Query databases via EF
  4. LINQ to XML: Query XML documents
  5. LINQ to DataSet: Query DataSets and DataTables
  6. Parallel LINQ (PLINQ): Parallel query execution

3. LINQ Query Syntax vs Method Syntax

Q4: What is the difference between query syntax and method syntax?

Query Syntax:

var query = from item in items
            where item.Price > 100
            orderby item.Name
            select item;

Method Syntax (Lambda):

var query = items
    .Where(item => item.Price > 100)
    .OrderBy(item => item.Name)
    .Select(item => item);

Key Differences:

  • Query syntax is converted to method syntax by the compiler
  • Method syntax offers more operators
  • Query syntax is more readable for complex joins
  • Method syntax is more flexible for chaining
Tip: Both syntaxes compile to the same IL. Choose based on readability and the specific scenario.

4. Deferred vs Immediate Execution

Q5: What is the difference between deferred and immediate execution?

Deferred Execution:

The query is not executed until you iterate over the results. Operators like Where, Select, OrderBy use deferred execution.

var query = items.Where(item => item.Price > 100);
// Query not executed yet

foreach (var item in query) // Executed here
{
    Console.WriteLine(item.Name);
}

Immediate Execution:

The query is executed immediately. Operators like Count, First, ToList, ToArray use immediate execution.

var count = items.Count(item => item.Price > 100);
// Executed immediately

var list = items.Where(item => item.Price > 100).ToList();
// Executed immediately
Q6: What operators cause immediate execution?
  • Aggregate operators: Count, Sum, Average, Min, Max
  • Element operators: First, FirstOrDefault, Last, LastOrDefault, Single, SingleOrDefault
  • Conversion operators: ToList, ToArray, ToDictionary, ToLookup
  • Equality operators: SequenceEqual
  • Quantifiers: All, Any, Contains

7. LINQ to SQL / Entity Framework

Q10: How does LINQ work with Entity Framework?

LINQ to Entities translates LINQ queries into SQL queries that are executed on the database server.

// LINQ to Entities example
using var context = new AppDbContext();

var products = context.Products
    .Where(p => p.Price > 100)
    .OrderBy(p => p.Name)
    .Include(p => p.Category)
    .ToList();

// Translated to SQL:
// SELECT p.*, c.*
// FROM Products p
// LEFT JOIN Categories c ON p.CategoryId = c.Id
// WHERE p.Price > 100
// ORDER BY p.Name
Important: Not all LINQ operators are supported in LINQ to Entities. Unsupported operators will throw NotSupportedException at runtime.

9. Performance Considerations

Q13: What are the performance considerations when using LINQ?
  • Deferred execution: Be aware of when queries execute
  • Multiple enumerations: Avoid enumerating the same query multiple times
  • Closure capture: Be careful with variable capture in lambdas
  • Database round-trips: Minimize database calls with IQueryable
  • Large collections: Consider streaming for large datasets
  • Complex queries: May be slower than optimized loops
// Bad - multiple enumerations
var query = items.Where(item => item.Price > 100);
var count = query.Count(); // Executes
var first = query.First(); // Executes again

// Good - cache the result
var query = items.Where(item => item.Price > 100).ToList();
var count = query.Count();
var first = query.First();
Q14: How does PLINQ differ from LINQ?

PLINQ (Parallel LINQ) enables parallel execution of LINQ queries on multi-core processors.

// LINQ - sequential execution
var result = items.Where(item => item.Price > 100).ToList();

// PLINQ - parallel execution
var result = items.AsParallel()
    .Where(item => item.Price > 100)
    .ToList();

// With degree of parallelism
var result = items.AsParallel()
    .WithDegreeOfParallelism(4)
    .Where(item => item.Price > 100)
    .ToList();

// Preserve order
var result = items.AsParallel()
    .AsOrdered()
    .Where(item => item.Price > 100)
    .ToList();
Important: PLINQ is not always faster. Use it for CPU-bound operations on large collections. For small collections or I/O-bound operations, sequential LINQ may be faster.

  • Join: Inner join - returns matching pairs from both collections
  • GroupJoin: Left outer join - returns each element from the first collection with matching elements from the second collection grouped

    user => user.Id,

    });

11. LINQ to Entities Deep Dive (Advanced Level)

Q31: What is the difference between eager loading, lazy loading, and explicit loading in Entity Framework?
  • Eager Loading: Load related entities with the main query using Include(). Good when you know you need the data.
  • Lazy Loading: Load related entities when accessed. Requires virtual navigation properties. Good when you might not need the data.
  • Explicit Loading: Load related entities on demand using Load() or Query(). Good when you need fine-grained control.
// Eager loading
var products = context.Products
    .Include(p => p.Category)
    .Include(p => p.Reviews)
    .ToList();

// Lazy loading (requires virtual properties)
public class Product
{
    public virtual Category Category { get; set; }
}

// Explicit loading
var product = context.Products.First();
context.Entry(product)
    .Reference(p => p.Category)
    .Load();

context.Entry(product)
    .Collection(p => p.Reviews)
    .Query()
    .Where(r => r.Rating > 4)
    .Load();

  • Use AsNoTracking: For read-only queries to avoid change tracking overhead
  • Select specific columns: Avoid fetching unnecessary data

  • Use pagination: Skip() and Take() for large datasets
  • Use compiled queries: For frequently executed queries
  • Avoid N+1 problem: Use Include() or projection
  • Use proper indexing: Ensure database indexes support your queries
  • Use stored procedures: For complex queries

Q33: What is the N+1 query problem and how do you solve it?

The N+1 query problem occurs when you execute N additional queries to fetch related data for each of N entities returned by the main query.

// Problematic - N+1 queries
var categories = context.Categories.ToList();
foreach (var category in categories)
{
    var products = context.Products
        .Where(p => p.CategoryId == category.Id)
        .ToList(); // N additional queries
}

// Solution 1 - Eager loading
var categories = context.Categories
    .Include(c => c.Products)
    .ToList(); // Single query with JOIN

// Solution 2 - Projection
var categories = context.Categories
    .Select(c => new
    {
        c.Name,
        Products = c.Products.Select(p => new { p.Name, p.Price })
    })
    .ToList();

// Solution 3 - Explicit loading with Select
var categories = context.Categories.ToList();
var categoryIds = categories.Select(c => c.Id).ToList();
var products = context.Products
    .Where(p => categoryIds.Contains(p.CategoryId))
    .ToList();

12. Parallel LINQ (Advanced Level)

Q36: What is PLINQ and when should you use it?

PLINQ (Parallel LINQ) enables parallel execution of LINQ queries on multi-core processors using the Task Parallel Library (TPL).

When to use PLINQ:

  • CPU-bound operations on large collections
  • Independent operations that can run in parallel
  • When order of results doesn't matter initially
  • When performance is critical and collection is large

When NOT to use PLINQ:

  • Small collections (overhead outweighs benefits)
  • I/O-bound operations (use async/await instead)
  • When order must be preserved (use AsOrdered())
  • When operations have side effects
// Basic PLINQ
var result = numbers.AsParallel()
    .Where(n => n % 2 == 0)
    .Select(n => n * n)
    .ToList();

// With degree of parallelism
var result = numbers.AsParallel()
    .WithDegreeOfParallelism(4)
    .Where(n => n % 2 == 0)
    .ToList();

// Preserve order
var result = numbers.AsParallel()
    .AsOrdered()
    .Where(n => n % 2 == 0)
    .ToList();

// With cancellation
var cts = new CancellationTokenSource();
var result = numbers.AsParallel()
    .WithCancellation(cts.Token)
    .Where(n => n % 2 == 0)
    .ToList();
Q37: How do you handle exceptions in PLINQ? 
// PLINQ exceptions are wrapped in AggregateException
try
{
    var result = numbers.AsParallel()
        .Select(n => ProcessNumber(n))
        .ToList();
}
catch (AggregateException ae)
{
    foreach (var e in ae.InnerExceptions)
    {
        Console.WriteLine($"Error: {e.Message}");
    }
}

// Handle exceptions within the query
var result = numbers.AsParallel()
    .Select(n =>
    {
        try
        {
            return ProcessNumber(n);
        }
        catch (Exception ex)
        {
            return -1; // Return error indicator
        }
    })
    .ToList();
Q38: What is the difference between AsParallel and AsSequential?
  • AsParallel: Converts the query to run in parallel. Subsequent operators run in parallel.
  • AsSequential: Converts back to sequential execution. Used when you need to switch from parallel to sequential mid-query.
// Start parallel, then switch to sequential
var result = numbers.AsParallel()
    .Where(n => n % 2 == 0) // Parallel
    .AsSequential() // Switch to sequential
    .Take(10) // Sequential
    .ToList();

13. LINQ Extension Methods (Advanced Level)

Q39: How do you create custom LINQ extension methods? 
// Custom extension method with deferred execution
public static IEnumerable<T> WhereIf<T>(
    this IEnumerable<T> source,
    bool condition,
    Func<T, bool> predicate)
{
    if (condition)
    {
        return source.Where(predicate);
    }
    return source;
}

// Usage
var result = items.WhereIf(applyFilter, item => item.Price > 100);

// Custom extension method with immediate execution
public static int CountIf<T>(
    this IEnumerable<T> source,
    Func<T, bool> predicate)
{
    int count = 0;
    foreach (var item in source)
    {
        if (predicate(item))
            count++;
    }
    return count;
}

// Batch extension method
public static IEnumerable<IEnumerable<T>> Batch<T>(
    this IEnumerable<T> source,
    int batchSize)
{
    var batch = new List<T>(batchSize);
    foreach (var item in source)
    {
        batch.Add(item);
        if (batch.Count == batchSize)
        {
            yield return batch;
            batch = new List<T>(batchSize);
        }
    }
    if (batch.Count > 0)
        yield return batch;
}

// Usage
var batches = items.Batch(100);

// Percentile calculation
public static double Percentile(this IEnumerable<double> source, int percentile)
{
    var sorted = source.OrderBy(x => x).ToList();
    int count = sorted.Count;
    
    if (count == 0)
        throw new InvalidOperationException("Sequence contains no elements");
    
    if (percentile < 0 || percentile > 100)
        throw new ArgumentOutOfRangeException(nameof(percentile));
    
    double index = (count - 1) * percentile / 100.0;
    int lower = (int)Math.Floor(index);
    int upper = (int)Math.Ceiling(index);
    
    if (lower == upper)
        return sorted[lower];
    
    double weight = index - lower;
    return sorted[lower] * (1 - weight) + sorted[upper] * weight;
}

        }
    }

// Pearson correlation coefficient
public static double CorrelationCoefficient(
    IEnumerable<double> x,
    IEnumerable<double> y)
{
    var xList = x.ToList();
    var yList = y.ToList();
    
    if (xList.Count != yList.Count)
        throw new ArgumentException("Sequences must have same length");
    
    int n = xList.Count;
    double sumX = xList.Sum();
    double sumY = yList.Sum();
    double sumXY = xList.Zip(yList, (xi, yi) => xi * yi).Sum();
    double sumX2 = xList.Sum(xi => xi * xi);
    double sumY2 = yList.Sum(yi => yi * yi);
    
    double numerator = n * sumXY - sumX * sumY;
    double denominator = Math.Sqrt((n * sumX2 - sumX * sumX) * (n * sumY2 - sumY * sumY));
    
    if (denominator == 0)
        return 0;
    
    return numerator / denominator;
}

public static (double Slope, double Intercept) LinearRegression(
    IEnumerable<double> x,
    IEnumerable<double> y)
{
    var paired = x.Zip(y, (xi, yi) => new { X = xi, Y = yi }).ToList();

// Moving standard deviation
public static IEnumerable<double> MovingStandardDeviation(
    this IEnumerable<double> source,
    int windowSize)
{
    var queue = new Queue<double>();
    double sum = 0;
    double sumSquares = 0;
    
    foreach (var value in source)
    {
        queue.Enqueue(value);
        sum += value;
        sumSquares += value * value;
        
        if (queue.Count > windowSize)
        {
            var removed = queue.Dequeue();
            sum -= removed;
            sumSquares -= removed * removed;
        }
        
        if (queue.Count == windowSize)
        {
            double mean = sum / windowSize;
            double variance = (sumSquares / windowSize) - (mean * mean);
            yield return Math.Sqrt(variance);
        }
    }
}

// Bollinger Bands
public static IEnumerable<(double Upper, double Middle, double Lower)> BollingerBands(
    this IEnumerable<double> source,
    int period,
    double stdDevMultiplier = 2)
{
    var list = source.ToList();
    
    for (int i = period - 1; i < list.Count; i++)
    {
        var window = list.Skip(i - period + 1).Take(period);
        var middle = window.Average();
        var stdDev = window.StandardDeviation();
        var upper = middle + stdDevMultiplier * stdDev;
        var lower = middle - stdDevMultiplier * stdDev;
        
        yield return (upper, middle, lower);
    }
}
