1. ML Fundamentals

Q1: What is the difference between supervised, unsupervised, and reinforcement learning?

Supervised Learning: Uses labeled data to learn mapping from inputs to outputs. Examples: classification, regression.

Unsupervised Learning: Uses unlabeled data to discover patterns. Examples: clustering, dimensionality reduction.

Reinforcement Learning: Learns through trial and error by receiving rewards/penalties. Examples: game playing, robotics.

Q2: What is the bias-variance tradeoff?

Bias: Error from oversimplified models (underfitting). High bias models miss relevant patterns.

Variance: Error from over-sensitivity to training data (overfitting). High variance models capture noise.

Tradeoff: Reducing bias typically increases variance, and vice versa. Goal is to find optimal balance minimizing total error.

Total Error = Bias² + Variance + Irreducible Error

Q3: What are overfitting and underfitting? How do you prevent them?

Overfitting: Model learns training data too well, including noise. Performs great on training data but poorly on new data.

Underfitting: Model is too simple to capture underlying patterns. Performs poorly on both training and test data.

Prevention methods:

  • More training data
  • Regularization (L1/L2, dropout)
  • Cross-validation
  • Early stopping
  • Feature selection
  • Ensemble methods
Q4: What is cross-validation and why is it important?

Cross-validation: Technique to assess model performance by splitting data into multiple subsets.

Why important:

  • Provides more reliable performance estimate
  • Reduces overfitting to specific train/test split
  • Better use of limited data
  • Helps in hyperparameter tuning

K-fold cross-validation: Data split into K folds, each fold used once as validation while K-1 folds for training.

Stratified K-fold: Preserves class distribution in each fold (important for imbalanced datasets).

Q5: What is the difference between parametric and non-parametric models?
Aspect Parametric Non-parametric
Parameters Fixed number of parameters Grows with training data
Flexibility Less flexible, strong assumptions More flexible, fewer assumptions
Data Requirements Less data needed More data needed
Examples Linear regression, logistic regression KNN, decision trees, SVM
Q6: What is the curse of dimensionality?

Curse of Dimensionality: As number of features increases, data becomes sparse, making it harder for algorithms to find patterns.

Problems:

  • Exponential increase in required data
  • Distance metrics become less meaningful
  • Computational complexity increases
  • Overfitting becomes more likely

Solutions: Dimensionality reduction (PCA, t-SNE, UMAP), feature selection, regularization.

Q7: What is the difference between generative and discriminative models?

Generative Models: Learn joint distribution P(X,Y) to generate new samples. Can generate new data.

  • Examples: Naive Bayes, GANs, VAEs
  • Useful when: Need to generate data, handle missing values

Discriminative Models: Learn conditional distribution P(Y|X) directly for classification.

  • Examples: Logistic regression, SVM, neural networks
  • Useful when: Only need classification, typically better accuracy

2. Supervised Learning

Q8: What is the difference between linear regression and logistic regression?
Aspect Linear Regression Logistic Regression
Output Continuous values Probabilities (0-1)
Task Regression Classification
Activation None (linear) Sigmoid
Loss Function MSE Log loss / Cross-entropy
Q9: What is the difference between L1 and L2 regularization?

L1 Regularization (Lasso):

  • Adds absolute value of coefficients as penalty
  • Can drive coefficients to exactly zero (feature selection)
  • Useful when you believe many features are irrelevant

L2 Regularization (Ridge):

  • Adds squared value of coefficients as penalty
  • Shrinks coefficients but rarely to exactly zero
  • Useful when you believe all features are relevant

Elastic Net: Combines both L1 and L2 regularization.

Q10: How does a decision tree work? What are its advantages and disadvantages?

How it works: Splits data based on feature values to maximize information gain or minimize impurity (Gini/entropy).

Advantages:

  • Easy to interpret and visualize
  • Handles both numerical and categorical data
  • Requires little data preprocessing
  • Non-parametric (no assumptions about data distribution)

Disadvantages:

  • Prone to overfitting
  • Unstable (small data changes can cause large tree changes)
  • Can create biased trees if some classes dominate
Q11: What is Random Forest? How does it work?

Random Forest: Ensemble of decision trees using bagging (bootstrap aggregating).

How it works:

  1. Train multiple decision trees on different bootstrap samples
  2. At each split, only consider random subset of features
  3. Average predictions (regression) or majority vote (classification)

Advantages:

  • Reduces overfitting compared to single tree
  • Handles high-dimensional data well
  • Provides feature importance
  • Robust to noise and outliers

Q13: How does SVM work? What is the kernel trick?

SVM (Support Vector Machine): Finds optimal hyperplane that maximizes margin between classes.

Key concepts:

  • Support vectors: Data points closest to decision boundary
  • Margin: Distance between decision boundary and support vectors
  • C parameter: Regularization parameter (tradeoff between margin width and misclassification)

Kernel Trick: Maps data to higher-dimensional space using kernel function without explicit computation.

Common kernels: Linear, Polynomial, RBF (Gaussian), Sigmoid.

Q14: What is gradient descent and its variants?

Gradient Descent: Optimization algorithm to minimize loss function by iteratively moving towards minimum.

Batch Gradient Descent:

  • Uses entire dataset for each update
  • Stable but computationally expensive
  • Can get stuck in local minima

Stochastic Gradient Descent (SGD):

  • Uses one random sample for each update
  • Faster but noisy updates
  • Can escape local minima

Mini-batch Gradient Descent:

  • Uses small batch of samples for each update
  • Balance between stability and speed
  • Most commonly used in practice
Q15: What is K-Nearest Neighbors (KNN)? How do you choose K?

KNN: Non-parametric algorithm that classifies based on K nearest neighbors.

How it works:

  1. Calculate distance between new point and all training points
  2. Select K nearest neighbors
  3. For classification: majority vote
  4. For regression: average of K values

Choosing K:

  • Small K: More flexible, prone to overfitting
  • Large K: Smoother decision boundary, prone to underfitting
  • Common approach: Try K = sqrt(n) or use cross-validation
  • Odd K for binary classification to avoid ties

3. Unsupervised Learning

Q16: What is K-Means clustering? How does it work?

K-Means: Partitions data into K clusters based on feature similarity.

Algorithm:

  1. Initialize K centroids randomly
  2. Assign each point to nearest centroid
  3. Recalculate centroids as mean of assigned points
  4. Repeat until convergence

Choosing K: Elbow method, silhouette score, gap statistic.

Limitations:

  • Requires specifying K
  • Sensitive to initial centroids
  • Assumes spherical clusters
  • Sensitive to outliers

Q18: What is PCA (Principal Component Analysis)?

PCA: Dimensionality reduction technique that transforms data into orthogonal components capturing maximum variance.

How it works:

  1. Standardize the data
  2. Compute covariance matrix
  3. Compute eigenvectors and eigenvalues
  4. Sort eigenvectors by eigenvalues
  5. Project data onto top k eigenvectors

Use cases: Visualization, noise reduction, feature extraction, speeding up algorithms.

Limitations: Linear technique, assumes orthogonal components, loses interpretability.

4. Model Evaluation

Q19: What are precision, recall, F1-score, and when to use each?
Metric Formula When to Use
Precision TP / (TP + FP) When false positives are costly (spam detection)
Recall TP / (TP + FN) When false negatives are costly (disease detection)
F1-Score 2 * (Precision * Recall) / (Precision + Recall) When you need balance between precision and recall
AUC-ROC Area under ROC curve When comparing models across different thresholds
Q20: What is a confusion matrix and what can you derive from it?

Confusion Matrix: Table showing actual vs predicted classifications.

Predicted Positive Predicted Negative
Actual Positive True Positive (TP) False Negative (FN)
Actual Negative False Positive (FP) True Negative (TN)

Derived metrics: Accuracy, Precision, Recall, F1-Score, Specificity, AUC-ROC.

Q21: What is ROC-AUC vs PR-AUC? When does each metric mislead?

ROC-AUC: Area under Receiver Operating Characteristic curve. Plots TPR vs FPR.

PR-AUC: Area under Precision-Recall curve. Plots Precision vs Recall.

When ROC-AUC misleads: With highly imbalanced datasets, ROC-AUC can be overly optimistic.

When PR-AUC is preferred: For imbalanced datasets where positive class is rare. PR-AUC focuses on performance on positive class.

Important: For imbalanced datasets (fraud detection, rare diseases), always use PR-AUC or F1-score instead of accuracy or ROC-AUC.
Q22: How do you handle imbalanced datasets?

Resampling techniques:

  • Oversampling minority class (SMOTE)
  • Undersampling majority class
  • Hybrid approaches

Algorithmic approaches:

  • Use appropriate metrics (F1, AUC-ROC, PR-AUC)
  • Class weights in loss function
  • Ensemble methods with balanced subsampling
  • Anomaly detection techniques

5. Feature Engineering

Q23: What is feature engineering and why is it important?

Feature Engineering: Process of creating new features from existing data to improve model performance.

Techniques:

  • Scaling/normalization (Min-Max, StandardScaler)
  • One-hot encoding
  • Label encoding
  • Binning/discretization
  • Polynomial features
  • Interaction features
  • Domain-specific transformations

Importance: Good features can make simple models perform better than complex models with poor features.

Q24: How do you handle missing data?

Deletion methods:

  • Listwise deletion (remove rows with missing values)
  • Pairwise deletion (use available data for each calculation)

Imputation methods:

  • Mean/median/mode imputation
  • KNN imputation
  • Regression imputation
  • Multiple imputation
  • Domain-specific imputation
Tip: Always understand why data is missing before choosing imputation method. Missing data patterns can be informative.
Q25: What is feature selection? What techniques do you use?

Feature Selection: Process of selecting most relevant features to improve model performance and reduce complexity.

Techniques:

  • Filter methods: Statistical tests (chi-square, ANOVA, correlation)
  • Wrapper methods: Recursive Feature Elimination (RFE), forward/backward selection
  • Embedded methods: Lasso, Ridge, tree-based feature importance
  • Dimensionality reduction: PCA, t-SNE, UMAP

6. Deep Learning

Q26: What is a neural network and how does it work?

Neural Network: Computing system inspired by biological neural networks.

Components:

  • Neurons (processing units)
  • Weights (connection strengths)
  • Biases (thresholds)
  • Activation functions (non-linearity)

Learning: Adjusts weights through backpropagation to minimize error.

Activation functions: ReLU, Sigmoid, Tanh, Softmax, Leaky ReLU, ELU.

Q27: What is backpropagation?

Backpropagation: Algorithm to train neural networks by calculating gradients of loss function with respect to weights.

Process:

  1. Forward pass: Compute predictions
  2. Calculate error (loss)
  3. Backward pass: Compute gradients using chain rule
  4. Update weights using gradient descent
Q28: What is the vanishing gradient problem?

Vanishing Gradient: Gradients become extremely small as they propagate backward through deep networks, preventing earlier layers from learning.

Solutions:

  • ReLU activation function
  • Batch normalization
  • Residual connections (ResNet)
  • LSTM/GRU for RNNs
  • Proper weight initialization (Xavier, He)
Q29: What is a CNN (Convolutional Neural Network)?

CNN: Neural network designed for image processing using convolutional layers.

Key Components:

  • Convolutional layers (detect features)
  • Pooling layers (reduce spatial dimensions)
  • Fully connected layers (classification)

Advantages: Translation invariance, parameter sharing, hierarchical feature learning.

Applications: Image classification, object detection, image segmentation.

Q30: What is an RNN (Recurrent Neural Network)?

RNN: Neural network designed for sequential data with internal memory.

How it works: Maintains hidden state that captures information from previous time steps.

Limitations:

  • Vanishing/exploding gradients
  • Difficulty learning long-term dependencies
  • Sequential processing (can't be parallelized)

Solutions: LSTM, GRU, attention mechanisms.

7. ML System Design

Q31: How do you design an ML pipeline for production?

Key components:

  1. Data Ingestion: Collect and validate data from sources
  2. Data Preprocessing: Clean, transform, and feature engineer
  3. Model Training: Train and validate models
  4. Model Evaluation: Test on holdout data with relevant metrics
  5. Model Deployment: Serve model predictions via API
  6. Monitoring: Track performance, data drift, concept drift
  7. Retraining: Schedule periodic retraining with new data
  8. Logging: Track predictions and feature values for debugging
Tip: Use tools like MLflow, Kubeflow, or SageMaker for production ML pipelines.
Q32: What is data drift vs concept drift?

Data Drift: Input data distribution changes over time (e.g., user demographics shift).

Concept Drift: Relationship between input and output changes (e.g., user behavior pattern changes).

Detection:

  • Monitor input distribution (KS test, PSI)
  • Monitor model performance over time
  • Track prediction distribution

Mitigation: Regular retraining, online learning, ensemble methods.

Q33: How do you monitor ML models in production?

Key metrics to monitor:

  • Model performance (accuracy, F1, AUC)
  • Prediction distribution
  • Feature distribution
  • Latency and throughput
  • Error rates and types
  • Data quality metrics

Tools: Prometheus, Grafana, MLflow, Evidently AI, Arize.

Q34: What is A/B testing in ML?

A/B Testing: Controlled experiment comparing two versions (A: control, B: treatment) to determine which performs better.

Process:

  1. Define hypothesis and metrics
  2. Randomly assign users to groups
  3. Run experiment for sufficient duration
  4. Analyze results with statistical significance tests

Considerations: Sample size, statistical significance, practical significance, seasonality.

Q35: What is model interpretability? Why is it important?

Model Interpretability: Ability to understand and explain model predictions.

Importance:

  • Debugging and improving models
  • Regulatory compliance (GDPR, CCPA)
  • Building user trust
  • Identifying bias and fairness issues

Techniques: LIME, SHAP, feature importance, partial dependence plots, attention visualization.

Memory Requires storing all data Can process data once and discard

Q37: What is transfer learning?

Transfer Learning: Using knowledge from pre-trained model on one task to improve performance on related task.

Benefits:

  • Faster training
  • Better performance with less data
  • Leverages learned features

Common approach: Freeze early layers, fine-tune later layers on new task.

Examples: ImageNet models for computer vision, BERT for NLP.

Q38: What is ensemble learning?

Ensemble Learning: Combining multiple models to improve performance.

Types:

  • Bagging: Random Forest (reduces variance)
  • Boosting: XGBoost, LightGBM (reduces bias and variance)
  • Stacking: Meta-model on base models (combines strengths)
  • Voting: Hard voting (majority) or soft voting (weighted probabilities)

Benefits: Improved accuracy, robustness, reduced overfitting.

Q39: What is hyperparameter tuning? What techniques do you use?

Hyperparameter Tuning: Process of finding optimal hyperparameters for machine learning model.

Techniques:

  • Grid Search: Exhaustive search over specified parameter values
  • Random Search: Random sampling from parameter space (more efficient)
  • Bayesian Optimization: Uses probabilistic model to guide search (SMOTE, TPE)
  • Genetic Algorithms: Evolutionary approach

Tools: Scikit-learn GridSearchCV, Optuna, Hyperopt, Ray Tune.

Q40: What is the difference between bagging, boosting, and stacking?

Bagging (Bootstrap Aggregating):

  • Train multiple models independently on different bootstrap samples
  • Average predictions (regression) or vote (classification)
  • Example: Random Forest
  • Reduces variance

Boosting:

  • Train models sequentially, each correcting errors of previous
  • Weight models based on performance
  • Example: XGBoost, LightGBM, CatBoost
  • Reduces both bias and variance

Stacking:

  • Train multiple base models
  • Train meta-model on base model predictions
  • Meta-model learns to combine base models optimally
  • Can achieve better performance but more complex
Q41: What is learning curve and how do you use it?

Learning Curve: Plot of model performance vs training set size.

Interpretation:

  • High bias: Training and validation errors converge at high value → need more features or complex model
  • High variance: Training error low, validation error high → need more data or regularization
  • Good fit: Both errors low and close → model is well-tuned

Use cases: Diagnose underfitting/overfitting, determine if more data would help, tune model complexity.

Q43: What is SMOTE for imbalanced datasets?

SMOTE (Synthetic Minority Over-sampling Technique): Generates synthetic samples for minority class.

How it works:

  1. For each minority sample, find K nearest neighbors
  2. Generate synthetic sample along line connecting sample and neighbor
  3. Repeat until desired balance achieved

Advantages: Better than simple oversampling (reduces overfitting).

Limitations: Can generate noisy samples, sensitive to outliers.

Q46: What is attention mechanism in deep learning?

Attention: Mechanism that allows model to focus on relevant parts of input when producing output.

How it works:

  1. Compute query, key, value vectors
  2. Calculate attention scores (query · key)
  3. Apply softmax to get weights
  4. Weighted sum of values

Types: Self-attention, cross-attention, multi-head attention.

Applications: Transformers, machine translation, image captioning, speech recognition.

Q48: What is dropout and how does it prevent overfitting?

Dropout: Regularization technique that randomly drops neurons during training.

How it works:

  • During training, randomly set some neuron outputs to zero
  • Forces network to learn redundant representations
  • Reduces co-adaptation between neurons

Benefits:

  • Reduces overfitting
  • Improves generalization
  • Acts as model ensemble

Typical dropout rates: 0.2-0.5 for hidden layers, 0.5-0.8 for input layers.

Q52: What is DBSCAN clustering?

DBSCAN (Density-Based Spatial Clustering): Clustering algorithm based on density of data points.

Key parameters:

  • Epsilon (ε): Maximum distance between two samples to be considered neighbors
  • MinPts: Minimum number of samples in a neighborhood to form a cluster

Advantages:

  • Can find arbitrarily shaped clusters
  • Does not require specifying number of clusters
  • Handles outliers well

Disadvantages: Sensitive to parameter selection, struggles with varying densities.

Q54: What is the difference between correlation and covariance?

Covariance: Measure of how two variables change together. Scale-dependent.

Correlation: Normalized covariance (between -1 and 1). Scale-independent.

Formula: Correlation = Covariance(X,Y) / (σX * σY)

Use case: Correlation is preferred for comparison as it's unitless and bounded.

Q55: What is the difference between Type I and Type II errors?
Error Type Description Example
Type I (False Positive) Rejecting true null hypothesis Healthy person diagnosed with disease
Type II (False Negative) Failing to reject false null hypothesis Sick person diagnosed as healthy

Tradeoff: Reducing Type I error typically increases Type II error, and vice versa.

Q56: What is p-value and statistical significance?

p-value: Probability of obtaining results at least as extreme as observed, assuming null hypothesis is true.

Statistical significance: Typically p < 0.05 (5% significance level).

Interpretation:

  • p < 0.05: Reject null hypothesis (statistically significant)
  • p ≥ 0.05: Fail to reject null hypothesis (not statistically significant)
Important: Statistical significance does not imply practical significance. Large datasets can yield statistically significant results for trivial effects.

Q58: What is the difference between ANOVA and t-test?

t-test: Compares means of two groups.

ANOVA (Analysis of Variance): Compares means of three or more groups.

Why ANOVA instead of multiple t-tests:

  • Multiple t-tests increase Type I error rate
  • ANOVA controls overall error rate
  • More efficient for multiple comparisons

Types: One-way ANOVA (one factor), Two-way ANOVA (two factors).

Q61: What is the difference between RobustScaler and StandardScaler?

StandardScaler: Uses mean and standard deviation. Sensitive to outliers.

RobustScaler: Uses median and interquartile range (IQR). Robust to outliers.

Formula: (x - median) / IQR

Use case: RobustScaler is preferred when data contains outliers.

Q62: What is the difference between accuracy and balanced accuracy?

Accuracy: (TP + TN) / (TP + TN + FP + FN). Can be misleading for imbalanced datasets.

Balanced Accuracy: Average of recall for each class. Accounts for class imbalance.

Formula: (Recall_class1 + Recall_class2 + ...) / n_classes

Use case: Balanced accuracy is preferred for imbalanced datasets.

Q63: What is the difference between macro, micro, and weighted F1-score?

Macro F1: Average of F1-scores for each class (treats all classes equally).

Micro F1: Calculates F1 globally by counting total TP, FP, FN (treats all instances equally).

Weighted F1: Average of F1-scores weighted by support (number of instances per class).

Use case:

  • Macro: When all classes are equally important
  • Micro: When class imbalance is not a concern
  • Weighted: When you want to account for class imbalance
Q64: What is the difference between log loss and cross-entropy?

Log Loss (Logarithmic Loss): Measures performance of classification models where output is probability.

Cross-Entropy: Generalization of log loss for multi-class classification.

Binary classification: Log loss = Cross-entropy

Multi-class classification: Cross-entropy extends log loss to multiple classes

Properties: Lower is better, penalizes confident wrong predictions heavily.

Q65: What is the difference between MSE, MAE, and RMSE?
Metric Formula Characteristics
MSE (Mean Squared Error) Σ(y - ŷ)² / n Penalizes large errors heavily, differentiable
MAE (Mean Absolute Error) Σ|y - ŷ| / n Robust to outliers, not differentiable at 0
RMSE (Root MSE) √MSE Same units as target, interpretable
Q66: What is the difference between R² and Adjusted R²?

R² (R-squared): Proportion of variance explained by model. Always increases with more features.

Adjusted R²: R² adjusted for number of predictors. Penalizes adding irrelevant features.

Formula: Adjusted R² = 1 - (1-R²)(n-1)/(n-p-1)

Use case: Adjusted R² is preferred for comparing models with different numbers of features.

Q67: What is the difference between Huber loss and MSE?

MSE: Quadratic loss for all errors. Sensitive to outliers.

Huber Loss: Quadratic for small errors, linear for large errors. Robust to outliers.

Formula:

  • If |error| ≤ δ: 0.5 * error²
  • If |error| > δ: δ * (|error| - 0.5 * δ)

Use case: Huber loss is preferred when data contains outliers.

Q68: What is the difference between L1 and L2 distance metrics?
Metric Formula Characteristics
L1 (Manhattan) Σ|x - y| Robust to outliers, sparse solutions
L2 (Euclidean) √Σ(x - y)² Sensitive to outliers, smooth

Use case: L1 for feature selection, L2 for smooth optimization.

Q69: What is the difference between cosine similarity and cosine distance?

Cosine Similarity: Measures cosine of angle between vectors. Range: [-1, 1]. Higher = more similar.

Cosine Distance: 1 - Cosine Similarity. Range: [0, 2]. Lower = more similar.

Formula: Cosine Similarity = (A · B) / (||A|| * ||B||)

Use case: Text analysis, recommendation systems, document similarity.

Q70: What is the difference between Jaccard similarity and cosine similarity?
Aspect Jaccard Similarity Cosine Similarity
Definition Intersection / Union Cosine of angle
Range [0, 1] [-1, 1]
Use case Set similarity, binary data Vector similarity, text data
Q71: What is the difference between silhouette score and Davies-Bouldin index?

Silhouette Score: Measures how similar an object is to its own cluster compared to other clusters. Range: [-1, 1]. Higher = better.

Davies-Bouldin Index: Average similarity between each cluster and its most similar cluster. Lower = better.

Use case: Both used for evaluating clustering quality. Silhouette is more intuitive, Davies-Bouldin is faster to compute.

Q72: What is the difference between elbow method and silhouette method for choosing K?

Elbow Method: Plots within-cluster sum of squares (WCSS) vs K. Choose K at "elbow" point.

Silhouette Method: Plots silhouette score vs K. Choose K with highest score.

Comparison:

  • Elbow: Subjective, may not have clear elbow
  • Silhouette: More objective, considers both cohesion and separation
