1. AWS Fundamentals

Q1: What is AWS and what are its key features?

Amazon Web Services (AWS) is a comprehensive cloud platform offering over 200 fully featured services from data centers globally.

Key Features:

  • Global infrastructure: Regions, Availability Zones, Edge Locations
  • Pay-as-you-go: Pay only for what you use
  • Scalability: Auto-scaling capabilities
  • Security: Shared responsibility model
  • Reliability: 99.99% SLA for many services
  • Flexibility: Wide range of services
  • Ecosystem: Large partner network

Q6: What are the different S3 storage classes?
  • Standard: Frequently accessed data
  • Intelligent-Tiering: Automatic cost optimization
  • Standard-IA: Infrequently accessed data
  • One Zone-IA: Infrequently accessed, single AZ
  • Glacier: Long-term archival (minutes retrieval)
  • Glacier Deep Archive: Long-term archival (hours retrieval)
Important: Choose the appropriate storage class based on access patterns and cost requirements. S3 lifecycle policies can automatically move data between classes.

7. AWS Serverless

Q13: What is AWS Lambda?

AWS Lambda is a serverless compute service that runs code in response to events and automatically manages the underlying compute resources.

Trigger Types:

  • S3: Object events
  • DynamoDB Streams: Database changes
  • Kinesis: Streaming data
  • SNS: Pub/sub messaging
  • SQS: Queue messages
  • API Gateway: HTTP endpoints
  • CloudWatch Events: Scheduled events
// Node.js Lambda function
exports.handler = async (event) => {
  console.log('Event:', JSON.stringify(event, null, 2));
  
  const response = {
    statusCode: 200,
    body: JSON.stringify({
      message: 'Hello from Lambda!',
      input: event
    })
  };
  
  return response;
};

10. Common Interview Questions

Q19: What is the difference between S3 and EBS?
  • S3: Object storage, unlimited size, accessible from anywhere, pay per GB/month. Best for static assets, backups.
  • EBS: Block storage, attached to EC2, limited size per volume, pay per GB/month + IOPS. Best for OS, databases.
Q20: What is the difference between public and private subnets?
  • Public subnet: Has route to Internet Gateway. Resources can access internet directly. Used for web servers, load balancers.
  • Private subnet: No route to Internet Gateway. Resources access internet via NAT Gateway. Used for databases, application servers.
Q21: What is the difference between NAT Gateway and Internet Gateway?

Aspect SQS SNS

Use Case Decoupling, buffering Broadcasting, notifications
Q25: What is the difference between CloudFront and ELB?
  • CloudFront: Global CDN that caches content at edge locations. Reduces latency for static content.
  • ELB: Load balancer that distributes traffic across multiple targets in a region. Provides health checks and scaling.
Q26: What is the difference between IAM role and IAM user?
  • IAM User: Represents a person or application. Has long-term credentials (access keys, password). Used for programmatic access or console access.
  • IAM Role: Temporary credentials. Used by AWS services or applications to assume permissions. No long-term credentials.
Q27: What is the difference between ECS and EKS?
Aspect ECS EKS
Orchestrator AWS proprietary Kubernetes
Portability AWS only Portable
Ecosystem AWS ecosystem Kubernetes ecosystem
Learning Curve Lower Higher
Q28: What is the difference between Kinesis and SQS?
  • Kinesis: Streaming data service. Real-time processing. Ordered data. Good for analytics, real-time processing.
  • SQS: Message queue service. Decoupling. No order guarantee. Good for buffering, task queues.
Q29: What is the difference between CloudTrail and CloudWatch?
  • CloudTrail: Logs API calls for auditing and compliance. Who did what and when. Focus on security and compliance.
  • CloudWatch: Monitoring and observability. Metrics, logs, alarms. Focus on operational health and performance.
Q30: What is the difference between Elastic IP and Public IP?
  • Public IP: Dynamic IP assigned to instance. Changes when instance is stopped/started. Free while instance is running.
  • Elastic IP: Static IP that you own. Can be remapped to different instances. Charged when not attached to running instance.

11. AWS Cost Management (All Levels Level)

Q35: How do you optimize AWS costs?
  • Right-sizing: Choose appropriate instance types
  • Reserved Instances: Commit to 1-3 years for discounts
  • Savings Plans: Flexible pricing model for compute
  • Spot Instances: Use unused capacity at up to 90% discount
  • Auto-scaling: Scale down during off-peak hours
  • S3 lifecycle policies: Move data to cheaper storage tiers
  • EBS snapshots: Delete old snapshots
  • Cost allocation tags: Track costs by department/project
Q36: What is AWS Cost Explorer?

AWS Cost Explorer is a tool for visualizing, understanding, and managing your AWS costs and usage.

Features:

  • Cost analysis: View costs by service, region, tag
  • Forecasting: Predict future spending
  • Anomalies: Detect unusual spending patterns
  • Recommendations: Get cost optimization recommendations
  • Reports: Generate custom reports
Q37: What is AWS Budgets?

AWS Budgets allows you to set custom budgets that alert you when your costs or usage exceed your thresholds.

Features:

  • Cost budgets: Set spending limits
  • Usage budgets: Set usage limits
  • Alerts: Email and SNS notifications
  • Actions: Automate responses to budget breaches
  • Forecasting: Include forecasted costs

12. AWS Hybrid Solutions (All Levels Level)

Q38: What is AWS Direct Connect?

AWS Direct Connect is a dedicated network connection between your on-premises network and AWS.

Benefits:

  • Consistent performance: Lower latency and jitter
  • Higher bandwidth: Up to 100 Gbps
  • Private connection: Doesn't traverse public internet
  • Cost savings: Reduced data transfer costs
  • Security: Private, encrypted connection
Q39: What is AWS Storage Gateway?

AWS Storage Gateway is a hybrid cloud storage service that connects on-premises applications with cloud storage.

Types:

  • File Gateway: NFS/SMB file share to S3
  • Volume Gateway: iSCSI block storage to S3/EBS
  • Tape Gateway: Virtual tape library to S3/Glacier
Q40: What is AWS Outposts?

AWS Outposts is a fully managed service that extends AWS infrastructure, services, APIs, and tools to customer premises.

Key Features:

  • Consistent AWS experience: Same APIs and services
  • Low latency: On-premises for latency-sensitive workloads
  • Data residency: Keep data on-premises
  • Hybrid cloud: Seamless integration with AWS
  • Managed service: AWS manages hardware and software

Q52: How do you implement AWS Transit Gateway for multi-VPC networking?

Transit Gateway provides hub-and-spoke networking across VPCs and on-premise.

# Create Transit Gateway
aws ec2 create-transit-gateway \
  --description "Main Transit Gateway" \
  --options AmazonSideAsn=64512

# Create VPC attachment
aws ec2 create-transit-gateway-vpc-attachment \
  --transit-gateway-id tgw-1234 \
  --vpc-id vpc-1234 \
  --subnet-ids subnet-1234 subnet-5678

# Create VPN attachment
aws ec2 create-transit-gateway-vpn-attachment \
  --transit-gateway-id tgw-1234 \
  --vpn-connection-id vpn-1234

# Route table association
aws ec2 associate-transit-gateway-route-table \
  --transit-gateway-route-table-id tgw-rt-1234 \
  --transit-gateway-attachment-id tgw-attach-1234

# Static route
aws ec2 create-transit-gateway-route \
  --transit-gateway-route-table-id tgw-rt-1234 \
  --destination-cidr 10.0.0.0/16 \
  --transit-gateway-attachment-id tgw-attach-1234

Q54: How do you implement AWS EventBridge for event-driven architecture?

EventBridge enables event-driven architecture with real-time event routing.

# Create event bus
aws events create-event-bus --name "OrderEvents"

# Create rule
aws events put-rule \
  --name "OrderCreatedRule" \
  --event-bus-name "OrderEvents" \
  --event-pattern file://event-pattern.json

# Event pattern
{
  "source": ["com.myapp.orders"],
  "detail-type": ["OrderCreated"],
  "detail": {
    "status": ["pending"]
  }
}

# Add target to rule
aws events put-targets \
  --rule "OrderCreatedRule" \
  --event-bus-name "OrderEvents" \
  --targets file://targets.json

# Targets configuration
[
  {
    "Id": "1",
    "Arn": "arn:aws:lambda:us-east-1:123456789012:function:ProcessOrder",
    "InputTransformer": {
      "InputPathsMap": {
        "orderId": "$.detail.orderId"
      },
      "InputTemplate": "{\"orderId\": <orderId>}"
    }
  }
]

# Cross-account event routing
aws events put-rule \
  --name "CrossAccountRule" \
  --event-bus-name "default" \
  --event-pattern '{"account":["123456789012"]}'

# Archive events
aws events put-archive-policy \
  --event-bus-name "OrderEvents" \
  --retention-days 30

CDK (Cloud Development Kit) enables infrastructure as code with programming languages.

// TypeScript CDK example
import * as cdk from 'aws-cdk-lib';
import { Construct } from 'constructs';
import * as ec2 from 'aws-cdk-lib/aws-ec2';
import * as rds from 'aws-cdk-lib/aws-rds';

export class MyStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {

    // VPC

          subnetType: ec2.SubnetType.PRIVATE_WITH_EGRESS,
          name: 'Private'
        }
      ]
    });

    // RDS instance
    const database = new rds.DatabaseInstance(this, 'Database', {

      storageEncrypted: true,
      deletionProtection: true
    });

    // Output database endpoint
    new cdk.CfnOutput(this, 'DatabaseEndpoint', {
      value: database.instanceEndpoint.hostname

  }
}
