1. AWS Fundamentals

Q1: What is AWS?

Amazon Web Services (AWS) is a comprehensive cloud platform offering over 200 services including compute, storage, databases, networking, analytics, machine learning, and more.

Key Features:

  • Global Infrastructure: Regions, Availability Zones, Edge Locations
  • Pay-as-you-go: Pay only for what you use
  • Scalability: Auto-scaling and elastic resources
  • Security: Comprehensive security features
  • Services: 200+ cloud services

Q3: What are AWS Regions and Availability Zones?

AWS Regions are geographical areas with multiple isolated data centers called Availability Zones. Each region is completely independent and isolated from other regions to provide fault tolerance and stability. Availability Zones (AZs) are one or more discrete data centers with redundant power, networking, and connectivity within an AWS Region. Each AZ is isolated from failures in other AZs, providing high availability and fault tolerance.

Key Concepts:

  • Regions: Geographical locations (e.g., us-east-1, eu-west-2) with multiple AZs
  • Availability Zones: Isolated data centers within a region
  • Edge Locations: Points of presence for CloudFront (100+ locations)
  • Local Zones: Extensions of AWS Regions in specific geographic areas
  • Wavelength Zones: AWS infrastructure deployed at telecommunications providers' 5G edge locations

Best Practices:

  • Deploy resources across multiple AZs for high availability
  • Use multiple regions for disaster recovery and latency reduction
  • Consider data residency requirements when choosing regions
  • Leverage edge locations for content delivery and low-latency applications
list_regions.py
import boto3 # List all AWS regions ec2 = boto3.client('ec2') regions = ec2.describe_regions() for region in regions['Regions']: print(f"Region: {region['RegionName']}")
Q4: What is the AWS Shared Responsibility Model?

AWS Shared Responsibility Model defines security responsibilities: AWS manages security of the cloud, while customers manage security in the cloud.

AWS Responsibilities:

  • Physical security of data centers
  • Network infrastructure
  • Hypervisor and compute infrastructure

Customer Responsibilities:

  • Account management
  • Network configuration
  • Operating system and application security
  • Data encryption

AWS Identity and Access Management (IAM) is a web service for securely controlling access to AWS services. It provides authentication and authorization.

Q6: What is an AWS Account?

AWS Account is a container for AWS resources. It provides a billing boundary and access control. Each account has a unique root user with full access to all resources and services. AWS Organizations allows you to manage multiple AWS accounts centrally, enabling you to consolidate billing, apply policies, and implement governance across your organization.

Account Features:

  • Root User: Complete access to all AWS resources and billing
  • IAM Users: Individual users with specific permissions
  • Billing: Consolidated billing for all services used
  • Resource Isolation: Logical separation of resources
  • Security: Account-level security settings and policies

Best Practices:

  • Never use root credentials for daily operations
  • Enable MFA on the root account
  • Use AWS Organizations for multi-account management
  • Implement strong password policies
  • Regularly rotate access keys

2. AWS Compute Services

Q7: What is EC2?

Amazon EC2 (Elastic Compute Cloud) provides scalable computing capacity in the cloud. It allows users to launch virtual servers with various configurations. EC2 eliminates the need to invest in hardware up front, so you can develop and deploy applications faster. You can launch as many or as few virtual servers as you need, configure security and networking, and manage storage.

EC2 Key Features:

  • Instance Types: Optimized for different use cases (compute, memory, storage, GPU)
  • AMIs (Amazon Machine Images): Pre-configured templates for instances
  • Elastic IPs: Static IP addresses for dynamic cloud computing
  • Security Groups: Virtual firewalls controlling inbound/outbound traffic
  • Auto Scaling: Automatically scale capacity based on demand
  • Placement Groups: Low-latency networking for instances
ec2_launch.sh
# Launch EC2 instance using AWS CLI aws ec2 run-instances \ --image-id ami-0c55b159cbfafe1f0 \ --instance-type t2.micro \ --key-name my-key-pair \ --security-group-ids sg-903004f8 \ --subnet-id subnet-12345678 \ --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=MyInstance}]' # List running instances aws ec2 describe-instances \ --filters Name=instance-state-name,Values=running \ --query 'Reservations[*].Instances[*].[InstanceId,State.Name,Tags[?Key==`Name`].Value]' \ --output table
Q8: What is Lambda?

AWS Lambda is a serverless compute service that runs code in response to events. It automatically manages compute resources, so you don't need to provision or manage servers. Lambda executes your code only when needed and scales automatically from a few requests per day to thousands per second. You pay only for the compute time you consume.

Lambda Key Features:

  • Serverless: No server management or provisioning
  • Event-Driven: Triggered by AWS services, HTTP requests, or custom events
  • Auto-Scaling: Automatically scales based on request volume
  • Pay-per-Use: Pay only for compute time (milliseconds)
  • Multiple Languages: Supports Python, Node.js, Java, Go, C#, Ruby, and custom runtimes
  • Integration: Works with 200+ AWS services
lambda.py
import json import boto3 def lambda_handler(event, context): # Process S3 event for record in event['Records']: bucket = record['s3']['bucket']['name'] key = record['s3']['object']['key'] print(f"File uploaded: {key} in {bucket}") return { 'statusCode': 200, 'body': json.dumps({'message': 'Processing complete'}) }
Q9: What is ECS?

Amazon ECS (Elastic Container Service) is a highly scalable container orchestration service for running Docker containers on AWS. It simplifies running containerized applications at scale without managing the underlying infrastructure. ECS integrates with other AWS services like ELB, CloudWatch, and IAM for comprehensive container management.

ECS Key Features:

  • Task Definitions: Blueprints for your containers (similar to Docker Compose)
  • Tasks: Running instances of task definitions
  • Services: Long-running tasks that are automatically replaced if they fail
  • Clusters: Logical grouping of tasks or services
  • Launch Types: Fargate (serverless) or EC2 (self-managed)
  • Auto Scaling: Automatically adjust number of tasks based on metrics

{ "family": "my-web-app", "containerDefinitions": [ { "name": "web", "image": "nginx:latest", "memory": 512, "cpu": 256, "essential": true, "portMappings": [ { "containerPort": 80 } ] } ], "requiresCompatibilities": ["FARGATE"], "networkMode": "awsvpc", "cpu": "256", "memory": "512" }
Q10: What is EKS?

Amazon EKS (Elastic Kubernetes Service) is a managed Kubernetes service for running Kubernetes on AWS without managing the control plane. EKS automatically manages the availability and scalability of the Kubernetes control plane across multiple AWS Availability Zones, detects and replaces unhealthy control plane nodes, and provides on-demand upgrades and patching for the control plane.

EKS Key Features:

  • Managed Control Plane: AWS manages Kubernetes API server and etcd
  • High Availability: Control plane runs across multiple AZs
  • Serverless Option: Run Kubernetes on AWS Fargate
  • Integration: Works with AWS IAM, VPC, CloudWatch, and other services
  • Upgrades: Automated Kubernetes version upgrades
  • Security: VPC isolation, IAM authentication, secrets encryption
eks_cluster.yaml
apiVersion: eksctl.io/v1alpha5 kind: ClusterConfig metadata: name: my-cluster region: us-east-1 nodeGroups: - name: nodegroup-1 instanceType: t2.micro desiredCapacity: 2 minSize: 1 maxSize: 3
Q11: What is Elastic Beanstalk?

AWS Elastic Beanstalk is a Platform as a Service (PaaS) for deploying and scaling web applications. It supports multiple programming languages including Java, .NET, PHP, Node.js, Python, Ruby, Go, and Docker. Elastic Beanstalk handles the infrastructure provisioning, load balancing, auto-scaling, and application health monitoring, allowing developers to focus on writing code rather than managing infrastructure.

Elastic Beanstalk Features:

  • Easy Deployment: Upload code and Beanstalk handles the rest
  • Auto Scaling: Automatically scales based on traffic
  • Load Balancing: Integrated with ELB for traffic distribution
  • Health Monitoring: Automatic health checks and alerts
  • Environment Management: Multiple environments (dev, test, prod)
  • Rolling Updates: Zero-downtime deployments

# Deploy application to Elastic Beanstalk aws elasticbeanstalk create-application \ --application-name my-app aws elasticbeanstalk create-environment \ --application-name my-app \ --environment-name my-env \ --solution-stack-name "64bit Amazon Linux 2 v5.8.0 running Python 3.8" # Deploy application version aws elasticbeanstalk create-application-version \ --application-name my-app \ --version-label v1 \ --source-bundle S3Bucket=my-bucket,S3Key=app.zip aws elasticbeanstalk update-environment \ --environment-name my-env \ --version-label v1
Q12: What is Fargate?

AWS Fargate is a serverless compute engine for containers that works with both ECS and EKS, removing the need to manage servers. With Fargate, you don't need to provision, configure, or scale clusters of virtual machines to run containers. You just define your application's resource requirements, pay for the resources the application uses, and get improved security through isolation by design.

Fargate Key Features:

  • No Server Management: No EC2 instances to manage or patch
  • Pay-per-Use: Pay only for vCPU and memory resources consumed
  • Security: Isolated compute environment for each task/pod
  • Integration: Works with ECS and EKS
  • Simplified Scaling: Define resource requirements, Fargate handles the rest
  • High Availability: Built-in multi-AZ support

3. AWS Storage

Q13: What are AWS Storage services?

Storage Services:

  • S3: Object storage for any data
  • EBS: Block storage for EC2
  • EFS: File storage for EC2 and Lambda
  • Glacier: Long-term archival storage
  • Storage Gateway: Hybrid storage integration
Q14: What is S3?

Amazon S3 (Simple Storage Service) is object storage for any amount of data. It offers high durability, availability, and scalability. S3 is designed for 99.999999999% (11 9's) durability and 99.99% availability, making it ideal for storing data lakes, websites, mobile applications, backup and restore, archive, and big data analytics.

S3 Key Features:

  • Object Storage: Store any amount of data as objects
  • Storage Classes: Multiple tiers for different access patterns and cost optimization
  • Versioning: Keep multiple versions of objects
  • Encryption: Server-side and client-side encryption
  • Lifecycle Policies: Automatic transition between storage classes
  • Static Website Hosting: Host static websites directly from S3
s3_operations.py
import boto3 s3 = boto3.client('s3') # Create bucket s3.create_bucket(Bucket='my-unique-bucket') # Upload file s3.upload_file( 'local_file.txt', 'my-unique-bucket', 'remote_file.txt' ) # Download file s3.download_file( 'my-unique-bucket', 'remote_file.txt', 'downloaded_file.txt' ) # List objects objects = s3.list_objects_v2(Bucket='my-unique-bucket') for obj in objects['Contents']: print(obj['Key'])
Q15: What is EBS?

Amazon EBS (Elastic Block Store) provides block-level storage volumes for EC2 instances. It offers persistent storage that survives instance termination. EBS volumes are like hard disks that you can attach to your EC2 instances. They provide consistent and low-latency performance, and you can choose from different volume types optimized for different workloads.

EBS Volume Types:

  • gp2/gp3: General Purpose SSD - balance price and performance
  • io1/io2: Provisioned IOPS SSD - highest performance for I/O-intensive workloads
  • st1: Throughput Optimized HDD - big data, data warehouses
  • sc1: Cold HDD - infrequent access data
ebs_volume.py
import boto3 ec2 = boto3.client('ec2') # Create EBS volume volume = ec2.create_volume( AvailabilityZone='us-east-1a', VolumeType='gp2', Size=20 ) # Attach volume to instance ec2.attach_volume( VolumeId=volume['VolumeId'], InstanceId='i-1234567890abcdef0', Device='/dev/sdh' )
Q16: What is EFS?

Amazon EFS (Elastic File System) provides scalable file storage for EC2 instances and Lambda functions. It supports NFS protocol and provides a simple, scalable, fully managed elastic NFS file system for use with AWS Cloud services and on-premises resources. EFS grows and shrinks automatically as you add and remove files, eliminating the need to provision and manage capacity.

EFS Key Features:

  • Scalable: Automatically grows and shrinks with your data
  • Shared Access: Multiple instances can access the same file system concurrently
  • NFS Protocol: Standard NFS v4 protocol for easy integration
  • Performance Modes: General Purpose and Max I/O for different workloads
  • Throughput Modes: Bursting and Provisioned throughput options
  • Encryption: Data at rest and in transit encryption
efs_create.sh
# Create EFS file system aws efs create-file-system \ --creation-token my-efs \ --performance-mode generalPurpose \ --throughput-mode bursting # Create mount target aws efs create-mount-target \ --file-system-id fs-12345678 \ --subnet-id subnet-abcdef0 \ --security-group-ids sg-12345678

Q18: What is S3 versioning?

S3 Versioning keeps multiple versions of an object in the same bucket. It provides protection against accidental deletion and overwrites. When versioning is enabled on a bucket, S3 preserves all versions of every object, allowing you to recover from unintended user actions or application failures. Each object version has a unique ID, and you can retrieve any previous version of an object.

Versioning Features:

  • Protection: Recover from accidental deletions and overwrites
  • Retention: Keep all versions or configure lifecycle rules to expire old versions
  • Cost: You pay for storage for all versions stored
  • MFA Delete: Require MFA to permanently delete versioned objects
  • Suspension: Can suspend versioning but existing versions remain
s3_versioning.py
import boto3 s3 = boto3.resource('s3') bucket = s3.Bucket('my-bucket') # Enable versioning bucket.Versioning().enable() # List object versions for version in bucket.object_versions.all(): print(f"{version.key}: {version.id}")
Q19: What is S3 lifecycle policy?

S3 Lifecycle Policy automatically transitions objects between storage classes or expires them based on rules, optimizing storage costs. You can define lifecycle rules to transition objects to less expensive storage classes when they are accessed less frequently, or expire objects after a certain period. This helps reduce storage costs while maintaining the appropriate performance characteristics for your data access patterns.

Lifecycle Policy Actions:

  • Transition: Move objects between storage classes (e.g., Standard to IA)
  • Expiration: Delete objects after a specified time period
  • Noncurrent Version: Manage old versions of objects
  • Incomplete Multipart Uploads: Clean up failed uploads
lifecycle_policy.json
{ "Rules": [ { "ID": "TransitionToIA", "Status": "Enabled", "Filter": {"Prefix": "logs/"}, "Transitions": [ { "Days": 30, "StorageClass": "STANDARD_IA" }, { "Days": 90, "StorageClass": "GLACIER" } ] } ] }
Q20: What is Glacier?

Amazon S3 Glacier provides secure, durable, and low-cost storage for data archiving and long-term backup. It offers three retrieval options: Expedited (1-5 minutes), Standard (3-5 hours), and Bulk (5-12 hours). Glacier is ideal for data that is rarely accessed and can tolerate retrieval times. It provides the same durability as S3 Standard (99.999999999%) but at a significantly lower cost.

Glacier Features:

  • Low Cost: Cheapest storage class for long-term archival
  • Three Retrieval Options: Expedited, Standard, and Bulk
  • Vaults: Organize archives into vaults for access control
  • Encryption: Encrypt data at rest using AES-256
  • Vault Lock: Implement WORM (Write Once Read Many) compliance
glacier_archive.py
import boto3 glacier = boto3.client('glacier') # Upload archive to vault response = glacier.upload_archive( vaultName='my-vault', archiveDescription='Backup data', body=open('backup.zip', 'rb') ) # Initiate retrieval job glacier.initiate_job( vaultName='my-vault', jobParameters={'Type': 'archive-retrieval'} )

4. AWS Networking

Q21: What is VPC?

Amazon VPC (Virtual Private Cloud) is a logically isolated network in the AWS cloud. It provides control over network environment including IP ranges, subnets, and routing. A VPC is similar to a traditional on-premises network, with the benefits of using AWS's scalable infrastructure. You can define your own IP address range, create subnets, configure route tables, and set up network gateways.

VPC Key Components:

  • Subnets: Segments of the VPC's IP address range (public and private)
  • Route Tables: Control network traffic routing between subnets and internet
  • Internet Gateway: Enables internet access for public subnets
  • NAT Gateway: Enables private subnets to access the internet
  • Security Groups: Virtual firewalls for instances
  • Network ACLs: Network-level firewall rules for subnets
vpc_create.py
import boto3 ec2 = boto3.resource('ec2') # Create VPC vpc = ec2.create_vpc(CidrBlock='10.0.0.0/16') vpc.create_tags(Tags=[{'Key': 'Name', 'Value': 'MyVPC'}]) # Create subnet subnet = vpc.create_subnet(CidrBlock='10.0.1.0/24') # Create internet gateway igw = ec2.create_internet_gateway() vpc.attach_internet_gateway(InternetGatewayId=igw.id)

Q23: What is Internet Gateway?

Internet Gateway allows communication between resources in a VPC and the internet. It enables instances in public subnets to access the internet. An Internet Gateway is a horizontally scaled, redundant, and highly available VPC component that allows communication between instances in your VPC and the internet. It imposes no availability risks or bandwidth constraints on your network traffic.

Internet Gateway Features:

  • Public Access: Enables public subnets to connect to the internet
  • High Availability: Redundant and highly available within the region
  • No Bandwidth Constraints: Scales automatically to handle network traffic
  • One per VPC: Only one internet gateway can be attached to a VPC
  • SNAT Support: Supports Source Network Address Translation
igw_create.py
import boto3 ec2 = boto3.resource('ec2') # Create internet gateway igw = ec2.create_internet_gateway() igw.create_tags(Tags=[{'Key': 'Name', 'Value': 'MyIGW'}]) # Attach to VPC vpc = ec2.Vpc('vpc-12345678') vpc.attach_internet_gateway(InternetGatewayId=igw.id)
Q24: What is NAT Gateway?

NAT Gateway enables instances in private subnets to connect to services outside the VPC but prevents external services from initiating connections. It's a managed service that provides high availability and bandwidth. NAT Gateway is used to allow instances in private subnets to connect to the internet for software updates, patching, or accessing external APIs, while maintaining security by preventing unsolicited inbound connections.

NAT Gateway Features:

  • Outbound Only: Allows outbound internet access, blocks inbound
  • High Availability: Automatically scales and provides redundancy
  • Elastic IP: Requires a static IP address
  • Bandwidth: Automatically scales up to 45 Gbps
  • Availability Zone: Deployed in specific AZ for zone-level isolation
nat_gateway.py
import boto3 ec2 = boto3.client('ec2') # Allocate Elastic IP eip_response = ec2.allocate_address(Domain='vpc') # Create NAT Gateway nat_response = ec2.create_nat_gateway( SubnetId='subnet-12345678', AllocationId=eip_response['AllocationId'], TagSpecifications=[{ 'ResourceType': 'natgateway', 'Tags': [{'Key': 'Name', 'Value': 'MyNAT'}] }] )
Q25: What is Route 53?

Amazon Route 53 is a highly available and scalable cloud Domain Name System (DNS) web service. It provides domain registration and DNS routing. Route 53 connects user requests to infrastructure running in AWS or outside of AWS. It offers low latency and high availability through a global network of DNS servers. Route 53 also provides health checking and DNS failover for high availability applications.

Route 53 Features:

  • Domain Registration: Register domain names directly with AWS
  • DNS Routing: Route traffic to AWS resources or external endpoints
  • Health Checks: Monitor the health of your resources
  • Routing Policies: Simple, Weighted, Latency, Failover, Geolocation
  • Alias Records: Route to AWS resources like ELB, S3, CloudFront
  • Private DNS: Private hosted zones for VPC internal DNS

Q26: What is CloudFront?

Amazon CloudFront is a content delivery network (CDN) that securely delivers data, videos, applications, and APIs to customers globally with low latency. CloudFront integrates with other AWS products to give developers and businesses an easy way to accelerate content to end users with no minimum usage commitments. It uses a global network of edge locations to cache content closer to users, reducing latency and improving performance.

CloudFront Features:

  • Edge Locations: 400+ edge locations worldwide for low-latency delivery
  • Caching: Intelligent caching to reduce origin load and improve performance
  • Security: AWS Shield, WAF integration, and field-level encryption
  • Dynamic Content: Lambda@Edge for customizing content at the edge
  • Origins: S3, EC2, ELB, or external HTTP servers
  • Real-time Logs: Detailed logging for monitoring and analytics

Q27: What is ELB?

Elastic Load Balancing (ELB) distributes incoming traffic across multiple targets. It includes ALB (Application), NLB (Network), and CLB (Classic). ELB automatically distributes incoming application traffic across multiple targets, such as EC2 instances, containers, IP addresses, and Lambda functions. It monitors the health of registered targets and routes traffic only to healthy targets.

ELB Types:

  • ALB (Application Load Balancer): Layer 7 (HTTP/HTTPS) for web applications
  • NLB (Network Load Balancer): Layer 4 (TCP/UDP) for high performance
  • CLB (Classic Load Balancer): Legacy load balancer for EC2-Classic
  • GLB (Gateway Load Balancer): Layer 3 for virtual appliances
alb_create.py
import boto3 elbv2 = boto3.client('elbv2') # Create Application Load Balancer response = elbv2.create_load_balancer( Name='my-alb', Subnets=['subnet-12345', 'subnet-67890'], SecurityGroups=['sg-abcdef'], Scheme='internet-facing', Type='application' ) alb_arn = response['LoadBalancers'][0]['LoadBalancerArn'] # Create target group tg = elbv2.create_target_group( Name='my-tg', Protocol='HTTP', Port=80, VpcId='vpc-12345678' )

Q29: What is VPC Peering?

VPC Peering allows two VPCs to communicate as if they are on the same network. It can be within the same region or across regions. VPC Peering is a networking connection between two VPCs that enables you to route traffic between them using private IP addresses. Instances in either VPC can communicate with each other as if they are in the same network.

VPC Peering Features:

  • Same Region: Peering between VPCs in the same AWS Region
  • Cross-Region: Peering between VPCs in different AWS Regions
  • Same Account: Peering between VPCs owned by the same account
  • Different Accounts: Peering between VPCs owned by different AWS accounts
  • No Transitive Peering: Peering connections are non-transitive
  • Routing: Update route tables to route traffic between VPCs
vpc_peering.py
import boto3 ec2 = boto3.client('ec2') # Create VPC peering connection response = ec2.create_vpc_peering_connection( VpcId='vpc-12345678', PeerVpcId='vpc-87654321', PeerRegion='us-west-2' ) peering_id = response['VpcPeeringConnection']['VpcPeeringConnectionId'] # Accept the peering request ec2.accept_vpc_peering_connection( VpcPeeringConnectionId=peering_id )
Q30: What is Direct Connect?

AWS Direct Connect provides dedicated network connections from on-premises to AWS, bypassing the public internet for more consistent network performance. Direct Connect links your internal network to an AWS Direct Connect location over a standard Ethernet fiber-optic cable. This provides a more consistent network experience than internet-based connections, with lower latency, higher throughput, and more consistent network performance.

Direct Connect Features:

  • Dedicated Connection: Private, dedicated network connection to AWS
  • Consistent Performance: Lower latency and higher throughput than internet
  • Secure: Private connection bypassing the public internet
  • Connection Types: 1Gbps, 10Gbps, or 100Gbps dedicated connections
  • Hosted Connections: Partners provide connectivity to AWS Direct Connect locations
  • Direct Connect Gateway: Access multiple VPCs and AWS Regions
direct_connect.py
import boto3 direct_connect = boto3.client('directconnect') # Create Direct Connect connection response = direct_connect.create_connection( location='EqDC2', bandwidth='1Gbps', connectionName='MyDirectConnect', lagId='lag-12345678' # Optional: Link Aggregation Group ) connection_id = response['connectionId'] print(f"Connection ID: {connection_id}")

5. AWS Database Services

Q31: What is RDS?

Amazon RDS (Relational Database Service) is a managed relational database service supporting MySQL, PostgreSQL, Oracle, SQL Server, and Aurora. It provides cost-efficient and resizable capacity while managing time-consuming database administration tasks like provisioning, patching, backup, recovery, failure detection, and repair. RDS offers Multi-AZ deployments for high availability and read replicas for scaling read workloads.

RDS Key Features:

  • Managed Service: Automated backups, patching, and maintenance
  • Multi-AZ: High availability with synchronous replication
  • Read Replicas: Scale read workloads across multiple instances
  • Encryption: Data at rest and in transit encryption
  • Backups: Automated backups and point-in-time recovery
  • Aurora: AWS-optimized database engine with 5x performance
rds_create.py
import boto3 rds = boto3.client('rds') # Create RDS instance response = rds.create_db_instance( DBInstanceIdentifier='mydb-instance', DBInstanceClass='db.t3.micro', Engine='mysql', MasterUsername='admin', MasterUserPassword='password123', AllocatedStorage=20, VpcSecurityGroupIds=['sg-12345678'], MultiAZ=True )
Q32: What is DynamoDB?

Amazon DynamoDB is a fully managed NoSQL database service that provides fast and predictable performance with seamless scalability. It's a key-value and document database that delivers single-digit millisecond performance at any scale. DynamoDB can handle more than 10 trillion requests per day and can support peaks of more than 20 million requests per second.

DynamoDB Key Features:

  • Serverless: No servers to provision, patch, or manage
  • Auto Scaling: Automatically scales read/write capacity
  • Global Tables: Multi-region, multi-active replication
  • Encryption: Data at rest and in transit encryption
  • TTL: Time to Live for automatic data expiration

AWS Config is a service that enables you to assess, audit, and evaluate the configurations of your AWS resources. Config continuously monitors and records your AWS resource configurations, allowing you to automate the evaluation of your recorded configurations against desired configurations. It helps you assess your overall compliance and risk status, and simplifies compliance auditing, security analysis, change management, and operational troubleshooting.

AWS Config Features:

  • Configuration History: Track configuration changes over time
  • Resource Inventory: Discover all AWS resources in your account
  • Compliance Rules: Evaluate resources against predefined or custom rules
  • Change Notifications: Get notified when resources change
  • Multi-Account: Aggregate data across multiple AWS accounts
  • Relationship Mapping: Visualize resource dependencies
Q65: What is AWS Trusted Advisor?

AWS Trusted Advisor provides real-time guidance to help provision resources following AWS best practices for security, performance, and cost optimization. Trusted Advisor inspects your AWS environment and makes recommendations when opportunities exist to save money, improve system availability and performance, or close security gaps. It provides recommendations in five categories: Cost Optimization, Performance, Security, Fault Tolerance, and Service Limits.

Trusted Advisor Features:

  • Cost Optimization: Identify unused resources and cost-saving opportunities
  • Performance: Improve application and infrastructure performance

  • Fault Tolerance: Improve availability and disaster recovery
  • Service Limits: Monitor and manage service quotas
  • Actionable Recommendations: Prioritized recommendations with remediation steps
Q66: What is AWS Organizations?

AWS Organizations helps centrally manage and govern your environment as you grow and scale your AWS resources. It enables you to consolidate multiple AWS accounts into an organization that you create as a single entity for billing and administration purposes. Organizations provides policy-based management that enables you to centrally control accounts, apply policies, and automate account creation.

AWS Organizations Features:

  • Centralized Management: Manage multiple AWS accounts centrally
  • Consolidated Billing: Single bill for all accounts in the organization
  • Service Control Policies (SCPs): Central policy control for all accounts

Q67: What is AWS Budgets?

AWS Budgets enables you to set custom budgets that alert you when your costs or usage exceed your budgeted amount. You can set budgets for your overall costs or specific cost categories, such as EC2 instances or S3 storage. AWS Budgets can track your usage and costs for specific services, and can send alerts via email or SNS notifications when you exceed your budget threshold.

AWS Budgets Features:

  • Cost Budgets: Set spending limits for your AWS account
  • Usage Budgets: Set limits for resource usage (e.g., hours, GB)
  • Alerts: Email or SNS notifications when thresholds are exceeded

  • Granular Control: Set budgets by service, tag, or account

It helps you manage hybrid environments, collect inventory, run shell scripts, and automate patching.

Q70: What is AWS Resource Groups?

AWS Resource Groups helps organize and manage AWS resources. You can group resources based on tags, applications, or criteria. Resource Groups allows you to manage and automate tasks on a group of resources together. You can create groups based on tags, AWS CloudFormation stacks, or other criteria, and then perform operations on all resources in the group at once.

Resource Groups Features:

  • Tag-based Grouping: Group resources by tags or tag values
  • Stack-based Grouping: Group resources by CloudFormation stacks
  • Bulk Operations: Perform actions on multiple resources at once
  • Query Editor: Search and filter resources
  • Automation: Automate tasks on resource groups
  • Integration: Works with Systems Manager and other AWS services
