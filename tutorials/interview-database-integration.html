1. Database Fundamentals

Q1: What is the difference between SQL and NoSQL databases?

SQL databases are relational, table-based databases that use structured query language. NoSQL databases are non-relational, document, key-value, column-family, or graph databases.

Aspect SQL NoSQL
Structure Tables with fixed schema Documents, key-value, graphs
Scalability Vertical scaling Horizontal scaling
Consistency ACID compliant BASE model (eventual)
Examples MySQL, PostgreSQL, SQL Server MongoDB, Cassandra, Redis

Q2: What are ACID properties in databases?

ACID stands for Atomicity, Consistency, Isolation, Durability - the four properties that guarantee database transactions are processed reliably.

  • Atomicity: All operations in a transaction succeed or all fail (no partial updates)
  • Consistency: Database moves from one valid state to another valid state
  • Isolation: Concurrent transactions don't interfere with each other
  • Durability: Once committed, transactions persist even after system failure
acid_transaction.sql
BEGIN TRANSACTION; UPDATE accounts SET balance = balance - 100 WHERE id = 1; UPDATE accounts SET balance = balance + 100 WHERE id = 2; COMMIT;
Q3: What is database normalization and its forms?

Normalization is the process of organizing data to reduce redundancy and improve data integrity by dividing large tables into smaller, related tables.

Normalization Forms:

  • 1NF: Atomic values, no repeating groups
  • 2NF: 1NF + no partial dependencies
  • 3NF: 2NF + no transitive dependencies
  • BCNF: Stricter version of 3NF
normalization.sql
-- After Normalization (3NF) CREATE TABLE customers (id INT PRIMARY KEY, name VARCHAR(255)); CREATE TABLE orders (id INT PRIMARY KEY, customer_id INT); CREATE TABLE order_items (order_id INT, product_id INT);

Q5: What are database indexes and how do they work?

Indexes are data structures that improve query performance by allowing faster data retrieval. They work like a book index, pointing to where data is stored.

Q6: What is a foreign key and referential integrity?

Foreign Key is a field that uniquely identifies a row in another table, establishing a relationship between tables. Referential Integrity ensures that relationships between tables remain consistent.

foreign_key.sql
CREATE TABLE employees ( id INT PRIMARY KEY, name VARCHAR(255), department_id INT, FOREIGN KEY (department_id) REFERENCES departments(id) ON DELETE CASCADE );
Q7: What are database joins and their types?

Joins combine rows from two or more tables based on a related column between them.

joins.sql

Q9: What is a database view and when to use it?

View is a virtual table based on the result-set of an SQL statement. It contains no data itself but stores the query definition.

Stored Procedure is a prepared SQL code that you can save and reuse. It's stored in the database and can accept parameters.

2. SQL Basics

Q11: What is the difference between WHERE and HAVING clauses?

WHERE filters rows before aggregation, while HAVING filters groups after aggregation.

where_having.sql
-- WHERE filters before aggregation SELECT department_id, COUNT(*) AS employee_count FROM employees WHERE salary > 50000 GROUP BY department_id; -- HAVING filters after aggregation SELECT department_id, COUNT(*) AS employee_count FROM employees GROUP BY department_id HAVING COUNT(*) > 5;
Q12: What are UNION and UNION ALL?

UNION combines result sets and removes duplicates. UNION ALL combines result sets but keeps all rows including duplicates.

union.sql
-- UNION removes duplicates SELECT name FROM customers_usa UNION SELECT name FROM customers_europe; -- UNION ALL keeps duplicates (faster) SELECT name FROM customers_usa UNION ALL SELECT name FROM customers_europe;
Q13: What are SQL aggregate functions?

Aggregate functions perform calculations on a set of values and return a single value.

  • COUNT() - Returns the number of rows
  • SUM() - Returns the sum of values
  • AVG() - Returns the average value
  • MIN() - Returns the minimum value
  • MAX() - Returns the maximum value
aggregate_functions.sql
SELECT department_id, COUNT(*) AS employee_count, AVG(salary) AS avg_salary FROM employees GROUP BY department_id;
Q14: What is a subquery and its types?

Subquery is a query nested inside another query. It can be used in SELECT, FROM, WHERE, or HAVING clauses.

subqueries.sql
-- Scalar subquery in WHERE clause SELECT * FROM employees WHERE salary > (SELECT AVG(salary) FROM employees); -- Table subquery in FROM clause SELECT * FROM ( SELECT department_id, AVG(salary) AS avg_sal FROM employees GROUP BY department_id ) AS dept_avg;
Q15: What are SQL window functions?

Window functions perform calculations across a set of table rows related to the current row without collapsing the rows.

  • ROW_NUMBER() - Assigns unique sequential numbers
  • RANK() - Assigns rank with gaps for ties
  • DENSE_RANK() - Assigns rank without gaps
  • LAG() - Access data from previous row
  • LEAD() - Access data from next row
window_functions.sql
SELECT name, salary, ROW_NUMBER() OVER (ORDER BY salary DESC) AS row_num, LAG(salary) OVER (ORDER BY salary) AS prev_salary FROM employees;

DENSE_RANK() Same rank for ties No gaps
Q17: What are Common Table Expressions (CTE)?

CTE is a temporary result set defined within the execution scope of a single SELECT, INSERT, UPDATE, DELETE statement. It improves readability and enables recursive queries.

cte.sql
WITH DepartmentStats AS ( SELECT department_id, COUNT(*) AS emp_count FROM employees GROUP BY department_id ) SELECT d.name, ds.emp_count FROM departments d JOIN DepartmentStats ds ON d.id = ds.department_id;
Q18: What is the difference between EXISTS and IN?

EXISTS checks for existence of rows and stops at first match (faster for large datasets). IN checks all values in the list/subquery.

exists_in.sql
-- EXISTS (stops at first match) SELECT * FROM customers c WHERE EXISTS ( SELECT 1 FROM orders o WHERE o.customer_id = c.id );
Q19: What is the COALESCE function?

COALESCE returns the first non-null value in a list. It's useful for providing default values when data might be NULL.

coalesce.sql
SELECT name, COALESCE(phone, mobile, 'N/A') AS contact_number FROM products;
Q20: What are CASE statements in SQL?

CASE provides conditional logic in SQL queries, similar to IF-ELSE statements in programming languages.

case_statement.sql
SELECT name, score, CASE WHEN score >= 90 THEN 'A' WHEN score >= 80 THEN 'B' ELSE 'F' END AS grade FROM students;

Q23: What is the GROUP BY clause?

GROUP BY arranges identical data into groups and is typically used with aggregate functions to perform calculations on each group.

group_by.sql
SELECT department_id, COUNT(*) AS count, SUM(salary) AS total_salary FROM employees GROUP BY department_id;
Q24: What is the ORDER BY clause?

ORDER BY sorts the result set in ascending or descending order based on one or more columns.

order_by.sql
-- Sort by single column SELECT * FROM employees ORDER BY salary DESC; -- Sort by multiple columns SELECT * FROM employees ORDER BY department_id ASC, salary DESC;
Q25: What is the LIMIT clause?

LIMIT constrains the number of rows returned by a query. It's useful for pagination and limiting result sets.

limit.sql
-- Get first 10 rows SELECT * FROM employees LIMIT 10; -- Pagination (skip 20, get next 10) SELECT * FROM employees LIMIT 10 OFFSET 20;

3. ORM and Data Access

Q26: What is an ORM and its benefits?

ORM (Object-Relational Mapping) is a technique that converts data between incompatible systems using object-oriented programming languages. It maps database tables to classes and rows to objects.

Benefits:

  • Eliminates need for raw SQL queries
  • Provides type safety and compile-time checking
  • Automates CRUD operations
  • Handles database relationships easily
  • Database-agnostic code (switch databases easily)
UserModel.cs
// Entity Framework ORM Example using System.ComponentModel.DataAnnotations; using System.ComponentModel.DataAnnotations.Schema; public class User { [Key] public int Id { get; set; } [MaxLength(255)] public string Name { get; set; } [MaxLength(255)] [Index(IsUnique = true)] public string Email { get; set; } }
Q27: What is the difference between ORM and raw SQL?
Aspect ORM Raw SQL
Performance Slower due to abstraction Faster, optimized queries
Development Faster development Slower, more code
Maintenance Easier, type-safe Harder, error-prone
Complex Queries Can be difficult Full control
Q28: What is lazy loading vs eager loading in ORM?

Lazy loading loads related data only when accessed. Eager loading loads related data immediately with the main query.

LoadingStrategies.cs
// Lazy loading (default) - N+1 query problem var users = context.Users.ToList(); // Separate query for each user's orders foreach (var user in users) { var orders = user.Orders.ToList(); } // Eager loading - single query var users = context.Users.Include(u => u.Orders).ToList();
Q29: What is the N+1 query problem?

N+1 query problem occurs when you execute 1 query to fetch N records, then N additional queries to fetch related data for each record. This causes severe performance issues.

NPlusOneProblem.cs
// Problem: N+1 queries var users = context.Users.ToList(); // 1 query foreach (var user in users) { var orders = user.Orders.ToList(); // N queries } // Solution: Eager loading var users = context.Users.Include(u => u.Orders).ToList(); // 1 query
Q30: What is database migration?

Database migration is the process of managing and applying incremental changes to database schema over time, allowing version control for database structure.

Migration.cs
// Entity Framework migration example public partial class AddEmailColumn : Migration { protected override void Up(MigrationBuilder migrationBuilder) { migrationBuilder.AddColumn<string>( name: "Email", table: "Users", type: "nvarchar(255)", nullable: false); migrationBuilder.AddIndex( name: "IX_Users_Email", table: "Users", column: "Email", unique: true); } protected override void Down(MigrationBuilder migrationBuilder) { migrationBuilder.DropIndex(name: "IX_Users_Email", table: "Users"); migrationBuilder.DropColumn(name: "Email", table: "Users"); } }
Q31: What is the difference between active record and data mapper patterns?

Active Record couples domain objects with database operations (objects save themselves). Data Mapper separates domain objects from persistence logic (mapper handles persistence).

Patterns.cs
// Active Record (object saves itself) var user = new User { Name = "John", Email = "john@example.com" }; context.Users.Add(user); context.SaveChanges(); // Object handles persistence // Data Mapper (separate mapper) var user = new User { Name = "John", Email = "john@example.com" }; userRepository.Save(user); // Mapper handles persistence
Q32: What is connection pooling?

Connection pooling maintains a cache of database connections to reduce the overhead of creating new connections for each request.

ConnectionPool.cs
// Entity Framework connection pool var optionsBuilder = new DbContextOptionsBuilder<AppDbContext>(); optionsBuilder.UseSqlServer("Server=localhost;Database=myDb;User Id=user;Password=pass;", options => { options.EnableRetryOnFailure(maxRetryCount: 3); options.CommandTimeout(30); });
Q33: What is the repository pattern?

Repository pattern mediates between the domain and data mapping layers, acting like an in-memory domain object collection. It abstracts data access logic.

Repository.cs
public class UserRepository : IUserRepository { private readonly AppDbContext _context; public UserRepository(AppDbContext context) { _context = context; } public User FindById(int userId) { return _context.Users.FirstOrDefault(u => u.Id == userId); } public void Save(User user) { _context.Users.Add(user); _context.SaveChanges(); } }
Q34: What is the Unit of Work pattern?

Unit of Work maintains a list of objects affected by a business transaction and coordinates writing out changes as a single transaction.

UnitOfWork.cs
public class UnitOfWork : IDisposable { private readonly AppDbContext _context; private readonly IUserRepository _userRepository; public UnitOfWork(AppDbContext context) { _context = context; _userRepository = new UserRepository(context); } public IUserRepository Users => _userRepository; public void Commit() { _context.SaveChanges(); } public void Dispose() { _context.Dispose(); } } // Usage using (var uow = new UnitOfWork(context)) { uow.Users.Add(newUser); uow.Commit(); }
Q35: What is the difference between SQL and NoSQL in terms of schema?

SQL databases use rigid, predefined schemas that require migrations for changes. NoSQL databases have flexible or schema-less designs, allowing dynamic data structures.

schema_comparison.json
// NoSQL (MongoDB) - flexible schema { "_id": "123", "name": "John", "email": "john@example.com", "tags": ["developer", "csharp"] } // Another document can have different fields { "_id": "124", "name": "Jane", "age": 30 }

4. Connection Management

Q36: What is database connection string?

Connection string contains all information needed to establish a connection to a database, including host, port, database name, credentials, and connection parameters.

ConnectionStrings.cs
// PostgreSQL connection string string postgresUrl = "Host=localhost;Port=5432;Database=mydb;Username=user;Password=password"; // MySQL connection string string mysqlUrl = "Server=localhost;Port=3306;Database=mydb;User Id=user;Password=password"; // MongoDB connection string string mongoUrl = "mongodb://localhost:27017/mydb";
Q37: What is the difference between persistent and non-persistent connections?

Persistent connections stay open after a request completes and are reused for subsequent requests. Non-persistent connections close after each request and must be reopened.

ConnectionTypes.cs
// Persistent connection (recommended) var optionsBuilder = new DbContextOptionsBuilder<AppDbContext>(); optionsBuilder.UseSqlServer(connectionString); // Non-persistent connection (not recommended) using (var connection = new SqlConnection(connectionString)) { connection.Open(); }
Q38: What is connection timeout?

Connection timeout is the maximum time a client waits for a connection to be established before giving up. It prevents indefinite waiting when the database is unresponsive.

Timeout.cs
// Entity Framework with timeout var optionsBuilder = new DbContextOptionsBuilder<AppDbContext>(); optionsBuilder.UseSqlServer(connectionString, options => { options.CommandTimeout(30); options.EnableRetryOnFailure(maxRetryCount: 3); });
Q39: What is connection leak?

Connection leak occurs when database connections are opened but never closed, eventually exhausting the connection pool and causing application failures.

ConnectionLeak.cs
// Bad: Connection leak public void BadQuery() { var connection = new SqlConnection(connectionString); connection.Open(); var command = connection.CreateCommand(); command.CommandText = "SELECT * FROM users"; command.ExecuteReader(); // Connection never closed! } // Good: Properly closed with using public void GoodQuery() { using (var connection = new SqlConnection(connectionString)) { connection.Open(); using (var command = connection.CreateCommand()) { command.CommandText = "SELECT * FROM users"; command.ExecuteReader(); } } }
Q40: What is connection health check?

Connection health check verifies that a database connection is still valid before using it, detecting stale connections that may have been closed by the server.

HealthCheck.cs
// Entity Framework connection health check var optionsBuilder = new DbContextOptionsBuilder<AppDbContext>(); optionsBuilder.UseSqlServer(connectionString, options => { options.EnableRetryOnFailure(maxRetryCount: 3); // Test connections before use }); // Manual health check public bool CheckConnection(SqlConnection connection) { try { if (connection.State != ConnectionState.Open) connection.Open(); return true; } catch { return false; } }

5. Transaction Management

Q41: What is a database transaction?

Transaction is a sequence of database operations performed as a single logical unit of work. All operations must succeed or all must fail (atomicity).

Transaction.cs
public void TransferMoney(int fromId, int toId, decimal amount) { using (var transaction = _context.Database.BeginTransaction()) { try { var fromAccount = _context.Accounts.FirstOrDefault(a => a.Id == fromId); var toAccount = _context.Accounts.FirstOrDefault(a => a.Id == toId); fromAccount.Balance -= amount; toAccount.Balance += amount; _context.SaveChanges(); transaction.Commit(); } catch { transaction.Rollback(); throw; } } }
Q42: What are transaction isolation levels?

Isolation levels define how transactions interact with each other, controlling phenomena like dirty reads, non-repeatable reads, and phantom reads.

Level Dirty Reads Non-Repeatable Reads Phantom Reads
Read Uncommitted Yes Yes Yes
Read Committed No Yes Yes
Repeatable Read No No Yes
Serializable No No No
isolation.sql
-- Set isolation level SET TRANSACTION ISOLATION LEVEL READ COMMITTED; SET TRANSACTION ISOLATION LEVEL REPEATABLE READ; SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;
Q43: What is a dirty read?

Dirty read occurs when a transaction reads data that has been modified but not yet committed by another transaction. If the other transaction rolls back, the read data is invalid.

dirty_read.sql
-- Transaction 1 BEGIN; UPDATE accounts SET balance = 1000 WHERE id = 1; // Not committed -- Transaction 2 (dirty read) BEGIN; SELECT balance FROM accounts WHERE id = 1; // Reads uncommitted 1000
Q44: What is a non-repeatable read?

Non-repeatable read occurs when a transaction reads the same row twice and gets different values because another transaction modified and committed the data between reads.

Q45: What is a phantom read?

Phantom read occurs when a transaction executes the same query twice and gets different sets of rows because another transaction inserted or deleted rows that match the query criteria.

Q46: What is optimistic concurrency control?

Optimistic concurrency assumes conflicts are rare. It allows concurrent reads but checks for conflicts before committing, using version numbers or timestamps.

Optimistic.cs
public class Product { public int Id { get; set; } public string Name { get; set; } public int Stock { get; set; } [Timestamp] public byte[] Version { get; set; } } // Update with version check var result = context.Products.Where(p => p.Id == 1 && p.Version == currentVersion); if (result.Any()) { var product = result.First(); product.Stock = newStock; context.SaveChanges(); }
Q47: What is pessimistic concurrency control?

Pessimistic concurrency assumes conflicts are likely. It locks resources when reading to prevent other transactions from modifying them until the lock is released.

pessimistic.sql
-- Pessimistic locking with SELECT FOR UPDATE BEGIN; SELECT * FROM products WHERE id = 1 FOR UPDATE; -- Other transactions must wait UPDATE products SET stock = stock - 1 WHERE id = 1; COMMIT;
Q48: What is a distributed transaction?

Distributed transaction involves multiple database resources or nodes, requiring coordination across all participants using protocols like 2PC (Two-Phase Commit).

Distributed.cs
using (var scope = new TransactionScope(TransactionScopeOption.Required, new TransactionOptions { IsolationLevel = IsolationLevel.Serializable })) { try { // Update database 1 using (var context1 = new AppDbContext1()) { var account = context1.Accounts.FirstOrDefault(a => a.Id == accountId); account.Balance = newBalance; context1.SaveChanges(); } // Update database 2 using (var context2 = new AppDbContext2()) { var order = context2.Orders.FirstOrDefault(o => o.Id == orderId); order.Status = "completed"; context2.SaveChanges(); } scope.Complete(); } catch { // Transaction automatically rolls back } }
Q49: What is savepoint in transactions?

Savepoint is a marker within a transaction that allows partial rollback. You can rollback to a specific savepoint without undoing all previous work in the transaction.

savepoint.sql
BEGIN; UPDATE accounts SET balance = balance - 100 WHERE id = 1; SAVEPOINT sp1; UPDATE accounts SET balance = balance + 50 WHERE id = 2; // Something went wrong ROLLBACK TO sp1; // Only rollback second update COMMIT;
Q50: What is dead lock?

Deadlock occurs when two or more transactions wait indefinitely for each other to release locks, preventing any of them from proceeding.

deadlock.sql
-- Transaction 1 BEGIN; SELECT * FROM accounts WHERE id = 1 FOR UPDATE; -- Waits for Transaction 2 to release id=2 SELECT * FROM accounts WHERE id = 2 FOR UPDATE; -- Transaction 2 BEGIN; SELECT * FROM accounts WHERE id = 2 FOR UPDATE; -- Waits for Transaction 1 to release id=1 SELECT * FROM accounts WHERE id = 1 FOR UPDATE;
Q51: How to prevent deadlocks?

Deadlock Prevention Strategies:

  • Access resources in consistent order - Always lock tables in the same sequence
  • Keep transactions short - Minimize the time locks are held
  • Use appropriate isolation levels - Lower isolation levels reduce locking
  • Add timeout - Set lock timeout to abort waiting transactions
  • Retry logic - Catch deadlocks and retry the transaction
PreventDeadlock.cs
// Always access in same order public void SafeTransfer(int fromId, int toId, decimal amount) { // Ensure consistent ordering var minId = Math.Min(fromId, toId); var maxId = Math.Max(fromId, toId); // Lock in order using (var transaction = _context.Database.BeginTransaction()) { var account1 = _context.Accounts.FirstOrDefault(a => a.Id == minId); var account2 = _context.Accounts.FirstOrDefault(a => a.Id == maxId); // Perform transfer transaction.Commit(); } }
Q52: What is transaction rollback?

Rollback undoes all changes made during a transaction, returning the database to its state before the transaction began. It's used when an error occurs or the transaction cannot complete.

Rollback.cs
try { using (var transaction = _context.Database.BeginTransaction()) { var account = _context.Accounts.FirstOrDefault(a => a.Id == accountId); account.Balance -= amount; if (account.Balance < 0) throw new InvalidOperationException("Insufficient funds"); _context.SaveChanges(); transaction.Commit(); } } catch { // Transaction automatically rolls back throw; }
Q53: What is nested transaction?

Nested transaction is a transaction within another transaction. Changes in the nested transaction can be committed or rolled back independently, but are only durable if the outer transaction commits.

Nested.cs
using (var scope = new TransactionScope()) { // Outer transaction var account1 = _context.Accounts.FirstOrDefault(a => a.Id == 1); account1.Balance -= 100; // Nested transaction using TransactionScope using (var nestedScope = new TransactionScope(TransactionScopeOption.Suppress)) { // Nested transaction - can rollback independently var account2 = _context.Accounts.FirstOrDefault(a => a.Id == 2); account2.Balance += 100; _context.SaveChanges(); nestedScope.Complete(); } _context.SaveChanges(); scope.Complete(); }
Q54: What is autocommit mode?

Autocommit mode automatically commits each individual SQL statement as a separate transaction. No explicit BEGIN/COMMIT is needed.

Autocommit.cs
// Entity Framework with auto-commit (default behavior) using (var context = new AppDbContext()) { var user = new User { Name = "John" }; context.Users.Add(user); context.SaveChanges(); // Auto-committed }
Q55: What is the difference between commit and flush?

Flush sends SQL statements to the database but doesn't commit the transaction. Commit permanently saves changes and ends the transaction.

FlushVsCommit.cs
// ChangeTracker tracks changes but doesn't send to DB using (var context = new AppDbContext()) { var user = new User { Name = "John" }; context.Users.Add(user); // Changes tracked but not sent to DB yet } // SaveChanges - sends SQL and commits using (var context = new AppDbContext()) { var user = new User { Name = "John" }; context.Users.Add(user); context.SaveChanges(); // INSERT sent and committed }
Q56: What is two-phase commit (2PC)?

Two-Phase Commit is a protocol for ensuring all participants in a distributed transaction either commit or rollback together.

Phases:

  • Prepare Phase: Coordinator asks all participants if they can commit
  • Commit Phase: If all vote yes, coordinator sends commit command; otherwise sends rollback
Q57: What is transaction scope?

Transaction scope defines the boundaries of a transaction - when it begins and ends. Proper scoping ensures transactions are neither too long (causing locks) nor too short (losing atomicity).

Q58: What is a long-running transaction?

Long-running transaction stays open for an extended period, holding locks and preventing other transactions from accessing resources. It should be avoided as it can cause deadlocks and performance issues.

Anti-pattern: Waiting for user input during a transaction. Instead, use optimistic concurrency with version checks.

Q59: What is transaction logging?

Transaction log (write-ahead log) records all database changes before they're written to data files. It ensures durability and enables recovery after failures.

Q60: What is the difference between implicit and explicit transactions?

Implicit transactions are automatically started and committed by the database (single-statement transactions). Explicit transactions are manually controlled with BEGIN/COMMIT/ROLLBACK statements.

transactions.sql
-- Implicit transaction (autocommit) UPDATE users SET name = 'John' WHERE id = 1; // Auto-committed -- Explicit transaction BEGIN TRANSACTION; UPDATE users SET name = 'John' WHERE id = 1; UPDATE users SET email = 'john@example.com' WHERE id = 1; COMMIT;

6. Performance Optimization

Q61: What is query optimization?

Query optimization is the process of improving database query performance by analyzing and modifying queries, indexes, and database schema to reduce execution time and resource usage.

query_optimization.sql
-- Before: Full table scan SELECT * FROM orders WHERE order_date > '2024-01-01'; -- After: Use index on order_date CREATE INDEX idx_order_date ON orders(order_date); SELECT * FROM orders WHERE order_date > '2024-01-01';
Q62: What is EXPLAIN and how is it used?

EXPLAIN shows the execution plan for a query, revealing how the database will retrieve data, which indexes will be used, and the estimated cost.

explain.sql
-- Analyze query execution plan EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'test@example.com'; -- Simple explain EXPLAIN SELECT * FROM orders WHERE customer_id = 123;
Q63: What is database partitioning?

Partitioning divides large tables into smaller, more manageable pieces while keeping them logically as a single table. It improves query performance on large datasets.

partitioning.sql
-- Range partitioning by date CREATE TABLE orders ( id INT, order_date DATE, customer_id INT ) PARTITION BY RANGE (order_date); CREATE TABLE orders_2024 PARTITION OF orders FOR VALUES FROM ('2024-01-01') TO ('2025-01-01');
Q64: What is database sharding?

Sharding distributes data across multiple database instances (shards) based on a shard key. It enables horizontal scaling by spreading load across multiple servers.

Sharding.cs
public int GetShard(int userId, int numShards) { // Consistent hashing for shard selection return userId % numShards; } // Route to appropriate shard int shardId = GetShard(userId, 4); var connection = shardConnections[shardId];
Q65: What is denormalization?

Denormalization intentionally introduces redundancy into a database to improve read performance by reducing the number of joins required for common queries.

denormalization.sql
-- Normalized (requires join) SELECT o.total, c.name FROM orders o JOIN customers c ON o.customer_id = c.id; -- Denormalized (no join needed) CREATE TABLE orders_denorm ( id INT, total DECIMAL, customer_name VARCHAR(255) -- Redundant );
Q66: What is query caching?

Query caching stores the results of frequently executed queries in memory, returning cached results instead of re-executing the query when the same request is made.

QueryCache.cs
public class QueryCache { private readonly Dictionary<string, object> _cache = new(); public object Get(string query) { if (_cache.ContainsKey(query)) return _cache[query]; var result = ExecuteQuery(query); _cache[query] = result; return result; } private object ExecuteQuery(string query) { // Execute query logic return null; } }
Q67: What is materialized view?

Materialized view stores the result of a query physically, unlike regular views which store only the query definition. It's refreshed periodically to maintain data currency.

materialized_view.sql
-- Create materialized view CREATE MATERIALIZED VIEW order_summary AS SELECT customer_id, COUNT(*) AS order_count, SUM(total) AS total_spent FROM orders GROUP BY customer_id; -- Refresh materialized view REFRESH MATERIALIZED VIEW order_summary;
Q68: What is database indexing best practices?

Indexing Best Practices:

  • Index columns used in WHERE, JOIN, ORDER BY
  • Avoid over-indexing - indexes slow down INSERT/UPDATE/DELETE
  • Use composite indexes for multi-column queries
  • Consider selective indexes - high cardinality columns benefit more
  • Monitor index usage - remove unused indexes

7. Security Best Practices

Q69: What is SQL injection and how to prevent it?

SQL injection is a vulnerability where malicious SQL code is inserted into input fields, allowing attackers to manipulate database queries.

SqlInjection.cs
// Bad: Vulnerable to SQL injection string query = $"SELECT * FROM users WHERE name = '{userInput}'"; // SqlCommand executes this directly - DANGEROUS! // Good: Parameterized query (prevents injection) string query = "SELECT * FROM users WHERE name = @Name"; using (var command = new SqlCommand(query, connection)) { command.Parameters.AddWithValue("@Name", userInput); command.ExecuteReader(); }
Q70: What is parameterized query?

Parameterized query uses placeholders for user input, separating SQL code from data. The database treats parameters as data, not executable code, preventing SQL injection.

Parameterized.cs

Q71: What is database encryption?

Database encryption protects data at rest (stored data) and in transit (data being transmitted). It uses encryption algorithms to make data unreadable without decryption keys.

Type Description Example
At Rest Encrypts stored data files TDE, disk encryption
In Transit Encrypts network traffic TLS/SSL, SSH
Column-Level Encrypts specific columns PGP, AES encryption
Q72: What is the principle of least privilege?

Principle of least privilege grants users and applications only the minimum permissions necessary to perform their tasks, reducing the impact of compromised accounts.

least_privilege.sql
-- Create user with limited permissions CREATE USER app_user WITH PASSWORD 'secure_password'; -- Grant only necessary permissions GRANT SELECT, INSERT ON orders TO app_user; GRANT SELECT ON customers TO app_user; # Revoke unnecessary permissions REVOKE ALL ON sensitive_data FROM app_user;
Q73: What is database auditing?

Database auditing tracks and logs database activities for security compliance, troubleshooting, and forensic analysis. It records who accessed what data and when.

auditing.sql
-- PostgreSQL audit trigger CREATE TABLE audit_log ( id SERIAL PRIMARY KEY, table_name VARCHAR(255), action VARCHAR(10), user_name VARCHAR(255), timestamp TIMESTAMP DEFAULT NOW() ); CREATE TRIGGER audit_trigger AFTER INSERT OR UPDATE OR DELETE ON users FOR EACH ROW EXECUTE FUNCTION audit_function();
Q74: What is row-level security?

Row-level security (RLS) restricts which rows a user can access based on policies defined at the database level, providing fine-grained access control.

rls.sql
-- Enable row-level security ALTER TABLE orders ENABLE ROW LEVEL SECURITY; -- Create policy: users only see their own orders CREATE POLICY user_orders_policy ON orders FOR SELECT USING (customer_id = current_user_id());
Q75: What is data masking?

Data masking obscures sensitive data while maintaining its format, allowing developers to work with realistic data without exposing actual sensitive information.

data_masking.sql
-- Mask sensitive columns in view CREATE VIEW users_masked AS SELECT id, name, SUBSTRING(email, 1, 3) || '***@' || SUBSTRING(email, POSITION('@' IN email) + 1) AS email, '***-**-****' AS phone FROM users;

8. Database Design Patterns

Q76: What is the star schema?

Star schema is a database design used in data warehousing with a central fact table surrounded by dimension tables, optimized for query performance in analytics.

star_schema.sql
-- Fact table (center) CREATE TABLE sales ( date_id INT, product_id INT, store_id INT, amount DECIMAL(10,2) ); -- Dimension tables (surrounding) CREATE TABLE dim_date (id INT, date DATE, quarter VARCHAR); CREATE TABLE dim_product (id INT, name VARCHAR, category VARCHAR); CREATE TABLE dim_store (id INT, location VARCHAR, region VARCHAR);
Q77: What is the snowflake schema?

Snowflake schema is an extension of star schema where dimension tables are normalized into multiple related tables, reducing redundancy but increasing query complexity.

Q78: What is the master-detail pattern?

Master-detail pattern represents a one-to-many relationship where a master record has multiple detail records, commonly used for orders and line items, invoices and invoice lines.

master_detail.sql
-- Master table CREATE TABLE invoices ( id INT PRIMARY KEY, customer_id INT, invoice_date DATE, total DECIMAL(10,2) ); -- Detail table CREATE TABLE invoice_lines ( id INT PRIMARY KEY, invoice_id INT, product_id INT, quantity INT, price DECIMAL(10,2), FOREIGN KEY (invoice_id) REFERENCES invoices(id) );
Q79: What is the soft delete pattern?

Soft delete marks records as deleted without actually removing them, using a flag or timestamp. This allows data recovery and maintains referential integrity.

soft_delete.sql
-- Add soft delete columns ALTER TABLE users ADD COLUMN deleted BOOLEAN DEFAULT FALSE, ADD COLUMN deleted_at TIMESTAMP; -- Soft delete instead of hard delete UPDATE users SET deleted = TRUE, deleted_at = NOW() WHERE id = 123; -- Query excluding deleted records SELECT * FROM users WHERE deleted = FALSE;
Q80: What is the audit trail pattern?

Audit trail pattern maintains a history of all changes to data, recording who made the change, when, and what the old and new values were.

audit_trail.sql
-- Audit table CREATE TABLE users_audit ( id SERIAL PRIMARY KEY, user_id INT, action VARCHAR(10), old_values JSONB, new_values JSONB, changed_by VARCHAR(255), changed_at TIMESTAMP DEFAULT NOW() );

Q82: What are the main types of NoSQL databases?
