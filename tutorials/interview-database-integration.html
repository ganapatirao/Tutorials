1. Database Fundamentals

Q1: What is database integration?

Database Integration Definition: The process of connecting an application to a database to store, retrieve, and manipulate data efficiently and securely.

Key Components:

  • Connection management (connection strings, pooling)
  • Query execution (SQL, NoSQL queries)
  • Data mapping (ORM, data transfer objects)
  • Transaction management
  • Error handling and retry logic
Q2: What are the types of databases?
Type Description Examples
Relational (SQL) Structured data with tables, rows, columns MySQL, PostgreSQL, SQL Server
Document (NoSQL) JSON-like documents MongoDB, CouchDB
Key-Value Simple key-value pairs Redis, DynamoDB
Column-Family Wide-column storage Cassandra, HBase
Graph Nodes and relationships Neo4j, ArangoDB
Q3: What is a connection string?

Connection String Definition: A string that specifies database connection parameters including server, database, credentials, and options.

// SQL Server Connection String
"Server=myServerAddress;Database=myDataBase;User Id=myUsername;Password=myPassword;"

// PostgreSQL Connection String
"Host=localhost;Port=5432;Database=mydb;Username=myuser;Password=mypassword;"

// MongoDB Connection String
"mongodb://username:password@localhost:27017/mydatabase"
Q4: What is connection pooling?

Connection Pooling Definition: Reusing database connections instead of creating new ones for each request.

Benefits:

  • Improved performance (no connection overhead)
  • Reduced resource usage
  • Better scalability
  • Faster response times

Configuration: Pool size, timeout, max connections, idle timeout

Q5: What is the difference between SQL and NoSQL?
Aspect SQL NoSQL
Data Model Structured, relational Flexible, schema-less
Schema Fixed schema Dynamic schema
Scaling Vertical scaling Horizontal scaling
Query Language SQL Various (JSON, etc.)
Use Case Complex queries, transactions Big data, rapid development

2. SQL Integration

Q6: How do you integrate with SQL databases?

Integration Methods:

  • Native drivers (ADO.NET, JDBC, psycopg2)
  • ORM frameworks (Entity Framework, Hibernate, SQLAlchemy)
  • Micro-ORMs (Dapper, MyBatis)
  • Query builders (Knex.js, jOOQ)
// Python - Using SQLAlchemy ORM
from sqlalchemy import create_engine, Column, Integer, String
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker

engine = create_engine('postgresql://user:password@localhost/db')
Base = declarative_base()

class User(Base):
    __tablename__ = 'users'
    id = Column(Integer, primary_key=True)
    name = Column(String)
    email = Column(String)

Session = sessionmaker(bind=engine)
session = Session()

# Query
users = session.query(User).filter(User.name == 'John').all()
Q7: What are database transactions?

Transaction Definition: A sequence of operations performed as a single logical unit of work.

ACID Properties:

  • Atomicity: All or nothing execution
  • Consistency: Database remains consistent
  • Isolation: Transactions don't interfere
  • Durability: Committed changes persist
// Transaction Example
try:
    connection.begin()
    cursor.execute("UPDATE accounts SET balance = balance - 100 WHERE id = 1")
    cursor.execute("UPDATE accounts SET balance = balance + 100 WHERE id = 2")
    connection.commit()
except Exception as e:
    connection.rollback()
    raise e
Q8: What is SQL injection and how to prevent it?

SQL Injection Definition: Attack where malicious SQL is injected into queries.

Prevention:

  • Use parameterized queries
  • Use ORM frameworks
  • Input validation and sanitization
  • Least privilege database access
// Vulnerable
query = f"SELECT * FROM users WHERE name = '{user_input}'"

// Safe - Parameterized
query = "SELECT * FROM users WHERE name = %s"
cursor.execute(query, (user_input,))
Q9: What are database indexes?

Index Definition: Data structures that improve query speed by allowing fast data lookup.

Types:

  • B-Tree indexes (default for most databases)
  • Hash indexes (equality comparisons)
  • Full-text indexes (text search)
  • Composite indexes (multiple columns)

Tradeoffs: Faster reads, slower writes, increased storage

Q10: What is database normalization?

Normalization Definition: Process of organizing data to reduce redundancy and improve integrity.

Normal Forms:

  • 1NF: Atomic values, no repeating groups
  • 2NF: 1NF + no partial dependencies
  • 3NF: 2NF + no transitive dependencies
  • BCNF: Stricter version of 3NF

3. NoSQL Integration

Q11: How do you integrate with MongoDB? 
// Python - Using PyMongo
from pymongo import MongoClient

client = MongoClient('mongodb://localhost:27017/')
db = client['mydatabase']
collection = db['users']

# Insert
user = {'name': 'John', 'email': 'john@example.com'}
collection.insert_one(user)

# Query
users = collection.find({'name': 'John'})

# Update
collection.update_one({'name': 'John'}, {'': {'email': 'new@email.com'}})

# Delete
collection.delete_one({'name': 'John'})
Q12: How do you integrate with Redis? 
// Python - Using redis-py
import redis

r = redis.Redis(host='localhost', port=6379, db=0)

# Set
r.set('key', 'value')
r.setex('session:123', 3600, 'user_data')  # Expires in 1 hour

# Get
value = r.get('key')

# Hash operations
r.hset('user:1', 'name', 'John')
r.hset('user:1', 'email', 'john@example.com')
user_data = r.hgetall('user:1')

# List operations
r.lpush('queue', 'task1')
r.lpush('queue', 'task2')
task = r.rpop('queue')
Q13: What is the difference between document and key-value databases?
Aspect Document Key-Value
Data Structure Complex nested documents Simple key-value pairs
Querying Rich query capabilities Key-based only
Schema Flexible schema No schema
Use Case Complex data, content management Caching, sessions, simple lookups
Q14: How do you handle relationships in NoSQL?

Relationship Strategies:

  • Embedding: Store related data in same document (denormalization)
  • Referencing: Store references/IDs to other documents
  • Hybrid: Combination of both approaches

Tradeoffs: Embedding = faster reads, harder updates; Referencing = slower reads, easier updates

Q15: What is eventual consistency in NoSQL?

Eventual Consistency Definition: System guarantees that if no new updates are made, eventually all accesses will return the last updated value.

Key Points:

  • Accepts temporary inconsistency
  • Converges to consistency over time
  • Common in distributed NoSQL databases
  • Tradeoff: consistency vs availability (CAP theorem)

4. ORM and Data Access

Q16: What is an ORM?

ORM Definition: Object-Relational Mapping - technique to convert data between incompatible systems using object-oriented programming languages.

Benefits:

  • Eliminates most SQL code
  • Type safety and compile-time checking
  • Easier to maintain and refactor
  • Database independence

Drawbacks: Performance overhead, complex queries can be difficult

Q17: What are popular ORM frameworks?
Language ORM Frameworks
Python SQLAlchemy, Django ORM, Peewee
Java Hibernate, JPA, MyBatis
C# Entity Framework, Dapper, NHibernate
Node.js Sequelize, TypeORM, Mongoose
PHP Eloquent, Doctrine
Q18: What is the difference between ORM and micro-ORM?
Aspect ORM Micro-ORM
Abstraction High-level, hides SQL Low-level, exposes SQL
Performance Slower (overhead) Faster (less overhead)
Control Less control More control
Learning Curve Steeper Easier
Q19: What is lazy loading in ORM?

Lazy Loading Definition: Loading related data only when it's accessed, not when the parent object is loaded.

Benefits:

  • Reduced initial query time
  • Load only needed data
  • Better memory usage

Drawbacks: N+1 query problem, potential performance issues

Q20: What is the N+1 query problem?

N+1 Problem Definition: When fetching N parent objects, the ORM executes N additional queries to fetch related data.

Solution:

  • Eager loading (fetch related data upfront)
  • Batch loading (fetch in batches)
  • Join queries
// N+1 Problem
users = session.query(User).all()  # 1 query
for user in users:
    print(user.orders)  # N queries

// Solution - Eager Loading
users = session.query(User).options(joinedload(User.orders)).all()

5. Cloud Databases

Q21: What are cloud databases?

Cloud Database Definition: Database services running on cloud platforms, offering managed database solutions.

Benefits:

  • No server management
  • Automatic scaling
  • High availability and backups
  • Pay-as-you-go pricing
Q22: What are popular cloud database services?
Provider Services
AWS RDS, DynamoDB, Aurora, ElastiCache, Redshift
Azure SQL Database, Cosmos DB, Cache for Redis
Google Cloud Cloud SQL, Firestore, Bigtable, Memorystore
Oracle Autonomous Database, NoSQL Database Service
Q23: How do you connect to AWS RDS? 
// Python - Connecting to AWS RDS PostgreSQL
import psycopg2
from psycopg2 import pool

# Connection string with SSL
connection_string = "postgresql://user:password@rds-endpoint.amazonaws.com:5432/dbname?sslmode=require"

# Connection pool
connection_pool = psycopg2.pool.SimpleConnectionPool(
    minconn=1,
    maxconn=10,
    dsn=connection_string
)

# Get connection
connection = connection_pool.getconn()
cursor = connection.cursor()
cursor.execute("SELECT * FROM users")
results = cursor.fetchall()
connection_pool.putconn(connection)
Q24: How do you integrate with Azure Cosmos DB? 
// Python - Using Azure Cosmos DB SDK
from azure.cosmos import CosmosClient

# Initialize client
client = CosmosClient(
    "https://account.documents.azure.com:443/",
    credential="your-master-key"
)

# Get database and container
database = client.get_database_client("mydatabase")
container = database.get_container_client("users")

# Insert item
user = {"id": "1", "name": "John", "email": "john@example.com"}
container.create_item(body=user)

// Query
users = container.query_items(
    query="SELECT * FROM users WHERE name = @name",
    parameters=[{"name": "name", "value": "John"}]
)
Q25: What is serverless database?

Serverless Database Definition: Database that automatically scales compute and storage based on demand, with pay-per-use pricing.

Examples:

  • Aurora Serverless (AWS)
  • Cosmos DB Serverless (Azure)
  • Cloud SQL Serverless (Google Cloud)
  • Neon Serverless PostgreSQL

Benefits: No capacity planning, auto-scaling, cost-effective for sporadic workloads

Q26: What is database as a service (DBaaS)?

DBaaS Definition: Cloud service where the provider manages database infrastructure, including provisioning, scaling, backups, and maintenance.

Features:

  • Automated backups and point-in-time recovery
  • High availability with multi-AZ deployments
  • Read replicas for scaling reads
  • Automated patching and upgrades
  • Monitoring and alerting
Q27: How do you handle database migrations in cloud?

Migration Tools:

  • AWS Database Migration Service (DMS)
  • Azure Database Migration Service
  • Google Cloud Data Migration Service
  • Schema conversion tools

Process: Assessment, schema conversion, data migration, validation, cutover

Q28: What is read replica in cloud databases?

Read Replica Definition: Copy of the primary database that serves read traffic, improving performance and scalability.

Benefits:

  • Offload read traffic from primary
  • Geographic distribution
  • Reporting and analytics without affecting primary
  • Disaster recovery (promote to primary if needed)
Q29: What is multi-region database deployment?

Multi-Region Definition: Deploying database across multiple geographic regions for performance, compliance, and disaster recovery.

Strategies:

  • Active-Active: Multiple primary regions, read-write
  • Active-Passive: One primary, replicas in other regions
  • Global Tables: Automatic replication across regions
Q30: How do you secure cloud databases?

Security Measures:

  • VPC/private network isolation
  • Encryption at rest and in transit
  • IAM roles and policies
  • Network security groups/firewalls
  • Auditing and logging
  • SSL/TLS connections

6. Performance & Optimization

Q31: How do you optimize database performance?

Optimization Techniques:

  • Proper indexing strategy
  • Query optimization (avoid SELECT *)
  • Connection pooling
  • Caching (Redis, Memcached)
  • Read replicas for scaling reads
  • Partitioning and sharding
Q32: What is database sharding?

Sharding Definition: Horizontal partitioning of data across multiple database instances.

Sharding Strategies:

  • Range-based: Data range determines shard
  • Hash-based: Hash of key determines shard
  • Geographic: Location determines shard

Benefits: Horizontal scaling, improved performance, better fault isolation

Q33: What is database partitioning?

Partitioning Definition: Dividing large tables into smaller, more manageable pieces.

Types:

  • Range partitioning: Based on value ranges
  • List partitioning: Based on value lists
  • Hash partitioning: Based on hash function

Benefits: Faster queries, easier maintenance, better performance

Q34: What is query optimization?

Query Optimization Techniques:

  • Use EXPLAIN/EXPLAIN ANALYZE to analyze queries
  • Avoid SELECT *, select only needed columns
  • Use appropriate indexes
  • Avoid functions on indexed columns in WHERE clause
  • Use LIMIT/OFFSET for pagination
  • Optimize JOINs
Q35: What is database caching?

Caching Definition: Storing frequently accessed data in fast storage to reduce database load.

Caching Strategies:

  • Read-through: Cache miss loads from DB, populates cache
  • Write-through: Write to cache and DB synchronously
  • Write-back: Write to cache, async write to DB
  • Cache-aside: Application manages cache

7. Security & Best Practices

Q36: How do you secure database connections?

Security Best Practices:

  • Use SSL/TLS for encrypted connections
  • Store credentials securely (environment variables, secret managers)
  • Use least privilege principle
  • Rotate credentials regularly
  • Use connection strings with authentication tokens
Q37: What is database encryption?

Encryption Types:

  • At rest: Encrypt stored data (TDE, file system encryption)
  • In transit: Encrypt data during transfer (SSL/TLS)
  • Application-level: Encrypt sensitive data before storage

Key Management: Use cloud KMS services (AWS KMS, Azure Key Vault, Google KMS)

Q38: What is database auditing?

Auditing Definition: Tracking and logging database activities for security, compliance, and troubleshooting.

Audit Events:

  • Login/logout attempts
  • Query execution
  • Data modifications
  • Schema changes
  • Privilege changes
Q39: What is database backup and recovery?

Backup Strategies:

  • Full backup: Complete database backup
  • Incremental backup: Changes since last backup
  • Differential backup: Changes since last full backup

Recovery: Point-in-time recovery, backup retention policies, disaster recovery planning

Q40: What are database integration best practices?

Best Practices:

  • Use connection pooling
  • Implement retry logic for transient failures
  • Use parameterized queries to prevent SQL injection
  • Implement proper error handling
  • Use transactions for data consistency
  • Monitor database performance
  • Implement caching strategies
  • Use environment-specific configurations
  • Implement database migrations
  • Document database schema and relationships
