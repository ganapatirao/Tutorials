1. Docker Fundamentals

Q1: What is Docker and why is it used?

Docker is an open-source platform for developing, shipping, and running applications in containers. It enables application isolation, portability, and consistency across environments.

Key Benefits:

  • Consistency: Same environment across development, testing, and production
  • Isolation: Applications run in isolated containers
  • Portability: Containers run anywhere (Linux, Windows, Cloud)
  • Efficiency: Lightweight compared to VMs
  • Speed: Fast startup and deployment
  • Scalability: Easy to scale horizontally
Q2: What is a container?

Container is a lightweight, standalone package that includes everything needed to run an application: code, runtime, system tools, libraries, and settings.

Key Characteristics:

  • Shares host OS kernel
  • Isolated from other containers
  • Portable across environments
  • Lightweight and fast
  • Immutable infrastructure

2. Docker Architecture

Q4: What are the key components of Docker architecture?

Docker Components:

  • Docker Daemon (dockerd): Background service that manages Docker objects (images, containers, networks, volumes)
  • Docker Client (docker): CLI tool that communicates with Docker Daemon
  • Docker Registry: Stores and distributes Docker images (Docker Hub, private registries)
  • Docker Objects: Images, containers, networks, volumes, services
Q5: What is the difference between Docker Daemon and Docker Client?
  • Docker Daemon: Background service that manages Docker objects. Listens for Docker API requests. Performs container operations.
  • Docker Client: CLI tool that sends commands to Docker Daemon. Provides user interface. Communicates via REST API.

3. Docker Images and Containers

Q6: What is a Docker image?

Docker Image is a read-only template for creating containers. It contains the application code, libraries, dependencies, and runtime.

Key Characteristics:

  • Read-only
  • Layered file system
  • Immutable
  • Can be versioned
  • Stored in registries
Q7: What is the difference between Docker image and container?
  • Docker Image: Read-only template. Blueprint for containers. Can be shared and stored.
  • Docker Container: Running instance of an image. Writable layer. Isolated runtime environment.
Q8: What are Docker image layers?

Docker Image Layers are read-only filesystems stacked on top of each other to form the final image.

Benefits:

  • Efficiency: Layers are cached and reused
  • Speed: Only changed layers need to be rebuilt
  • Size: Shared layers reduce storage
  • Versioning: Each layer has a unique ID

4. Dockerfile and Building Images

Q9: What is a Dockerfile?

Dockerfile is a text document that contains all the commands to assemble a Docker image. It's like a recipe for building images.

`r`n# Example Dockerfile FROM node:18-alpine WORKDIR /app COPY package*.json ./ RUN npm install COPY . . EXPOSE 3000 CMD ["npm", "start"]
Q10: What are the common Dockerfile instructions?
Instruction Purpose
FROM Base image to build from
RUN Execute commands during build
CMD Default command to run on container start
ENTRYPOINT Executable that runs on container start
COPY Copy files from host to image
ADD Copy files and URLs, auto-extract
ENV Set environment variables
EXPOSE Document exposed ports
VOLUME Create mount point for volumes
WORKDIR Set working directory
USER Set user for RUN, CMD, ENTRYPOINT
Q11: What is the difference between CMD and ENTRYPOINT?
  • CMD: Default command that can be overridden. Provides default arguments. Can be replaced at runtime.
  • ENTRYPOINT: Main executable that cannot be overridden. Provides entry point. Can only be overridden with --entrypoint flag.

5. Docker Compose

Q12: What is Docker Compose?

Docker Compose is a tool for defining and running multi-container Docker applications. It uses YAML files to configure application services.

`r`n# docker-compose.yml version: '3.8' services: web: build: . ports: - "3000:3000" db: image: postgres:13 environment: POSTGRES_PASSWORD: example
Q13: What are the common Docker Compose commands?
  • docker-compose up: Create and start containers
  • docker-compose down: Stop and remove containers
  • docker-compose build: Build or rebuild services
  • docker-compose ps: List containers
  • docker-compose logs: View output from containers
  • docker-compose exec: Execute command in running container

6. Docker Networking

Q14: What are Docker network types?
  • Bridge: Default network for containers. Isolated on host. Containers can communicate via IP.
  • Host: No network isolation. Container uses host network. Good for high performance.
  • None: No network access. Completely isolated. Good for security.
  • Overlay: Multi-host networking. Connects containers across multiple hosts. Used in Swarm.
  • Macvlan: Assign MAC address to container. Direct access to physical network.
Q15: What is the difference between bridge and overlay networks?
Aspect Bridge Overlay
Scope Single host Multi-host
Use Case Local development Swarm, multi-host
Performance Better Slightly lower

7. Docker Storage and Volumes

Q16: What are Docker storage drivers?

Docker Storage Drivers manage how containers store and access data on the host filesystem.

Common Drivers:

  • overlay2: Default for Linux. Best performance.
  • btrfs: Btrfs filesystem support.
  • zfs: ZFS filesystem support.
  • vfs: For testing, no copy-on-write.
Q17: What is a Docker volume?

Docker Volume is a persistent data storage mechanism that exists independently of containers. It allows data to survive container deletion.

Volume Types:

  • Named volumes: Managed by Docker. Created with docker volume create.
  • Bind mounts: Map host directory to container. Managed by user.
  • tmpfs mounts: Stored in host memory. Non-persistent.

8. Docker Registry

Q19: What is a Docker registry?

Docker Registry is a storage and distribution system for Docker images. It stores and serves Docker images.

Types:

  • Docker Hub: Public registry by Docker. Free and private repositories.
  • Private Registry: Self-hosted registry. For security and compliance.
  • Third-party registries: AWS ECR, Azure Container Registry, Google GCR.
Q20: What are the common Docker registry commands?
  • docker login: Authenticate with registry
  • docker pull: Pull image from registry
  • docker push: Push image to registry
  • docker search: Search for images in Docker Hub
  • docker tag: Tag image for registry

9. Docker Security

Q21: What are Docker security best practices?
  • Use official images: Prefer official images from Docker Hub
  • Scan images: Use tools like Trivy, Clair to scan for vulnerabilities
  • Use non-root user: Run containers as non-root user
  • Minimize image size: Use alpine images, multi-stage builds
  • Use secrets management: Don't hardcode secrets in images
  • Enable Content Trust: Verify image signatures
  • Use Docker Bench: Run Docker security benchmark
  • Limit capabilities: Drop unnecessary Linux capabilities
  • Use read-only filesystem: Make container filesystem read-only
  • Network isolation: Use proper network segmentation
Q22: What is Docker Content Trust?

Docker Content Trust (DCT) enables digital signing of Docker images. It ensures the integrity and authenticity of images pulled from or pushed to registries.

Key Features:

  • Image signing
  • Signature verification
  • Prevents image tampering
  • Ensures image authenticity

Docker Swarm is Docker's native clustering and orchestration tool. It turns a pool of Docker hosts into a single virtual host.

11. Docker Performance Optimization

Q25: How do you optimize Docker image size?
  • Use alpine images: Smaller base images
  • Multi-stage builds: Separate build and runtime environments
  • Minimize layers: Combine RUN commands
  • Use .dockerignore: Exclude unnecessary files
  • Clean up package cache: Remove cache after installation
  • Use specific versions: Avoid latest tag
  • Remove unnecessary files: Delete temp files, documentation
Q26: What is .dockerignore?

.dockerignore is a file that specifies patterns to exclude when building a Docker image. Similar to .gitignore.

`r`n# .dockerignore example node_modules npm-debug.log .git .env .DS_Store *.md

12. Docker Multi-Stage Builds

Q27: What are multi-stage builds?

Multi-stage builds allow you to use multiple FROM statements in a Dockerfile. Each FROM starts a new build stage, and you can copy artifacts from one stage to another.

`r`n# Multi-stage build example # Build stage FROM node:18-alpine AS builder WORKDIR /app COPY package*.json ./ RUN npm install COPY . . RUN npm run build # Runtime stage FROM node:18-alpine WORKDIR /app COPY --from=builder /app/dist ./dist COPY package*.json ./ RUN npm install --production EXPOSE 3000 CMD ["npm", "start"]
Q28: What are the benefits of multi-stage builds?
  • Smaller final image: Only runtime dependencies included
  • Better security: Build tools not in final image
  • Faster deployment: Smaller images download faster
  • Clean separation: Build and runtime environments separate

14. Common Interview Questions

Q31: What is the difference between docker run and docker start?
  • docker run: Creates a new container and starts it. Combines create and start. Used for first time.
  • docker start: Starts an existing stopped container. Used for restarting containers.
Q32: What is the difference between docker stop and docker kill?
  • docker stop: Gracefully stops container. Sends SIGTERM, then SIGTERM after timeout. Allows cleanup.
  • docker kill: Immediately stops container. Sends SIGKILL. No cleanup.
Q33: What is the difference between docker pause and docker stop?
  • docker pause: Suspends all processes in container. Keeps container running but frozen. Can be resumed.
  • docker stop: Stops container completely. Processes terminated. Cannot be resumed, only restarted.
Q34: What is the difference between docker exec and docker attach?
  • docker exec: Run command in running container. Starts new process. Good for debugging.
  • docker attach: Attach to container's main process. View container output. Good for monitoring.
Q35: What is the difference between docker ps and docker inspect?
  • docker ps: List running containers. Shows basic information. Good for overview.
  • docker inspect: Detailed information about container. Shows configuration, state. Good for debugging.
Q36: What is the difference between docker logs and docker events?
  • docker logs: View container logs. Shows stdout/stderr. Good for application debugging.
  • docker events: View Docker daemon events. Shows container lifecycle events. Good for monitoring.
Q37: What is the difference between docker commit and docker build?
  • docker commit: Create image from container. Captures container state. Not reproducible.
  • docker build: Build image from Dockerfile. Reproducible builds. Best practice.
Q38: What is the difference between docker export and docker save?
  • docker export: Export container filesystem to tar. Creates flat image. Loses history and metadata.
  • docker save: Save image to tar. Preserves layers and metadata. Can be loaded back.
Q39: What is the difference between docker import and docker load?
  • docker import: Import tar as image. Creates new image from export. No layers.
  • docker load: Load image from tar. Restores saved image with layers.
Q40: What is the difference between docker rmi and docker rm?
  • docker rmi: Remove Docker image. Deletes image from local storage. Must stop containers first.
  • docker rm: Remove Docker container. Deletes container. Must stop container first.
Q41: What is the difference between COPY and ADD in Dockerfile?
  • COPY: Copy files from host to image. Simple and transparent. Recommended for most cases.
  • ADD: Copy files and URLs. Auto-extracts archives. Can download from URLs. More complex.
Q42: What is the difference between ENTRYPOINT and CMD in Dockerfile?
  • ENTRYPOINT: Executable that runs on container start. Cannot be overridden easily. Defines container's main process.
  • CMD: Default arguments for ENTRYPOINT. Can be overridden. Provides default command.
Q43: What is the difference between ENV and ARG in Dockerfile?
  • ENV: Environment variable available at runtime. Persists in container. Can be overridden at runtime.
  • ARG: Build-time variable. Only available during build. Not available at runtime.
Q44: What is the difference between ONBUILD and STOPSIGNAL in Dockerfile?
  • ONBUILD: Trigger instruction when image used as base. Good for base images. Deferred execution.
  • STOPSIGNAL: Signal to send to container on stop. Default is SIGTERM. Can be overridden.
Q45: What is the difference between HEALTHCHECK and SHELL in Dockerfile?
  • HEALTHCHECK: Define health check command. Monitors container health. Can trigger restarts.
  • SHELL: Override default shell. Changes shell used for RUN, CMD, ENTRYPOINT.
Q46: What are the common Docker troubleshooting scenarios?
  • Container won't start: Check image, logs, resource limits
  • Container exits immediately: Check entrypoint, command, logs
  • Network issues: Check network configuration, DNS, firewall
  • Volume mounting issues: Check permissions, paths, SELinux
  • Image pull failures: Check registry, authentication, network
  • Resource exhaustion: Check CPU, memory, disk space
  • Permission denied: Check user permissions, SELinux
  • Port conflicts: Check port bindings, host ports
Q47: What is Docker Bench Security?

Docker Bench Security is a script that checks for common security best practices in Docker deployments. It's based on the CIS Docker Benchmark.

Checks:

  • Host configuration
  • Docker daemon configuration
  • Docker daemon configuration files
  • Container images and build files
  • Container runtime
  • Docker security operations
Q48: What is Docker Scout?

Docker Scout is a tool for analyzing and securing Docker images. It provides vulnerability scanning, image analysis, and best practice recommendations.

Features:

  • Vulnerability scanning
  • Image analysis
  • Base image recommendations
  • License compliance
  • CI/CD integration
Q49: What is Docker BuildKit?

Docker BuildKit is the next-generation builder for Docker. It provides improved performance, better caching, and more features.

Features:

  • Parallel build execution
  • Better caching
  • Secret management
  • SSH agent forwarding
  • Custom build outputs
Q50: What are the common Docker monitoring tools?
  • cAdvisor: Container metrics collection
  • Prometheus: Metrics storage and querying
  • Grafana: Visualization and dashboards
  • ELK Stack: Log aggregation and analysis
  • Docker Stats: Built-in container stats
  • Portainer: Docker management UI
