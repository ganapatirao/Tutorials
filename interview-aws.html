1. AWS Fundamentals

Q1: What is AWS and what are its key features?

Amazon Web Services (AWS) is a comprehensive cloud platform offering over 200 fully featured services from data centers globally.

Key Features:

  • Global infrastructure: Regions, Availability Zones, Edge Locations
  • Pay-as-you-go: Pay only for what you use
  • Scalability: Auto-scaling capabilities
  • Security: Shared responsibility model
  • Reliability: 99.99% SLA for many services
  • Flexibility: Wide range of services
  • Ecosystem: Large partner network
Q2: What are the different AWS service models?

IaaS Infrastructure as a Service EC2, EBS, VPC
PaaS Platform as a Service Elastic Beanstalk, Lambda
SaaS Software as a Service WorkSpaces, Chime
FaaS Function as a Service Lambda, EventBridge

3. AWS Storage

Service Description Use Case

EBS Block storage EC2 volumes, databases

Q6: What are the different S3 storage classes?
  • Standard: Frequently accessed data
  • Intelligent-Tiering: Automatic cost optimization
  • Standard-IA: Infrequently accessed data
  • One Zone-IA: Infrequently accessed, single AZ
  • Glacier: Long-term archival (minutes retrieval)
  • Glacier Deep Archive: Long-term archival (hours retrieval)
Important: Choose the appropriate storage class based on access patterns and cost requirements. S3 lifecycle policies can automatically move data between classes.

5. AWS Database Services

Q9: What are the main AWS database services?
  • RDS: Relational Database Service (SQL Server, MySQL, PostgreSQL, Oracle, MariaDB, Aurora)
  • DynamoDB: NoSQL key-value and document database
  • ElastiCache: In-memory cache (Redis, Memcached)
  • Neptune: Graph database
  • DocumentDB: MongoDB-compatible document database

  • Redshift: Data warehousing

Q10: What is DynamoDB?

DynamoDB is a fully managed NoSQL database service that provides fast and predictable performance with seamless scalability.

Key Features:

  • Key-value and document: Flexible data model
  • Single-digit millisecond latency: Fast performance
  • Automatic scaling: Handles traffic spikes
  • Global tables: Multi-region replication
  • DAX: In-memory acceleration
  • TTL: Automatic data expiration
  • Streams: Change data capture

6. AWS Security

Q11: What are the key AWS security services?
  • IAM (Identity and Access Management): Access control
  • KMS (Key Management Service): Encryption keys
  • Secrets Manager: Secrets management
  • Shield: DDoS protection
  • WAF (Web Application Firewall): Web application protection
  • GuardDuty: Threat detection
  • Security Hub: Security posture management
  • Macie: Data classification
Q12: What is the Shared Responsibility Model?

The Shared Responsibility Model defines security responsibilities between AWS and the customer.

AWS Responsibilities:

  • Physical security of data centers
  • Network infrastructure
  • Virtualization layer
  • Region and AZ security

Customer Responsibilities:

  • Data encryption
  • Operating system security
  • Network configuration (VPC, security groups)
  • Identity and access management (IAM)
  • Application security

7. AWS Serverless

Q13: What is AWS Lambda?

AWS Lambda is a serverless compute service that runs code in response to events and automatically manages the underlying compute resources.

Trigger Types:

  • S3: Object events
  • DynamoDB Streams: Database changes
  • Kinesis: Streaming data
  • SNS: Pub/sub messaging
  • SQS: Queue messages
  • API Gateway: HTTP endpoints
  • CloudWatch Events: Scheduled events

Q14: What is API Gateway?

AWS API Gateway is a fully managed service that makes it easy to create, publish, maintain, monitor, and secure APIs.

Features:

  • REST APIs: RESTful API endpoints
  • HTTP APIs: Low-latency REST APIs
  • WebSocket APIs: Real-time communication
  • Throttling: Request throttling
  • Authorization: IAM, Cognito, Lambda authorizers
  • Caching: API response caching
  • Stage variables: Environment-specific configuration

10. Common Interview Questions

Q19: What is the difference between S3 and EBS?
  • S3: Object storage, unlimited size, accessible from anywhere, pay per GB/month. Best for static assets, backups.
  • EBS: Block storage, attached to EC2, limited size per volume, pay per GB/month + IOPS. Best for OS, databases.
Q20: What is the difference between public and private subnets?
  • Public subnet: Has route to Internet Gateway. Resources can access internet directly. Used for web servers, load balancers.
  • Private subnet: No route to Internet Gateway. Resources access internet via NAT Gateway. Used for databases, application servers.

Q25: What is the difference between CloudFront and ELB?
  • CloudFront: Global CDN that caches content at edge locations. Reduces latency for static content.
  • ELB: Load balancer that distributes traffic across multiple targets in a region. Provides health checks and scaling.
Q26: What is the difference between IAM role and IAM user?
  • IAM User: Represents a person or application. Has long-term credentials (access keys, password). Used for programmatic access or console access.
  • IAM Role: Temporary credentials. Used by AWS services or applications to assume permissions. No long-term credentials.
Q27: What is the difference between ECS and EKS?
Aspect ECS EKS
Orchestrator AWS proprietary Kubernetes
Portability AWS only Portable
Ecosystem AWS ecosystem Kubernetes ecosystem
Learning Curve Lower Higher
Q28: What is the difference between Kinesis and SQS?
  • Kinesis: Streaming data service. Real-time processing. Ordered data. Good for analytics, real-time processing.
  • SQS: Message queue service. Decoupling. No order guarantee. Good for buffering, task queues.
Q29: What is the difference between CloudTrail and CloudWatch?
  • CloudTrail: Logs API calls for auditing and compliance. Who did what and when. Focus on security and compliance.
  • CloudWatch: Monitoring and observability. Metrics, logs, alarms. Focus on operational health and performance.
Q30: What is the difference between Elastic IP and Public IP?
  • Public IP: Dynamic IP assigned to instance. Changes when instance is stopped/started. Free while instance is running.
  • Elastic IP: Static IP that you own. Can be remapped to different instances. Charged when not attached to running instance.

Q33: What is AWS Well-Architected Framework?

AWS Well-Architected Framework provides best practices for designing and operating reliable, secure, efficient, and cost-effective systems.

Six Pillars:

  • Operational Excellence: Operations, monitoring, automation
  • Security: Identity, data protection, infrastructure protection
  • Reliability: Recovery, scalability, distributed systems
  • Performance Efficiency: Selection, review, monitoring, trade-offs
  • Cost Optimization: Cost-awareness, optimization, resource management
  • Sustainability: Reduce environmental impact
Q34: What is AWS Step Functions?

AWS Step Functions is a serverless orchestration service that lets you coordinate distributed applications using visual workflows.

Key Features:

  • Visual workflows: State machine definition
  • Integration: 200+ AWS service integrations
  • Error handling: Built-in retry and catch
  • Parallel execution: Parallel and dynamic parallel
  • Serverless: No infrastructure to manage

11. AWS Cost Management (Expert Level)

Q35: How do you optimize AWS costs?
  • Right-sizing: Choose appropriate instance types
  • Reserved Instances: Commit to 1-3 years for discounts
  • Savings Plans: Flexible pricing model for compute
  • Spot Instances: Use unused capacity at up to 90% discount
  • Auto-scaling: Scale down during off-peak hours
  • S3 lifecycle policies: Move data to cheaper storage tiers
  • EBS snapshots: Delete old snapshots
  • Cost allocation tags: Track costs by department/project
Q36: What is AWS Cost Explorer?

AWS Cost Explorer is a tool for visualizing, understanding, and managing your AWS costs and usage.

Features:

  • Cost analysis: View costs by service, region, tag
  • Forecasting: Predict future spending
  • Anomalies: Detect unusual spending patterns
  • Recommendations: Get cost optimization recommendations
  • Reports: Generate custom reports
Q37: What is AWS Budgets?

AWS Budgets allows you to set custom budgets that alert you when your costs or usage exceed your thresholds.

Features:

  • Cost budgets: Set spending limits
  • Usage budgets: Set usage limits
  • Alerts: Email and SNS notifications
  • Actions: Automate responses to budget breaches
  • Forecasting: Include forecasted costs

12. AWS Hybrid Solutions (Expert Level)

Q38: What is AWS Direct Connect?

AWS Direct Connect is a dedicated network connection between your on-premises network and AWS.

Benefits:

  • Consistent performance: Lower latency and jitter
  • Higher bandwidth: Up to 100 Gbps
  • Private connection: Doesn't traverse public internet
  • Cost savings: Reduced data transfer costs
  • Security: Private, encrypted connection
Q39: What is AWS Storage Gateway?

AWS Storage Gateway is a hybrid cloud storage service that connects on-premises applications with cloud storage.

Types:

  • File Gateway: NFS/SMB file share to S3
  • Volume Gateway: iSCSI block storage to S3/EBS
  • Tape Gateway: Virtual tape library to S3/Glacier
Q40: What is AWS Outposts?

AWS Outposts is a fully managed service that extends AWS infrastructure, services, APIs, and tools to customer premises.

Key Features:

  • Consistent AWS experience: Same APIs and services
  • Low latency: On-premises for latency-sensitive workloads
  • Data residency: Keep data on-premises
  • Hybrid cloud: Seamless integration with AWS
  • Managed service: AWS manages hardware and software

AWS Organizations is a service for consolidating multiple AWS accounts into an organization that you create and centrally manage.

  • Organizational Units (OUs): Group accounts
  • Service Control Policies (SCPs): Control access across accounts
  • Account management: Create and manage accounts
  • Tag policies: Enforce consistent tagging
Q43: What is AWS CloudFormation?

AWS CloudFormation is a service for infrastructure as code (IaC) that lets you model and provision AWS resources.

Key Concepts:

  • Templates: JSON or YAML resource definitions
  • Stacks: Collection of resources
  • Change sets: Preview changes before applying
  • Drift detection: Detect configuration drift
  • Nested stacks: Reusable template components

Q45: What is AWS CodePipeline?

AWS CodePipeline is a fully managed continuous integration and continuous delivery (CI/CD) service.

Stages:

  • Source: CodeCommit, GitHub, Bitbucket
  • Build: CodeBuild, Jenkins
  • Test: CodeBuild, third-party tools
  • Deploy: CodeDeploy, ECS, Lambda, S3
  • Approval: Manual approval stages
Q46: What is AWS ECR?

Amazon Elastic Container Registry (ECR) is a fully managed Docker container registry.

Features:

  • Private registry: Secure image storage
  • Image scanning: Vulnerability scanning
  • Lifecycle policies: Auto-delete old images
  • Cross-region replication: Replicate images across regions
  • Integration: ECS, EKS, CodeBuild
Q47: What is AWS Global Accelerator?

AWS Global Accelerator is a service that improves the availability and performance of your applications with static IP addresses.

Features:

  • Static IP addresses: Fixed IPs for your applications
  • Global network: AWS global network backbone
  • Health checks: Automatic failover
  • Traffic steering: Route traffic to closest endpoint
  • DNS independence: No DNS changes needed
Q48: What is AWS WAF?

AWS WAF (Web Application Firewall) protects web applications from common web exploits.

Features:

  • Rules: Define custom rules for traffic filtering
  • Managed rules: Pre-configured rule groups
  • Bot control: Block malicious bots
  • Rate limiting: Limit request rates
  • Integration: CloudFront, ALB, API Gateway
Q49: What is AWS Shield?

AWS Shield is a managed Distributed Denial of Service (DDoS) protection service.

Tiers:

  • Shield Standard: Automatic protection at no extra cost
  • Shield Advanced: Advanced protection with DDoS Response Team (DRT)
Q50: What are the best practices for AWS security?
  • Use IAM roles: Avoid long-term credentials
  • Enable MFA: Multi-factor authentication for root and IAM users
  • Least privilege: Grant minimum required permissions
  • Encrypt data: Use KMS for encryption at rest and in transit
  • Network security: Use security groups and NACLs
  • Enable CloudTrail: Log all API calls
  • Use GuardDuty: Threat detection
  • Security Hub: Centralized security management
  • Patch management: Keep systems updated
  • Backup: Implement backup and disaster recovery
Interview Tip: When discussing AWS, emphasize understanding of the shared responsibility model, service selection based on use cases, cost optimization strategies, and security best practices. Be prepared to discuss real-world architectures and design decisions. For senior positions, demonstrate knowledge of hybrid solutions, cost management strategies, and advanced architecture patterns.
Key Takeaways for Interview (Beginner to Expert):
  • Beginner (0-3 years): Understand AWS service models (IaaS, PaaS, SaaS), know compute services (EC2, Lambda), understand storage options (S3, EBS), know networking basics (VPC, subnets, security groups), understand database services (RDS, DynamoDB), know IAM basics.
  • Intermediate (4-7 years): Understand CloudFormation and IaC, know DevOps services (CodePipeline, CodeBuild), implement monitoring with CloudWatch, use KMS for encryption, understand VPC peering and VPN, implement Lambda triggers and API Gateway, know S3 lifecycle policies.
  • Advanced (8-10 years): Design serverless architectures with Lambda and EventBridge, implement ECS/EKS for container orchestration, use Step Functions for workflow orchestration, implement AWS Security Hub and GuardDuty, optimize costs with Reserved Instances and Savings Plans, implement hybrid solutions with Direct Connect and Storage Gateway.
  • Expert (11-13 years): Design multi-region architectures for disaster recovery, implement complex hybrid solutions with Outposts, design cost optimization strategies at scale, implement Zero Trust architecture with AWS services, design enterprise-grade landing zones with Organizations and Control Tower, implement advanced networking with Transit Gateway, design cloud migration strategies.
