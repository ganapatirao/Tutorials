1. Entity Framework Fundamentals

Q1: What is Entity Framework and what are its benefits?

Entity Framework (EF) is an Object-Relational Mapper (ORM) that enables .NET developers to work with relational data using domain-specific objects. It eliminates the need for most of the data-access code that developers usually need to write.

Benefits:

  • Productivity: Reduces data access code
  • Maintainability: Less code to maintain
  • Type Safety: Compile-time checking
  • Abstraction: Database agnostic
  • Change Tracking: Automatic change detection
  • LINQ Support: Query using LINQ
  • Migrations: Database schema management
Q2: What is the difference between EF6 and EF Core?
Aspect EF6 EF Core
Platform .NET Framework only Cross-platform (.NET Core/.NET 5+)
Performance Good Significantly faster
Features More features Subset of features, growing
Database Support SQL Server, Oracle, etc. SQL Server, PostgreSQL, MySQL, SQLite, etc.
Migrations Yes Yes, improved
Lazy Loading Default Opt-in

2. DbContext and DbSet

Q3: What is DbContext?

DbContext is the primary class that coordinates the Entity Framework functionality for a given data model. It:

  • Manages entity objects during runtime
  • Handles database connections
  • Tracks changes to entities
  • Saves changes to the database
  • Executes LINQ queries against the database
public class AppDbContext : DbContext { public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { } public DbSet<Product> Products { get; set; } public DbSet<Category> Categories { get; set; } protected override void OnModelCreating(ModelBuilder modelBuilder) { // Fluent API configuration } }
Q4: What is DbSet?

DbSet<TEntity> represents a collection of entities of a specific type that can be queried from the database. It:

  • Represents a table in the database
  • Enables CRUD operations
  • Supports LINQ queries
  • Tracks changes to entities
// CRUD operations // Create context.Products.Add(new Product { Name = "Laptop" }); await context.SaveChangesAsync(); // Read var products = await context.Products.ToListAsync(); var product = await context.Products.FindAsync(1); // Update product.Name = "Updated Laptop"; await context.SaveChangesAsync(); // Delete context.Products.Remove(product); await context.SaveChangesAsync();

4. Migrations

Q7: What are Entity Framework migrations?

Migrations enable you to apply incremental changes to the database schema to keep it in sync with your EF Core model.

Migration Commands:

// Add migration dotnet ef migrations add InitialCreate // Update database dotnet ef database update // Remove last migration dotnet ef migrations remove // List migrations dotnet ef migrations list // Generate SQL script dotnet ef migrations script

Migration Steps:

  1. Define or modify your entity classes
  2. Add a migration to capture the changes
  3. Update the database to apply the migration
Q8: How do you handle migrations in production?
// Generate SQL script for production dotnet ef migrations script 2023-01-01-InitialCreate --output migration.sql // Apply migrations programmatically public static void ApplyMigrations(IApplicationBuilder app) { using var scope = app.ApplicationServices.CreateScope(); var context = scope.ServiceProvider.GetRequiredService<AppDbContext>(); context.Database.Migrate(); } // In Program.cs var app = builder.Build(); ApplyMigrations(app);
Important: Never use automatic migrations in production. Always generate SQL scripts and review them before applying.

5. Change Tracking

Q9: How does Entity Framework track changes?

EF Core tracks changes to entities using a change tracker. When you query entities, EF Core:

  • Takes a snapshot of the entity's original values
  • Monitors the entity for changes
  • Detects changes when SaveChanges is called
  • Generates appropriate SQL UPDATE/INSERT/DELETE statements
// Change tracking states var entry = context.Entry(product); Console.WriteLine(entry.State); // Added, Modified, Deleted, Unchanged, Detached // Detect changes manually context.ChangeTracker.DetectChanges(); // Query change tracker var modifiedEntities = context.ChangeTracker.Entries() .Where(e => e.State == EntityState.Modified) .ToList();
Q10: What are the different entity states?
State Description
Added Entity is new and will be inserted
Unchanged Entity hasn't been modified
Modified Entity has been modified
Deleted Entity is marked for deletion
Detached Entity is not being tracked

6. LINQ to Entities

Q11: How does LINQ work with Entity Framework?

LINQ queries in EF Core are translated to SQL queries. The query is executed when:

  • You iterate over the results (foreach)
  • You call a conversion operator (ToList, ToArray, etc.)
  • You call an aggregate operator (Count, Sum, etc.)
// Query is translated to SQL var products = context.Products .Where(p => p.Price > 100) .OrderBy(p => p.Name) .ToList(); // Translated SQL: // SELECT * FROM Products // WHERE Price > 100 // ORDER BY Name
Important: Not all LINQ operators are supported in LINQ to Entities. Unsupported operators will throw NotSupportedException.
Q12: What is the difference between eager loading, lazy loading, and explicit loading?
Loading Type Description When to Use
Eager Loading Load related entities with the main query When you know you need the data
Lazy Loading Load related entities when accessed When you might not need the data
Explicit Loading Load related entities on demand When you need fine-grained control
// Eager loading with Include var products = await context.Products .Include(p => p.Category) .Include(p => p.Reviews) .ToListAsync(); // Lazy loading (requires virtual navigation properties) public class Product { public virtual Category Category { get; set; } } // Explicit loading var product = await context.Products.FindAsync(1); await context.Entry(product) .Reference(p => p.Category) .LoadAsync();

7. Performance Optimization

Q13: How do you optimize Entity Framework performance?
  • Use AsNoTracking: For read-only queries
  • Select only needed columns: Avoid fetching unnecessary data
  • Use eager loading wisely: Avoid over-including
  • Use pagination: For large datasets
  • Use compiled queries: For frequently executed queries
  • Batch updates: Reduce round-trips
  • Use connection pooling: Optimize connection settings
  • Average query complexity: Keep queries simple
// AsNoTracking for read-only var products = await context.Products .AsNoTracking() .ToListAsync(); // Select specific columns var names = await context.Products .Select(p => p.Name) .ToListAsync(); // Pagination var page = await context.Products .Skip((page - 1) * pageSize) .Take(pageSize) .ToListAsync(); // Compiled query private static readonly Func<AppDbContext, int, Task<Product>> GetProductById = EF.CompileAsyncQuery((AppDbContext db, int id) => db.Products.FirstOrDefault(p => p.Id == id)); var product = await GetProductById(context, 1);
Q14: What is the N+1 query problem?

The N+1 query problem occurs when you execute N additional queries to fetch related data for each of N entities returned by the main query.

// Problematic - N+1 queries var categories = context.Categories.ToList(); foreach (var category in categories) { var products = context.Products .Where(p => p.CategoryId == category.Id) .ToList(); // N additional queries } // Solution - eager loading var categories = context.Categories .Include(c => c.Products) .ToList(); // Single query with JOIN

8. Advanced Features

Q15: What are global query filters?

Global query filters apply a filter to all queries for a given entity type. They're useful for implementing soft delete, multi-tenancy, and data isolation.

// Configure global query filter protected override void OnModelCreating(ModelBuilder modelBuilder) { modelBuilder.Entity<Product>() .HasQueryFilter(p => !p.IsDeleted); modelBuilder.Entity<Product>() .HasQueryFilter(p => p.TenantId == _currentTenantId); } // Ignore filter when needed var allProducts = await context.Products .IgnoreQueryFilters() .ToListAsync();
Q16: What are owned entities?

Owned entities are entity types that don't have their own identity and are always mapped to the same table as their owner. They're useful for value objects.

// Owned entity public class Address { public string Street { get; set; } public string City { get; set; } public string ZipCode { get; set; } } public class User { public int Id { get; set; } public string Name { get; set; } public Address Address { get; set; } } // Configuration protected override void OnModelCreating(ModelBuilder modelBuilder) { modelBuilder.Entity<User>() .OwnsOne(u => u.Address); }
Q17: What is shadow property?

Shadow properties are properties that are not defined in your .NET entity class but are defined in the EF Core model. They're useful for data that doesn't need to be exposed in the domain model.

// Configure shadow property protected override void OnModelCreating(ModelBuilder modelBuilder) { modelBuilder.Entity<Product>() .Property<DateTime>("LastModified"); } // Access shadow property var lastModified = context.Entry(product) .Property<DateTime>("LastModified") .CurrentValue; // Query using shadow property var products = context.Products .Where(p => EF.Property<DateTime>(p, "LastModified") > DateTime.Now.AddDays(-7)) .ToList();

9. Common Interview Questions

Q18: What is the difference between Add, Attach, and Update?
  • Add: Marks entity as Added (will be inserted)
  • Attach: Marks entity as Unchanged (no changes)
  • Update: Marks entity as Modified (will be updated)
// Add - inserts new entity context.Products.Add(new Product { Name = "New" }); // Attach - tracks existing entity without changes context.Products.Attach(existingProduct); // Update - tracks as modified context.Products.Update(existingProduct);
Q19: How do you handle concurrency conflicts?
// Configure concurrency token public class Product { public int Id { get; set; } public string Name { get; set; } [Timestamp] public byte[] RowVersion { get; set; } } // Handle concurrency exception try { await context.SaveChangesAsync(); } catch (DbUpdateConcurrencyException ex) { var entry = ex.Entries.Single(); var databaseValues = await entry.GetDatabaseValuesAsync(); var clientValues = entry.CurrentValues; // Resolve conflict entry.OriginalValues.SetValues(databaseValues); await context.SaveChangesAsync(); }
Q20: What is the difference between HasData and Seed Data?
  • HasData: Used in OnModelCreating for seed data that is part of the model
  • Seed Data (Migration): Used in migrations for initial data population
// HasData in OnModelCreating protected override void OnModelCreating(ModelBuilder modelBuilder) { modelBuilder.Entity<Category>().HasData( new Category { Id = 1, Name = "Electronics" }, new Category { Id = 2, Name = "Books" } ); } // Seed data in migration protected override void Up(MigrationBuilder migrationBuilder) { migrationBuilder.Sql("INSERT INTO Categories (Name) VALUES ('Electronics'), ('Books')"); }
Q21: How do you implement repository pattern with EF Core?
// Generic repository interface public interface IRepository<T> where T : class { Task<T> GetByIdAsync(int id); Task<IEnumerable<T>> GetAllAsync(); Task AddAsync(T entity); Task UpdateAsync(T entity); Task DeleteAsync(T entity); } // Generic repository implementation public class Repository<T> : IRepository<T> where T : class { private readonly AppDbContext _context; private readonly DbSet<T> _dbSet; public Repository(AppDbContext context) { _context = context; _dbSet = context.Set<T>(); } public async Task<T> GetByIdAsync(int id) { return await _dbSet.FindAsync(id); } public async Task<IEnumerable<T>> GetAllAsync() { return await _dbSet.ToListAsync(); } public async Task AddAsync(T entity) { await _dbSet.AddAsync(entity); await _context.SaveChangesAsync(); } // ... other methods }
Q22: What is the difference between Find and SingleOrDefault?
  • Find: First checks the context for the entity before querying the database. Returns null if not found.
  • SingleOrDefault: Always queries the database. Throws if more than one match.
// Find - checks context first var product = await context.Products.FindAsync(1); // SingleOrDefault - always queries database var product = await context.Products.SingleOrDefaultAsync(p => p.Id == 1);
Q23: How do you execute raw SQL queries?
// FromSqlRaw for entity types var products = await context.Products .FromSqlRaw("SELECT * FROM Products WHERE Price > {0}", 100) .ToListAsync(); // ExecuteSqlRaw for non-query await context.Database.ExecuteSqlRawAsync( "UPDATE Products SET Price = Price * 1.1 WHERE CategoryId = {0}", categoryId); // ExecuteSqlInterpolated (safer) var price = 100; var products = await context.Products .FromSqlInterpolated($"SELECT * FROM Products WHERE Price > {price}") .ToListAsync();
Important: Always use parameterized queries to prevent SQL injection. Never concatenate user input into SQL strings.
Q24: How do you implement soft delete?
// Add IsDeleted property public class Product { public int Id { get; set; } public string Name { get; set; } public bool IsDeleted { get; set; } } // Global query filter protected override void OnModelCreating(ModelBuilder modelBuilder) { modelBuilder.Entity<Product>() .HasQueryFilter(p => !p.IsDeleted); } // Soft delete public async Task DeleteAsync(int id) { var product = await context.Products.FindAsync(id); if (product != null) { product.IsDeleted = true; await context.SaveChangesAsync(); } }
Q25: What is the difference between DbContext scope and lifetime?
  • Scoped: Default lifetime - one DbContext per HTTP request
  • Transient: New DbContext each time - not recommended
  • Singleton: Single DbContext for app lifetime - not recommended (context not thread-safe)
// Scoped (recommended) builder.Services.AddDbContext<AppDbContext>(options => options.UseSqlServer(connectionString)); // For background services, create scoped context using var scope = serviceProvider.CreateScope(); var context = scope.ServiceProvider.GetRequiredService<AppDbContext>();
Q26: How do you implement multi-tenancy with EF Core?
// Add TenantId to entities public class Product { public int Id { get; set; } public string Name { get; set; } public int TenantId { get; set; } } // Global query filter for tenant isolation protected override void OnModelCreating(ModelBuilder modelBuilder) { modelBuilder.Entity<Product>() .HasQueryFilter(p => p.TenantId == _currentTenantId); } // Set tenant ID on save public override async Task SaveChangesAsync(CancellationToken cancellationToken = default) { foreach (var entry in ChangeTracker.Entries<Product>()) { if (entry.State == EntityState.Added) { entry.Entity.TenantId = _currentTenantId; } } return await base.SaveChangesAsync(cancellationToken); }
Q27: What is the difference between Include and ThenInclude?
  • Include: Loads related entities at the first level
  • ThenInclude: Loads related entities at deeper levels
// Include - first level var products = await context.Products .Include(p => p.Category) .ToListAsync(); // ThenInclude - deeper levels var products = await context.Products .Include(p => p.Category) .ThenInclude(c => c.Products) .ToListAsync();
Q28: How do you implement audit fields?
// Base entity with audit fields public abstract class AuditableEntity { public DateTime CreatedAt { get; set; } public string CreatedBy { get; set; } public DateTime? UpdatedAt { get; set; } public string UpdatedBy { get; set; } } // Override SaveChanges public override async Task SaveChangesAsync(CancellationToken cancellationToken = default) { foreach (var entry in ChangeTracker.Entries<AuditableEntity>()) { if (entry.State == EntityState.Added) { entry.Entity.CreatedAt = DateTime.UtcNow; entry.Entity.CreatedBy = _currentUser; } else if (entry.State == EntityState.Modified) { entry.Entity.UpdatedAt = DateTime.UtcNow; entry.Entity.UpdatedBy = _currentUser; } } return await base.SaveChangesAsync(cancellationToken); }
Q29: What is the difference between HasDefaultSchema and ToTable?
  • HasDefaultSchema: Sets the default schema for all tables in the context
  • ToTable: Sets the table name for a specific entity
// Set default schema for all tables protected override void OnModelCreating(ModelBuilder modelBuilder) { modelBuilder.HasDefaultSchema("dbo"); } // Set table name for specific entity modelBuilder.Entity<Product>() .ToTable("Products", "dbo");
Q30: How do you implement database view mapping?
// Create entity for view public class ProductSummary { public int Id { get; set; } public string Name { get; set; } public decimal AveragePrice { get; set; } } // Map to view protected override void OnModelCreating(ModelBuilder modelBuilder) { modelBuilder.Entity<ProductSummary>() .ToView("vw_ProductSummary") .HasNoKey(); } // Query the view var summaries = await context.ProductSummaries.ToListAsync();

10. Database Design Patterns (Expert Level)

Q31: What is the Repository pattern and how do you implement it with EF Core?

The Repository pattern mediates between the domain and data mapping layers, acting like an in-memory domain object collection.

// Generic repository interface public interface IRepository<T> where T : class { Task<T> GetByIdAsync(int id); Task<IEnumerable<T>> GetAllAsync(); Task<IEnumerable<T>> FindAsync(Expression<Func<T, bool>> predicate); Task<T> AddAsync(T entity); Task UpdateAsync(T entity); Task DeleteAsync(T entity); } // Generic repository implementation public class Repository<T> : IRepository<T> where T : class { protected readonly DbContext Context; protected readonly DbSet<T> DbSet; public Repository(DbContext context) { Context = context; DbSet = context.Set<T>(); } public async Task<T> GetByIdAsync(int id) { return await DbSet.FindAsync(id); } public async Task<IEnumerable<T>> GetAllAsync() { return await DbSet.ToListAsync(); } public async Task<IEnumerable<T>> FindAsync(Expression<Func<T, bool>> predicate) { return await DbSet.Where(predicate).ToListAsync(); } public async Task<T> AddAsync(T entity) { await DbSet.AddAsync(entity); await Context.SaveChangesAsync(); return entity; } public async Task UpdateAsync(T entity) { DbSet.Update(entity); await Context.SaveChangesAsync(); } public async Task DeleteAsync(T entity) { DbSet.Remove(entity); await Context.SaveChangesAsync(); } }
Q32: What is the Unit of Work pattern and how does it relate to EF Core?

The Unit of Work pattern maintains a list of objects affected by a business transaction and coordinates the writing out of changes. EF Core's DbContext is essentially a Unit of Work implementation.

// DbContext is already a Unit of Work // It tracks changes and commits them in a single transaction // Custom Unit of Work wrapper public class UnitOfWork : IUnitOfWork { private readonly AppDbContext _context; private bool _disposed = false; public UnitOfWork(AppDbContext context) { _context = context; } public async Task<int> CommitAsync() { return await _context.SaveChangesAsync(); } public async Task<IDbContextTransaction> BeginTransactionAsync() { return await _context.Database.BeginTransactionAsync(); } public async Task RollbackAsync() { await _context.Database.RollbackTransactionAsync(); } protected virtual void Dispose(bool disposing) { if (!_disposed) { if (disposing) { _context.Dispose(); } _disposed = true; } } public void Dispose() { Dispose(true); GC.SuppressFinalize(this); } }
Q33: What is the Specification pattern and how do you implement it with EF Core?

The Specification pattern encapsulates business rules as reusable, composable predicates.

// Specification interface public interface ISpecification<T> { Expression<Func<T, bool>> ToExpression(); } // Base specification public abstract class Specification<T> : ISpecification<T> { public abstract Expression<Func<T, bool>> ToExpression(); public bool IsSatisfiedBy(T entity) { var predicate = ToExpression().Compile(); return predicate(entity); } } // Concrete specifications public class ExpensiveProductSpecification : Specification<Product> { private readonly decimal _minPrice; public ExpensiveProductSpecification(decimal minPrice) { _minPrice = minPrice; } public override Expression<Func<Product, bool>> ToExpression() { return p => p.Price > _minPrice; } } // Usage var spec = new ExpensiveProductSpecification(100); var products = await context.Products .Where(spec.ToExpression()) .ToListAsync();
Q34: What is the CQRS pattern and how do you implement it with EF Core?

CQRS (Command Query Responsibility Segregation) separates read and write operations.

// Command side (write) public interface ICommand<TResult> { } public interface ICommandHandler<TCommand, TResult> where TCommand : ICommand<TResult> { Task<TResult> HandleAsync(TCommand command); } // Query side (read) public interface IQuery<TResult> { } public interface IQueryHandler<TQuery, TResult> where TQuery : IQuery<TResult> { Task<TResult> HandleAsync(TQuery query); } // Command example public class CreateProductCommand : ICommand<int> { public string Name { get; set; } public decimal Price { get; set; } } public class CreateProductCommandHandler : ICommandHandler<CreateProductCommand, int> { private readonly AppDbContext _context; public CreateProductCommandHandler(AppDbContext context) { _context = context; } public async Task<int> HandleAsync(CreateProductCommand command) { var product = new Product { Name = command.Name, Price = command.Price }; _context.Products.Add(product); await _context.SaveChangesAsync(); return product.Id; } }

11. Multi-Tenancy and Global Filters (Expert Level)

Q35: How do you implement multi-tenancy in EF Core?
// Tenant entity public class Tenant { public int Id { get; set; } public string Name { get; set; } } // Base entity with tenant ID public abstract class TenantAwareEntity { public int TenantId { get; set; } public Tenant Tenant { get; set; } } // DbContext with tenant filtering public class AppDbContext : DbContext { private readonly int _currentTenantId; public AppDbContext(DbContextOptions<AppDbContext> options, ITenantProvider tenantProvider) : base(options) { _currentTenantId = tenantProvider.GetTenantId(); } protected override void OnModelCreating(ModelBuilder modelBuilder) { // Apply global query filter for all tenant-aware entities foreach (var entityType in modelBuilder.Model.GetEntityTypes()) { if (typeof(TenantAwareEntity).IsAssignableFrom(entityType.ClrType)) { modelBuilder.Entity(entityType.ClrType) .HasQueryFilter(e => EF.Property<int>(e, "TenantId") == _currentTenantId); } } } public override async Task<int> SaveChangesAsync(CancellationToken cancellationToken = default) { // Set tenant ID on new entities foreach (var entry in ChangeTracker.Entries<TenantAwareEntity>()) { if (entry.State == EntityState.Added) { entry.Entity.TenantId = _currentTenantId; } } return await base.SaveChangesAsync(cancellationToken); } }
Q36: How do you implement soft delete with global filters?
// Soft delete entity public abstract class SoftDeleteEntity { public bool IsDeleted { get; set; } public DateTime? DeletedAt { get; set; } public string DeletedBy { get; set; } } // DbContext with soft delete filter public class AppDbContext : DbContext { protected override void OnModelCreating(ModelBuilder modelBuilder) { // Apply global query filter for soft delete foreach (var entityType in modelBuilder.Model.GetEntityTypes()) { if (typeof(SoftDeleteEntity).IsAssignableFrom(entityType.ClrType)) { modelBuilder.Entity(entityType.ClrType) .HasQueryFilter(e => EF.Property<bool>(e, "IsDeleted") == false); } } } public override async Task<int> SaveChangesAsync(CancellationToken cancellationToken = default) { // Handle soft delete foreach (var entry in ChangeTracker.Entries<SoftDeleteEntity>()) { if (entry.State == EntityState.Deleted) { entry.State = EntityState.Modified; entry.Entity.IsDeleted = true; entry.Entity.DeletedAt = DateTime.UtcNow; entry.Entity.DeletedBy = _currentUser; } } return await base.SaveChangesAsync(cancellationToken); } } // Include deleted entities when needed var allProducts = await context.Products .IgnoreQueryFilters() .Where(p => p.IsDeleted) .ToListAsync();
Q37: How do you implement audit trails with EF Core?
// Audit entity public class AuditLog { public int Id { get; set; } public string EntityName { get; set; } public string EntityId { get; set; } public string Action { get; set; } public string Changes { get; set; } public string ChangedBy { get; set; } public DateTime ChangedAt { get; set; } } // DbContext with audit logging public class AppDbContext : DbContext { private readonly string _currentUser; public DbSet<AuditLog> AuditLogs { get; set; } public override async Task<int> SaveChangesAsync(CancellationToken cancellationToken = default) { var auditEntries = OnBeforeSaveChanges(); var result = await base.SaveChangesAsync(cancellationToken); await OnAfterSaveChanges(auditEntries); return result; } private List<AuditEntry> OnBeforeSaveChanges() { ChangeTracker.DetectChanges(); var entries = ChangeTracker.Entries() .Where(e => e.State == EntityState.Added || e.State == EntityState.Modified || e.State == EntityState.Deleted) .ToList(); var auditEntries = new List<AuditEntry>(); foreach (var entry in entries) { var auditEntry = new AuditEntry(entry) { TableName = entry.Metadata.GetTableName(), UserId = _currentUser, Action = entry.State.ToString(), Time = DateTime.UtcNow }; foreach (var property in entry.Properties) { string propertyName = property.Metadata.Name; if (property.Metadata.IsPrimaryKey()) { auditEntry.KeyValues[propertyName] = property.CurrentValue; continue; } switch (entry.State) { case EntityState.Added: auditEntry.NewValues[propertyName] = property.CurrentValue; break; case EntityState.Deleted: auditEntry.OldValues[propertyName] = property.OriginalValue; break; case EntityState.Modified: if (property.IsModified) { auditEntry.OldValues[propertyName] = property.OriginalValue; auditEntry.NewValues[propertyName] = property.CurrentValue; } break; } } auditEntries.Add(auditEntry); } foreach (var auditEntry in auditEntries) { AuditLogs.Add(auditEntry.ToAuditLog()); } return auditEntries; } }

12. Concurrency and Transactions (Expert Level)

Q38: How do you handle concurrency conflicts in EF Core?

Optimistic Concurrency assumes conflicts are rare and checks for them when saving.

// Add concurrency token public class Product { public int Id { get; set; } public string Name { get; set; } public decimal Price { get; set; } [Timestamp] public byte[] RowVersion { get; set; } } // Handle concurrency exception try { await context.SaveChangesAsync(); } catch (DbUpdateConcurrencyException ex) { var entry = ex.Entries.Single(); var databaseValues = await entry.GetDatabaseValuesAsync(); var clientValues = entry.CurrentValues; // Resolve conflict foreach (var property in entry.Metadata.GetProperties()) { var proposedValue = clientValues[property]; var databaseValue = databaseValues[property]; // Choose which value to keep if (property.Name == "Price") { // Keep database value clientValues[property] = databaseValue; } } // Refresh original values to avoid next concurrency issue entry.OriginalValues.SetValues(databaseValues); await context.SaveChangesAsync(); }
Q39: How do you implement transactions in EF Core?
// Explicit transaction using var transaction = await context.Database.BeginTransactionAsync(); try { // Multiple operations var product = new Product { Name = "Laptop", Price = 999.99 }; context.Products.Add(product); var category = new Category { Name = "Electronics" }; context.Categories.Add(category); await context.SaveChangesAsync(); await transaction.CommitAsync(); } catch { await transaction.RollbackAsync(); throw; } // Transaction with isolation level using var transaction = await context.Database.BeginTransactionAsync( IsolationLevel.Serializable);
Q40: What are the different isolation levels and when should you use each?
Isolation Level Description Use Case
Read Uncommitted Can read uncommitted data (dirty reads) Rarely used - only for non-critical data
Read Committed Cannot read uncommitted data Default for most databases
Repeatable Read Same query returns same data within transaction When data consistency is critical
Serializable Highest isolation - prevents all concurrency issues Critical financial operations
Snapshot Reads from snapshot of committed data High concurrency read scenarios

Q51: What is the difference between Add, Attach, and Update?
  • Add: Marks entity as Added. Will be inserted on SaveChanges.
  • Attach: Attaches entity to context in Unchanged state. No changes tracked.
  • Update: Marks entity as Modified. All properties will be updated on SaveChanges.
// Add - new entity context.Products.Add(new Product { Name = "Laptop" }); // Attach - existing entity, no changes var product = new Product { Id = 1, Name = "Laptop" }; context.Products.Attach(product); // Update - existing entity, mark all as modified var product = new Product { Id = 1, Name = "Laptop" }; context.Products.Update(product); // Update specific properties var product = await context.Products.FindAsync(1); product.Price = 999.99; context.Entry(product).Property(p => p.Price).IsModified = true;
