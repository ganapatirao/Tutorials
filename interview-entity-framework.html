1. Entity Framework Fundamentals

Q1: What is Entity Framework and what are its benefits?

Entity Framework (EF) is an Object-Relational Mapper (ORM) that enables .NET developers to work with relational data using domain-specific objects. It eliminates the need for most of the data-access code that developers usually need to write.

Benefits:

  • Productivity: Reduces data access code
  • Maintainability: Less code to maintain
  • Type Safety: Compile-time checking
  • Abstraction: Database agnostic
  • Change Tracking: Automatic change detection
  • LINQ Support: Query using LINQ
  • Migrations: Database schema management

2. DbContext and DbSet

Q3: What is DbContext?

DbContext is the primary class that coordinates the Entity Framework functionality for a given data model. It:

  • Manages entity objects during runtime
  • Handles database connections
  • Tracks changes to entities
  • Saves changes to the database
  • Executes LINQ queries against the database
public class AppDbContext : DbContext
{
  public AppDbContext(DbContextOptions<AppDbContext> options)
    : base(options)
  {
  }

  public DbSet<Product> Products { get; set; }
  public DbSet<Category> Categories { get; set; }

  protected override void OnModelCreating(ModelBuilder modelBuilder)
  {
    // Fluent API configuration
  }
}
Q4: What is DbSet?

DbSet<TEntity> represents a collection of entities of a specific type that can be queried from the database. It:

  • Represents a table in the database
  • Enables CRUD operations
  • Supports LINQ queries
  • Tracks changes to entities
// CRUD operations
// Create
context.Products.Add(new Product { Name = "Laptop" });
await context.SaveChangesAsync();

// Read
var products = await context.Products.ToListAsync();
var product = await context.Products.FindAsync(1);

// Update
product.Name = "Updated Laptop";
await context.SaveChangesAsync();

// Delete
context.Products.Remove(product);
await context.SaveChangesAsync();

4. Migrations

Q7: What are Entity Framework migrations?

Migrations enable you to apply incremental changes to the database schema to keep it in sync with your EF Core model.

Migration Commands:

// Add migration
dotnet ef migrations add InitialCreate

// Update database
dotnet ef database update

// Remove last migration
dotnet ef migrations remove

// List migrations
dotnet ef migrations list

// Generate SQL script
dotnet ef migrations script

Migration Steps:

  1. Define or modify your entity classes
  2. Add a migration to capture the changes
  3. Update the database to apply the migration
Q8: How do you handle migrations in production? 
// Generate SQL script for production
dotnet ef migrations script 2023-01-01-InitialCreate --output migration.sql

// Apply migrations programmatically
public static void ApplyMigrations(IApplicationBuilder app)
{
  using var scope = app.ApplicationServices.CreateScope();
  var context = scope.ServiceProvider.GetRequiredService<AppDbContext>();
  context.Database.Migrate();
}

// In Program.cs
var app = builder.Build();
ApplyMigrations(app);
Important: Never use automatic migrations in production. Always generate SQL scripts and review them before applying.

5. Change Tracking

Q9: How does Entity Framework track changes?

EF Core tracks changes to entities using a change tracker. When you query entities, EF Core:

  • Takes a snapshot of the entity's original values
  • Monitors the entity for changes
  • Detects changes when SaveChanges is called
  • Generates appropriate SQL UPDATE/INSERT/DELETE statements
// Change tracking states
var entry = context.Entry(product);
Console.WriteLine(entry.State); // Added, Modified, Deleted, Unchanged, Detached

// Detect changes manually
context.ChangeTracker.DetectChanges();

// Query change tracker
var modifiedEntities = context
  .ChangeTracker.Entries()
  .Where(e => e.State == EntityState.Modified)
  .ToList();

6. LINQ to Entities

Q11: How does LINQ work with Entity Framework?

LINQ queries in EF Core are translated to SQL queries. The query is executed when:

  • You iterate over the results (foreach)
  • You call a conversion operator (ToList, ToArray, etc.)
  • You call an aggregate operator (Count, Sum, etc.)
// Query is translated to SQL
var products = context
  .Products
  .Where(p => p.Price > 100)
  .OrderBy(p => p.Name)
  .ToList();

// Translated SQL:
// SELECT * FROM Products
// WHERE Price > 100
// ORDER BY Name
Important: Not all LINQ operators are supported in LINQ to Entities. Unsupported operators will throw NotSupportedException.

Loading Type Description When to Use
Eager Loading Load related entities with the main query When you know you need the data
Lazy Loading Load related entities when accessed When you might not need the data
Explicit Loading Load related entities on demand When you need fine-grained control

await context
  .Entry(product)

7. Performance Optimization

Q13: How do you optimize Entity Framework performance?
  • Use AsNoTracking: For read-only queries
  • Select only needed columns: Avoid fetching unnecessary data
  • Use eager loading wisely: Avoid over-including
  • Use pagination: For large datasets
  • Use compiled queries: For frequently executed queries
  • Batch updates: Reduce round-trips
  • Use connection pooling: Optimize connection settings
  • Average query complexity: Keep queries simple
// AsNoTracking for read-only
var products = await context
  .Products
  .AsNoTracking()
  .ToListAsync();

// Select specific columns
var names = await context
  .Products
  .Select(p => p.Name)
  .ToListAsync();

// Pagination
var page = await context
  .Products
  .Skip((page - 1) * pageSize)
  .Take(pageSize)
  .ToListAsync();

// Compiled query
private static readonly Func<AppDbContext, int, Task<Product>>
  GetProductById = EF.CompileAsyncQuery(
    (AppDbContext db, int id) => db.Products.FirstOrDefault(p => p.Id == id)
  );

var product = await GetProductById(context, 1);
Q14: What is the N+1 query problem?

The N+1 query problem occurs when you execute N additional queries to fetch related data for each of N entities returned by the main query.

// Problematic - N+1 queries
var categories = context.Categories.ToList();
foreach (var category in categories)
{
  var products = context
    .Products
    .Where(p => p.CategoryId == category.Id)
    .ToList(); // N additional queries
}

// Solution - eager loading
var categories = context
  .Categories
  .Include(c => c.Products)
  .ToList(); // Single query with JOIN

8. Advanced Features

Q15: What are global query filters?

Global query filters apply a filter to all queries for a given entity type. They're useful for implementing soft delete, multi-tenancy, and data isolation.

// Configure global query filter
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
  modelBuilder.Entity<Product>().HasQueryFilter(p => !p.IsDeleted);

  modelBuilder
    .Entity<Product>()
    .HasQueryFilter(p => p.TenantId == _currentTenantId);
}

// Ignore filter when needed
var allProducts = await context
  .Products
  .IgnoreQueryFilters()
  .ToListAsync();
Q16: What are owned entities?

Owned entities are entity types that don't have their own identity and are always mapped to the same table as their owner. They're useful for value objects.

// Owned entity
public class Address
{
  public string Street { get; set; }
  public string City { get; set; }
  public string ZipCode { get; set; }
}

public class User
{
  public int Id { get; set; }
  public string Name { get; set; }
  public Address Address { get; set; }
}

// Configuration
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
  modelBuilder.Entity<User>().OwnsOne(u => u.Address);
}
Q17: What is shadow property?

Shadow properties are properties that are not defined in your .NET entity class but are defined in the EF Core model. They're useful for data that doesn't need to be exposed in the domain model.

// Configure shadow property
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
  modelBuilder.Entity<Product>().Property<DateTime>("LastModified");
}

// Access shadow property
var lastModified = context
  .Entry(product)
  .Property<DateTime>("LastModified")
  .CurrentValue;

// Query using shadow property
var products = context
  .Products
  .Where(
    p =>
      EF.Property<DateTime>(p, "LastModified")
      > DateTime.Now.AddDays(-7)
  )
  .ToList();

Q18: What is the difference between Add, Attach, and Update?
  • Add: Marks entity as Added (will be inserted)
  • Attach: Marks entity as Unchanged (no changes)
  • Update: Marks entity as Modified (will be updated)
// Add - inserts new entity
context.Products.Add(new Product { Name = "New" });

// Attach - tracks existing entity without changes
context.Products.Attach(existingProduct);

// Update - tracks as modified

// Configure concurrency token
public class Product
{

  public string Name { get; set; }

  [Timestamp]
  public byte[] RowVersion { get; set; }
}

// Handle concurrency exception
try
{
  await context.SaveChangesAsync();
}
catch (DbUpdateConcurrencyException ex)
{
  var entry = ex.Entries.Single();
  var databaseValues = await entry.GetDatabaseValuesAsync();
  var clientValues = entry.CurrentValues;

  // Resolve conflict
  entry.OriginalValues.SetValues(databaseValues);
  await context.SaveChangesAsync();
}

Q22: What is the difference between Find and SingleOrDefault?
  • Find: First checks the context for the entity before querying the database. Returns null if not found.
  • SingleOrDefault: Always queries the database. Throws if more than one match.

// SingleOrDefault - always queries database
var product = await context.Products.SingleOrDefaultAsync(p => p.Id == 1);

// FromSqlRaw for entity types
var products = await context
  .Products
  .FromSqlRaw("SELECT * FROM Products WHERE Price > {0}", 100)

await context.Database.ExecuteSqlRawAsync(

public abstract class AuditableEntity

  public string UpdatedBy { get; set; }

)
{

      entry.Entity.UpdatedBy = _currentUser;
    }
  }
  return await base.SaveChangesAsync(cancellationToken);
}

10. Database Design Patterns (Expert Level)

Q31: What is the Repository pattern and how do you implement it with EF Core?

The Repository pattern mediates between the domain and data mapping layers, acting like an in-memory domain object collection.

// Generic repository interface
public interface IRepository<T>
  where T : class
{
  Task<T> GetByIdAsync(int id);
  Task<IEnumerable<T>> GetAllAsync();
  Task<IEnumerable<T>> FindAsync(
    Expression<Func<T, bool>> predicate
  );
  Task<T> AddAsync(T entity);
  Task UpdateAsync(T entity);
  Task DeleteAsync(T entity);
}

// Generic repository implementation
public class Repository<T> : IRepository<T>
  where T : class
{
  protected readonly DbContext Context;
  protected readonly DbSet<T> DbSet;

  public Repository(DbContext context)
  {
    Context = context;
    DbSet = context.Set<T>();
  }

  public async Task<T> GetByIdAsync(int id)
  {
    return await DbSet.FindAsync(id);
  }

  public async Task<IEnumerable<T>> GetAllAsync()
  {
    return await DbSet.ToListAsync();
  }

  public async Task<IEnumerable<T>> FindAsync(
    Expression<Func<T, bool>> predicate
  )
  {
    return await DbSet.Where(predicate).ToListAsync();
  }

  public async Task<T> AddAsync(T entity)
  {
    await DbSet.AddAsync(entity);
    await Context.SaveChangesAsync();
    return entity;
  }

  public async Task UpdateAsync(T entity)
  {
    DbSet.Update(entity);
    await Context.SaveChangesAsync();
  }

  public async Task DeleteAsync(T entity)
  {
    DbSet.Remove(entity);
    await Context.SaveChangesAsync();
  }
}
Q32: What is the Unit of Work pattern and how does it relate to EF Core?

The Unit of Work pattern maintains a list of objects affected by a business transaction and coordinates the writing out of changes. EF Core's DbContext is essentially a Unit of Work implementation.

// DbContext is already a Unit of Work
// It tracks changes and commits them in a single transaction

// Custom Unit of Work wrapper
public class UnitOfWork : IUnitOfWork
{
  private readonly AppDbContext _context;
  private bool _disposed = false;

  public UnitOfWork(AppDbContext context)
  {
    _context = context;
  }

  public async Task<int> CommitAsync()
  {
    return await _context.SaveChangesAsync();
  }

  public async Task<IDbContextTransaction> BeginTransactionAsync()
  {
    return await _context.Database.BeginTransactionAsync();
  }

  public async Task RollbackAsync()
  {
    await _context.Database.RollbackTransactionAsync();
  }

  protected virtual void Dispose(bool disposing)
  {
    if (!_disposed)
    {
      if (disposing)
      {
        _context.Dispose();
      }
      _disposed = true;
    }
  }

  public void Dispose()
  {
    Dispose(true);
    GC.SuppressFinalize(this);
  }
}
Q33: What is the Specification pattern and how do you implement it with EF Core?

The Specification pattern encapsulates business rules as reusable, composable predicates.

// Specification interface
public interface ISpecification<T>
{
  Expression<Func<T, bool>> ToExpression();
}

// Base specification
public abstract class Specification<T> : ISpecification<T>
{
  public abstract Expression<Func<T, bool>> ToExpression();

  public bool IsSatisfiedBy(T entity)
  {
    var predicate = ToExpression().Compile();
    return predicate(entity);
  }
}

// Concrete specifications
public class ExpensiveProductSpecification : Specification<Product>
{
  private readonly decimal _minPrice;

  public ExpensiveProductSpecification(decimal minPrice)
  {
    _minPrice = minPrice;
  }

  public override Expression<Func<Product, bool>> ToExpression()
  {
    return p => p.Price > _minPrice;
  }
}

// Usage
var spec = new ExpensiveProductSpecification(100);
var products = await context
  .Products
  .Where(spec.ToExpression())
  .ToListAsync();
Q34: What is the CQRS pattern and how do you implement it with EF Core?

CQRS (Command Query Responsibility Segregation) separates read and write operations.

// Command side (write)
public interface ICommand<TResult> {}

public interface ICommandHandler<TCommand, TResult>
  where TCommand : ICommand<TResult>
{
  Task<TResult> HandleAsync(TCommand command);
}

// Query side (read)
public interface IQuery<TResult> {}

public interface IQueryHandler<TQuery, TResult>
  where TQuery : IQuery<TResult>
{
  Task<TResult> HandleAsync(TQuery query);
}

// Command example
public class CreateProductCommand : ICommand<int>
{
  public string Name { get; set; }
  public decimal Price { get; set; }
}

public class CreateProductCommandHandler
  : ICommandHandler<CreateProductCommand, int>
{
  private readonly AppDbContext _context;

  public CreateProductCommandHandler(AppDbContext context)
  {
    _context = context;
  }

  public async Task<int> HandleAsync(CreateProductCommand command)
  {
    var product = new Product
    {
      Name = command.Name,
      Price = command.Price
    };

    _context.Products.Add(product);
    await _context.SaveChangesAsync();

    return product.Id;
  }
}

11. Multi-Tenancy and Global Filters (Expert Level)

Q35: How do you implement multi-tenancy in EF Core? 
// Tenant entity
public class Tenant
{
  public int Id { get; set; }
  public string Name { get; set; }
}

// Base entity with tenant ID
public abstract class TenantAwareEntity
{
  public int TenantId { get; set; }
  public Tenant Tenant { get; set; }
}

// DbContext with tenant filtering
public class AppDbContext : DbContext
{
  private readonly int _currentTenantId;

  public AppDbContext(
    DbContextOptions<AppDbContext> options,
    ITenantProvider tenantProvider
  )
    : base(options)
  {
    _currentTenantId = tenantProvider.GetTenantId();
  }

  protected override void OnModelCreating(ModelBuilder modelBuilder)
  {
    // Apply global query filter for all tenant-aware entities
    foreach (
      var entityType in modelBuilder.Model.GetEntityTypes()
    )
    {
      if (typeof(TenantAwareEntity).IsAssignableFrom(entityType.ClrType))
      {
        modelBuilder
          .Entity(entityType.ClrType)
          .HasQueryFilter(
            e => EF.Property<int>(e, "TenantId") == _currentTenantId
          );
      }
    }
  }

  public override async Task<int> SaveChangesAsync(
    CancellationToken cancellationToken = default
  )
  {
    // Set tenant ID on new entities
    foreach (
      var entry in ChangeTracker.Entries<TenantAwareEntity>()
    )
    {
      if (entry.State == EntityState.Added)
      {
        entry.Entity.TenantId = _currentTenantId;
      }
    }

    return await base.SaveChangesAsync(cancellationToken);
  }
}

// Audit entity
public class AuditLog
{
  public int Id { get; set; }
  public string EntityName { get; set; }
  public string EntityId { get; set; }
  public string Action { get; set; }
  public string Changes { get; set; }
  public string ChangedBy { get; set; }
  public DateTime ChangedAt { get; set; }
}

// DbContext with audit logging
public class AppDbContext : DbContext
{
  private readonly string _currentUser;

  public DbSet<AuditLog> AuditLogs { get; set; }

  public override async Task<int> SaveChangesAsync(
    CancellationToken cancellationToken = default
  )
  {
    var auditEntries = OnBeforeSaveChanges();
    var result = await base.SaveChangesAsync(cancellationToken);
    await OnAfterSaveChanges(auditEntries);
    return result;
  }

  private List<AuditEntry> OnBeforeSaveChanges()
  {
    ChangeTracker.DetectChanges();
    var entries = ChangeTracker
      .Entries()
      .Where(
        e =>
          e.State == EntityState.Added
          || e.State == EntityState.Modified
          || e.State == EntityState.Deleted
      )
      .ToList();

    var auditEntries = new List<AuditEntry>();

    foreach (var entry in entries)
    {
      var auditEntry = new AuditEntry(entry)
      {
        TableName = entry.Metadata.GetTableName(),
        UserId = _currentUser,
        Action = entry.State.ToString(),
        Time = DateTime.UtcNow
      };

      foreach (var property in entry.Properties)
      {
        string propertyName = property.Metadata.Name;
        if (property.Metadata.IsPrimaryKey())
        {
          auditEntry.KeyValues[propertyName] = property.CurrentValue;
          continue;
        }

        switch (entry.State)
        {
          case EntityState.Added:
            auditEntry.NewValues[propertyName] = property.CurrentValue;
            break;
          case EntityState.Deleted:
            auditEntry.OldValues[propertyName] = property.OriginalValue;
            break;
          case EntityState.Modified:
            if (property.IsModified)
            {
              auditEntry.OldValues[propertyName] = property.OriginalValue;
              auditEntry.NewValues[propertyName] = property.CurrentValue;
            }
            break;
        }
      }

      auditEntries.Add(auditEntry);
    }

    foreach (var auditEntry in auditEntries)
    {
      AuditLogs.Add(auditEntry.ToAuditLog());
    }

    return auditEntries;
  }
}

  await context.SaveChangesAsync();

catch (DbUpdateConcurrencyException ex)
{
  var entry = ex.Entries.Single();
  var databaseValues = await entry.GetDatabaseValuesAsync();
  var clientValues = entry.CurrentValues;

  // Resolve conflict
  foreach (var property in entry.Metadata.GetProperties())
  {
    var proposedValue = clientValues[property];
    var databaseValue = databaseValues[property];

    // Choose which value to keep
    if (property.Name == "Price")
    {
      // Keep database value
      clientValues[property] = databaseValue;
    }
  }

  // Refresh original values to avoid next concurrency issue
  entry.OriginalValues.SetValues(databaseValues);
  await context.SaveChangesAsync();
}

  if (string.IsNullOrEmpty(email))

  var parts = email.Split('@');
  if (parts.Length != 2)

  var username = parts[0];

    username.Length > 2

      : new string('*', username.Length);

}

Q52: How do you implement the Specification pattern with EF Core?

Specification pattern encapsulates business rules and query logic.

// Specification interface
public interface ISpecification<T>
{
  Expression<Func<T, bool>> ToExpression();
  bool IsSatisfiedBy(T entity);
}

// Base specification
public abstract class Specification<T> : ISpecification<T>
{
  public abstract Expression<Func<T, bool>> ToExpression();

  public bool IsSatisfiedBy(T entity)
  {
    var predicate = ToExpression();
    return predicate.Compile()(entity);
  }
}

// Concrete specifications
public class ActiveProductSpecification : Specification<Product>
{
  public override Expression<Func<Product, bool>> ToExpression()
  {
    return p => !p.IsDeleted && p.IsActive;
  }
}

public class ExpensiveProductSpecification : Specification<Product>
{
    private readonly decimal _minPrice;
    
    public ExpensiveProductSpecification(decimal minPrice)
    {
        _minPrice = minPrice;
  }

  public override Expression<Func<Product, bool>> ToExpression()
  {
    return p => p.Price >= _minPrice;
  }
}

// Composite specification (AND)
public class AndSpecification<T> : Specification<T>
{
  private readonly ISpecification<T> _left;
  private readonly ISpecification<T> _right;

  public AndSpecification(ISpecification<T> left, ISpecification<T> right)
  {
    _left = left;
    _right = right;
  }

  public override Expression<Func<T, bool>> ToExpression()
  {
    var leftExpr = _left.ToExpression();
    var rightExpr = _right.ToExpression();

    var parameter = Expression.Parameter(typeof(T));
    var leftBody = Expression.Invoke(leftExpr, parameter);
    var rightBody = Expression.Invoke(rightExpr, parameter);
        
        return Expression.Lambda<Func<T, bool>>(
  Expression.AndAlso(leftBody, rightBody),
  parameter
);
}
}

{
    var query = context.Set<T>().Where(specification.ToExpression());
    return await query.ToListAsync();

// SaveChanges override in DbContext
public override async Task<int> SaveChangesAsync(
  CancellationToken cancellationToken = default
)
{
  var auditEntries = OnBeforeSaveChanges();
  var result = await base.SaveChangesAsync(cancellationToken);
  await OnAfterSaveChangesAsync(auditEntries);
  return result;
}

private List<AuditEntry> OnBeforeSaveChanges()
{
  ChangeTracker.DetectChanges();
  var entries = ChangeTracker
    .Entries()
    .Where(
      e =>
        e.State == EntityState.Added
        || e.State == EntityState.Modified
        || e.State == EntityState.Deleted
    )
    .ToList();

  var auditEntries = new List<AuditEntry>();

  foreach (var entry in entries)
  {
    var auditEntry = new AuditEntry(entry)
{
  TableName = entry.Entity.GetType().Name,
  UserId = _currentUserService.UserId,
  Action = entry.State.ToString(),
  Timestamp = DateTime.UtcNow
};

foreach (var property in entry.Properties)
{
  string propertyName = property.Metadata.Name;

  if (property.Metadata.IsPrimaryKey)
  {
    auditEntry.KeyValues[propertyName] = property.CurrentValue;
    continue;
  }

  switch (entry.State)
  {
    case EntityState.Added:
      auditEntry.NewValues[propertyName] = property.CurrentValue;
      break;
    case EntityState.Deleted:
      auditEntry.OldValues[propertyName] = property.OriginalValue;
      break;
    case EntityState.Modified:
      if (property.IsModified)
      {
        auditEntry.OldValues[propertyName] = property.OriginalValue;
        auditEntry.NewValues[propertyName] = property.CurrentValue;
      }
      break;
  }
}

auditEntries.Add(auditEntry);
}
    
    foreach (var auditEntry in auditEntries)
{
  Audits.Add(auditEntry.ToAudit());
}

return auditEntries.Where(e => e.HasTemporaryProperties).ToList();
}

private Task OnAfterSaveChangesAsync(List<AuditEntry> auditEntries)
{
  if (auditEntries == null || auditEntries.Count == 0)
    return Task.CompletedTask;

  foreach (var auditEntry in auditEntries)
  {
    // Get final values
    foreach (var prop in auditEntry.Entry.Properties)
    {
      if (prop.IsTemporary)
      {
        auditEntry.KeyValues[prop.Metadata.Name] = prop.CurrentValue;
      }
    }
  }

  // Save audits
  return SaveChangesAsync();
}

public class EncryptionService
{
    private readonly byte[] _encryptionKey;
    private readonly byte[] _iv;
    
    public string Encrypt(string plainText)
    {
        using var aes = Aes.Create();
        aes.Key = _encryptionKey;
        aes.IV = _iv;
        
        using var encryptor = aes.CreateEncryptor();
        using var msEncrypt = new MemoryStream();
        
        using (var csEncrypt = new CryptoStream(msEncrypt, encryptor, CryptoStreamMode.Write))
        using (var swEncrypt = new StreamWriter(csEncrypt))
        {
            swEncrypt.Write(plainText);
        }
        
        return Convert.ToBase64String(msEncrypt.ToArray());
    }
    
    public string Decrypt(string cipherText)
    {
        using var aes = Aes.Create();
        aes.Key = _encryptionKey;
        aes.IV = _iv;
        
        using var decryptor = aes.CreateDecryptor();
        using var msDecrypt = new MemoryStream(Convert.FromBase64String(cipherText));
        
        using var csDecrypt = new CryptoStream(msDecrypt, decryptor, CryptoStreamMode.Read);
        using var srDecrypt = new StreamReader(csDecrypt);
        
        return srDecrypt.ReadToEnd();
    }
}

            .FirstOrDefaultAsync(o => o.Id == orderId);

        }

public class MigrationRunner
{
    private readonly AppDbContext _context;
    
    public async Task RunMigrationsAsync()
    {
        var appliedVersions = await _context.DatabaseVersions
            .OrderBy(v => v.Id)
            .Select(v => v.Version)
            .ToListAsync();
        
        var pendingMigrations = GetPendingMigrations(appliedVersions);
        
        foreach (var migration in pendingMigrations)
        {
            await ExecuteMigrationAsync(migration);
            
            _context.DatabaseVersions.Add(new DatabaseVersion
            {
                Version = migration.Version,
                AppliedAt = DateTime.UtcNow,
                Description = migration.Description
            });
            
            await _context.SaveChangesAsync();
        }
    }
    
    private async Task ExecuteMigrationAsync(MigrationScript migration)
    {
        try
        {
            await _context.Database.ExecuteSqlRawAsync(migration.UpScript);
        }
        catch (Exception ex)
        {
            // Rollback
            await _context.Database.ExecuteSqlRawAsync(migration.DownScript);
            throw;
        }
    }
}

// Blue-green deployment strategy
public class DatabaseDeploymentService
{
    public async Task DeployWithZeroDowntimeAsync(string newVersion)
    {
        // 1. Create new database copy
        await CreateDatabaseCopyAsync($"MyDb_{newVersion}");
        
        // 2. Apply migrations to new database
        await ApplyMigrationsAsync($"MyDb_{newVersion}");
        
        // 3. Sync data from old to new database
        await SyncDataAsync("MyDb", $"MyDb_{newVersion}");
        
        // 4. Switch connection string
        await SwitchDatabaseAsync($"MyDb_{newVersion}");
        
        // 5. Keep old database for rollback
    }
    
    private async Task SyncDataAsync(string sourceDb, string targetDb)
    {
        var syncScript = $@"
            INSERT INTO {targetDb}.dbo.Products
            SELECT * FROM {sourceDb}.dbo.Products
            WHERE NOT EXISTS (
                SELECT 1 FROM {targetDb}.dbo.Products 
                WHERE Id = {sourceDb}.dbo.Products.Id
            )";
        
        await _context.Database.ExecuteSqlRawAsync(syncScript);
    }
}
