1. Angular Fundamentals

Angular is a TypeScript-based open-source web application framework led by Google. It's a complete rewrite of AngularJS (Angular 1.x).

Key Features:

  • TypeScript: Built with TypeScript for better tooling and type safety

2. Components and Templates

Q3: What is an Angular component?

A Component is the fundamental building block of Angular applications. It consists of:

  • Template: HTML view
  • Class: TypeScript class with logic
  • Styles: CSS styles
  • Metadata: Decorator that configures the component
@Component({

export class ProductComponent implements OnInit {

Q4: What is the component lifecycle?

Angular components have lifecycle hooks that allow you to tap into key moments:

  • ngOnChanges: Called when input properties change
  • ngOnInit: Called after constructor, when inputs are set
  • ngDoCheck: Called during every change detection
  • ngAfterContentInit: Called after content projection
  • ngAfterContentChecked: Called after content check
  • ngAfterViewInit: Called after view initialization
  • ngAfterViewChecked: Called after view check
  • ngOnDestroy: Called before component destruction

5. Pipes

Q8: What are pipes and how do you use them?

Pipes transform data in templates. Angular provides built-in pipes and allows custom pipes.

Built-in Pipes:

  • DatePipe - date formatting
  • UpperCasePipe / LowerCasePipe - case transformation
  • CurrencyPipe - currency formatting
  • DecimalPipe - number formatting

// Custom pipe
import { Pipe, PipeTransform } from '@angular/core';

@Pipe({
  name: 'truncate'
})
export class TruncatePipe implements PipeTransform {
  transform(value: string, limit: number): string {
    return value.length > limit ? value.substring(0, limit) + '...' : value;
  }
}

6. Services and Dependency Injection

Q9: What is dependency injection in Angular?

Dependency Injection (DI) is a design pattern where dependencies are provided to a class rather than the class creating them. Angular has a built-in DI system.

Providing Services:

  • root: Singleton across the app
  • any: Singleton per module
  • Component: New instance per component
// Service
@Injectable({
  providedIn: 'root'
})
export class ProductService {
  constructor(private http: HttpClient) {}
  
  getProducts(): Observable<Product[]> {
    return this.http.get<Product[]>('/api/products');
  }
}

// Component
@Component({...})
export class ProductComponent {
  constructor(private productService: ProductService) {}
}
Q10: What are the different injectors in Angular?
  • Root injector: App-level injector
  • Module injector: NgModule-level injector
  • Element injector: Component-level injector
  • Platform injector: Platform-level injector
Important: Angular's DI system follows a hierarchical injector tree. When a dependency is requested, Angular searches up the injector tree until it finds the provider.

7. Modules

Q11: What is an NgModule?

NgModule is a class decorated with @NgModule that groups related components, directives, pipes, and services.

NgModule Properties:

  • declarations: Components, directives, pipes belonging to this module
  • imports: Other modules whose exports are needed
  • exports: Components, directives, pipes to be used by other modules

  • bootstrap: Root component to bootstrap

  • declarations: Components, directives, and pipes that belong to this module. They must be declared in exactly one module.
  • imports: Other modules whose exported components, directives, and pipes are needed in this module's templates.

Q14: What are route guards?

Route guards control navigation to and from routes.

  • CanActivate: Can a user navigate to a route?
  • CanActivateChild: Can a user navigate to child routes?
  • CanDeactivate: Can a user navigate away from a route?
  • CanLoad: Can a module be loaded?
  • CanMatch: Can a route match?
@Injectable({ providedIn: 'root' })
export class AuthGuard implements CanActivate {
  constructor(private authService: AuthService, private router: Router) {}
  
  canActivate(): boolean {
    if (this.authService.isLoggedIn()) {
      return true;
    }
    this.router.navigate(['/login']);
    return false;
  }
}

// Usage

Q17: What are HTTP interceptors?

HTTP interceptors intercept HTTP requests and responses to add common logic like authentication, logging, or error handling.

@Injectable()
export class AuthInterceptor implements HttpInterceptor {
  constructor(private authService: AuthService) {}
  
  intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    const token = this.authService.getToken();
    const authReq = req.clone({
      setHeaders: { Authorization: `Bearer ${token}` }
    });
    return next.handle(authReq);
  }
}

11. Observables and RxJS

Q18: What is an Observable?

Observable is a representation of any set of values over any amount of time. It's the primary data structure in Angular for handling asynchronous operations.

Key Operators:

  • map: Transform values
  • filter: Filter values
  • switchMap: Cancel previous observable
  • mergeMap: Merge observables
  • debounceTime: Delay emissions
  • distinctUntilChanged: Only emit distinct values
  • catchError: Handle errors
  • tap: Side effects
import { Observable, of } from 'rxjs';
import { map, filter, catchError } from 'rxjs/operators';

this.http
  .get<Item[]>('/api/items')
  .pipe(
    map(items => items.filter(item => item.active)),
    catchError(error => {
      console.error(error);
      return of([]);
    })
  )
  .subscribe(items => {
    this.items = items;
  });

12. Change Detection

Q20: How does Angular change detection work?

Angular uses zone.js to detect changes. When an event occurs (click, HTTP response, timer), Zone triggers change detection. Angular then checks all components for changes.

Change Detection Strategies:

  • Default: Checks entire component tree on every change
  • OnPush: Only checks when input references change or events occur in component
@Component({
  selector: 'app-product',
  changeDetection: ChangeDetectionStrategy.OnPush
})
export class ProductComponent {
  @Input() product: Product;
}
Tip: Use OnPush for better performance in large applications. Combine with immutable data patterns.

Q25: What is the difference between Subject and BehaviorSubject?
  • Subject: Basic observable that multicasts to multiple observers. No initial value.
  • BehaviorSubject: Subject that has an initial value and emits the current value to new subscribers.
// Subject
const subject = new Subject<string>();
subject.next('hello');
subject.subscribe(value => console.log(value)); // Won't receive 'hello'

// BehaviorSubject
const behaviorSubject = new BehaviorSubject<string>('initial');
behaviorSubject.next('hello');
behaviorSubject.subscribe(value => console.log(value)); // Receives 'hello'
Q26: What is the difference between switchMap and mergeMap?
  • switchMap: Cancels previous observable when new value arrives. Good for search, type-ahead.
  • mergeMap: Keeps all observables active. Good for parallel operations.
// switchMap - cancels previous

).subscribe(results => {
  this.results = results;
});

// mergeMap - keeps all active
ids.pipe(
  mergeMap(id => this.http.getItem(id))
).subscribe(item => {
  this.items.push(item);
});

Q33: How do you use Angular Signals for state management?

Angular Signals provide a reactive way to manage state with fine-grained reactivity. They are simpler than RxJS and work well for component state.


import { signal, computed, effect, Injectable } from '@angular/core';

@Injectable({ providedIn: 'root' })
export class ProductService {
  // Writable signal
  private products = signal<Product[]>([]);
  private loading = signal(false);
  private error = signal('');

  // Computed signal
  readonly products$ = this.products.asReadonly();
  readonly loading$ = this.loading.asReadonly();
  readonly error$ = this.error.asReadonly();

  readonly productCount = computed(() => this.products().length);
  readonly expensiveProducts = computed(() =>
    this.products().filter(p => p.price > 100)
  );

  // Effect
  constructor() {
    effect(() => {
      if (this.error()) {
        console.error('Product error:', this.error());
      }
    });
  }

  async loadProducts() {
    this.loading.set(true);
    try {
      const data = await this.http.get<Product[]>('/api/products').toPromise();
      this.products.set(data);
    } catch (err) {
      this.error.set(err.message);
    } finally {
      this.loading.set(false);
    }
  }

  addProduct(product: Product) {
    this.products.update(products => [...products, product]);
  }

  updateProduct(id: number, updates: Partial<Product>) {
    this.products.update(products =>
      products.map(p => p.id === id ? { ...p, ...updates } : p)
    );
  }
}

// Usage in component
@Component({
  selector: 'app-products',
  template: `
    <div *ngIf="loading()">Loading...</div>
    <div *ngIf="error()">{{ error() }}</div>
    <div>Total: {{ productCount() }}</div>
    <div *ngFor="let product of products()">{{ product.name }}</div>
  `
})
export class ProductsComponent {
  products = this.productService.products$;
  loading = this.productService.loading$;
  productCount = this.productService.productCount;

  constructor(private productService: ProductService) {
    this.productService.loadProducts();
  }
}
Q34: How do you implement service-based state management with RxJS?

Service-based state management with RxJS uses BehaviorSubjects to hold state and expose it as Observables. This is a common pattern for managing shared state across components.


import { Injectable } from '@angular/core';
import { BehaviorSubject, Observable, tap, catchError } from 'rxjs';

@Injectable({ providedIn: 'root' })
export class CartService {
  private cartItems = new BehaviorSubject<CartItem[]>([]);
  cartItems$ = this.cartItems.asObservable();

  private total = new BehaviorSubject<number>(0);
  total$ = this.total.asObservable();

  constructor(private http: HttpClient) {
    this.loadCart();
  }

  private loadCart() {
    this.http.get<CartItem[]>('/api/cart').subscribe(items => {
      this.cartItems.next(items);
      this.calculateTotal();
    });
  }

  addToCart(item: CartItem): Observable<CartItem[]> {
    return this.http.post<CartItem[]>('/api/cart', item).pipe(
      tap(items => {
        this.cartItems.next(items);
        this.calculateTotal();
      }),
      catchError(error => {
        console.error('Error adding to cart:', error);
        throw error;
      })
    );
  }

  removeFromCart(itemId: number): Observable<CartItem[]> {
    return this.http.delete<CartItem[]>(`/api/cart/${itemId}`).pipe(
      tap(items => {
        this.cartItems.next(items);
        this.calculateTotal();
      })
    );
  }

  private calculateTotal() {
    const currentItems = this.cartItems.value;
    const total = currentItems.reduce((sum, item) => sum + item.price * item.quantity, 0);
    this.total.next(total);
  }

  clearCart() {
    this.cartItems.next([]);
    this.total.next(0);
  }
}

  name: 'truncate',
  pure: true // Set to false for impure pipe (runs on every change detection)
})
export class TruncatePipe implements PipeTransform {
  transform(value: string, limit: number = 20, trail: string = '...'): string {
    if (!value) return '';
    return value.length > limit ? value.substring(0, limit) + trail : value;
  }
}

@Pipe({
  name: 'asyncError',
  pure: false
})
export class AsyncErrorPipe implements PipeTransform {
  transform(obs: Observable<any>): any {
    let latestValue: any;
    let error: any;

    obs.subscribe({
      next: value => latestValue = value,
      error: err => error = err
    });

    if (error) throw error;
    return latestValue;
  }
}

        } else if (error.status === 500) {
          // Handle server error
          this.showError('Server error occurred');
        }
        return throwError(() => error);
      })
    );
  }
}

export class AuthInterceptor implements HttpInterceptor {
  constructor(private authService: AuthService) {}

  intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    const token = this.authService.getToken();
    
    if (token) {
      req = req.clone({
        setHeaders: {
          'Authorization': `Bearer ${token}`
        }
      });
    }

      catchError(error => {

          this.authService.logout();
        }
        return throwError(() => error);
      })
    );
  }

export class DynamicFormService {
  buildForm(controls: FormControlConfig[]): FormGroup {
    const group: any = {};
    
    controls.forEach(config => {
      group[config.name] = this.createControl(config);
    });
    
    return new FormGroup(group);
  }
  
  private createControl(config: FormControlConfig): AbstractControl {
    let validators = [];
    let asyncValidators = [];
    
    if (config.required) validators.push(Validators.required);
    if (config.minLength) validators.push(Validators.minLength(config.minLength));
    if (config.maxLength) validators.push(Validators.maxLength(config.maxLength));
    if (config.pattern) validators.push(Validators.pattern(config.pattern));
    if (config.email) validators.push(Validators.email);
    
    if (config.uniqueEmail) {
      asyncValidators.push(uniqueEmailValidator(this.userService));
    }
    
    return new FormControl(config.value || '', validators, asyncValidators);
  }
}
