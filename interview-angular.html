1. Angular Fundamentals

Q1: What is Angular and what are its key features?

Angular is a TypeScript-based open-source web application framework led by Google. It's a complete rewrite of AngularJS (Angular 1.x).

Key Features:

  • TypeScript: Built with TypeScript for better tooling and type safety
  • Component-based: Everything is a component
  • Dependency Injection: Built-in DI system
  • Two-way data binding: Automatic synchronization of data
  • Directives: Extend HTML with custom attributes
  • Pipes: Transform data in templates
  • Routing: Client-side routing
  • Forms: Template-driven and reactive forms
  • HTTP Client: Built-in HTTP service
  • Testing: Comprehensive testing support

2. Components and Templates

Q3: What is an Angular component?

A Component is the fundamental building block of Angular applications. It consists of:

  • Template: HTML view
  • Class: TypeScript class with logic
  • Styles: CSS styles
  • Metadata: Decorator that configures the component
@Component({ selector: 'app-product', templateUrl: './product.component.html', styleUrls: ['./product.component.css'] }) export class ProductComponent implements OnInit { @Input() product: Product; @Output() productSelected = new EventEmitter<Product>(); constructor(private productService: ProductService) {} ngOnInit(): void { this.loadProduct(); } onSelect(): void { this.productSelected.emit(this.product); } }
Q4: What is the component lifecycle?

Angular components have lifecycle hooks that allow you to tap into key moments:

  • ngOnChanges: Called when input properties change
  • ngOnInit: Called after constructor, when inputs are set
  • ngDoCheck: Called during every change detection
  • ngAfterContentInit: Called after content projection
  • ngAfterContentChecked: Called after content check
  • ngAfterViewInit: Called after view initialization
  • ngAfterViewChecked: Called after view check
  • ngOnDestroy: Called before component destruction

4. Directives

Q6: What are the different types of directives?
  • Components: Directives with a template
  • Attribute directives: Change appearance or behavior of elements
  • Structural directives: Change DOM layout (add/remove elements)

5. Pipes

Q8: What are pipes and how do you use them?

Pipes transform data in templates. Angular provides built-in pipes and allows custom pipes.

Built-in Pipes:

  • DatePipe - date formatting
  • UpperCasePipe / LowerCasePipe - case transformation
  • CurrencyPipe - currency formatting
  • DecimalPipe - number formatting
  • JsonPipe - JSON stringify
  • SlicePipe - array slicing
  • AsyncPipe - unwrap observables
// Built-in pipes {{ birthday | date:'shortDate' }} {{ name | uppercase }} {{ price | currency:'USD' }} {{ value | number:'1.2-2' }} {{ object | json }} {{ items | slice:0:5 }} {{ observable$ | async }} // Custom pipe import { Pipe, PipeTransform } from '@angular/core'; @Pipe({ name: 'truncate' }) export class TruncatePipe implements PipeTransform { transform(value: string, limit: number): string { return value.length > limit ? value.substring(0, limit) + '...' : value; } } // Usage {{ description | truncate:50 }}

6. Services and Dependency Injection

Q9: What is dependency injection in Angular?

Dependency Injection (DI) is a design pattern where dependencies are provided to a class rather than the class creating them. Angular has a built-in DI system.

Providing Services:

  • root: Singleton across the app
  • any: Singleton per module
  • Component: New instance per component
// Service @Injectable({ providedIn: 'root' }) export class ProductService { constructor(private http: HttpClient) {} getProducts(): Observable<Product[]> { return this.http.get<Product[]>('/api/products'); } } // Component @Component({...}) export class ProductComponent { constructor(private productService: ProductService) {} }
Q10: What are the different injectors in Angular?
  • Root injector: App-level injector
  • Module injector: NgModule-level injector
  • Element injector: Component-level injector
  • Platform injector: Platform-level injector
Important: Angular's DI system follows a hierarchical injector tree. When a dependency is requested, Angular searches up the injector tree until it finds the provider.

7. Modules

Q11: What is an NgModule?

NgModule is a class decorated with @NgModule that groups related components, directives, pipes, and services.

@NgModule({ declarations: [AppComponent, ProductComponent], imports: [BrowserModule, HttpClientModule, AppRoutingModule], providers: [ProductService], bootstrap: [AppComponent] }) export class AppModule { }

NgModule Properties:

  • declarations: Components, directives, pipes belonging to this module
  • imports: Other modules whose exports are needed
  • exports: Components, directives, pipes to be used by other modules
  • providers: Services to be provided at module level
  • bootstrap: Root component to bootstrap
Q12: What is the difference between declarations and imports?
  • declarations: Components, directives, and pipes that belong to this module. They must be declared in exactly one module.
  • imports: Other modules whose exported components, directives, and pipes are needed in this module's templates.

8. Routing

Q13: How does Angular routing work?

Angular's Router enables navigation from one view to another as users perform application tasks.

// Route configuration const routes: Routes = [ { path: '', component: HomeComponent }, { path: 'products', component: ProductsComponent }, { path: 'products/:id', component: ProductDetailComponent }, { path: '**', component: NotFoundComponent } ]; @NgModule({ imports: [RouterModule.forRoot(routes)], exports: [RouterModule] }) export class AppRoutingModule { } // Router outlet <router-outlet></router-outlet> // Navigation <a routerLink="/products">Products</a> <a [routerLink]="['/products', product.id]">{{ product.name }}</a> // Programmatic navigation constructor(private router: Router) {} navigateToProduct(id: number) { this.router.navigate(['/products', id]); }
Q14: What are route guards?

Route guards control navigation to and from routes.

  • CanActivate: Can a user navigate to a route?
  • CanActivateChild: Can a user navigate to child routes?
  • CanDeactivate: Can a user navigate away from a route?
  • CanLoad: Can a module be loaded?
  • CanMatch: Can a route match?
@Injectable({ providedIn: 'root' }) export class AuthGuard implements CanActivate { constructor(private authService: AuthService, private router: Router) {} canActivate(): boolean { if (this.authService.isLoggedIn()) { return true; } this.router.navigate(['/login']); return false; } } // Usage { path: 'admin', component: AdminComponent, canActivate: [AuthGuard] }

11. Observables and RxJS

Q18: What is an Observable?

Observable is a representation of any set of values over any amount of time. It's the primary data structure in Angular for handling asynchronous operations.

Key Operators:

  • map: Transform values
  • filter: Filter values
  • switchMap: Cancel previous observable
  • mergeMap: Merge observables
  • debounceTime: Delay emissions
  • distinctUntilChanged: Only emit distinct values
  • catchError: Handle errors
  • tap: Side effects
`nimport { Observable, of } from 'rxjs'; import { map, filter, catchError } from 'rxjs/operators'; this.http.get<Item[]>('/api/items') .pipe( map(items => items.filter(item => item.active)), catchError(error => { console.error(error); return of([]); }) ) .subscribe(items => { this.items = items; });

12. Change Detection

Q20: How does Angular change detection work?

Angular uses zone.js to detect changes. When an event occurs (click, HTTP response, timer), Zone triggers change detection. Angular then checks all components for changes.

Change Detection Strategies:

  • Default: Checks entire component tree on every change
  • OnPush: Only checks when input references change or events occur in component
@Component({ selector: 'app-product', changeDetection: ChangeDetectionStrategy.OnPush }) export class ProductComponent { @Input() product: Product; }
Tip: Use OnPush for better performance in large applications. Combine with immutable data patterns.

13. Common Interview Questions

Q21: What is the difference between @Input and @Output?
  • @Input: Receives data from parent component to child component
  • @Output: Sends data from child component to parent component via events
// Parent <app-child [data]="parentData" (dataChanged)="onDataChanged($event)"></app-child> // Child @Input() data: any; @Output() dataChanged = new EventEmitter<any>(); onChange() { this.dataChanged.emit(this.data); }
Q22: What is Content Projection?

Content Projection allows you to project content from a parent component into a child component using ng-content.

// Child component <div class="card"> <ng-content select="header"></ng-content> <ng-content></ng-content> <ng-content select="footer"></ng-content> </div> // Parent component <app-card> <header>Card Title</header> <p>Card content</p> <footer>Card footer</footer> </app-card>
Q23: What is the difference between ViewChild and ContentChild?
  • ViewChild: Access child components or elements defined in the component's template
  • ContentChild: Access projected content (ng-content) from parent
@ViewChild(ChildComponent) childComponent: ChildComponent; @ViewChild('myInput') inputElement: ElementRef; @ContentChild(ProjectedComponent) projectedComponent: ProjectedComponent; ngAfterViewInit() { this.childComponent.doSomething(); this.inputElement.nativeElement.focus(); }
Q24: What is a pure pipe vs impure pipe?
  • Pure pipe: Only re-evaluates when input reference changes. Default behavior.
  • Impure pipe: Re-evaluates on every change detection cycle. Use with caution for performance.
// Pure (default) @Pipe({ name: 'truncate', pure: true }) // Impure @Pipe({ name: 'async', pure: false })
Q25: What is the difference between Subject and BehaviorSubject?
  • Subject: Basic observable that multicasts to multiple observers. No initial value.
  • BehaviorSubject: Subject that has an initial value and emits the current value to new subscribers.
// Subject const subject = new Subject<string>(); subject.next('hello'); subject.subscribe(value => console.log(value)); // Won't receive 'hello' // BehaviorSubject const behaviorSubject = new BehaviorSubject<string>('initial'); behaviorSubject.next('hello'); behaviorSubject.subscribe(value => console.log(value)); // Receives 'hello'
Q26: What is the difference between switchMap and mergeMap?
  • switchMap: Cancels previous observable when new value arrives. Good for search, type-ahead.
  • mergeMap: Keeps all observables active. Good for parallel operations.
// switchMap - cancels previous searchTerms.pipe( switchMap(term => this.http.search(term)) ).subscribe(results => { this.results = results; }); // mergeMap - keeps all active ids.pipe( mergeMap(id => this.http.getItem(id)) ).subscribe(item => { this.items.push(item); });
Q27: What is lazy loading in Angular?

Lazy loading loads feature modules on demand, improving initial load time.

// Route configuration const routes: Routes = [ { path: 'products', loadChildren: () => import('./products/products.module').then(m => m.ProductsModule) } ]; // Products module @NgModule({ imports: [RouterModule.forChild([ { path: '', component: ProductsComponent } ])] }) export class ProductsModule { }
Q28: What is the difference between ngOnInit and constructor?
  • Constructor: Used for dependency injection. Don't do heavy initialization here.
  • ngOnInit: Called after constructor when inputs are set. Use for initialization logic.
Important: Constructor is for DI, ngOnInit is for initialization. Always use ngOnInit for component initialization.
Q29: What is the difference between ng-container and ng-template?
  • ng-container: Grouping element that doesn't render to DOM. Used with structural directives.
  • ng-template: Template that doesn't render by default. Used with ngIf, ngFor, or ngTemplateOutlet.
// ng-container <ng-container *ngIf="condition"> <p>Content</p> </ng-container> // ng-template <ng-template #myTemplate> <p>Template content</p> </ng-template> <ng-container *ngTemplateOutlet="myTemplate"></ng-container>

Q33: How do you use Angular Signals for state management?

Angular Signals provide a reactive way to manage state with fine-grained reactivity. They are simpler than RxJS and work well for component state.

import { signal, computed, effect, Injectable } from '@angular/core'; @Injectable({ providedIn: 'root' }) export class ProductService { // Writable signal private products = signal<Product[]>([]); private loading = signal(false); private error = signal(''); // Computed signal readonly products$ = this.products.asReadonly(); readonly loading$ = this.loading.asReadonly(); readonly error$ = this.error.asReadonly(); readonly productCount = computed(() => this.products().length); readonly expensiveProducts = computed(() => this.products().filter(p => p.price > 100) ); // Effect constructor() { effect(() => { if (this.error()) { console.error('Product error:', this.error()); } }); } async loadProducts() { this.loading.set(true); try { const data = await this.http.get<Product[]>('/api/products').toPromise(); this.products.set(data); } catch (err) { this.error.set(err.message); } finally { this.loading.set(false); } } addProduct(product: Product) { this.products.update(products => [...products, product]); } updateProduct(id: number, updates: Partial<Product>) { this.products.update(products => products.map(p => p.id === id ? { ...p, ...updates } : p) ); } } // Usage in component @Component({ selector: 'app-products', template: ` <div *ngIf="loading()">Loading...</div> <div *ngIf="error()">{{ error() }}</div> <div>Total: {{ productCount() }}</div> <div *ngFor="let product of products()">{{ product.name }}</div> ` }) export class ProductsComponent { products = this.productService.products$; loading = this.productService.loading$; productCount = this.productService.productCount; constructor(private productService: ProductService) { this.productService.loadProducts(); } }
Q34: How do you implement service-based state management with RxJS?

Service-based state management with RxJS uses BehaviorSubjects to hold state and expose it as Observables. This is a common pattern for managing shared state across components.

import { Injectable } from '@angular/core'; import { BehaviorSubject, Observable, tap, catchError } from 'rxjs'; @Injectable({ providedIn: 'root' }) export class CartService { private cartItems = new BehaviorSubject<CartItem[]>([]); cartItems$ = this.cartItems.asObservable(); private total = new BehaviorSubject<number>(0); total$ = this.total.asObservable(); constructor(private http: HttpClient) { this.loadCart(); } private loadCart() { this.http.get<CartItem[]>('/api/cart').subscribe(items => { this.cartItems.next(items); this.calculateTotal(); }); } addToCart(item: CartItem): Observable<CartItem[]> { return this.http.post<CartItem[]>('/api/cart', item).pipe( tap(items => { this.cartItems.next(items); this.calculateTotal(); }), catchError(error => { console.error('Error adding to cart:', error); throw error; }) ); } removeFromCart(itemId: number): Observable<CartItem[]> { return this.http.delete<CartItem[]>(`/api/cart/${itemId}`).pipe( tap(items => { this.cartItems.next(items); this.calculateTotal(); }) ); } private calculateTotal() { const currentItems = this.cartItems.value; const total = currentItems.reduce((sum, item) => sum + item.price * item.quantity, 0); this.total.next(total); } clearCart() { this.cartItems.next([]); this.total.next(0); } }

15. Performance Optimization (Expert Level)

Q39: How do you optimize Angular application performance?
  • Change Detection Strategy: Use OnPush for components with immutable data
  • TrackBy in ngFor: Improve list rendering performance
  • Lazy Loading: Load modules on demand
  • Preloading Strategy: Preload modules in background
  • Avoid Memory Leaks: Unsubscribe from Observables
  • Virtual Scrolling: Use CDK virtual scroll for large lists
  • Production Build: Use AOT compilation and tree-shaking
  • Bundle Analysis: Use webpack-bundle-analyzer
  • Service Workers: Use PWA for caching
  • Server-Side Rendering: Use Angular Universal for SEO
Q40: How do you implement OnPush change detection?
`r`nimport { Component, ChangeDetectionStrategy, Input } from '@angular/core'; @Component({ selector: 'app-product-item', template: ` <div class="product"> <h3>{{ product.name }}</h3> <p>{{ product.price | currency }}</p> </div> `, changeDetection: ChangeDetectionStrategy.OnPush }) export class ProductItemComponent { @Input() product: Product; // Component only checks for changes when: // 1. Input reference changes // 2. Event from component or its children // 3. Values Emits single value Emits multiple values over time

Operators No operators Rich set of operators (map, filter, etc.)

Q54: What is the difference between Subject and BehaviorSubject?
  • Subject: Basic multicast Observable. Subscribers only receive values emitted after subscription.
  • BehaviorSubject: Has initial value. New subscribers receive the latest value immediately upon subscription.
  • ReplaySubject: Can replay N previous values to new subscribers.
  • AsyncSubject: Emits last value only when complete.
Subject const subject = new Subject<number>(); subject.next(1); subject.subscribe(x => console.log(x)); // Nothing subject.next(2); // Logs: 2 // BehaviorSubject const behavior = new BehaviorSubject<number>(0); behavior.next(1); behavior.subscribe(x => console.log(x)); // Logs: 1 behavior.next(2); // Logs: 2
