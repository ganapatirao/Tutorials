Chapter 1: Introduction to AWS

1.1 What is AWS?

Amazon Web Services (AWS) is a comprehensive cloud computing platform provided by Amazon that offers over 200 fully featured services from data centers globally.

1.2 AWS Regions and Availability Zones

  • Region: A geographic area with multiple isolated data centers
  • Availability Zone: One or more discrete data centers within a region
  • Edge Location: Site for CloudFront content delivery

1.3 AWS Pricing Models

  • On-Demand: Pay for what you use, no commitments
  • Reserved: Commit to 1-3 years for significant discount
  • Spot Instances: Bid for unused capacity at low prices
  • Savings Plans: Flexible pricing model with discounts

Chapter 4: RDS (Relational Database Service)

4.1 Supported Database Engines

  • MySQL
  • PostgreSQL
  • MariaDB
  • Oracle
  • SQL Server
  • Aurora (AWS proprietary)

4.2 RDS Features

  • Multi-AZ: High availability with standby replica
  • Read Replicas: Scale read operations
  • Automated Backups: Point-in-time recovery
  • Encryption: At rest and in transit
  • Performance Insights: Performance monitoring

Chapter 5: VPC (Virtual Private Cloud)

5.1 VPC Components

  • VPC: Isolated network in AWS cloud
  • Subnet: Segmentation of IP address range
  • Route Table: Controls network traffic routing
  • Internet Gateway: Enable internet access
  • NAT Gateway: Enable outbound internet access
  • Security Groups: Stateful firewall
  • Network ACLs: Stateless firewall

5.2 VPC Design Patterns

  • Public Subnet: Resources with public IP addresses
  • Private Subnet: Resources without public IP addresses
  • VPC Peering: Connect VPCs in same or different regions
  • VPC Endpoints: Private connectivity to AWS services

Chapter 6: Lambda (Serverless Computing)

6.1 Lambda Concepts

  • Function: Code that runs in response to events
  • Trigger: Event source that invokes function
  • Runtime: Programming language environment
  • Layers: Libraries and dependencies
  • Concurrency: Simultaneous executions

6.2 Lambda Use Cases

  • Web applications with API Gateway
  • Data processing with S3 and DynamoDB
  • Real-time file processing
  • IoT backend processing
  • Chatbots and voice assistants

Chapter 7: IAM (Identity and Access Management)

7.1 IAM Components

  • User: Entity that represents a person or application
  • Group: Collection of users
  • Role: Identity with permissions for AWS services
  • Policy: Document that defines permissions
  • MFA: Multi-factor authentication

7.2 IAM Best Practices

  • Use MFA for root account
  • Use IAM roles instead of access keys
  • Apply least privilege principle
  • Rotate credentials regularly
  • Use AWS Organizations for multi-account management

Chapter 8: CloudFormation (Infrastructure as Code)

8.1 CloudFormation Basics

CloudFormation allows you to model and set up AWS resources using templates written in YAML or JSON.

8.2 Template Structure

AWSTemplateFormatVersion: '2010-09-09'
Description: Sample template

Resources:
  MyEC2Instance:
    Type: AWS::EC2::Instance
    Properties:
      InstanceType: t2.micro
      ImageId: ami-0c55b159cbfafe1f0

Outputs:
  InstanceId:
    Value: !Ref MyEC2Instance

9.1 AWS Security Services

  • Shield: DDoS protection

Chapter 10: Cost Optimization

10.1 Cost Optimization Strategies

  • Use appropriate instance types
  • Implement auto-scaling
  • Use reserved instances for steady workloads
  • Use spot instances for fault-tolerant workloads
  • Review and delete unused resources
  • Use S3 lifecycle policies
  • Enable CloudWatch billing alerts
