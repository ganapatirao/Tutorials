Chapter 1: Introduction to Docker

1.1 What is Docker?

Docker is a platform that enables developers to build, ship, and run applications in containers. Containers are lightweight, standalone packages that include everything needed to run an application: code, runtime, system tools, libraries, and settings.

1.2 Containers vs VMs

  • VMs: Full OS with hypervisor, heavier, slower startup
  • Containers: Share host OS kernel, lighter, faster startup

1.3 Benefits of Docker

  • Consistent environments across development and production
  • Rapid application deployment
  • Resource efficiency
  • Isolation and security
  • Scalability

Chapter 2: Docker Architecture

2.1 Docker Components

  • Docker Daemon: Background service managing containers
  • Docker CLI: Command-line interface to interact with Docker
  • Docker Registry: Storage for Docker images (Docker Hub)
  • Docker Client: User interface to Docker Daemon

2.2 Docker Objects

  • Images: Read-only templates for containers
  • Containers: Running instances of images
  • Services: Define desired state of containers
  • Stacks: Group of interrelated services

Chapter 3: Docker Installation

3.1 Installing Docker

# Ubuntu/Debian
sudo apt-get update
sudo apt-get install docker.io

# Windows/Mac
# Download Docker Desktop from docker.com

# Verify installation
docker --version
docker run hello-world

3.2 Docker Commands

# List running containers
docker ps

# List all containers
docker ps -a

# List images
docker images

# Pull an image
docker pull nginx:latest

# Run a container
docker run -d -p 80:80 --name web nginx

# Stop a container
docker stop web

# Remove a container
docker rm web

# Remove an image
docker rmi nginx

Chapter 4: Docker Images

4.1 Building Images

# Build from Dockerfile
docker build -t myapp:1.0 .

# Build with build arguments
docker build --build-arg VERSION=1.0 -t myapp:1.0 .

# Build from remote Dockerfile
docker build -t myapp https://github.com/user/repo.git

4.2 Image Layers

  • Each instruction in Dockerfile creates a layer
  • Layers are cached for faster builds
  • Final image is combination of all layers

Chapter 5: Dockerfile

5.1 Dockerfile Instructions

# Base image
FROM mcr.microsoft.com/dotnet/sdk:7.0 AS build

# Working directory
WORKDIR /src

# Copy files
COPY ["MyApp.csproj", "./"]

# Restore dependencies
RUN dotnet restore

# Copy all files
COPY . .

# Build application
RUN dotnet build -c Release

# Runtime image
FROM mcr.microsoft.com/dotnet/aspnet:7.0 AS runtime
WORKDIR /app
COPY --from=build /src/publish .

# Expose port
EXPOSE 80

# Entry point
ENTRYPOINT ["dotnet", "MyApp.dll"]

5.2 Best Practices

  • Use official base images
  • Minimize number of layers
  • Use multi-stage builds
  • Leverage build cache
  • Don't store sensitive data in images

Chapter 6: Docker Compose

6.1 Compose File

version: '3.8'

services:
  web:
    build: .
    ports:
      - "80:80"
    depends_on:
      - db
    environment:
      - ConnectionString=Server=db;Database=MyDb
  
  db:
    image: mcr.microsoft.com/mssql/server:2022-latest
    environment:
      - ACCEPT_EULA=Y
      - SA_PASSWORD=YourPassword123
    volumes:
      - dbdata:/var/opt/mssql

volumes:
  dbdata:

6.2 Compose Commands

# Start services
docker-compose up -d

# Stop services
docker-compose down

# View logs
docker-compose logs -f

# Scale services
docker-compose up -d --scale web=3

Chapter 7: Docker Networks

7.1 Network Drivers

  • Bridge: Default for standalone containers
  • Host: No network isolation
  • Overlay: For multi-host networking
  • Macvlan: Assign MAC addresses to containers

7.2 Network Commands

# Create network
docker network create mynetwork

# Connect container to network
docker network connect mynetwork mycontainer

# List networks
docker network ls

# Inspect network
docker network inspect mynetwork

Chapter 8: Docker Volumes

8.1 Volume Types

  • Named volumes: Managed by Docker
  • Bind mounts: Mount host directory into container
  • tmpfs mounts: Stored in host memory

8.2 Volume Commands

# Create volume
docker volume create myvolume

# Mount volume
docker run -v myvolume:/data myapp

# List volumes
docker volume ls

# Remove volume
docker volume rm myvolume

Chapter 9: Docker Security

9.1 Security Best Practices

  • Use non-root user in containers
  • Scan images for vulnerabilities
  • Use content trust
  • Limit container capabilities
  • Use secrets management

9.2 Security Scanning

# Scan image with Trivy
trivy image myapp:1.0

# Scan with Docker Scout
docker scout quickview myapp:1.0

Chapter 10: Docker Best Practices

10.1 General Best Practices

  • Keep images small
  • Use specific tags, not latest
  • Implement health checks
  • Use resource limits
  • Monitor container performance
Important: Never run containers as root user. Always create a non-root user in your Dockerfile for better security.
