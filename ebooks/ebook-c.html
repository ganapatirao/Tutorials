Chapter 1: Introduction to C

1.1 What is C?

C is a general-purpose, procedural programming language created by Dennis Ritchie at Bell Labs in 1972. It is one of the most widely used programming languages of all time and has influenced many other languages including C++, Java, and Python.

1.2 History of C

C was developed as a system programming language for the Unix operating system. It evolved from an earlier language called B, which was developed by Ken Thompson. C has gone through several standards:

  • K&R C (1978) - Original version by Kernighan and Ritchie
  • C89/C90 (1989/1990) - First ANSI standard
  • C99 (1999) - Added new features like inline functions
  • C11 (2011) - Added multi-threading support
  • C17 (2018) - Latest standard

1.3 Why Learn C?

  • Foundation: Basis for many modern programming languages
  • Performance: Produces fast and efficient code
  • Portability: Code can run on different platforms
  • Low-level Access: Direct memory manipulation via pointers
  • System Programming: Ideal for operating systems and embedded systems
  • Industry Standard: Widely used in embedded systems, game development, and more

1.4 C Applications

  • Operating Systems (Unix, Linux, Windows)
  • Embedded Systems (Microcontrollers, IoT devices)
  • Database Systems (MySQL, PostgreSQL)
  • Game Development (Game engines, game logic)
  • Compilers and Interpreters
  • Device Drivers

2.3 Compilation Process

C source code goes through several stages:

  1. Preprocessing: Handles #include and #define directives
  2. Compilation: Converts source code to assembly code
  3. Assembly: Converts assembly code to machine code (object file)

Chapter 4: Operators

4.1 Arithmetic Operators

int a = 10, b = 3;

printf("%d\n", a + b); // Addition: 13
printf("%d\n", a - b); // Subtraction: 7
printf("%d\n", a * b); // Multiplication: 30
printf("%d\n", a / b); // Division: 3 (integer division)
printf("%d\n", a % b); // Modulus: 1

4.2 Relational Operators

int a = 10, b = 5;

printf("%d\n", a == b); // Equal: 0 (false)
printf("%d\n", a != b); // Not equal: 1 (true)
printf("%d\n", a > b);  // Greater than: 1 (true)
printf("%d\n", a < b);  // Less than: 0 (false)
printf("%d\n", a >= b); // Greater or equal: 1 (true)
printf("%d\n", a <= b); // Less or equal: 0 (false)

4.3 Logical Operators

int a = 1, b = 0;

printf("%d\n", a && b); // Logical AND: 0
printf("%d\n", a || b); // Logical OR: 1
printf("%d\n", !a);     // Logical NOT: 0

4.4 Increment/Decrement Operators

int x = 5;

// Pre-increment
int y = ++x; // x becomes 6, y becomes 6

// Post-increment
int z = x++; // z becomes 6, x becomes 7

// Pre-decrement
int w = --x; // x becomes 6, w becomes 6

// Post-decrement
int v = x--; // v becomes 6, x becomes 5

6.2 Call by Value vs Call by Reference

// Call by value
void swapByValue(int a, int b)
{
    int temp = a;
    a = b;
    b = temp;
    // Original values unchanged
}

// Call by reference (using pointers)
void swapByReference(int *a, int *b)
{
    int temp = *a;
    *a = *b;
    *b = temp;
    // Original values changed
}

int main()
{
    int x = 10, y = 20;
    
    swapByValue(x, y);
    printf("After swapByValue: x=%d, y=%d\n", x, y); // x=10, y=20
    
    swapByReference(&x, &y);
    printf("After swapByReference: x=%d, y=%d\n", x, y); // x=20, y=10
    
    return 0;
}

6.3 Recursion

// Factorial using recursion
int factorial(int n)
{
    if (n <= 1) // Base case
        return 1;
    return n * factorial(n - 1); // Recursive case
}

// Fibonacci using recursion
int fibonacci(int n)
{
    if (n <= 1)
        return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
Important: Always have a base case in recursion to prevent infinite recursion, which can cause stack overflow.

    for (int j = 0; j < 3; j++)
    {
        printf("%d ", matrix[i][j]);
    }
    printf("\n");
}

// Function with array parameter

{

    {

    printf("\n");
}

// Using pointer notation
void printArrayPointer(int *arr, int size)
{
    for (int i = 0; i < size; i++)
    {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main()
{
    int numbers[] = {1, 2, 3, 4, 5};
    printArray(numbers, 5);
    return 0;
}

Chapter 8: Pointers

8.1 Pointer Basics

int num = 42;
int *ptr;      // Pointer declaration
ptr = #    // Store address of num

printf("Value of num: %d\n", num);        // 42
printf("Address of num: %p\n", &num);     // Address
printf("Value of ptr: %p\n", ptr);        // Address
printf("Value pointed by ptr: %d\n", *ptr); // 42

*ptr = 100;    // Modify value through pointer

int arr[] = {10, 20, 30, 40, 50};

printf("*(ptr+2): %d\n", *(ptr+2)); // 30

ptr++; // Move to next integer
printf("After ptr++: %d\n", *ptr); // 20

8.3 Pointers and Arrays

// Pointer arithmetic

    printf("%d ", *(ptr + i));
}

Chapter 9: Strings

9.1 String Basics

In C, strings are arrays of characters terminated by a null character ('\0').

// String literal
char str1[] = "Hello";
// H e l l o \0

// Manual string
char str2[] = {'H', 'i', '\0'};

// String functions
#include 

char str[] = "Hello";
printf("Length: %zu\n", strlen(str)); // 5
printf("Copy: %s\n", strcpy(str2, str));
printf("Concatenate: %s\n", strcat(str1, str2));
printf("Compare: %d\n", strcmp(str1, str2));
Important: Always ensure strings are null-terminated. Functions like strlen, strcpy, etc. rely on the null terminator.

Chapter 11: Memory Management

11.1 Dynamic Memory Allocation

#include 

// malloc - uninitialized
int *ptr1 = (int*)malloc(5 * sizeof(int));
if (ptr1 == NULL) {
    printf("Memory allocation failed\n");
    return 1;
}

// calloc - initialized to zero
int *ptr2 = (int*)calloc(5, sizeof(int));

// realloc - resize
ptr1 = (int*)realloc(ptr1, 10 * sizeof(int));

// free - deallocate
free(ptr1);
free(ptr2);
Important: Always check if malloc/calloc returns NULL (allocation failed). Always free allocated memory to prevent memory leaks.

11.2 Memory Layout

  • Text Segment: Contains compiled code
  • Initialized Data Segment: Global and static variables
  • Uninitialized Data Segment (BSS): Uninitialized global/static variables
  • Stack: Local variables, function calls
  • Heap: Dynamic memory allocation

    printf("%s", buffer);

12.2 File Modes

Mode Description
"r" Read - opens file for reading (file must exist)
"w" Write - creates new file or truncates existing
"a" Append - opens file for appending (creates if not exists)
"r+" Read/Write - opens for reading and writing (file must exist)
"w+" Read/Write - creates or truncates file
"a+" Read/Append - opens for reading and appending

Chapter 14: Best Practices

14.1 Coding Standards

  • Use meaningful variable and function names
  • Follow consistent naming conventions
  • Add comments to explain complex logic
  • Use functions to break down complex problems
  • Initialize variables before use
  • Check return values from functions
  • Avoid magic numbers - use named constants

14.2 Memory Safety

  • Always free allocated memory
  • Check for NULL after malloc/calloc
  • Avoid buffer overflows
  • Initialize pointers to NULL
  • Use strncpy instead of strcpy
  • Be careful with pointer arithmetic

14.3 Error Handling

// Check file operations
FILE *file = fopen("data.txt", "r");
if (file == NULL) {
    perror("Error opening file");
    return 1;
}

// Check memory allocation
int *ptr = (int*)malloc(size * sizeof(int));
if (ptr == NULL) {
    fprintf(stderr, "Memory allocation failed\n");
    return 1;
}
Tip: Use static analysis tools like clang-tidy, cppcheck, or valgrind to detect memory leaks and other issues.
Important: C gives you great power and flexibility, but with that comes responsibility. Always be careful with memory management and pointer operations.
