Chapter 1: Introduction to Generative AI

1.1 What is Generative AI?

Generative AI refers to artificial intelligence systems that can create new content, including text, images, audio, and code. Unlike traditional AI that analyzes existing data, generative AI learns patterns and generates novel outputs.

Key Insight: Generative AI models don't memorize training data; they learn underlying patterns to create new, original content.

1.2 Types of Generative AI

  • Text Generation: GPT, Claude, LLaMA
  • Image Generation: DALL-E, Midjourney, Stable Diffusion
  • Audio Generation: MusicLM, AudioLM
  • Code Generation: GitHub Copilot, CodeLlama
  • Video Generation: Sora, Runway

1.3 Key Concepts

  • Training: Learning from large datasets
  • Inference: Generating new content
  • Tokens: Basic units of text processing
  • Parameters: Model weights and configurations
  • Context Window: Maximum input length

Chapter 2: Large Language Models

2.1 What are LLMs?

Large Language Models are neural networks trained on vast amounts of text data. They understand and generate human-like text by predicting the next token in a sequence.

2.2 Popular LLMs

  • GPT-4: OpenAI's advanced language model
  • Claude: Anthropic's safety-focused model
  • LLaMA: Meta's open-source model family
  • Mistral: Efficient open-source models

2.3 LLM Capabilities

  • Text generation and completion
  • Question answering
  • Summarization
  • Translation
  • Code generation
  • Reasoning and analysis

Chapter 3: Transformer Architecture

3.1 Attention Mechanism

The attention mechanism allows models to focus on relevant parts of the input when generating each part of the output.

# Simplified attention mechanism
import torch
import torch.nn.functional as F

def attention(query, key, value):
    scores = torch.matmul(query, key.transpose(-2, -1))
    scores = scores / torch.sqrt(torch.tensor(key.size(-1)))
    weights = F.softmax(scores, dim=-1)
    return torch.matmul(weights, value)

3.2 Transformer Components

  • Encoder: Processes input sequence
  • Decoder: Generates output sequence
  • Multi-Head Attention: Multiple attention mechanisms
  • Feed-Forward Networks: Non-linear transformations
  • Positional Encoding: Sequence position information

Chapter 4: Prompt Engineering

4.1 What is Prompt Engineering?

Prompt engineering is the art of crafting effective inputs to guide AI models toward desired outputs.

4.2 Prompt Techniques

  • Zero-shot: No examples provided
  • Few-shot: Provide examples in the prompt
  • Chain-of-Thought: Guide reasoning process
  • Role-playing: Assign specific roles to the AI
# Example prompts
zero_shot = "Explain quantum computing in simple terms."

few_shot = """Translate English to French:
Hello -> Bonjour
Goodbye -> Au revoir
Thank you -> """

chain_of_thought = "Solve step by step: If 3x + 7 = 22, what is x?"

4.3 Best Practices

  • Be specific and clear
  • Provide context and examples
  • Use structured formatting
  • Iterate and refine prompts
  • Test across different models

Chapter 5: Fine-Tuning

5.1 What is Fine-Tuning?

Fine-tuning adapts pre-trained models to specific tasks or domains by training on additional data.

5.2 Fine-Tuning Approaches

  • Full Fine-Tuning: Update all model parameters
  • LoRA: Low-Rank Adaptation for efficient tuning
  • PEFT: Parameter-Efficient Fine-Tuning
  • QLoRA: Quantized LoRA for memory efficiency
# LoRA fine-tuning example
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none"
)

model = AutoModelForCausalLM.from_pretrained("base-model")
model = get_peft_model(model, config)

Chapter 6: RAG (Retrieval-Augmented Generation)

6.1 What is RAG?

RAG combines retrieval systems with generative models to provide accurate, context-aware responses using external knowledge bases.

6.2 RAG Components

  • Document Store: Vector database for embeddings
  • Embedding Model: Converts text to vectors
  • Retriever: Finds relevant documents
  • Generator: Creates responses using retrieved context
# Simple RAG implementation
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings
from langchain.chains import RetrievalQA

# Create vector store
vectorstore = Chroma.from_documents(
    documents,
    OpenAIEmbeddings()
)

# Create RAG chain
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    retriever=vectorstore.as_retriever()
)
