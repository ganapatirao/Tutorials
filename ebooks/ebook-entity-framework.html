Chapter 1: Introduction to Entity Framework

1.1 What is Entity Framework?

Entity Framework (EF) is an Object-Relational Mapper (ORM) that enables .NET developers to work with relational data using domain-specific objects. It eliminates the need for most of the data-access code that developers usually need to write.

1.2 EF Core vs EF6

  • EF Core: Cross-platform, lightweight, modern
  • EF6: Windows-only, full-featured, legacy

1.3 Key Features

  • Model-first and Code-first approaches
  • Change tracking
  • LINQ support
  • Migrations
  • Multiple database providers

    {
        // Fluent API configurations

2.2 Entity Classes

public class Product
{
    public int Id { get; set; }
    public string Name { get; set; }
    public decimal Price { get; set; }
    
    public int CategoryId { get; set; }
    public Category Category { get; set; }
}

Chapter 3: Querying Data

3.1 LINQ Queries

// LINQ syntax
var products = context.Products
    .Where(p => p.Price > 100)
    .OrderBy(p => p.Name)
    .ToList();

// Method syntax
var products = context.Products
    .Where(p => p.Price > 100)
    .OrderBy(p => p.Name)
    .ToList();

// Find by primary key
var product = context.Products.Find(1);

3.2 Loading Related Data

  • Eager loading: Include()
  • Lazy loading: Virtual navigation properties
  • Explicit loading: Load()

Chapter 4: Creating and Updating Data

4.1 CRUD Operations

// Create
var product = new Product { Name = "New Product", Price = 99.99 };
context.Products.Add(product);
context.SaveChanges();

// Update
product.Price = 89.99;
context.SaveChanges();

// Delete
context.Products.Remove(product);
context.SaveChanges();

Chapter 5: Relationships

5.1 Relationship Types

  • One-to-Many: One category, many products
  • Many-to-Many: Products and tags
  • One-to-One: User and profile

5.2 Configuring Relationships

modelBuilder.Entity()
    .HasOne(p => p.Category)
    .WithMany(c => c.Products)
    .HasForeignKey(p => p.CategoryId);

Chapter 6: Migrations

6.1 Creating Migrations

# Add migration
dotnet ef migrations add InitialCreate

# Update database
dotnet ef database update

# Remove migration
dotnet ef migrations remove

Chapter 7: Performance Optimization

7.1 Query Optimization

  • Use AsNoTracking for read-only queries
  • Select only needed columns
  • Use proper indexing
  • Avoid N+1 query problem

Chapter 8: Best Practices

8.1 General Best Practices

  • Use async/await for database operations
  • Dispose DbContext properly
  • Use transactions for multiple operations
  • Implement proper error handling
  • Use connection pooling
Important: Always use async methods for database operations to prevent thread blocking and improve application scalability.
