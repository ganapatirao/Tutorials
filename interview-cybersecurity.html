1. Cybersecurity Fundamentals

Q1: What is the CIA Triad?

CIA Triad is a model for information security consisting of three core principles:

  • Confidentiality: Ensuring information is only accessible to authorized users
  • Integrity: Ensuring information is accurate and unaltered
  • Availability: Ensuring information and systems are available when needed
Q2: What is the difference between authentication and authorization?
Aspect Authentication Authorization
Purpose Verify identity Grant permissions
Question Who are you? What can you do?
First Step Yes No (after auth)
Example Login with password Access control list
Q3: What is the difference between vulnerability, threat, and risk?
  • Vulnerability: Weakness in a system that can be exploited. Example: Unpatched software.
  • Threat: Potential danger to an asset. Example: Hacker, malware.
  • Risk: Likelihood of threat exploiting vulnerability. Risk = Threat × Vulnerability × Impact.

2. Network Security

Q4: What is a firewall?

Firewall is a network security device that monitors and filters incoming and outgoing network traffic based on predetermined security rules.

Types:

  • Packet Filtering Firewall: Filters based on packet headers
  • Stateful Inspection Firewall: Tracks connection state
  • Proxy Firewall: Acts as intermediary
  • Next-Generation Firewall: Includes application awareness
Q5: What is a VPN?

VPN (Virtual Private Network) creates a secure, encrypted connection over a public network, allowing users to access private networks remotely.

Benefits:

  • Encryption: Encrypts data in transit
  • Anonymity: Hides IP address
  • Access: Access resources remotely
  • Security: Protects on public networks

3. Web Application Security

Q7: What is OWASP Top 10?

OWASP Top 10 is a standard awareness document for developers and web application security. It represents a broad consensus about the most critical security risks to web applications.

Current OWASP Top 10:

  1. Broken Access Control
  2. Cryptographic Failures
  3. Injection
  4. Insecure Design
  5. Security Misconfiguration
  6. Vulnerable and Outdated Components
  7. Identification and Authentication Failures
  8. Software and Data Integrity Failures
  9. Security Logging and Monitoring Failures
  10. Server-Side Request Forgery (SSRF)
Q8: What is SQL Injection?

SQL Injection (SQLi) is a vulnerability where malicious SQL code is inserted into a query, allowing attackers to view, modify, or delete data.

# Vulnerable code
query = "SELECT * FROM users WHERE username = '" + username + "'"

# Malicious input
username = "admin' OR '1'='1"

# Resulting query
SELECT * FROM users WHERE username = 'admin' OR '1'='1'

Prevention:

  • Use parameterized queries
  • Use prepared statements
  • Input validation
  • Use ORM frameworks
Q9: What is XSS (Cross-Site Scripting)?

XSS (Cross-Site Scripting) is a vulnerability where attackers inject malicious scripts into web pages viewed by other users.

Types:

  • Stored XSS: Malicious script stored on server
  • Reflected XSS: Malicious script reflected in response
  • DOM-based XSS: Vulnerability in client-side code

Prevention:

  • Output encoding
  • Input validation
  • Content Security Policy (CSP)
  • HTTP-only cookies
Q10: What is CSRF (Cross-Site Request Forgery)?

CSRF (Cross-Site Request Forgery) is an attack where unauthorized commands are transmitted from a user that the website trusts.

Prevention:

  • CSRF tokens
  • SameSite cookie attribute
  • Verify origin headers
  • Custom headers

Q12: What is hashing?

Hashing is a one-way function that converts data of arbitrary size to a fixed-size string. It's used for data integrity and password storage.

Properties:

  • One-way: Cannot reverse hash to original
  • Deterministic: Same input always produces same output
  • Fast to compute: Efficient to generate hash
  • Collision-resistant: Hard to find two inputs with same hash

Common Hash Algorithms:

  • SHA-256, SHA-512
  • MD5 (deprecated)
  • bcrypt, Argon2 (for passwords)
Q13: What is a digital signature?

Digital Signature is a cryptographic technique used to verify the authenticity and integrity of digital messages or documents.

How it works:

  1. Sender creates hash of message
  2. Hash is encrypted with sender's private key
  3. Receiver decrypts with sender's public key
  4. Receiver verifies hash matches message

5. Identity and Access Management

Q14: What is MFA (Multi-Factor Authentication)?

MFA (Multi-Factor Authentication) requires users to provide multiple forms of authentication to verify identity.

Factors:

  • Knowledge: Something you know (password, PIN)
  • Possession: Something you have (token, phone)
  • Inherence: Something you are (biometrics)
Q15: What is OAuth 2.0?

OAuth 2.0 is an authorization framework that enables applications to obtain limited access to user accounts on an HTTP service.

Roles:

  • Resource Owner: User who owns the data
  • Client: Application requesting access
  • Authorization Server: Issues access tokens
  • Resource Server: Hosts protected resources
Q16: What is SAML?

SAML (Security Assertion Markup Language) is an XML-based standard for exchanging authentication and authorization data between parties.

Use Cases:

  • Single Sign-On (SSO)
  • Federation
  • Identity management

6. Security Operations

Q17: What is SIEM?

SIEM (Security Information and Event Management) is a solution that provides real-time analysis of security alerts generated by applications and network hardware.

Components:

  • SIM: Security Information Management (log management)
  • SEM: Security Event Management (real-time monitoring)

Benefits:

  • Centralized log collection
  • Real-time threat detection
  • Compliance reporting
  • Incident response
Q18: What is IDS/IPS?

IDS/IPS (Intrusion Detection/Prevention System) monitors network traffic for suspicious activity and takes action.

Types:

  • Network-based (NIDS/NIPS): Monitors network traffic
  • Host-based (HIDS/HIPS): Monitors individual hosts

Detection Methods:

  • Signature-based: Matches known patterns
  • Anomaly-based: Detects deviations from baseline
Q19: What is incident response?

Incident Response is the process of handling security incidents systematically to minimize damage and recover quickly.

NIST Incident Response Phases:

  1. Preparation: Prepare for incidents
  2. Detection and Analysis: Identify and analyze incidents
  3. Containment: Limit damage
  4. Eradication: Remove threat
  5. Recovery: Restore systems
  6. Lessons Learned: Post-incident review

7. Cloud Security

Q20: What is the Shared Responsibility Model?

Shared Responsibility Model defines security responsibilities between cloud provider and customer based on the service model.

Service Model Provider Responsibility Customer Responsibility
IaaS Physical, network, virtualization OS, applications, data
PaaS Physical, network, virtualization, OS Applications, data
SaaS Everything except data Data, user access
Q21: What is Zero Trust Architecture?

Zero Trust Architecture is a security model based on the principle "never trust, always verify." It assumes no user or device is trusted by default.

Principles:

  • Verify explicitly
  • Use least privilege access
  • Assume breach
Q22: What is DevSecOps?

DevSecOps integrates security practices into the DevOps process, ensuring security is considered throughout the software development lifecycle.

Key Practices:

  • Security as code
  • Automated security testing
  • Shift left security
  • Continuous security monitoring

8. Compliance and Governance

Q23: What is GDPR?

GDPR (General Data Protection Regulation) is a European Union regulation on data protection and privacy.

Key Principles:

  • Lawfulness, fairness, transparency
  • Purpose limitation
  • Data minimization
  • Accuracy
  • Storage limitation
  • Integrity and confidentiality
Q24: What is HIPAA?

HIPAA (Health Insurance Portability and Accountability Act) is a US law that requires the creation of national standards to protect sensitive patient health information.

Key Rules:

  • Privacy Rule: Protects PHI
  • Security Rule: Protects ePHI
  • Breach Notification Rule: Requires breach notification
Q25: What is PCI DSS?

PCI DSS (Payment Card Industry Data Security Standard) is a set of security standards designed to ensure that all companies that accept, process, store, or transmit credit card information maintain a secure environment.

Requirements:

  • Build and maintain secure network
  • Protect cardholder data
  • Maintain vulnerability management program
  • Implement strong access control
  • Regularly monitor and test networks
  • Maintain information security policy

9. Threat Intelligence

Q26: What is threat intelligence?

Threat Intelligence is evidence-based knowledge about existing or emerging threats to assets, including context, mechanisms, indicators, implications, and actionable advice.

Types:

  • Strategic: High-level trends for executives
  • Tactical: TTPs (Tactics, Techniques, Procedures)
  • Operational: Specific campaigns and actors
  • Technical: IOCs (Indicators of Compromise)
Q27: What is MITRE ATT&CK?

MITRE ATT&CK is a globally-accessible knowledge base of adversary tactics and techniques based on real-world observations.

Components:

  • Tactics: Technical objectives (e.g., Initial Access)
  • Techniques: How adversaries achieve tactics
  • Procedures: Specific implementations
Q28: What is CVE?

CVE (Common Vulnerabilities and Exposures) is a list of entries for publicly known cybersecurity vulnerabilities.

Format:

  • CVE-Year-ID (e.g., CVE-2024-1234)
  • Provides unique identifier
  • Standardized description
  • Links to references and fixes

10. Security Architecture

Q29: What is Defense in Depth?

Defense in Depth is a layered security approach where multiple security controls are implemented at different levels to protect against threats.

Layers:

  • Physical: Physical access controls
  • Network: Firewalls, IDS/IPS
  • Endpoint: Antivirus, EDR
  • Application: WAF, secure coding
  • Data: Encryption, DLP
Q30: What is the principle of least privilege?

Principle of Least Privilege (PoLP) states that users and programs should only have the minimum permissions necessary to perform their functions.

Benefits:

  • Reduces attack surface
  • Limits damage from compromised accounts
  • Improves auditability
  • Reduces insider threat risk

11. Penetration Testing

Q31: What is penetration testing?

Penetration Testing (Pen Testing) is a simulated cyberattack against a computer system to check for exploitable vulnerabilities.

Phases:

  1. Reconnaissance: Gather information
  2. Scanning: Identify vulnerabilities
  3. Exploitation: Attempt to exploit
  4. Post-Exploitation: Maintain access
  5. Reporting: Document findings
Q32: What is the difference between black box, white box, and gray box testing?
Type Knowledge Use Case
Black Box No prior knowledge External attacker simulation
White Box Full knowledge Internal audit
Gray Box Partial knowledge Realistic scenario

12. Advanced Topics

Q33: What is Social Engineering?

Social Engineering is the use of deception to manipulate individuals into divulging confidential or personal information.

Types:

  • Phishing: Email-based deception
  • Vishing: Voice-based deception
  • Smishing: SMS-based deception
  • Pretexting: Creating a scenario
  • Baiting: Offering something enticing
Q34: What is Ransomware?

Ransomware is a type of malware that encrypts a victim's files and demands payment to restore access.

Prevention:

  • Regular backups
  • User education
  • Email filtering
  • Endpoint protection
  • Network segmentation
Q35: What is Supply Chain Attack?

Supply Chain Attack targets the supply chain of an organization by compromising a third-party vendor or software component.

Examples:

  • SolarWinds attack
  • Log4j vulnerability
  • Code repository compromise

Q46: What are the best practices for password security?
  • Strong Passwords: Minimum length, complexity
  • Password Hashing: Use bcrypt, Argon2
  • Salt: Use unique salt for each password
  • Multi-Factor: Enable MFA
  • Password Policy: Enforce password policies
  • Password Managers: Encourage password managers
  • Regular Changes: Periodic password rotation
  • Breach Checking: Check against breached passwords
Q47: What are the best practices for secure coding?
  • Input Validation: Validate all inputs
  • Output Encoding: Encode all outputs
  • Parameterized Queries: Prevent SQL injection
  • Error Handling: Don't expose sensitive info
  • Authentication: Implement strong auth
  • Authorization: Implement proper authorization
  • Logging: Log security events
  • Dependencies: Keep dependencies updated
Q48: What are the best practices for network security?
  • Network Segmentation: Segment networks
  • Firewalls: Implement firewalls
  • VPN: Use VPN for remote access
  • IDS/IPS: Deploy IDS/IPS
  • Encryption: Encrypt network traffic
  • Monitoring: Monitor network traffic
  • Access Control: Implement network ACLs
  • Regular Updates: Keep network devices updated
Q49: What are the best practices for cloud security?
  • Identity Management: Implement IAM
  • Encryption: Encrypt data at rest and in transit
  • Network Security: Use VPC, security groups
  • Monitoring: Enable cloud monitoring
  • Compliance: Ensure compliance with regulations
  • Backup: Regular backups
  • Least Privilege: Apply least privilege
