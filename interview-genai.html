1. Generative AI Fundamentals

Q1: What is Generative AI?

Generative AI is a type of artificial intelligence that can create new content, including text, images, audio, and code, by learning patterns from existing data.

Key Characteristics:

  • Content Creation: Generates new, original content
  • Pattern Learning: Learns from large datasets
  • Probabilistic Output: Produces probabilistic results
  • Context Awareness: Understands and maintains context
  • Multi-modal: Can handle text, images, audio, video
Q2: What is the difference between Generative AI and Discriminative AI?
Aspect Generative AI Discriminative AI
Purpose Create new content Classify or predict
Output New data samples Labels or predictions
Examples GPT, DALL-E, Midjourney Classification, Regression
Use Case Content generation Decision making
Q3: What are the types of Generative AI models?
  • Text Generation: GPT, BERT, T5, LLaMA
  • Image Generation: DALL-E, Midjourney, Stable Diffusion
  • Audio Generation: WaveNet, Tacotron, VALL-E
  • Video Generation: Sora, Runway, Pika
  • Code Generation: Codex, CodeLlama, StarCoder
  • Multi-modal: GPT-4V, Gemini, CLIP

2. Large Language Models

Q4: What is a Large Language Model (LLM)?

Large Language Model (LLM) is a neural network trained on vast amounts of text data to understand and generate human-like text.

Key Characteristics:

  • Scale: Billions of parameters
  • Training Data: Trained on internet-scale text
  • Capabilities: Translation, summarization, question answering
  • Few-shot Learning: Can learn from few examples
  • Emergent Abilities: Unexpected capabilities at scale

3. Transformer Architecture

Q7: What is the Transformer architecture?

Transformer is a neural network architecture introduced in "Attention Is All You Need" (2017). It uses self-attention mechanisms to process sequential data.

Key Components:

  • Self-Attention: Weighs importance of different words
  • Multi-Head Attention: Multiple attention heads
  • Positional Encoding: Encodes word position
  • Feed-Forward Networks: Non-linear transformations
  • Layer Normalization: Stabilizes training
Q8: What is self-attention?

Self-Attention is a mechanism that allows the model to weigh the importance of different words in a sequence when processing a particular word.

# Self-attention formula
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k))V

# Where:
# Q = Query matrix
# K = Key matrix
# V = Value matrix
# d_k = Dimension of keys

4. Training and Fine-tuning

Q10: What is pre-training?

Pre-training is the process of training a model on a large corpus of unlabeled data to learn general language patterns and knowledge.

Key Aspects:

  • Large Dataset: Trained on internet-scale text
  • Self-Supervised: No manual labeling required
  • General Knowledge: Learns broad language patterns
  • Computationally Expensive: Requires massive compute
Q11: What is fine-tuning?

Fine-tuning is the process of adapting a pre-trained model to a specific task or domain by training on a smaller, task-specific dataset.

Types of Fine-tuning:

  • Full Fine-tuning: Update all model parameters
  • Partial Fine-tuning: Update only some layers
  • LoRA (Low-Rank Adaptation): Efficient fine-tuning with low-rank matrices
  • QLoRA: Quantized LoRA for memory efficiency
Q12: What is LoRA?

LoRA (Low-Rank Adaptation) is a parameter-efficient fine-tuning method that adds trainable low-rank matrices to the model instead of updating all parameters.

Benefits:

  • Memory Efficient: Reduces memory usage significantly
  • Faster Training: Fewer parameters to update
  • Modular: Can swap LoRA adapters
  • Storage Efficient: Only store LoRA weights

5. Prompt Engineering

Q13: What is prompt engineering?

Prompt Engineering is the art of designing and optimizing prompts to get the best possible output from language models.

Key Techniques:

  • Zero-shot: No examples provided
  • One-shot: One example provided
  • Few-shot: Multiple examples provided
  • Chain-of-Thought: Step-by-step reasoning
  • Role-playing: Assign specific role to model
Q14: What is chain-of-thought prompting?

Chain-of-Thought (CoT) prompting encourages the model to break down complex problems into intermediate steps, improving reasoning performance.

# Example of Chain-of-Thought prompting
Prompt: "Solve this step by step: If a store sells 5 apples for $3, 
how much do 12 apples cost?"

Response: "Step 1: Find the cost per apple. $3 / 5 = $0.60 per apple.
Step 2: Calculate cost for 12 apples. 12 * $0.60 = $7.20.
Answer: 12 apples cost $7.20."

6. RAG (Retrieval-Augmented Generation)

Q16: What is RAG?

RAG (Retrieval-Augmented Generation) is a technique that combines retrieval-based and generation-based approaches by retrieving relevant information from a knowledge base and using it to generate responses.

Components:

  • Retriever: Searches knowledge base for relevant documents
  • Generator: LLM that generates response using retrieved context
  • Knowledge Base: Vector database of documents
  • Embedding Model: Converts text to vector embeddings
Q17: What are the benefits of RAG?
  • Reduced Hallucination: Grounded in retrieved facts
  • Up-to-date Information: Can access latest data
  • Domain Specific: Can use domain-specific knowledge
  • Transparent: Can show source documents
  • Cost Effective: No need to fine-tune for new knowledge

7. Vector Databases

Q19: What is a vector database?

Vector Database is a specialized database designed to store, index, and query high-dimensional vector embeddings efficiently.

Popular Vector Databases:

  • Pinecone: Managed vector database service
  • Weaviate: Open-source vector database
  • Chroma: Open-source embedding database
  • Milvus: Open-source vector database
  • Qdrant: Open-source vector similarity search
Q20: What is an embedding?

Embedding is a numerical representation of text, images, or other data in a high-dimensional vector space where similar items are close together.

Key Characteristics:

  • High-dimensional: Typically 256-4096 dimensions
  • Dense: Most dimensions are non-zero
  • Semantic: Captures semantic meaning
  • Comparable: Can use similarity metrics
Q21: What are the common similarity metrics?
  • Cosine Similarity: Measures angle between vectors. Range: -1 to 1. Most common.
  • Euclidean Distance: Measures straight-line distance. Range: 0 to infinity.
  • Dot Product: Measures magnitude and direction. Range: -infinity to infinity.
  • Manhattan Distance: Sum of absolute differences. Range: 0 to infinity.

8. LLM Deployment and Scaling

Q22: What are the methods for deploying LLMs?
  • API Services: OpenAI API, Anthropic API, Google AI
  • Self-hosting: vLLM, Text Generation Inference (TGI)
  • Serverless: AWS Lambda, Azure Functions
  • Edge Deployment: ONNX, TensorRT for edge devices
  • Managed Services: AWS Bedrock, Azure OpenAI, GCP Vertex AI
Q23: What is vLLM?

vLLM is a high-throughput and memory-efficient LLM serving engine that uses PagedAttention to optimize memory usage.

Key Features:

  • PagedAttention: Efficient attention computation
  • High Throughput: Optimized for serving
  • Memory Efficient: Reduces memory usage
  • Continuous Batching: Dynamic batching
Q24: What is quantization?

Quantization is the process of reducing the precision of model weights (e.g., from 16-bit to 8-bit or 4-bit) to reduce memory usage and improve inference speed.

Benefits:

  • Reduced Memory: Smaller model size
  • Faster Inference: Faster computation
  • Lower Cost: Reduced hardware requirements
  • Edge Deployment: Enables edge deployment

9. AI Safety and Ethics

Q25: What is AI alignment?

AI Alignment is the field of ensuring AI systems act in accordance with human values, intentions, and interests.

Key Approaches:

  • RLHF: Reinforcement Learning from Human Feedback
  • Constitutional AI: AI governed by principles
  • Value Learning: Learning human values
  • Interpretability: Understanding model decisions
Q26: What is RLHF?

RLHF (Reinforcement Learning from Human Feedback) is a technique for training AI models using human feedback to align model outputs with human preferences.

Process:

  • Collect Feedback: Humans rate model outputs
  • Train Reward Model: Model learns to predict human preferences
  • Optimize Policy: Use RL to optimize for reward
Q27: What are the common AI safety concerns?
  • Hallucination: Generating false information
  • Bias: Perpetuating societal biases
  • Privacy: Leaking sensitive information
  • Misuse: Malicious use of AI
  • Autonomy: Unintended autonomous behavior
  • Transparency: Lack of explainability

10. Popular LLM Frameworks

Q28: What is LangChain?

LangChain is a framework for developing applications powered by language models. It provides tools for chaining, memory, agents, and retrieval.

Key Components:

  • Chains: Sequences of operations
  • Agents: Autonomous decision-making
  • Memory: State management
  • Retrievers: Information retrieval
  • Tools: External API integrations
Q29: What is LlamaIndex?

LlamaIndex is a data framework for LLM applications, focused on connecting custom data sources to LLMs for RAG applications.

Key Features:

  • Data Connectors: Ingest data from various sources
  • Indices: Structure data for retrieval
  • Query Engines: Query and retrieve data
  • Chat Engines: Conversational interfaces

12. Model Evaluation

Q33: How do you evaluate LLM performance?
  • Perplexity: Measures prediction quality
  • BLEU Score: Machine translation quality
  • ROUGE Score: Summarization quality
  • Human Evaluation: Human raters assess quality
  • Benchmark Tests: Standardized tests (MMLU, GSM8K)
  • Task-Specific Metrics: Domain-specific evaluations
Q34: What is MMLU?

MMLU (Massive Multitask Language Understanding) is a benchmark for evaluating LLMs across 57 subjects including STEM, humanities, and social sciences.

Key Features:

  • Multi-domain: Covers diverse subjects
  • Multiple Choice: Standardized format
  • Zero-shot and Few-shot: Different evaluation modes
  • Widely Used: Industry standard benchmark

13. Advanced Topics

Q35: What is a multimodal model?

Multimodal Model is an AI model that can process and generate multiple types of data such as text, images, audio, and video.

Examples:

  • GPT-4V: Text and image understanding
  • Gemini: Text, image, audio, video
  • CLIP: Text and image similarity
  • DALL-E 3: Text to image generation
Q36: What is parameter-efficient fine-tuning (PEFT)?

PEFT (Parameter-Efficient Fine-Tuning) refers to methods that fine-tune only a small subset of parameters or add new parameters while keeping most of the model frozen.

Methods:

  • LoRA: Low-Rank Adaptation
  • Prefix Tuning: Learn prompt prefixes
  • P-Tuning: Learn prompt embeddings
  • Adapters: Add adapter layers
Q37: What is model distillation?

Model Distillation is the process of training a smaller, more efficient model (student) to mimic the behavior of a larger, more complex model (teacher).

Benefits:

  • Smaller Size: Reduced model size
  • Faster Inference: Improved speed
  • Lower Cost: Reduced deployment cost
  • Edge Deployment: Enables edge use

14. Common Interview Questions

Q38: What is the difference between temperature and top-p sampling?
  • Temperature: Controls randomness of output. Higher = more random, Lower = more deterministic. Range: 0-2.
  • Top-p (Nucleus Sampling): Samples from top p probability mass. Filters low-probability tokens. Range: 0-1.
Q39: What is the difference between tokenization and embedding?
  • Tokenization: Converting text to tokens (words, subwords). First step in processing. Creates discrete units.
  • Embedding: Converting tokens to vectors. Second step in processing. Creates continuous representations.
Q40: What is the difference between context window and context length?
  • Context Window: Maximum input + output tokens model can handle. Fixed limit determined by model architecture.
  • Context Length: Actual number of tokens in current input. Can vary up to context window limit.
Q41: What is the difference between inference and training?
  • Training: Learning model parameters from data. Computationally expensive. Requires large datasets. Updates model weights.
  • Inference: Using trained model to make predictions. Computationally lighter. Uses fixed weights. Generates output.
Q42: What is the difference between batch size and sequence length?
  • Batch Size: Number of samples processed in parallel. Affects memory usage and training speed. Larger = faster but more memory.
  • Sequence Length: Number of tokens in each sample. Affects context and memory. Longer = more context but more memory.
Q43: What is the difference between greedy decoding and beam search?
  • Greedy Decoding: Always pick highest probability token. Fast but can miss better sequences. No exploration.
  • Beam Search: Keep top k sequences at each step. Slower but finds better sequences. Explores multiple paths.
Q44: What is the difference between BPE and WordPiece tokenization?
  • BPE (Byte Pair Encoding): Merges most frequent byte pairs. Simple and effective. Used by GPT.
  • WordPiece: Similar to BPE but different merge criteria. Maximizes likelihood. Used by BERT.
Q45: What is the difference between causal and masked language modeling?
  • Causal Language Modeling: Predict next token given previous. Left-to-right only. Used by GPT. Good for generation.
  • Masked Language Modeling: Predict masked tokens given context. Bidirectional. Used by BERT. Good for understanding.
Q46: What is the difference between attention and self-attention?
  • Attention: General mechanism for focusing on relevant parts. Can be between different sequences. Used in encoder-decoder.
  • Self-Attention: Attention within same sequence. All positions attend to all positions. Used in Transformers.
Q47: What is the difference between position encoding and positional embedding?
  • Position Encoding: Add position information to embeddings. Can be sinusoidal or learned. Added to token embeddings.
  • Positional Embedding: Learned position vectors. Similar to word embeddings. Can be absolute or relative.
Q48: What are the best practices for prompt engineering?
  • Be Specific: Provide clear, detailed instructions
  • Use Examples: Provide few-shot examples
  • Chain-of-Thought: Encourage step-by-step reasoning
  • Role Assignment: Assign specific role to model
  • Format Requirements: Specify output format
  • Context: Provide relevant context
  • Iterate: Refine prompts based on results
  • Test: Test prompts with various inputs
Q49: What are the best practices for RAG implementation?
  • Chunking Strategy: Optimal chunk size for retrieval
  • Embedding Model: Choose appropriate embedding model
  • Vector Database: Use efficient vector database
  • Retrieval Strategy: Hybrid search (keyword + semantic)
  • Reranking: Rerank retrieved documents
  • Citation: Include source citations
  • Evaluation: Evaluate retrieval quality
  • Update Strategy: Regular knowledge base updates

  • Maintenance: Ongoing model updates
