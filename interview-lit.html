1. Lit Fundamentals

Q1: What is Lit and what are its key features?

Lit is a lightweight library for building fast, lightweight web components. It provides a simple base class for creating custom elements with reactive state and efficient rendering.

Key Features:

  • Lightweight: Small bundle size (~5KB)
  • Fast: Efficient rendering with declarative templates
  • Web Components: Built on Web Components standards
  • Reactive: Automatic updates when properties change
  • TypeScript support: First-class TypeScript support
  • Framework-agnostic: Works with any framework
  • Declarative templates: HTML template literals
Q2: What are Web Components?

Web Components are a set of web platform APIs that allow you to create new custom, reusable, encapsulated HTML tags to use in web pages and web apps.

Web Components Standards:

  • Custom Elements: Define new HTML elements
  • Shadow DOM: Encapsulate DOM and styles
  • HTML Templates: Define reusable markup
  • ES Modules: Import and export JavaScript

import { LitElement, html, css } from 'lit';

class MyElement extends LitElement {

  _increment() {
    this.count++;
  }
}

3. Reactive Properties

Q5: How do reactive properties work in Lit?

Lit's reactive properties automatically trigger a re-render when they change. Lit uses a reactive system that:

  • Tracks property changes
  • Schedules updates efficiently
  • Batches multiple changes
  • Updates only changed parts of the DOM
static properties = {
  items: { type: Array },
  selected: { type: Number }
};

render() {
  return html`
    <ul>
      ${this.items.map((item, index) => html`
        <li class="${index === this.selected ? 'selected' : ''}">
          ${item.name}
        </li>
      `)}
    </ul>
  `;
}
Q6: What is the difference between internal state and properties?
  • Properties: Public, reactive, can be set from outside. Triggers re-render when changed.
  • Internal state: Private, not reactive by default. Use for component-internal state that doesn't need to trigger re-render.
// Property - reactive, public
static properties = {
  count: { type: Number }
};

// Internal state - not reactive
private _internalState = {
  data: null
};

4. Templates and Rendering

Q7: How do you use Lit's template system?

Lit uses html template tag function to create templates. It supports:

  • Expression interpolation
  • Conditional rendering
  • Looping
  • Event handling
  • Directives
render() {
  return html`
    <div>
      <h1>${this.title}</h1>
      
      ${this.items.length > 0 ? html`
        <ul>
          ${this.items.map(item => html`
            <li>${item.name}</li>
          `)}
        </ul>
      ` : html`
        <p>No items</p>
      `}
      
      <button @click=${this.handleClick}>Click</button>
    </div>
  `;
}
Q8: What are Lit directives?

Directives are functions that modify template behavior. Lit provides several built-in directives:

  • repeat: Efficient list rendering with keying
  • map: Simple array mapping
  • classMap: Conditional class names
  • styleMap: Conditional styles
  • ifDefined: Render only if value is defined
  • unsafeHTML: Render raw HTML (use with caution)
import { repeat } from 'lit/directives/repeat.js';
import { classMap } from 'lit/directives/class-map.js';
import { styleMap } from 'lit/directives/style-map.js';

render() {
  return html`
    <div class="${classMap({ active: this.isActive })}">
      ${repeat(this.items, (item) => item.id, (item, index) => html`
        <div style="${styleMap({ color: item.color })}">
          ${item.name}
        </div>
      `)}
    </div>
  `;
}

5. Styling

Q9: How do you style Lit components?

Lit components use Shadow DOM by default, which encapsulates styles. Styles are defined in the static styles property.

import { css } from 'lit';

class MyElement extends LitElement {
  static styles = css`
    :host {
      display: block;
      padding: 16px;
      background: #f0f0f0;
    }
    
    h1 {
      color: #333;
      font-size: 24px;
    }
    
    button {
      background: #512bd4;
      color: white;
      border: none;
      padding: 8px 16px;
      cursor: pointer;
    }
    
    button:hover {
      background: #4230a3;
    }
  `;
}
Q10: What is the difference between scoped and global styles?
  • Scoped styles: Defined in static styles property. Encapsulated to component's Shadow DOM. Don't leak out or get affected by external styles.
  • Global styles: Can be applied to light DOM or shared across components. Use constructable stylesheets or global CSS.
// Scoped (Shadow DOM)
static styles = css`
  :host { display: block; }
`;

// Global (Light DOM)
const globalStyle = css`
  .global-class { color: red; }
`;

render() {
  return html`
    <style>${globalStyle}</style>
    <div class="global-class">Global styled</div>
  `;
}

6. Lifecycle

Q11: What are the Lit component lifecycle methods?
  • constructor(): Called when element is created. Set initial state.
  • connectedCallback(): Called when element is added to DOM. Set up event listeners, timers.
  • disconnectedCallback(): Called when element is removed from DOM. Clean up resources.
  • attributeChangedCallback(): Called when observed attributes change.
  • willUpdate(): Called before update. Can modify properties before render.
  • updated(): Called after update. Post-render logic.
  • firstUpdated(): Called after first update. One-time setup.
class MyElement extends LitElement {
  connectedCallback() {
    super.connectedCallback();
    // Set up event listeners, timers
    this._interval = setInterval(() => this._tick(), 1000);
  }
  
  disconnectedCallback() {
    super.disconnectedCallback();
    // Clean up
    clearInterval(this._interval);
  }
  
  willUpdate(changedProperties) {
    // Modify properties before render
    if (changedProperties.has('value')) {
      this._processedValue = this._process(this.value);
    }
  }
  
  updated(changedProperties) {
    // Post-render logic
    if (changedProperties.has('items')) {
      this._scrollToBottom();
    }
  }
}

7. Testing

Q12: How do you test Lit components?

Lit components can be tested with standard web component testing tools like Web Test Runner or @web/tester.

import { fixture, expect } from '@open-wc/testing';
import { MyElement } from './my-element.js';

describe('MyElement', () => {
  it('renders a greeting', async () => {
    const el = await fixture('<my-element name="World"></my-element>');
    
    const h1 = el.shadowRoot.querySelector('h1');
    expect(h1.textContent).to.equal('Hello, World!');
  });
  
  it('increments count on button click', async () => {
    const el = await fixture('<my-element></my-element>');
    const button = el.shadowRoot.querySelector('button');
    
    button.click();
    await el.updateComplete;
    
    expect(el.count).to.equal(1);
  });
});

9. Common Interview Questions

Q15: What is Shadow DOM and why is it important?

Shadow DOM is a web standard that provides encapsulation for DOM and styles. It:

  • Isolates component styles from the page
  • Prevents style conflicts
  • Enables component composition
  • Provides true encapsulation
Important: Lit uses Shadow DOM by default, but you can opt out to use Light DOM if you need global style penetration.
Q16: What is the difference between render and firstUpdated?
  • render: Called every time properties change. Returns the template to render.
  • firstUpdated: Called only once after the first render. Used for one-time setup like initializing third-party libraries.
Q17: How do you handle events in Lit? 
// Event handling in template
render() {
  return html`
    <button @click=${this.handleClick}>Click</button>
    <input @input=${this.handleInput}>
    <div @mouseover=${this.handleHover}>Hover</div>
  `;
}

handleClick(event) {
  console.log('Clicked:', event);
}

handleInput(event) {
  this.value = event.target.value;
}

handleHover(event) {

}
Q18: What is the difference between @event and @event:capture?
  • @event: Bubbling phase. Event bubbles up from target to ancestors.
  • @event:capture: Capture phase. Event flows down from ancestors to target.
<div @click=${this.handleParentClick}>
  <button @click=${this.handleChildClick}>Click</button>
</div>

<div @click:capture=${this.handleParentClick}>
  <button @click=${this.handleChildClick}>Click</button>
</div>

9. Advanced Lit Patterns (Expert Level)

Q31: How do you implement a compound component pattern in Lit? 
 Parent component
class Tabs extends LitElement {
  static properties = {
    active: { type: Number }
  };

  static styles = css`
    :host { display: block; }
    ::slotted(.tab) { padding: 10px; cursor: pointer; }
    ::slotted(.tab.active) { background: #ddd; }
  `;

  render() {
    return html`
      <slot @slotchange="${this._handleSlotChange}"></slot>
      <slot name="panel"></slot>
    `;
  }

  _handleSlotChange(e) {
    const tabs = e.target.assignedElements({ flatten: true });
    tabs.forEach((tab, index) => {
      tab.addEventListener('click', () => {
        this.active = index;
        this._updatePanels();
      });
    });
  }

  _updatePanels() {
    const panels = this.shadowRoot.querySelectorAll('slot[name="panel"]')
      .assignedElements({ flatten: true });
    panels.forEach((panel, index) => {
      panel.style.display = index === this.active ? 'block' : 'none';
    });
  }
}

// Usage
<my-tabs>
  <div class="tab">Tab 1</div>
  <div class="tab">Tab 2</div>
  <div slot="panel">Panel 1</div>
  <div slot="panel">Panel 2</div>
</my-tabs>

// Custom directive for event delegation
const delegate = directive((eventName, handler) => (part) => {
  const element = part.committer?.element;
  if (element) {
    element.addEventListener(eventName, (e) => {
      if (e.target.matches(part.selector)) {
        handler(e);
      }
    });
  }
});

Q34: How do you implement a controller in Lit? 
 Controller for managing child elements
class ResizeController {
  constructor(host, callback) {
    this.host = host;
    this.callback = callback;
    this.observer = new ResizeObserver((entries) => {
      this.callback(entries[0].contentRect);
    });
    this.host.addController(this);
  }

  hostConnected() {
    this.observer.observe(this.host);
  }

  hostDisconnected() {
    this.observer.disconnect();
  }
}

class ResizableElement extends LitElement {
  constructor() {
    super();
    new ResizeController(this, (rect) => {
      console.log('Size:', rect.width, rect.height);
    });
  }

  render() {
    return html`<div>Resize me</div>`;
  }
}

11. Performance Optimization (Expert Level)

Q38: How do you optimize Lit component performance?
  • Use @state for internal state: Only re-renders when state changes
  • Avoid expensive computations in render: Memoize results
  • Use directives for repeated patterns: Reduce template complexity
  • Lazy load components: Use dynamic imports
  • Virtualize long lists: Use windowing techniques
  • Batch property updates: Use requestUpdateAsync
  • Avoid unnecessary re-renders: Use hasChanged option
 Custom hasChanged for performance
class OptimizedElement extends LitElement {
  static properties = {
    data: {
      hasChanged: (newVal, oldVal) => {
        // Only update if data actually changed
        return JSON.stringify(newVal) !== JSON.stringify(oldVal);
      }
    }
  }

  render() {
    return html`<div>${this.data}</div>`;
  }
}

Q40: How do you implement lazy loading in Lit? 
class LazyLoadElement extends LitElement {
  static properties = {
    component: { type: String }
  };

  @state()
  private _loadedComponent = null;

  async _loadComponent() {
    if (!this._loadedComponent && this.component) {
      const module = await import(`./${this.component}.js`);
      this._loadedComponent = module.default;
    }
  }

  render() {
    if (!this._loadedComponent) {
      return html`<div>Loading...</div>`;
    }
    return html`<${this._loadedComponent}></${this._loadedComponent}>`;
  }

  updated(changedProperties) {
    super.updated(changedProperties);
    if (changedProperties.has('component')) {
      this._loadComponent();
    }
  }
}

Q46: How do you implement form validation in Lit? 
class ValidatedInput extends LitElement {
  static properties = {
    value: { type: String },
    required: { type: Boolean },
    pattern: { type: String },
    error: { type: String }
  };

  render() {
    return html`
      <input
        value="${this.value}"
        ?required="${this.required}"
        pattern="${this.pattern}"
        @input="${this._handleInput}"
        @invalid="${this._handleInvalid}"
      />
      ${this.error ? html`<span class="error">${this.error}</span>` : ''}
    `;
  }

  _handleInput(e) {
    this.value = e.target.value;
    this.error = '';
  }

  _handleInvalid(e) {
    this.error = e.target.validationMessage;
  }
}

Q48: How do you implement drag and drop in Lit? 
class DraggableElement extends LitElement {
  render() {
    return html`
      <div
        draggable="true"
        @dragstart="${this._handleDragStart}"
        @dragend="${this._handleDragEnd}"
      >
        Drag me
      </div>
    `;
  }

  _handleDragStart(e) {
    e.dataTransfer.setData('text/plain', 'dragged-data');
    this.classList.add('dragging');
  }

  _handleDragEnd() {
    this.classList.remove('dragging');
  }
}

class DropZone extends LitElement {
  render() {
    return html`
      <div
        @dragover="${this._handleDragOver}"
        @drop="${this._handleDrop}"
      >
        Drop here
      </div>
    `;
  }

  _handleDragOver(e) {
    e.preventDefault();
  }

  _handleDrop(e) {
    e.preventDefault();
    const data = e.dataTransfer.getData('text/plain');
    console.log('Dropped:', data);
  }
}
